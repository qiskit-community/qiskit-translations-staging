গ্রোভার অ্যালগরিদম এবং বিস্তার বিবর্ধন¶

লোভ গ্রোভার দ্বারা ১৯৯৬ সালে বানানো গ্রোভার অ্যালগরিদম [1] হচ্ছে কোয়ান্টাম অ্যালগরিদমগুলি দের মধ্যে যুক্তিযুক্তভাবে সবচেয়ে বিখ্যাত। এটি প্রথমে কাঠামোগত অনুসন্ধান সমস্যার (একটি কাঠামোহীন ডাটাবেসে চিহ্নিত উপাদান বের করার জন্য) জন্যে উত্থাপন করা হয়েছিল। ইদানিং এটি একাধিক অ্যালগরিদম এর মধ্যে মিলিত। উদাহরণ মূলক গ্রোভার অভিযোজিত অনুসন্ধান (গ্রোভার অ্যাডাপ্টিভ সার্চ) [2]। এর উপরে আরো জানতে Qiskit নোটবুকে গ্রোভার অ্যালগরিদম দেখুন।

Qiskit গ্রভারের অ্যালগরিদমকে Grover শ্রেণিতে সম্পাদন করে। এই শ্রেণিতে সাধারণকৃত সংস্করণ, বিস্তার পরিবর্ধন [3] অন্তর্ভুক্ত রয়েছে, এবং এটি গ্রোভারের অ্যালগরিদমে পৃথক পুনরাবৃত্তি এবং অন্যান্য মেটা-সেটিংস সেট করার অনুমতি দেয়।

রেফারেন্স সমূহ:

[1]: L. K. Grover, ডাটাবেস অনুসন্ধানের জন্য একটি দ্রুত কোয়ান্টাম মেকানিকাল অ্যালগরিদম। থিউরি অফ কম্পিউটিংয়ের ২৮ তম বার্ষিক সিম্পোজিয়ামের ক্রিয়াকলাপ (STOC) ১৯৯৬, pp. 212-219. https://arxiv.org/abs/quant-ph/9605043

[2]: A. Gilliam, S. Woerner, C. Gonciulea, গ্রোভার অ্যাডাপটিভ সার্চ ফর কনস্ট্রেনেড পলিনোমিয়াল বাইনারি অপটিমাইজেশন, https://arxiv.org/abs/1912.04088

[3]: Brassard, G., Hoyer, P., Mosca, M., & Tapp, A. (২০০০). কোয়ান্টাম বিস্তার পরিবর্ধন এবং অনুমান। http://arxiv.org/abs/quant-ph/0005055

গ্রোভার অ্যালগরিদম / ধরাক্রম¶

গ্রোভারের অ্যালগরিদম বিস্তার পরিবর্ধন করতে \(\mathcal{Q}\) অপারেটরটি ব্যাবহার করে:

\[\mathcal{Q} = \mathcal{A}\mathcal{S_0}\mathcal{A}^\dagger \mathcal{S_f}\]

এখানে, * \(\mathcal{A}\) হল অ্যালগরিদমটির প্রাথমিক অনুসন্ধানপূর্বক অবস্থা, যেটা কেবল গ্রোভার অনুসন্ধানের জন্য Hadamard গেটসমুহ \(H^{\otimes n}\) , কিন্তু বিস্তার পরিবর্ধনের জন্য বিস্তারযোগ্য, \(\mathcal{S_0}\) হল সবগুলো 0 অবস্থার প্রতিফলন।

\[\begin{split}|x\rangle \mapsto \begin{cases} -|x\rangle, &x \neq 0 \\ |x\rangle, &x = 0\end{cases}\end{split}\]

* \(\mathcal{S_f}\) ওরাকলটি প্রয়োগ করে,

\[|x\rangle \mapsto (-1)^{f(x)}|x\rangle\]

যেখানে \(f(x)\) হল 1 যদি \(x\) একটি সুষম অবস্থা হয়, আর তা নাহলে 0.

সংক্ষেপে, গ্রোভারের অ্যালগরিদম \(\mathcal{Q}\) বিভিন্ন সূচক প্রয়োগ করে এবং প্রতিটি সম্পাদন কার্যকর করার পরে একটি ভাল সমাধান পাওয়া গেছে কিনা তা পরীক্ষা করে।

গ্রোভারের অ্যালগরিদম চালানো¶

গ্রোভারের অ্যালগরিদমকে Grover শ্রেণীর সাথে চালাতে প্রথমে আমাদের গ্রোভারের অ্যালগরিদমের সার্কিটের জন্য একটি ওরাকল নির্ধারণ করতে হবে। নিম্নলিখিত উদাহরণে, আমরা গ্রোভারের অ্যালগরিদমের ওরাকল হিসাবে QuantumCircuit ব্যবহার করেছি। তবে, আরও কয়েকটি শ্রেণি রয়েছে যেগুলো আমরা গ্রোভারের অ্যালগরিদমের ওরাকল হিসাবে ব্যবহার করতে পারি। আমরা সেগুলো সম্পর্কে এই টিউটোরিয়ালের শেষেরদিকে আলোচনা করব।

লক্ষ্য কর Grover এর জন্য ওরাকলটি অবশ্যই একটি phase-flip ওরাকল হতে হবে। অর্থাৎ, এটা সুষম অবস্থার বিস্তারকে \(-1\) এর একটি গুনিতক দ্বারা গুন করে। কিভাবে একটি bit-flip ওরাকলকে একটি phase-flip ওরাকলে পরিণত করা যায় তা আমরা পরে আলোচনা করব।

[1]:

from qiskit import QuantumCircuit
from qiskit.algorithms import AmplificationProblem

# the state we desire to find is '11'
good_state = ['11']

# specify the oracle that marks the state '11' as a good solution
oracle = QuantumCircuit(2)
oracle.cz(0, 1)

# define Grover's algorithm
problem = AmplificationProblem(oracle, is_good_state=good_state)

# now we can have a look at the Grover operator that is used in running the algorithm
problem.grover_operator.draw(output='mpl')

[1]:

তারপরে, আমরা একটি ব্যাকএন্ড পছন্দ করি এবং সার্কিটগুলি সম্পাদন করতে ব্যাকএন্ড সহ Grover এর run মেথডটি কল করি। প্রত্যাবর্তিত ফলাফলের ধরণটি হল একটি GroverResult।

অনুসন্ধানটি সফল হলে, ফলাফলটির oracle_evaluation অ্যাট্রিবিউট True হবে। এই ক্ষেত্রে, সর্বাধিক নমুনাকৃত হিসাব top_measurement হল অনেকগুলো সূষম অবস্থার মধে একটি অবস্থা।

[2]:

from qiskit import Aer
from qiskit.utils import QuantumInstance
from qiskit.algorithms import Grover

aer_simulator = Aer.get_backend('aer_simulator')
grover = Grover(quantum_instance=aer_simulator)
result = grover.amplify(problem)
print('Result type:', type(result))
print()
print('Success!' if result.oracle_evaluation else 'Failure!')
print('Top measurement:', result.top_measurement)

Result type: <class 'qiskit.algorithms.amplitude_amplifiers.grover.GroverResult'>

Success!
Top measurement: 11

উদাহরণস্বরূপ, top_measurement এর ফলাফল হল 11, যা একটি “good state”। এভাবে, আমরা গ্রোভার ব্যবহার করে ফলাফলটি বের করতে পেরেছি।

ওরাকল হিসাবে Grover এর বিভিন্ন ধরণের ক্লাস ব্যবহার করা¶

যদিও, আমরা ওরাকল হিসাবে qiskit.quantum_info.Statevector ব্যবহার করতে পারতাম, উপরিউক্ত উদাহরনে QuantumCircuit কে Grover এর ওরাকল হিসাবে ব্যবহার করেছি। “good state” হিসেবে \(|11\rangle\) এর উদাহরণগুলো নিম্নরূপ,

[3]:

from qiskit.quantum_info import Statevector
oracle = Statevector.from_label('11')
problem = AmplificationProblem(oracle, is_good_state=['11'])

grover = Grover(quantum_instance=aer_simulator)
result = grover.amplify(problem)
print('Result type:', type(result))
print()
print('Success!' if result.oracle_evaluation else 'Failure!')
print('Top measurement:', result.top_measurement)

Result type: <class 'qiskit.algorithms.amplitude_amplifiers.grover.GroverResult'>

Success!
Top measurement: 11

ইন্টারনালি, স্টেকভেক্টরটি কোয়ান্টাম সার্কিটে ম্যাপ করা হয়:

[4]:

problem.grover_operator.oracle.draw(output='mpl')

[4]:

Qiskit আরও জটিল ওরাকলগুলি সহজেই নির্মাণের সুযোগ দেয়। যেমন লজিকাল এক্সপ্রেশন বাছাই করার জন্য * PhaseOracle, যদি এবং কেবল যদি '~a | b' হয়। এটি বিশেষত 3-SAT সমস্যা সমাধানের জন্য সহায়ক এবং এর সাথে সম্পৃক্ত Grover Examples টিউটোরিয়ালে প্রদর্শিত হবে।

এখানে আমরা সরল উদাহরণসরূপ \(|11\rangle\) অবস্থা অনুসন্ধানের জন্য PhaseOracle ব্যবহার করব, যখন 'a & b' হয়।

[5]:

from qiskit.circuit.library.phase_oracle import PhaseOracle
from qiskit.exceptions import MissingOptionalLibraryError

# `Oracle` (`PhaseOracle`) as the `oracle` argument
expression = '(a & b)'
try:
    oracle = PhaseOracle(expression)
    problem = AmplificationProblem(oracle)
    display(problem.grover_operator.oracle.draw(output='mpl'))
except MissingOptionalLibraryError as ex:
    print(ex)

আপনি পর্যবেক্ষণ করতে পারেন যে, এই ওরাকলটি দুটির পরিবর্তে তিনটি কিউবিট দিয়ে বাস্তবায়িত হয়েছে!

এর কারণ হল PhaseOracle কোনো ফেজ-ফ্লিপ ওরাকল নয় (যা সুষম অবস্থার ধাপটিকে বদলিয়ে দেয়) বিট-ফ্লিপ ওরাকল। অর্থাৎ এটি যদি অন্যান্য কিউবিটগুলো শর্তটি পূরণ করে তবে এটি একটি গৌণ কিউবিট এর অবস্থা ফ্লিপ করে। তাও গ্রোভারের অ্যালগরিদমের জন্য আমাদের একটি ফেজ-ফ্লিপ ওরাকল প্রয়োজন। যেমনটি আপনি উপরের সার্কিটটিতে দেখতে পাচ্ছেন, বিট-ফ্লিপ ওরাকলকে একটি ফেজ-ফ্লিপ ওরাকলে রূপান্তর করতে আমরা controled-NOT গেটকে \(X\) এবং \(H\) গেট দ্বারা পরিবর্তন করি।

Note: এভাবে বিট-ফ্লিপ থেকে একটি ফেজ-ফ্লিপ ওরাকল এ রূপান্তরটি সাধারণভাবে সংরক্ষিত হয় এবং আপনি আপনার ওরেকলকে যথার্থ প্রতিরূপে রূপান্তর করতে এটি ব্যবহার করতে পারেন।

বিস্তার বিবর্ধন¶

গ্রোভারের অ্যালগরিদম সবগুলো মৌলিক সুপারপজিশন তৈরি করতে গ্রোভার অপারেটর \(\mathcal{Q}\) এর শুরুতে হাদামারড গেটগুলি ব্যবহার করে। যদি সুষম অবস্থার ওপর কোন তথ্য থেকে থাকে, তবে একে মৌলিক সুপারপজিশনে শুরু না করিয়ে কেবল একটি নির্দিষ্ট অবস্থায় স্থির করলে উপকারি হতে পারে। গ্রভারের অ্যালগরিদমের এই সারাংশিত ভার্শনকে বিস্তার পরিবর্ধন বলে অভিহিত করা হয়।

কিস্কিটে প্রাথমিক সুপারপজিশন স্টেট খুব সহজেই state_preparation আরগুমেন্ট বসিয়ে রদবদল করে নেয়া যায়।

State preparation - অবস্থা ব্যবস্থাপন¶

state_preparation আর্গুমেন্টটি কোয়ান্টাম সার্কিট চিহ্নিত করতে ব্যবহৃত হয় যা বিস্তার পরিবর্ধনের প্রারম্ভিকতার জন্য কোয়ান্টাম স্টেট প্রস্তুত করে। প্রাথমিকভাবে, $H^{\otimes `n}$ সহ একটি সার্কিট মৌলিক সুপারপজিশন তৈরি করতে ব্যাহৃত হয়(যেন এটি একটি গ্রভার সার্চ হয়)। বিস্তার বিবর্ধনের ডিফিউশান সার্কিটটি স্বয়ংক্রিয়ভাবেই ``state_preparation` ব্যাবর্তন করে।

[6]:

import numpy as np

# Specifying `state_preparation`
# to prepare a superposition of |01>, |10>, and |11>
oracle = QuantumCircuit(3)
oracle.h(2)
oracle.ccx(0,1,2)
oracle.h(2)

theta = 2 * np.arccos(1 / np.sqrt(3))

Full flexibility-সম্পূর্ণ অভিযোজ্যতা¶

আরও বেশি তদতিরিক্ত ব্যবহারের জন্য, সম্পূর্ণ গ্রোভার অপারেটরটি grover_operator আর্গুমেন্ট দ্বারা নির্দিষ্ট করাও সম্ভব। আর এটা তখনি লাভবান হবে যখন আপনি জিরো রিফ্লেকশন, ওরাকল অথবা স্টেট প্রিপারেশোনের চেয়ে \(\mathcal{Q}\) এর জন্য বেশি ফলপ্রদ নিষ্পাদন জানবেন।

গ্রোভার অপারেটরের স্বয়ংক্রিয় উৎপাদনের জন্য qiskit.circuit.library.GroverOperator একটি উপযুক্ত সূত্রপাত হতে পারে। উদাহরণস্বরূপ, আপনি বাহ্যিকভাবে reflection_qubits * বসিয়ে কিউবিট জিরো রিফ্লেকশনের মাঝের কিউবিটসমুহকে উপেক্ষা করে oracle, zero_reflection এবং state_preparation আরগুমেন্টগুলো দ্বারা \(\mathcal{S_f}, \mathcal{S_0}\) এবং \(\mathcal{A}\) কে পরিবর্তন করতে পারেন।

উদাহরণস্বরূপ, মনে করুন সুষম অবস্থাটা হল তিন কিউবিটের একটা অবস্থা \(|111\rangle\), কিন্তু আমরাতো অতিরিক্ত ২টি কিউবিট গৌণ কিউবিট হিসেবে ব্যাবহার করেছি।

Grover এর oracle এবং state_preparation ছাড়াও অন্যান্য আরগুমেন্ট আছে। এই অনুচ্ছেদে আমরা তা ব্যাখ্যা করব।

good_state - সুষম_অবস্থা নির্দিষ্টকরন:¶

ইন্টারনালি পরিমাপন ফল সঠিক হচ্ছে কিনা তা যাচাই করতে good_state ব্যবহার করা হয়। এটা বাইনারি স্ট্রিং এর একটি লিস্ট হতে পারে, আবার ইন্টেজার, Statevector কিংবা কলেবল হতে পারে। যদি ইনপুট একটি ইন্টেজারের লিস্ট হয়, তাহলে প্রত্যেকটা ইন্টেজার সুষম অবস্থার ধারা হিসেবে \(|1\rangle\) দেখায়। আর যদি এটা একটা Statevector হয়, তাহলে সব সুষম অবস্থার সুপারপজিশন দেখায়।

পুনরাবৃত্তির সংখ্যা¶

গ্রোভারের অ্যালগরিদমের মাধ্যমে সঠিক ফলাফল পেতে গ্রোভার অপারেটর প্রয়োগের পুনরাবৃত্তির সংখ্যা গুরুত্বপূর্ণ। পুনরাবৃত্তির সংখ্যা Grover এর iteration আরগুমেন্ট দ্বারা সেট করা যেতে পারে। সাপোর্টেড ইনপুটগুলো নিম্নরূপ: গ্রোভার অপারেটরের সূচক নির্দিষ্ট করতে একটি *ইন্টেজার অথবা এটি *ইন্টেজারের লিস্ট, যেগুলোতে এইসব গ্রোভার অপারেটরের সূচকগুলো বারবার এবং প্রত্যেকবার আমাদের চেক করার পর সঠিক সমাধান খুঁজে পাওয়া গেছে কিনা জেনে সম্পাদিত হয়

এছাড়াও sample_from_iterations নামক একটা আরগুমেন্ট আছে। যখন এটা True হয়, গ্রোভারের অপারেটরের মত নির্দিষ্ট সূচকের পুনরাবৃত্তির বদলে 0 এবং পুনরাবৃত্তির মানের একটি র‍্যান্ডম ইন্টেজার এখানে ব্যাবহ্রিত হয়।

sample_from_iterations ব্যাবহার করে অ্যালগরিদমটির সম্পর্কে বিস্তারিত জানতে [4] দেখুন।

রেফারেন্স সমূহ:

[4]: Boyer et al., Tight bounds on quantum searching arxiv:quant-ph/9605034

post_processing এর ব্যাবহার¶

সহজে পড়ার জন্য আমরা সর্বশেষ মেজারমেন্টে একটি ঐচ্ছিক পোস্ট প্রসেসিং প্রয়োগ করতে পারি। এটা ব্যাবহ্রিত হতে পারে, যেমন: মেজারমেন্টের বিট-রিপ্রেজেন্টেশন [1, 0, 1] থেকে একটি DIMACS CNF রুপী [1, -2, 3]।

def to_DIAMACS_CNF_format(bit_rep):
    return [index+1 if val==1 else -1 * (index + 1) for index, val in enumerate(bit_rep)]

oracle = QuantumCircuit(2)
oracle.cz(0, 1)
problem = AmplificationProblem(oracle, is_good_state=['11'], post_processing=to_DIAMACS_CNF_format)
problem.post_processing([1, 0, 1])
