কোয়ান্টাম টমোগ্রাফি¶

ভূমিকা¶

কোয়ান্টাম টমোগ্রাফি হচ্ছে এমন একটা পরীক্ষার পদ্ধতি যেখানে নির্দিষ্ট কিছু মেজারমেন্টের ফলাফল থেকে কোয়ান্টাম সিস্টেমটির পুনঃউৎপাদন বর্ণনা করা হয়। Qiskit এ আমরা নিম্নোক্ত টমোগ্রাফি পদ্ধতি প্রয়োগ করব:

  1. কোয়ান্টাম স্টেট টমোগ্রাফি: একটি দশা বা স্টেট প্রস্তুতকারী বর্তনী যা দশার ডেনসিটি ম্যাট্রিক্স \(\rho\) থেকে দশার প্রকৃত অবস্থা সম্পর্কে ধারণা দেয়।

  2. কোয়ান্টাম প্রক্রিয়াকারী টমোগ্রাফি: এই সার্কিট সিস্টেম দ্বারা সম্পাদিত সার্কিটের অপারেটর সম্পর্কে ধারনা দেয়। এর বিবরণকালে সার্কিটটি একটি কোয়ান্টাম চ্যানেল \(\mathcal{E}\) ব্যবহার করে।

  3. কোয়ান্টাম যুক্তি বর্তনী টমোগ্রাফি: এই ধরনের টমোগ্রাফিক সার্কিট কিছু নির্দিষ্ট সার্কিটের উপর সেলফ কন্সিস্টেন্ট ভিত্তিতে কাজ করে। সেলফ কন্সিস্টেন্ট প্রক্রিয়ার একটি বিশেষ বৈশিষ্ট্য হচ্ছে- সার্কিটের উপর টমোগ্রাফিক প্রক্রিয়া দ্বারা কোয়ান্টাম নয়েজকেও এক্ষেত্রে বিবেচনায় নেয়া হয়।

এই নোটবুকটি তে ignis.verification.tomography এই মডিউলটির ব্যবহার ব্যাখ্যা করা হবে।

Here we are going to generate and run the state tomography circuits. By only passing in the 2 registers we want to measure the state tomography will only run on that reduced \(2^2\) Hilbert space. However, if we pass the whole register in the state tomography module will try and fit the full \(2^6\) space.

উচ্চদক্ষতা ভিত্তিক কিছু উদাহরণসমূহ¶

দৈবচয়ন ভিত্তিক দশা (র‍্যান্ডম স্টেটস) প্রস্তুতকরন এবং ফিটিং¶

এখন আমরা দৈবচয়ন ভিত্তিক একটি একক কিউবিট থেকে নির্মিত একক অভেদক (ইউনিটারি) সার্কিট u3 দ্বারা প্রস্তুতকৃত দশার অন্বয় (ফাংশন) তৈরি করব।

[11]:

def random_u_tomo(nq, shots):

    def rand_angles():
        return tuple(2 * np.pi * np.random.random(3) - np.pi)
    q = QuantumRegister(nq)
    circ = QuantumCircuit(q)
    for j in range(nq):
        circ.u(*rand_angles(), q[j])
    target_state = qi.Statevector.from_instruction(circ)

    qst_circs = state_tomography_circuits(circ, q)
    job = qiskit.execute(qst_circs, Aer.get_backend('qasm_simulator'),
                         shots=shots)
    tomo_data = StateTomographyFitter(job.result(), qst_circs)
    rho_fit = tomo_data.fit(method='lstsq')

    print('F = {:.5f}'.format(qi.state_fidelity(rho_fit, target_state)))

দুই-কিউবিট শর্তাধীন টমোগ্রাফি স্টেট¶

এই উদাহরনে আমরা তিন কিউবিট সিস্টেম ব্যবহার করে স্টেট টমোগ্রাফি প্রক্রিয়া সম্পাদন করব। এই তিনটি কিউবিটের মধ্যে একটি হবে এনসিলা কিউবিট বা সাহায্যকারী কিউবিট। এনসিলা কিউবিটটি টমোগ্রাফির উপর তখনই কাজ করবে যখন তৃতীয় কিউবিটটির দশা অবস্থা হবে “১” । এই যুক্তিবর্তনীটি এমনভাবে সম্পাদন করা হয় যাতে শর্তাধীন টমোগ্রাফী সম্পাদনের পর প্রথম দুইটি কিউবিট বেল দশায় (বেল স্টেট) এ থাকে।

প্রথমে কোনরকম ধ্রুপদী গণনাযন্ত্রের পরিমাপ সাহায্য ছাড়া একটি 3Q GHZ স্টেট সার্কিট বানাতে হবে।

যুক্তিবর্তনী ভিত্তিক টমোগ্রাফি¶

এক-কিউবিট মান বা অবস্থার টমোগ্রাফির উদাহরণ¶

যুক্তিবর্তনী ভিত্তিক টমোগ্রাফি (গেট সেট টমোগ্রাফি) এবং প্রক্রিয়াধীন টমোগ্রাফি (প্রসেস টমোগ্রাফি) এর মধ্যে প্রধান পার্থক্য হচ্ছে যুক্তিবর্তনী ভিত্তিক টমোগ্রাফীতে ইনপুট একগুচ্ছ সার্কিট ভিত্তিক বেসিস হিসেবে থাকে। এই গুচ্ছ সার্কিটগুলোকে (সেট অব গেটস) টমোগ্রাফির প্রারম্ভিকরন (ইনিশিয়ালাইজেশন) / পরিমাপ (মেজারমেন্ট) দশা হিসেবে ব্যবহার এবং সম্পাদন করা যাবে।

Qiskit এর নিজস্ব একগুচ্ছ যুক্তিবর্তনী ভিত্তিক বেসিস আছে। এই গুচ্ছ যুক্তিবর্তনী ভিত্তিক বেসিস ব্যবহার করে নতুন কিছু সার্কিট তৈরি করা যাবে এবং এই নতুন সার্কিটগুলো একই বেসিস ব্যবহার করতে পারবে। নিম্নে এই প্রক্রিয়া বর্ণনা করা হল:

from qiskit.ignis.verification.tomography.basis import default_gateset_basis

def collect_tomography_data(shots=10000,
                            noise_model=None,
                            gateset_basis='Standard GST'):
    backend_qasm = Aer.get_backend('qasm_simulator')
    circuits = gateset_tomography_circuits(gateset_basis=gateset_basis)
    qobj = assemble(circuits, shots=shots)
    result = backend_qasm.run(qobj, noise_model=noise_model).result()
    fitter = GatesetTomographyFitter(result, circuits, gateset_basis)
    return fitter

def gate_set_tomography(gate, noise_model=None):
    basis = default_gateset_basis()
    basis.add_gate(gate)
    fitter = collect_tomography_data(shots=10000, noise_model=noise_model, gateset_basis=basis)
    result_gates = fitter.fit()
    result_gate = result_gates[gate.name]
    return Choi(result_gate)
