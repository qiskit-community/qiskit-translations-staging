Usakinishaji¶

Qiskit inategemea Python 3.6 au toleo lingine la baadaye. Hata hivyo, Python na Qiskit ni mifumo ya ikolojia inayozidi kubadilika, na hivyo wakati mwingine, toleo jipya kutoka kwa Python au Qiskit huweza kusababisha tatizo la utangamanifu wao.

Tunapendekeza kusakinisha Anaconda, programu ya Python ya kufanyia hesabu za kisayansi. Jupyter, iliyojumuishwa katika Anaconda, inapendekezwa kwa kushirikiana na Qiskit.

Qiskit imejaribiwa na kuungwa mkono na mifumo ya 64-bit:

  • Ubuntu 16.04 au baadaye

  • macOS 10.12.6 au baadaye

  • Windows 7 au baadaye

Tunapendekeza utumie mazingira halisi ya Python kutenganisha Qiskit kutoka kwa programu zingine na uboreshe uzoefu wako.

Njia rahisi zaidi ya kutumia mazingira ni kwa kutumia amri ya conda iliyojumuishwa na Anaconda. Mazingira ya Conda hukuruhusu kutaja toleo maalum la Python na seti ya maktaba. Fungua terminal kwenye folda unayotaka kufanya kazi.

Inapendekezwa kuwa utumie prompt ya Anaconda iliyosanikishwa na Anaconda. Unachohitajika kufanya ni kuunda mazingira halisi ndani ya Anaconda na kuamilisha mazingira. Amri hizi zinaweza kuendeshwa katika prompt ya Anaconda bila kujali iwapo ni mashine ya Windows au Linux.

Tengeneza mazingira ya kiwango cha chini na Python iwe imesakinishwa ndani yake.

conda create -n ENV_NAME python=3

Amilisha mazingira yako mapya.

conda activate ENV_NAME

Ifuatayo, sakinisha kifurushi cha Qiskit.

pip install qiskit

Iwapo vifurushi vilisakinishwa sahihi, unaweza kuendesha conda list kuona vifurushi vyenye kutumika katika mazingira yako.

Iwapo unakusudia kutumia utendaji wa taswira au Jupyter notebooks unashauriwa kusakinisha Qiskit pamoja na msaada wa ziada wa visualization:

pip install qiskit[visualization]

Inafaa kutaja kuwa ikiwa wewe ni mtumiaji wa zsh (ambayo ni shell ya kaida kwenye matoleo mapya ya macOS), utahitaji kuweka qiskit[visualization] katika nukuu:

pip install 'qiskit[visualization]'

Wauzaji wafuatao wa huduma za wingu wana Qiskit iliyosanikishwa mapema katika mazingira yao:

IBM Quantum Lab

Build quantum applications and experiments with Qiskit in a cloud programming environment.

Strangeworks

A platform that enables users and organizations to easily apply quantum computing to their most pressing problems and research.

Kusakinisha vipengee kutoka chanzo hukuruhusu kupata toleo lililosasishwa la hivi karibuni la Qiskit badala ya kutumia toleo lilio kwenye repo ya Python Package Index (PyPI). Hii itakupa uwezo wa kukagua na kupanua toleo la hivi karibuni la msimbo wa Qiskit kwa ufanisi zaidi.

Wakati unasakinisha vipengee na visehemu kutoka kwa chanzo, kwa chaguo-msingi toleo la maendeleo (ambayo inalingana na tawi la git la master) litatumika, kinyume na toleo la thabiti (ambalo lina codebase sawa kama vifurushi vilivyochapishwa vya pip). Kwa kuwa matoleo ya maendeleo ya kifurushi au sehemu kawaida hujumuisha vipengee vipya na mabadiliko, kwa jumla zinahitaji kutumia toleo la maendeleo la vitu vingine pia.

Vifurushi vya Terra na Aer vinahitaji compiler ili kujenga kutoka kwa chanzo kabla ya kusakinisha. Ignis, Aqua na IBM Quantum Provider backend havihitaji compiler.

Kusakinisha vipengee kutoka kwa chanzo kunahitaji utaratibu ufuatao wa usakinishaji kuzuia kusakinisha matoleo ya vipengee ambavyo vinaweza kuwa chini kuliko inavyotakikana ikiwa toleo la pip liko nyuma ya matoleo ya chanzo:

  1. qiskit-terra

  2. qiskit-aer

  3. qiskit-ignis

  4. qiskit-aqua

  5. qiskit-ibmq-provider (ikiwa unataka kuunganishwa na vifaa vya IBM Quantum au mtambo ya kuiga wa mtandaoni)

Kufanya kazi na vifaa kadhaa na vipengele kwa wakati mmoja, tumia hatua zifuatazo kwa kila kipengele.

Note

Kutokana na utumizi wa Python wa namespace packaging, tahadhari sharti ichukuliwe katika kusakinisha vifurushi. Iwapo unakusudia kusakinisha vipengele kutoka chanzo, usitumie kifurushi cha meta cha qiskit. Pia, fuata mwongozo huu na utumie mazingira tofauti ya maendeleo. Ukichagua kuchanganya sakinisho lililopo na maendeleo yako, rejelea https://github.com/pypa/sample-namespace-packages/blob/master/table.md kwa seti ya muunganisho wa njia za usakinishaji zinazofanya kazi pamoja.

Set up the Virtual Development Environment

conda create -y -n QiskitDevenv python=3
conda activate QiskitDevenv

Installing Terra from Source

Kusakinisha kutoka chanzo kunahitaji uwe na compiler ya C++ inayounga mkono C++11 kwenye tarakilishi yako.

Katika majukwaa mengi ya Linux, compiler inayohitajika ya GCC huwa tayari imewekwa.

Ikiwa unatumia macOS, unaweza kusakinisha compiler ya Clang kwa kusakinisha XCode. Angalia iwapo umesakinisha XCode na Clang kwa kufungua taminali na kutia yafuatayo.

clang --version

Sakinisha XCode na Clang kwa kutumia amri ifuatayo.

xcode-select --install

Katika Windows, ni rahisi kusakinisha compiler ya Visual C++ kutoka Build Tools for Visual Studio 2019. Badala yake unaweza kusakinisha Visual Studio toleo la 2015 au 2017, ukihakikisha kuchagua chaguo la kusakinisha compiler ya C++.

Punde compiler zikisakinishwa, u tayari kusakinisha Qiskit Terra.

  1. Tengeneza pacha wa repo ya Terra.

    git clone https://github.com/Qiskit/qiskit-terra.git

  2. Kutengeneza pacha wa repo huunda folda ya kienyeji inayoitwa qiskit-terra.

    cd qiskit-terra

  3. Sakinisha faili za maktaba zinazohitajika na Python kutoka folda yako ya "qiskit-terra".

    pip install cython

  4. Ikiwa unataka kufanya majaribio au ukaguzi wa msimbo wa programu, sakinisha mahitaji ya msanidi programu.

    pip install -r requirements-dev.txt

  5. Sakinisha qiskit-terra.

    pip install .

Ikiwa unataka kuisakinisha katika hali inayoweza kuhaririwa, kumaanisha kuwa mabadiliko ya misimbo ya mradi hayahitaji usakinishaji mpya utumike, unaweza fanya hivyo kutumia:

pip install -e .

Basi unaweza kutekeleza mifano ya misimbo baada ya kisakinisha Terra. Unaweza kutekeleza mifano kwa kutumia amri ifuatayo.

python examples/python/using_qiskit_terra_level_0.py

Note

Ikiwa hautakusudia kusakinisha vifaa vingine, qiskit-terra itatoa onyo la RuntimeWarning kwamba qiskit-aer na qiskit-ibmq-provider hazijasakinishwa. Hii imefanywa kwa sababu watumiaji kwa kawaida hukusudia kutumia vifaa hivyo vya ziada lakini hawatambui kuwa havijasakinishwa au usakinishaji wa aidha Aer au IBM Quantum Provider haukufaulu kwa sababu fulani. Ukitaka kuzuia maonyo haya, ongeza:

import warnings
warnings.filterwarnings('ignore', category=RuntimeWarning,
                        module='qiskit')

kabla ya agiza zozote za qiskit katika msimbo wako. Huku kutazuia onyo kuhusu ukosefu wa qiskit-aer au qiskit-ibmq-provider lakini itaendelea kuonyesha maonyo mengine kutoka kwa qiskit au vifurushi vingine.

Installing Aer from Source

  1. Tengeneza pacha wa repo ya Aer.

    git clone https://github.com/Qiskit/qiskit-aer

  2. Sakinisha mahitaji ya ujenzi.

    pip install cmake scikit-build cython

Baada ya haya, hatua za kusakinisha Aer hutegemea mfumo endeshi wa tarakilishi unayotumia. Kwa kuwa Aer ni programu iliyokusanywa ya C++ yenye kiolesura cha Python, kuna utegemezi usio wa Python wa kujenga jozi ya Aer ambayo haiwezi kusakinishwa ulimwenguni kulingana na mfumo endeshi wa tarakilishi.

Linux

  1. Sakinisha mahitaji ya compiler.

    Ujenzi wa Aer huhitaji compiler ya C++ na development headers.

    Ikiwa unatumia Fedora au usambazaji sawa wa Linux, sakinisha kwa kutumia:

    dnf install @development-tools

    Kwa Ubuntu/Debian sakinisha kwa kutumia:

    apt-get install build-essential

  2. Sakinisha OpenBLAS development headers.

    Ikiwa unatumia Fedora au usambazaji sawa wa Linux, sakinisha kwa kutumia:

    dnf install openblas-devel

    Kwa Ubuntu/Debian sakinisha kwa kutumia:

    apt-get install libopenblas-dev
macOS

  1. Sakinisha vitegemezi.

    Ili kutumia complier ya Clang katika macOS, unahitaji kusakinisha faili ya maktaba ya kuunga mkono OpenMP. Unaweza tumia brew kusakinisha vitegemezi vingine.

    brew install libomp

  2. Kisha sakinisha tekelezo la BLAS; OpenBLAS ndio chaguo msingi.

    brew install openblas

    Ifuatayo, sakinisha Xcode Command Line Tools.

    xcode-select --install
Windows

Katika Windows utahitaji kutumia Anaconda3 au Miniconda3 ili kusakinisha vitegemezi vyote.

  1. Sakinisha mahitaji ya compiler.

    conda install --update-deps vs2017_win-64 vs2017_win-32 msvc_runtime

Qiskit Aer ni mfumo wa utendaji wa juu wa mtambo ya kuiga kwa saketi za kwanta. Inatoa backends kadhaa kufikia malengo tofauti ya uigaji.

conda install --update-deps -c conda-forge -y openblas cmake

  1. Jenga na usakinishe qiskit-aer moja kwa moja

    Iwapo una pip <19.0.0 iliyosakinishwa na mazingira yako hayahitaji ujenzi maalum, endesha:

    cd qiskit-aer
pip install .

    Huku kutajenga jozi hizo na kusakinisha Aer.

    Vinginevyo, ikiwa una pip mpya iliyosanikishwa, au una mahitaji fulani maalum, unaweza kujenga Python wheel kwa mikono.

    cd qiskit-aer
python ./setup.py bdist_wheel

    Iwapo unahitaji chaguo la kibinafsi wakati wa kujenga wheel, rejelea Chaguo za kibinafsi wakati wa kujenga wheel.

    Baada ya kujenga wheel ya Python, itahifadhiwa katika folda ya dist/ katika repo ya Aer. Toleo halisi litategemea

    cd dist
pip install qiskit_aer-*.whl

    Jina halisi la faili ya wheel iliyozalishwa inategemea toleo la sasa la Aer lililo chini ya maendeleo.

Custom options

Mfumo wa kujenga Aer unatumia scikit-build kutekeleza mkusanyiko wakati wa kuijenga kutumia kiolesura cha Python. Inafanya kazi kama kiolesura cha setuptools kuagiza CMake na kukusanya jozi zitakazotumiwa na tarakilishi yako.

Kwa sababu ya ugumu wa kukusanya binaries, itakubidi upitishe chaguzi katika sehemu fulani ya mchakato wa ujenzi. Jinsi ya kupitisha vigezo ni:

python setup.py bdist_wheel [skbuild_opts] [-- [cmake_opts] [-- build_tool_opts]]

pale ambapo kipengele killicho ndani ya mabano ya mraba [] ni hiari na skbuild_opts, cmake_opts, build_tool_opts zitabadilishwa na vigezo vya chaguo lako. Orodha ya chaguzi za CMake inapatikana hapa: https://cmake.org/cmake/help/v3.6/manual/cmake.1.html#options. Kwa mfano unzaweza kuendesha kitu kama:

python setup.py bdist_wheel -- -- -j8

Huku ni kupitisha kigezo -j8 kwa mfumo wa msingi wa ujenzi (ambao sasa hivi ni Automake), ukiiambia kwamba unataka kujenga sambamba ukitumia michakato 8.

Kwa mfano, matumizi ya kawaida ya vigezo hivi katika Linux huwa ni kutaja toleo maalum la compiler ya C++ litakalotumiwa (hususan iwapo la chaguo-msingi ni la kale sana):

python setup.py bdist_wheel -- -DCMAKE_CXX_COMPILER=g++-7

itakayoambia CMake kutumia amri ya g++-7 badala ya chaguo-msingi la g++ wakati wa kukusanya Aer.

Matumizi mengine ya kawaida ya hii, kulingana na mazingira yako, ni kwamba unaweza kuhitaji kutaja jina la platform yako na kusitisha uunganisho tuli (static linking).

python setup.py bdist_wheel --plat-name macosx-10.9-x86_64 \
-- -DSTATIC_LINKING=False -- -j8

Hapa, --plat-name ni kigezo kwa setuptools, cha kutajia jina la platform litakalotumiwa katika kifurushi cha metadata, -DSTATIC_LINKING ni kigezo cha kutumia CMake kusitisha uunganisho tuli, na -j8 ni kigezo cha kutumia Automake kutumia michakato 8 katika mkusanyiko.

Orodha ya chaguzi za kawaida kulingana na platform ni:

Jukwaa

Chombo

Chaguo

Matumizi

Zote

Automake

-j

Ikifuatiwa na idadi, huweka idadi ya michakato ya kutumia katika mkusanyiko.

Linux

CMake

-DCMAKE_CXX_COMPILER

Inatumika kutaja compiler maalum ya C++; hii inahitajika sanasana ikiwa g++ ya chaguo-msingi imezeeka sana.

OSX

setuptools

--plat-name

Inatumika kutajia jina la platform katika kifurushi cha pato la Python.

OSX

CMake

-DSTATIC_LINKING

Inatumika kutajia iwapo uunganisho tuli utumike au la.

Note

Baadhi ya chaguzi hizi sio maalum kwa platform. Platform hizi zimeorodheshwa kwa kuwa zinatumiwa kwa kawaida katika mazingira haya. Rejelea nyaraka kwa habari zaidi.
