糾纏驗證¶

GHZ 狀態簡介¶

The Greenberger-Horne-Zeilinger (GHZ) State is a \(n\)-qubit entangled state best defined by the following state vector:

\[|{\rm GHZ} \rangle = \frac{|0 \rangle ^{\otimes n} + |1 \rangle ^{\otimes n}}{\sqrt{2}} , n > 2\]

GHZ 狀態的測量對評估多量子位元的互動很有幫助，它的穩健性是開發大規模量子計算的關鍵。

測量一個量子態¶

Any mixed quantum state can be identified by a density matrix, defined as \(\rho = \sum_{i} p_i |\psi_{i} \rangle \langle \psi_{i}|\), where \(|\psi_{i} \rangle\) are the pure quantum states forming the mixture and \(0 < p_i \le 1\), \(\sum_{i} p_i = 1\) are the classical probabilities to be in state \(|\psi_{i} \rangle\). We denote the pure density matrix of an ideal GHZ State by \(\rho_{p} \equiv |{\rm GHZ} \rangle \langle {\rm GHZ}|\). We want to see how close this matrix is to the density matrix of a GHZ State as produced in an experiment, \(\rho_{T}\). One method to quantify this similarity is to calculate the fidelity between the states, \(F(\rho_{p},\rho_{T})\)

\[F = \Big[ Tr \sqrt{\sqrt{\rho_{p}}\rho_{T}\sqrt{\rho_{p}}} \Big] ^{2}\]

本教學的目的有兩方面：我們將探索測量 GHZ 狀態的方式，以及可以利用 Ignis 的誤差緩解工具以增加讀出保真度的方式，而不論其測量方法。

準備一個 GHZ 狀態¶

讓我們先一起來看如何準備一個 GHZ 狀態：

假設我們有一個 \(n\) 量子位元的系統，這些 \(n\) 個量子位元都被準備在 \(|0\rangle\) 狀態中：

\[|\psi \rangle = |00...0\rangle\]

我們將 Hadamard 閘套用至第一個量子位元：\(|0\rangle \longrightarrow \frac{1}{\sqrt{2}}(|0\rangle + |1\rangle)\)。 我們現在的狀態看起來如下：

\[|\psi \rangle = \frac{|00...0\rangle + |10...0\rangle}{\sqrt{2}}\]

把一個在第 \(n^{th}\) 和第 \((n+1)^{th}\) 對應於 \(n = 0 \ldots n-1\) CNOT 閘的序列套用到此狀態上，如果第 \(n^{th}\) 是處於 \(|0\rangle\) 則使得第 \(n+1\) 個量子位元處於 \(|0\rangle\)，如果第 \(n^{th}\) 是處於 \(|1\rangle\) 則使得第 \(n+1\) 個量子位元處於 \(|1\rangle\)。如此我們則可以建立一個 GHZ 態。

\[|GHZ \rangle = \frac{|00...0\rangle + |11...1\rangle}{\sqrt{2}} \equiv \frac{|0 \rangle ^{\otimes n} + |1 \rangle ^{\otimes n}}{\sqrt{2}}\]

以下函數會建立可以測量 GHZ 狀態的這個 “線性” 電路：

測量，第一部分¶

多重量子相干（MQC）¶

多重量子相干(MQC) 通過以下方式來執行：首先準備 \(n\) 量子位元 GHZ 狀態，並把每個量子位狀態以 z 軸為軸旋轉相位 \(\phi\) 。 接著，我們套用一個 X 閘，也就是一個以 x 軸旋轉 ​:math:pi 的脈衝。然後，我們採取相反於原先用來獲得 GHZ 狀態的操作。理想情況下，其最終狀態是 \(|\psi \rangle = \frac{|0 \rangle ^{\otimes n} + e^{i n \phi}|1 \rangle ^{\otimes n}}{\sqrt{2}}\)。我們可以理想地觀察通過將 \(|\psi \rangle\) 投影到狀態 \(|0 \rangle ^{\otimes n}\) 所上獲得的相位。此方法類似迴聲序列，並且已被證明可以顯著改善讀取時的保真度。

以下函數會建立一個線性 MQC 電路。 與這裡的每一個電路一樣，您可以變更 full_measurement 選項，以在完整測量所有量子位元或僅測量控制量子位元之間切換。 完整的測量會產生最精確的結果，但如果是超過 7 個量子位元的情況，建議將它設為 false ，並只觀察 '0' 與 '1' 狀態之間的振盪。

在這個 MQC 電路上做完實驗後，可以選擇一個狀態來觀察當我們把角度 \(\phi\) 從 \(0\) 掃到 \(2 \pi\) 的振盪。理論上，我們的訊號應該遵循 \(S(\phi) = \frac{1}{2}(1+\cos(n \phi))\)。然後我們執行一個離散傅立葉變換(DFT: \(I_{v}=(1/N)|\sum_{\phi}e^{iv\phi}S(\phi)\)) ，以汲取該狀態被邊界條件所定義的保真度 \(2\sqrt{I_{n}} \leq F \leq \sqrt{I_{0}/2}+ \sqrt{I_{n}}\); 想要的話，我們也可以獲得一個保真度的實際值 \(F = \frac{1}{2}(P_{00...0}+P_{11...1})+\sqrt{I_{n}})\) (arXiv:1905.05720)。

宇稱振盪¶

用於測量 GHZ 狀態的下一種方法是宇稱振盪。 準備好一個 GHZ 狀態後，我們套用一個以 x 軸和 y 軸的旋轉組合以創建各種疊加狀態，該疊加狀態是一個函數，其自變數為 \(\phi\): \(U(\phi) = \otimes_{j}^{N} e^{i\frac{\pi}{4}(\cos(\phi)\sigma_{x}^{j}+\sin(\phi)\sigma_{y}^{j})}\)。我們接著測量期望值:math:langle otimes_{j}^{N} sigma_{z}^{j} rangle_{phi}，此期望值是一個函數，其自變數為 \(\phi\)，這在理論上應該會導致 1 和 -1 之間的宇稱振盪。

以下的函數會產生一個等效於上述 MQC 電路的宇稱振盪電路。

[8]:

circ_po = get_ghz_po_para(qn)

[9]:

circ_po[0].draw(output='mpl')

[9]:

從 \(F = \frac{1}{2}(P_{00...0}+P_{11...1}+C)\) 我們可以得到宇稱振盪的保真度 \(S_{\phi}\)，其中 \(C\)，遵循和在 MQC 方法中 DFT 的相同表達式。

斷層掃描¶

斷層掃描通過產生許多名義上相同的狀態並在不同基底中測量不同的狀態實例，以測量密度矩陣。 GHZ 狀態的理想結果是在張量積的四個角落的密度矩陣元素彼此相同，而所有其他元素消失不見。 雖然這種方法可以很容易地計算保真度，但此方法過於緩慢 (需要指數數量級的 n 次量測)，並且如果 n 大於 7 左右，則該方法需要長到難以實現的時間來運行。 不過，我們會在下面演示如何執行此方法，畢竟它在少量的量子位元情況下還是可行的。

並行化電路¶

上面的 “線性” 電路非常適合運行模擬，但當我們使用真實元件時，由於每個系統具有一個任意的拓撲 (toplogy) 並且會發生各種錯誤，我們該怎麼辦？我們的目標是真正的硬體，而不僅僅是 Aer 模擬。減少實際硬體影響的一種方法是通過並行化 CNOT 閘來建立較淺層的電路。這在效率和保真度方面都是非常有幫助的。parallelize 模組中的 BConfig 類別即可完成上述的任務。

首先，我們必須配置我們想要使用的最佳後台。 我們將考慮透過一個仿效後台``』FakeMelbourne』將一個模擬後台 ``'ibmq_qasm_simulator' 偽裝成一個真實元件``』ibmq_16_melbourne』``。

[10]:

from qiskit.providers.aer import QasmSimulator
from qiskit.test.mock import FakeMelbourne
backend = QasmSimulator()
backend_hardware = FakeMelbourne()

通過使用此 雜訊 模組，我們現在可以在 'ibmq_16_melbourne' 中定義一個雜訊模型並將之 “指派 (assign) ” 到 'ibmq_qasm_simulator'。

[11]:

noise_model = noise.NoiseModel.from_backend(backend_hardware)
coupling_map = backend_hardware._configuration.coupling_map
basis_gates=noise_model.basis_gates

And there we have it. From now on, in the tutorial, when using a real device, not a simulation, just take out every mention and assignment of noise_model and coupling_map, and assign the real device to backend. The simulator used from now on is no substitute for running a real device.

BConfig 為建立平行化電路勾勒其藍圖。 讓我們初始化一個物件(此物件會把我們剛才定義的真實元件作為輸入)，並將其命名為 protocirc。我們的所有實驗都將會使用它：

[12]:

protocirc = BConfig(backend_hardware)

誤差緩解¶

Qiskit Ignis 為輸入原始數據和輸出校準過的數據提供了非常準確的工具。 這是通過取得向量 \(v_{raw}\) 形式原始數據，以及取得一個校正矩陣 \(A\)。 則其輸出會是一個最佳化問題的解: \(argmin_{v_{cal}} ||Av_{cal}-v_{raw}||^{2}\)

現在我們建構 \(\otimes _{j}^{N} \sigma_z^{j}\):

[37]:

composite_sigma_z = sigma_z = np.array([[1, 0],[0, -1]])
for a in range(1,qn):
    composite_sigma_z = np.kron(composite_sigma_z,sigma_z)

Now we want to make sure that our list of counts is correctly ordered so that it coincides with the states of the \(\otimes _{j}^{N} \sigma_z^{j}\), so that calculating \(\langle \otimes _{j}^{N} \sigma_z^{j} \rangle\) will be as simple as taking the dot product of this ordered list with the diagonal of \(\otimes _{j}^{N} \sigma_z^{j}\). The composite_pauli_z_expvalue function does just that; it takes a circuit and appropriately orders the state vector counts. We can plot this dot product as a function of \(\phi\) to observe parity oscillations.

** 關於斷層掃描的補充說明 **

不要使用超過 5 個量子位元來執行量子斷層掃描

結論¶

結論是，我們看到在沒有誤差緩解的情況下， MQC 是測量 GHZ 狀態的優越方法。 然而在有誤差緩解的情況下，所有方法都可以達到了更高的保真度並且都接近相同值，至少對於只有少量的量子位元的情況是如此。 為了取得更準確的結果，除了使用真實的元件之外，將測量次數增加 4 到 8 倍是值得的。以保真度來衡量比較此筆記本中所使用的平行化電路與線性化電路也可能是值得的。
