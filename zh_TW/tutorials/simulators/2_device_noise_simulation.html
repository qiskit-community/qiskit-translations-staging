後端設備噪音模型模擬¶

簡介¶

此筆記本說明如何使用 Qiskit Aer``noise`` 模組自動生成 IBMQ 硬體裝置的基本噪聲模型，並使用此模型對 ``QuantumCircuits``進行噪音模擬，以研究在真實裝置上發生的錯誤的影響。

請注意，這些自動模型僅是在實際裝置上發生的實際錯誤的 *近似值 * ，因為它們必須從與閘上 *平均錯誤率相關的有限輸入參數建置。 關於真實裝置上的量子錯誤的研究是一個活躍的研究領域，我們討論了 Qiskit Aer 工具，用於在另一個筆記本中配置更詳細的雜訊模型。

[1]:

from qiskit import IBMQ, transpile
from qiskit import QuantumCircuit
from qiskit.providers.aer import AerSimulator
from qiskit.tools.visualization import plot_histogram

裝置後端雜訊模型¶

Qiskit Aer 元件雜訊模型會自動為真實元件生成簡化的雜訊模型。 此模型是使用在元件的 BackendProperties 中報告的校正資訊所產生的，且此模型考慮到

  • 每一個量子位元上的每個基底閘的*gatew_error* 機率。

  • 每一個量子位元上每一個基底閘的 gate_length 。

  • 各量子位元的 \(T_1\)，\(T_2\) 弛緩時間常數。

  • 每一個量子位元的讀出誤差機率。

Terra 模擬後台¶

我們會使用 Qiskit Terra 裡儲存的 IBM 量子裝置之真實雜訊數據，而此教程中我們用的是 ibmq_vigo 這個元件。

[2]:

from qiskit.test.mock import FakeVigo
device_backend = FakeVigo()

用於元件和模擬比較的測試電路¶

現在，我們建構一個測試電路來比較真實器件的輸出與 Qiskit Aer ``AerSimulator``上模擬出的具有雜訊的輸出。我們將在量子位元 0、 1 和 2上製備 3量子位元 GHZ 狀態 \(\frac{1}{\sqrt2}(|0,0,0\rangle + |1,1,1\rangle)\) 。在執行有雜訊的模擬或在真實裝置上運行之前，我們先顯示出沒有雜訊的理想預期輸出。

[3]:

# Construct quantum circuit
circ = QuantumCircuit(3, 3)
circ.h(0)
circ.cx(0, 1)
circ.cx(1, 2)
circ.measure([0, 1, 2], [0, 1, 2])

sim_ideal = AerSimulator()

# Execute and get counts
result = sim_ideal.run(transpile(circ, sim_ideal)).result()
counts = result.get_counts(0)
plot_histogram(counts, title='Ideal counts for 3-qubit GHZ state')

[3]:

產生模擬真實裝置的模擬器¶

我們呼叫 from_backend ，以建立 ` ` ibmq_vigo``的模擬器：

[4]:

sim_vigo = AerSimulator.from_backend(device_backend)

通過在 vigo_simulator 中儲存元件的特性，我們可以確保在編譯用於模擬的電路時，我們使用的基底閘和耦合映射是適當的，從而最仿真地模擬將在真實元件上執行的閘。此外 vigo_simulator 包含一個近似雜訊模型，此模型由以下三者構成：

  • **單量子位元閘誤差 ** 由單量子位元的去極化誤差和接續的單量子位元的熱弛緩誤差所組成。

  • 雙量子位元閘誤差 由一個雙量子位元去極化誤差，和接續發生在兩個量子位元上的閘中單量子位元熱弛緩誤差組成。

  • 個別量子位元測量所獲得的古典位元數值的**單量子位元讀出誤差**。

對於閘的誤差，通過使用來自 aer.noise.errors 模組中的 thermal_relaxation_error 函數以及其個別量子位元的 \(T_1\) and \(T_2\) 參數，以及來自元件後台特性的 gate_time 參數，我們可以推導出該熱弛緩的誤差參數。接著，我們可以設定去極化誤差的機率，使得來自去極化誤差和接續的熱弛緩的加總平均閘不保真度(combined average gate infidelity) 等同於來自其後台特性的 gate_error 值。

至於讀出誤差，量子位元的``readout_errors`` 會告訴我們測量之後得到的古典位元值，有多大的機率會翻轉成和真實結果相反。

執行雜訊模擬¶

一旦我們基於真實的元件建立了具有雜訊的模擬器後台，我們就可以使用它來執行具有雜訊的模擬。

重要: 執行具有雜訊的模擬時，幫後台 transpile 電路*非常重要*。電路必須被轉譯為該後台可以執行的基底閘的集合。

[5]:

# Transpile the circuit for the noisy basis gates
tcirc = transpile(circ, sim_vigo)

# Execute noisy simulation and get counts
result_noise = sim_vigo.run(tcirc).result()
counts_noise = result_noise.get_counts(0)
plot_histogram(counts_noise,
               title="Counts for 3-qubit GHZ state with device noise model")

[5]:

如果我們跳過了轉譯這個步驟，來自元件雜訊模型的雜訊將不會被套用於在此電路中的閘。此電路被該模擬器支援，但並不被仿真後台支援。
