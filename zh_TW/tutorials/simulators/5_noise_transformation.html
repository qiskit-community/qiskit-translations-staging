雜訊轉換¶

簡介¶

此筆記本會顯示如何使用 Qiskit Aer 的效能函數: approximate_quantum_error 和 approximate_noise_model 將量子雜訊通道轉換成不同的、更適合的雜訊通道。

我們的指導範例是 Clifford 模擬器。Clifford 模擬器可以有效地模擬量子計算，其中包括僅限於有限的、非通用的閘集合(Clifford 閘)中的量子計算。 並非所有的量子雜訊都可以加入這樣的模擬中；因此，我們的目標是找到一個可以在 Clifford 模擬器中模擬的「封閉」雜訊通道。

我們首先從 Aer 提供者公用程式導入轉換功能

[1]:

from qiskit.providers.aer.utils import approximate_quantum_error
from qiskit.providers.aer.utils import approximate_noise_model

「近似」 表示，這些函式會產生最接近（在the Hilbert-Schmidt公制度量中）於真實情況的誤差。

我們示範使用 Qiskit 中定義的數個標準誤差通道的近似值。

[2]:

import numpy as np

# Import Aer QuantumError functions that will be used
from qiskit.providers.aer.noise import amplitude_damping_error
from qiskit.providers.aer.noise import reset_error
from qiskit.providers.aer.noise import pauli_error

概述¶

1-量子位元通道:math:mathcal{C}:mathbb{C}^{2times2}tomathbb{C}^{2times2}`是對映密度算子的功能(確認影像是密度算子:math:mathcal{C}` is required to be completely positive and trace preserving, CPTP).

給定量子通道:math:mathcal{E}_{1},dots,mathcal{E}_{r}，還有機率:math:p_1, p_2, dots, p_r，滿足:math:0le p_i le 1 和 \(p_1+\dots +p_r = 1\)，而新的量子通道:math:mathcal{C}_mathcal{E}`可以基於:math:mathcal{C}_mathcal{E}(rho)`可以有:math:p_i`的機率選:math:mathcal{E}_i`通道，並且運用:math:`rho`的情況下建立。

噪音轉換方程式可以優化下述問題: 給定一個頻道:math:mathcal{C} （目標）還有一個頻道的列表:math:mathcal{E}_{1},dots,mathcal{E}_{r} ，根據某距離函數:math:`D`（在這裡我們使用Hilbert-Schmidt函數），找出機率:math:p_1, p_2, dots, p_r 可以最小化:math:D(mathcal{C}, mathcal{C}_mathcal{E})

為了確保近似值是真的，在意義上，近似誤差通道作為實際誤差通道的 「上界 」 ，我們增加額外的真實限制：

\[\text{F}(I,\mathcal{C})\ge F(I,\mathcal{C}_\mathcal{E})\]

:math:`text{F}`是忠實地測量而 :math:`I`是身分通道

範例：具有重設雜訊的近似振幅阻尼雜訊。¶

限幅阻尼 噪音由一個單一參數 \(0\le \le \le 1\) 說明，並由 Kraus 運算子提供：

\[\begin{split}\left(\begin{array}{cc} 1 & 0\\ 0 & \sqrt{1-\gamma} \end{array}\right),\left(\begin{array}{cc} 0 & \sqrt{\gamma}\\ 0 & 0 \end{array}\right)\end{split}\]

Reset 誤差的說明是由機率 \(0\le p， q\le 1\) ，例如： \(p+q\le 1\) ，並由 Kraus 算子提供：

\[\begin{split}\left(\begin{array}{cc} \sqrt{p} & 0\\ 0 & 0 \end{array}\right),\left(\begin{array}{cc} 0 & \sqrt{p}\\ 0 & 0 \end{array}\right),\left(\begin{array}{cc} 0 & 0\\ \sqrt{q} & 0 \end{array}\right),\left(\begin{array}{cc} 0 & 0\\ 0 & \sqrt{q} \end{array}\right)\end{split}\]

這可以被想成“重置”先前被影響的量子到:math:left|0rightrangle ，並有:math:p, to \(\left|1\right\rangle的機率，擁有機率:math:\)，這個動作不影響到機率:math:1-(p+q)

It is not too difficult to determine analytically the best values of \(p,q\) to approximate a \(\gamma\) amplitude damping channel, see the details here. The best approximation is:

\[p=\frac{1}{2}\left(1+\gamma-\sqrt{1-\gamma}\right), q=0\]
[3]:

gamma = 0.23
error = amplitude_damping_error(gamma)
results = approximate_quantum_error(error, operator_string="reset")

We only needed the above code to perform the actual approximation.

[4]:

print(results)

p = (1 + gamma - np.sqrt(1 - gamma)) / 2
q = 0

print("")
print("Expected results:")
print("P(0) = {}".format(1-(p+q)))
print("P(1) = {}".format(p))
print("P(2) = {}".format(q))

  P(0) = 0.8237482193044614, QasmQobjInstructions = [[{'name': 'id', 'qubits': [0]}]
  P(1) = 0.17625178069553862, QasmQobjInstructions = [[{'name': 'reset', 'qubits': [0]}]

Expected results:
P(0) = 0.8237482193696062

我們得到了解析解的預期結果。

不同的輸入類型¶

近似函數有兩個輸入： 該要被近似的誤差通道，以及一組可以用來建構近似的誤差通道。

The error channel to approximate can be given as any input that can be converted to the QuantumError object.

舉個例子，我們明確建構了振幅衰減的 Kraus 矩陣，並與之前的相同近似函式傳送：

[5]:

gamma = 0.23
K0 = np.array([[1,0],[0,np.sqrt(1-gamma)]])
K1 = np.array([[0,np.sqrt(gamma)],[0,0]])
results = approximate_quantum_error((K0, K1), operator_string="reset")
print(results)

QuantumError on 1 qubits. Noise circuits:
  P(0) = 0.8237482193044617, QasmQobjInstructions = [[{'name': 'id', 'qubits': [0]}]
  P(1) = 0.1762517806955383, QasmQobjInstructions = [[{'name': 'reset', 'qubits': [0]}]

用來建構近似通道的 誤差運算子 ，可定為清單、字典或指出硬編碼通道的字串。

Any channel can be either a list of Kraus operators, or ‘QuantumError’ objects.

身分通道不需要直接被傳遞；雖然沒有明確指出，但是預設上身分通道都有被使用。

舉個例子，為了重設的雜訊，我們通過使用明確的 Kraus 表示法來近似振幅阻尼。

[6]:

reset_to_0 = [np.array([[1,0],[0,0]]), np.array([[0,1],[0,0]])]
reset_to_1 = [np.array([[0,0],[1,0]]), np.array([[0,0],[0,1]])]
reset_kraus = (reset_to_0, reset_to_1)

gamma = 0.23
error = amplitude_damping_error(gamma)
results = approximate_quantum_error(error, operator_list=reset_kraus)
print(results)

QuantumError on 1 qubits. Noise circuits:
  P(0) = 0.8237482193044614, QasmQobjInstructions = [[{'name': 'id', 'qubits': [0]}]
  P(1) = 0.17625178069553862, QasmQobjInstructions = [[{'name': 'kraus', 'qubits': [0], 'params': [array([[1, 0],
       [0, 0]]), array([[0, 1],
       [0, 0]])]}]

Note the difference in the output channel: The probabilities are the same, but the input Kraus operators were converted to general Kraus channels, which cannot be used in a Clifford simulator. Hence, it is always better to pass a QuantumError object instead of the Kraus matrices, when possible.
