Conector Torch e QNNs híbridas¶

Este tutorial introduz a classe TorchConnector do Qiskit, e demonstra como o TorchConnector permite permite uma integração natural de qualquer NeuralNetwork do Qiskit Machine Learning em um fluxo de trabalho PyTorch. O TorchConnector recebe um Qiskit NeuralNetwork e o torna disponível como um Módulo PyTorch. O módulo resultante pode ser perfeitamente incorporado na arquitetura clássica do PyTorch e treinado em conjunto sem considerações adicionais, permitindo o desenvolvimento e o teste de relação híbrido quântico-classico na arquitetura de machine learning.

Conteúdo:¶

Parte 1: Classificação e Regressão Simples

A primeira parte deste tutorial mostra como redes neurais podem ser treinadas usando a ferramenta de diferenciação automática do PyTorch (torch.autograd, link) para tarefas simples de classificação e regressão.

Parte 2: Classificação MNIST, QNNs Híbrido

A segunda parte deste tutorial ilustra como embutir uma NeuralNetwork (Quântico) em um fluxo de trabalho alvo PyTorch (Neste caso, uma arquitetura típica CNN) para classificar dados MNIST de uma maneira híbrida quântica-clássica.

Otimizador¶

A escolha do otimizador para treinar qualquer modelo de machine learning pode ser crucial em determinar o sucesso do nosso resultado de treinamento. Quando usamos o TorchConnector, nós ganhamos acesso a todos os algoritmos otimizadores definidos no pacote [torch.optim (link). Alguns dos algoritmos mais famosos usados em populares arquiteturas de machine learning, incluindo Adam, SGD, ou Adagrad. Contudo, para este tutorial vamos usar o algoritmo L-BFGS (torch.optim.LBFGS), um dos mais bem conhecidos algoritmo otimizador de segunda-ordem para otimização numérica.

Funçao de Perda¶

Quanto a função de perda, também podemos tirar proveito dos módulos predefinidos do PyTorch do torch.nn, como o Cross-Entropy ou Mean Squared Error.

💡 Esclarecimento : Em um machine learning clássico, a regra geral é aplicar uma perda de entropia cruzada para classificar tarefas, e perda de MSE para regressão de tarefas. Contudo, esta recomendação é dada assumindo que a saída da rede de classificação é um valor de probabilidade de classe no intervalo de [0,1] (Geralmente isto se consegue por meio de uma camada Softmax). Porque o exemplo a seguir para TwoLayerQNN não inclui essa camada, e nós não aplicamos nenhum mapeamento para saída (A sessão a seguir apresenta um exemplo de aplicação de mapeamento de paridade com CircuitQNNs), as saídas QNNs podem tomar qualquer valor no intervalo [-1,1]. Caso você esteja se perguntando, está é a razão pela qual este exemplo em particular usa MSELoss para classificação apesar de não ser uma norma (Mas nós encorajamos você a experimentar com diferentes funções de perda e ver como elas podem impactar nos resultados do treino).

[5]:

# Define optimizer and loss
optimizer = LBFGS(model1.parameters())
f_loss = MSELoss(reduction='sum')

# Start training
model1.train()   # set model to training mode


# Note from (https://pytorch.org/docs/stable/optim.html):
# Some optimization algorithms such as LBFGS need to
# reevaluate the function multiple times, so you have to
# pass in a closure that allows them to recompute your model.
# The closure should clear the gradients, compute the loss,
# and return it.
def closure():
    optimizer.zero_grad()          # Initialize/clear gradients
    loss = f_loss(model1(X_), y_)  # Evaluate loss function
    loss.backward()                # Backward pass
    print(loss.item())             # Print loss
    return loss

# Run optimizer step4
optimizer.step(closure)

B. Classificação com PyTorch e CircuitQNN¶

Ligar o CircuitQNN ao PyTorch requer um pouco mais atenção que o OpflowQNN. Sem as configurações corretas, retropropagação não é possível.

Em particular, devemos ter certeza de que estamos retornando uma matriz densa de probabilidades no passe de encaminhamento de rede (sparse=False). Este parâmetro é configurado como False por padrão, então nós apenas precisamos ter certeza de que ele não foi alterado.

⚠️ Atenção: Se nós definirmos uma função de interpretação personalizada (No exemplo: paridade), nós devemos lembrar de fornecer explicitamente a forma de saída desejada (No exemplo: 2). Para mais informações sobre a configuração dos parâmetros iniciais para CircuitQNN, por favor vá até Documentação oficial do Qiskit.

[7]:

# Define feature map and ansatz
feature_map = ZZFeatureMap(num_inputs)
ansatz = RealAmplitudes(num_inputs, entanglement='linear', reps=1)

# Define quantum circuit of num_qubits = input dim
# Append feature map and ansatz
qc = QuantumCircuit(num_inputs)
qc.append(feature_map, range(num_inputs))
qc.append(ansatz, range(num_inputs))


# Define CircuitQNN and initial setup

# Define torch NN module

class Net(Module):

    def __init__(self):
        super().__init__()
        self.conv1 = Conv2d(1, 2, kernel_size=5)
        self.conv2 = Conv2d(2, 16, kernel_size=5)
        self.dropout = Dropout2d()
        self.fc1 = Linear(256, 64)
        self.fc2 = Linear(64, 2)         # 2-dimensional input to QNN
        self.qnn = TorchConnector(qnn4)  # Apply torch connector, weights chosen
                                         # uniformly at random from interval [-1,1].
        self.fc3 = Linear(1, 1)          # 1-dimensional output from QNN

    def forward(self, x):
        x = F.relu(self.conv1(x))
        x = F.max_pool2d(x, 2)
        x = F.relu(self.conv2(x))
        x = F.max_pool2d(x, 2)
        x = self.dropout(x)
        x = x.view(x.shape[0], -1)
        x = F.relu(self.fc1(x))
        x = self.fc2(x)
        x = self.qnn(x)  # apply QNN
        x = self.fc3(x)
        return cat((x, 1 - x), -1)

model4 = Net()

Passo 3: Treinamento¶

[20]:

# Define model, optimizer, and loss function
optimizer = optim.Adam(model4.parameters(), lr=0.001)
loss_func = NLLLoss()

# Start training
epochs = 10     # Set number of epochs
loss_list = []  # Store loss history
model4.train()  # Set model to training mode

for epoch in range(epochs):
    total_loss = []
    for batch_idx, (data, target) in enumerate(train_loader):
        optimizer.zero_grad(set_to_none=True)  # Initialize gradient
        output = model4(data)                  # Forward pass
        loss = loss_func(output, target)       # Calculate loss
        loss.backward()                        # Backward pass
        optimizer.step()                       # Optimize weights
        total_loss.append(loss.item())         # Store loss
    loss_list.append(sum(total_loss)/len(total_loss))
    print('Training [{:.0f}%]\tLoss: {:.4f}'.format(
        100. * (epoch + 1) / epochs, loss_list[-1]))
