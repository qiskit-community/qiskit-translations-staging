ക്വിസ്കിറ്റ്നെ പരുജയപെടാം¶

ക്വിസ്കിറ്റ് വ​ർക്ക്ഫ്ലൊയിൽ പ്രധാനമായി താഴെ പറയുന്ന നാല് ഘട്ടങൾ ആണ് ഉള്ളത്:

  • Build: അനുയോജ്യമായ ക്വാണ്ടം സർക്യൂട്ട് (കൾ) രൂപകൽപ്പന ചെയ്യുക.

  • Compile: ക്വാണ്ടം സിസ്റ്റങ്ങൾ അല്ലെങ്കിൽ ക്ലാസിക്കൽ സിമുലേറ്റർ പോലുള്ള ക്വാണ്ടം സേവനങ്ങളിൽ നിന്ന് അനുയോജ്യമായത് ഉപയോഗിച്ച് സർക്യൂട്ട് കംപൈൽ ചെയ്യുക.

  • Run: കംപൈൽ ചെയ്ത​ സർക്യൂട്ടുകൾ നിർദ്ദിഷ്ട ക്വാണ്ടം സേവനത്തിൽ (കൾ) പ്രവർത്തിപ്പിക്കുക. ഈ സേവനങ്ങൾ ക്ലൗഡ് അധിഷ്ഠിതമോ പ്രാദേശികമോ ആകാം.

  • Analyze: പരീക്ഷണങ്ങളുടെ ഫലങ്ങൾ ദൃശ്യവൽക്കരിക്കുകയും അവയി​​ൽ നിന്ന സ്റ്റാറ്റിസ്റ്റിക്ക​ൽ നിഘമനങൾ പുറപെടുവികുഖയും ചെയ്യുക.

ഒരു ഉദാഹരണം താഴെ കൊടുത്തിരിക്കുന്നു. ഉദാഹരണത്തിലെ ഓരോ ഘട്ടവും തുടർന്നുള്ള വിഭാഗങ്ങളിൽ വിശദമായി വിവരിക്കുന്നു:

import numpy as np
from qiskit import QuantumCircuit, transpile
from qiskit.providers.aer import QasmSimulator
from qiskit.visualization import plot_histogram

# Use Aer's qasm_simulator
simulator = QasmSimulator()

# Create a Quantum Circuit acting on the q register
circuit = QuantumCircuit(2, 2)

# Add a H gate on qubit 0
circuit.h(0)

# Add a CX (CNOT) gate on control qubit 0 and target qubit 1
circuit.cx(0, 1)

# Map the quantum measurement to the classical bits
circuit.measure([0,1], [0,1])

# compile the circuit down to low-level QASM instructions
# supported by the backend (not needed for simple circuits)
compiled_circuit = transpile(circuit, simulator)

# Execute the circuit on the qasm simulator
job = simulator.run(compiled_circuit, shots=1000)

# Grab results from the job
result = job.result()

# Returns counts
counts = result.get_counts(compiled_circuit)
print("\nTotal count for 00 and 11 are:",counts)

# Draw the circuit
circuit.draw()


Total count for 00 and 11 are: {'00': 493, '11': 507}

q_0: ┤ H ├──■──┤M├───
     └───┘┌─┴─┐└╥┘┌─┐
q_1: ─────┤ X ├─╫─┤M├

c: 2/═══════════╩══╩═
                0  1 
# Plot a histogram
plot_histogram(counts)

ഉദാഹരണത്തിന്റെ വിശദീകരണം¶

മുകളിലുള്ള ഉദാഹരണം ആറ് ഘട്ടങ്ങളായി തിരിക്കാം:

  1. ലൈബ്രറികൾ/പാക്കേജുകൾ ഇമ്ബൊർട്ട് ചെയ്യൽ

  2. വേരിയബിൾസ് അസ്യിൻ ചെയ്യൽ

  3. ഗേറ്റുകൾ ചേർക്കുന്നു

  4. സർക്യൂട്ട് പ്രിന്റ ചെയുന്നു

  5. Simulate the experiment

  6. ഗ്രാഫ്/ചാർട്ട് പ്രിന്റ ചെയുന്നു

Step 1 : Import Packages¶

നിങ്ങളുടെ പ്രോഗ്രാമിന് ആവശ്യമായ അടിസ്ഥാന ലൈബ്രറികൾ/പാക്കേജുകൾ ചുവടെ കൊടുത്തിരിക്കുന്നതുപോലെ ഇമ്ബൊർട്ട് ചെയാം:

import numpy as np
from qiskit import QuantumCircuit
from qiskit.providers.aer import QasmSimulator
from qiskit.visualization import plot_histogram

പാക്കേജുകൾ-കൂടുതൽ വിശദാംശങ്ങൾ

  • QuantumCircuit: ക്വാണ്ടം സിസ്റ്റത്തിന് ആവശ്യമായ നിർദ്ദേശങ്ങൾ അടങ്ങിയ പാക്കേജ്. നിങ്ങൾക്ക് ആവശ്യമായ എല്ലാ ക്വാണ്ടം ഓപ്രേഷന്സും ഇതിൽ അടങ്ങിയിരിക്കുന്നു.

  • QasmSimulator: is the Aer high performance circuit simulator.

  • plot_histogram: ഹിസ്റ്റോഗ്രാം ഉപയോഗിക്കുന്നതിന് ആവശ്യമായ പാക്കേജ്.

Step 2 : Initialize Variables¶

ഉദാഹരണത്തിന്

circuit = QuantumCircuit(2, 2)

ഇവിടെ, നിങ്ങൾ 0 സ്റ്റേയ്റ്റ്ലുള്ള​ 2 ക്വിബിറ്റും 2 ക്ലാസിക്കൽ ബിറ്റും അടങ്ങിയിരിക്കുന്ന ഒരു ക്വാണ്ടം സർക്യൂട്ട് circuit എന്ന പേരിൽ ഇനിഷ്യലയ്സ് ചെയ്യുന്നു.

സിന്റാക്സ്:

  • QuantumCircuit(int, int)

Step 3 : Add Gates¶

സർക്യൂട്ടിൽ ഗേറ്റുകൾ(ഓപ്രേഷന്സ്) ചേർകുനതിലൂടേ രജിസ്റ്ററുകൾ(ക്വിബിറ്റുകൾ) മാനിപുലേറ്റ് ചെയ്യാൻ കഴിയും.

ഉദാഹരണത്തിന്:

circuit.h(0)
circuit.cx(0, 1)
circuit.measure([0,1], [0,1])

The gates are added to the circuit one-by-one to form the Bell state

\[|\psi\rangle = \left(|00\rangle+|11\rangle\right)/\sqrt{2}.\]

മുകളിലുള്ള കോഡ് ഇനിപ്പറയുന്ന ഗേറ്റുകൾ അപ്ലയ് ചെയ്യുന്നു:

  • QuantumCircuit.h(0): ക്വിബിറ്റ് 0 ൽ ഹഡാമാർഡ് ഗേറ്റ് \(H\) അപ്ലയ് ചെയുബൊൾ, ക്വിബിറ്റ് **സൂപ്പർ പോസിഷൻ സ്റ്റേറ്റ്ല് ** ആവുന്നു.

  • QuantumCircuit.cx(0, 1):ക്വിബിറ്റ് 0(കോൺട്രോൾ) ക്വിബിറ്റ് 1(ടാർഗെറ്റ്) -മേൽ കോൺട്രോൾഡ്-നോട്ട് ഓപ്രേഷൻ(\(CNOT\)) ചെയുമ്ബൊൾ അവ എന്ടാങ്ൾ്ഡ് സ്ടേയ്റ്റിൽ ആവുന്നു. അതായത്, കോൺട്രോൾ ക്വിബിറ്റിന്റെ (ക്വിബിറ്റ് 0) മൂല്യം മാറുമ്പോൾ ടാർഗെറ്റ് ക്വിബിറ്റിന്റെ (ക്വിബിറ്റ് 1) മൂല്യം അതനുസരിച്ച് മാറുന്നു.

  • QuantumCircuit.measure([0,1], [0,1]): if you pass the entire quantum and classical registers to measure, the ith qubit’s measurement result will be stored in the ith classical bit.

Step 4 : Visualize the Circuit¶

You can use qiskit.circuit.QuantumCircuit.draw() to view the circuit that you have designed in the various forms used in many textbooks and research articles.

circuit.draw()

     └───┘┌─┴─┐└╥┘┌─┐
q_1: ─────┤ X ├─╫─┤M├
          └───┘ ║ └╥┘
c: 2/═══════════╩══╩═
                0  1

മുകളിലുള്ള സർക്യൂട്ടിൽ, ആദ്യം ക്വിബിറ്റും 0-യും തുടർന്ന് ക്വിബിറ്റും 1-യും പ്രിന്റ് ചെയ്തിരിക്കുന്നു. സർക്യൂട്ട് വായികെഡത്ത് ഇടത്തുനിന്ന് വലത്തോട്ടാണ്. അതാത്ത ആദ്യം ഉപയോഗിച്ച ഗേറ്റ് ആദ്യവും പിന്നീട് ഉപയോഗിച്ച ഗേറ്റ് പിന്നീടും പ്രിന്റ് ചെയ്തിരിക്കുന്നു.

QuantumCircuit.draw()-ലും``qiskit.visualization.circuit_drawer()``-ലും ഉപയോഗിക്കുന്ന ഡിഫോൾട്ട് ബാക്കെൻഡ് - ടെക്സ്റ്റ് ബാക്കെൻഡ് ആണ്.എന്നിരുന്നാലും, ലോക്കൽ എവിവർമെന്റിൽ കോഡ് ചെയ്യുമ്പോൾ അൽഗോരിതത്തിനു അനുയോജ്യമായ ബാക്കെൻഡ് തിരഞ്ഞെടുക്കാവുന്നതാണ്. ~/.qiskit/settings.conf` എന്ന ഫയിലിൽ ഉള്ള user config എന്ന ``.ini`ഫയ​ൽ മുഖാന്തരം ഇത ചെയാവുനതാണ്.

ഉദാ: Matplotlib എന്ന ബാക്കെൻഡ് സെലെക്റ്റ് ചെയുനതിന്നുള്ള​ ``settings.conf`ഫയ​ൽ താഴെ കൊടുത്തിരിക്കുന്നു.

[default]
circuit_drawer = mpl

You can use any of the valid circuit drawer backends as the value for this config, this includes text, mpl, latex, and latex_source.

സ്റ്റെപ്പ് 5: സ്ർക്യൂട്ട് സ്റ്റിമുലേറ്റ് ചെയ്യൽ¶

ക്വിസ്കിറ്റ് എഇർ - ക്വാണ്ടം സർക്യൂട്ടിനായി ഉപയോഗിക്കുന്ന ഹയ് പെർഫോർമന്സ് സിമുലേറ്റർ ഫ്രെയിം വർക്ക് ആണ്. ഇതിൽ സിമുലേഷനുകൾക്ക് അനുയോജ്യമായ പല​ ബാക്കെൻഡുകളും അടങ്ങിയിരിക്കുന്നു.

If you have issues installing Aer, you can alternatively use the Basic Aer provider by replacing Aer with BasicAer. Basic Aer is included in Qiskit Terra.

import numpy as np
from qiskit import QuantumCircuit, transpile
from qiskit.providers.basicaer import QasmSimulatorPy
...

മുകളിലുള്ള സർക്യൂട്ട് qasm സിമുലേറ്റർ ഉപയോഗിച്ചാണ് സിമുലേറ്റ് ചെയ്തിരികുനത്. ഈ സർക്യൂട്ട് ഓരൊ തവണ എക്സിക്യൂട്ട് ചെയുബോളൂം 00 അഥവാ 11 എന്ന ഉത്തരം ലഭികുനതാണ്.

simulator = QasmSimulator()
compiled_circuit = transpile(circuit, simulator)
job = simulator.run(compiled_circuit, shots=1000)
result = job.result()
counts = result.get_counts(circuit)
print("\nTotal count for 00 and 11 are:",counts)


Total count for 00 and 11 are: {'00': 502, '11': 498}

പ്രതീക്ഷിച്ച പോലെ, ഏതാണ്ട് പകുതിയോളം ഉത്തരം 00 ആണ്. എത്ര ഉത്തരങ്ങൾ ലഭിക്കണം എന്ന് തീരുമാനിക്കുന്നത്ത് shots ആണ്. ഇവിടെ ഷോട്സിന്റെ വാല്യു 1000 ആണ്(ഡിഫോൾട്ട് വാല്യു 1024 ആണ് ).

result-ന്റെ കൗണ്ട് കിട്ടുനതിനായി get_counts(circuit)` ഉപയോഗിക്കാം. ഇങനെ ചെയ്യ്ത് കൊണ്ട് നിങ്ങളുടെ കോഡ്ന്റെ മൊത്തം ഫലങ്ങൾ കരസ്ത്മാകാം.

Step 6 : Visualize the Results¶

ക്വിസ്കിറ്റിൽ റിസൽട്ട് നിരവധി 'ദൃശ്യവൽക്കരിക്കാൻ <apidoc/visualization.html>__ നിരവധി രീതികളുണ്ട്

റിസൽട്ട് കാണുനതിനയി plot_histogram ഏഡ് ചെയുക​.

plot_histogram(counts)

മൊത്തം ഷോട്ടുകളുടെ എണ്ണത്തെ ഓരൊ ഉതരതിന്റെയും എണ്ണം കൊണ്ട് ഹരിച്ചാണ് പ്രോബബിലിറ്റികൾ \(Pr(00)\) and \(Pr(11)\) കണക്കാക്കുന്നത്.

Note

run()-ലെ shots-ന്റെ മൂല്യത്തിനു മാറ്റം വരുത്തി അതിനു അനുസരിചു പ്രോബബിലിറ്റിയിൽ വരുന്ന മാറ്റം നോട്ട ചെയുക്ക​.

അടുത്തത് എന്ത്?¶

അടിസ്ഥാന പാഠങ്ങൾ പൂർത്തിയായി, മുന്നോട്ടു നീങ്ങാൻ താഴെ കൊടുത്തിരിക്കുന്ന ലിങ്കുകൾ സന്ദർശിക്കുക:

  • ക്വിസ്കിറ്റ് ട്യൂട്ടോറിയലുകൾ

  • `പാഠപുസ്തകം:ക്വിസ്കിറ്റ് ഉപയോഗിച്ച് ക്വാണ്ടം കമ്പ്യൂട്ടിംഗ് പഠിക്കാം<https://qiskit.org/textbook>`_

  • വീഡിയോ പരമ്പര:ക്വിസ്കിറ്റ് കോഡിംഗ്
