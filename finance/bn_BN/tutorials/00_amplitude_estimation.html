কোয়ান্টাম বিস্তার অনুমান¶

প্রদত্ত \(\mathcal{A}\) অপারেটরটি যেভাবে কাজ করে,

\[\mathcal{A}|0\rangle = \sqrt{1 - a}|\Psi_0\rangle + \sqrt{a}|\Psi_1\rangle\]

কোয়ান্টাম বিস্তার অনুমান (QAE) হল \(|\Psi_1\rangle\) অবস্থার \(a\) বিস্তারের জন্য একটি অনুমান খুঁজে বের করার কাজ:

\[a = |\langle\Psi_1 | \Psi_1\rangle|^2.\]

এই কাজটি প্রথমে ব্রাসার্ড et al. [১] দ্বারা 2000 সালে তদন্ত করা হয়েছে এবং তাদের অ্যালগরিদম গ্রোভার অপারেটরের সংমিশ্রণ ব্যবহার করে

\[\mathcal{Q} = \mathcal{A}\mathcal{S}_0\mathcal{A}^\dagger\mathcal{S}_{\Psi_1}\]

যেখানে \(\mathcal{S}_0\) এবং \(\mathcal{S}_{\Psi_1}\) এর প্রতিফলন হল \(|0\rangle\) এবং \(|\Psi_1\rangle\) অবস্থা, যথাক্রমে, এবং পর্যায় অনুমান। তবে এই অ্যালগরিদম, যাকে Qiskit এর AmplitudeEstimation বলা হয়, এর জন্য বড় সার্কিটের প্রয়োজন হয় এবং এটি কম্পিউটেশনালভাবে ব্যয়বহুল। অতএব, QAE এর অন্যান্য রূপগুলি প্রস্তাব করা হয়েছে, যা আমরা একটি সহজ উদাহরণের জন্য এই টিউটোরিয়ালে দেখাব।

আমাদের উদাহরণে, \(\mathcal{A}\) একটি বার্নোলি (Bernoulli) দৈব চল রাশি বর্ণনা করে (অজানা বলে ধরে নেওয়া হয়) সাফল্যের সম্ভাবনা \(p\):

\[\mathcal{A}|0\rangle = \sqrt{1 - p}|0\rangle + \sqrt{p}|1\rangle.\]

একটি কোয়ান্টাম কম্পিউটারে আমরা এই অপারেটরকে \(Y\)- একটি কিউবিটের অক্ষের চারদিকে ঘুরিয়ে মডেল করতে পারি

\[\mathcal{A} = R_Y(\theta_p), \theta_p = 2\sin^{-1}(\sqrt{p}).\]

এই ক্ষেত্রে গ্রোভার অপারেটর বিশেষভাবে সহজ

\[\mathcal{Q} = R_Y(2\theta_p),\]

যার ক্ষমতা গণনা করা খুব সহজ: \(\mathcal{Q}^k = R_Y(2k\theta_p)\)।

আমরা যে সম্ভাবনাটি অনুমান করতে চাই তা \(p = 0.2\) এ ঠিক করব।

[1]:

p = 0.2

Now we can define circuits for \(\mathcal{A}\) and \(\mathcal{Q}\).

[2]:

import numpy as np
from qiskit.circuit import QuantumCircuit

class BernoulliA(QuantumCircuit):
    """A circuit representing the Bernoulli A operator."""

    def __init__(self, probability):
        super().__init__(1)  # circuit on 1 qubit

        theta_p = 2 * np.arcsin(np.sqrt(probability))
        self.ry(theta_p, 0)


class BernoulliQ(QuantumCircuit):
    """A circuit representing the Bernoulli Q operator."""

    def __init__(self, probability):
        super().__init__(1)  # circuit on 1 qubit

        self._theta_p = 2 * np.arcsin(np.sqrt(probability))
        self.ry(2 * self._theta_p, 0)

    def power(self, k):
        # implement the efficient power of Q
        q_k = QuantumCircuit(1)
        q_k.ry(2 * k * self._theta_p, 0)
        return q_k

[3]:

A = BernoulliA(p)
Q = BernoulliQ(p)

Qiskit বিস্তার নির্ণয় কর্মধারা¶

Qiskit বেশ কয়েকটি QAE অ্যালগরিদম প্রয়োগ করে যা সমস্ত AmplitudeEstimator ইন্টারফেস থেকে প্রাপ্ত। ইনিশিয়ালাইজারে আমরা অ্যালগরিদম সুনির্দিষ্ট সেটিংস এবং estimate পদ্ধতি উল্লেখ করি, যা সমস্ত কাজ করে, একটি EstimationProblem ইনপুট হিসাবে নেয় এবং একটি AmplitudeEstimationResult অবজেক্ট প্রদান করে। যেহেতু সব QAE ভেরিয়েন্ট একই ইন্টারফেস অনুসরণ করে, তাই আমরা একই সমস্যা সমাধানের জন্য তাদের সব ব্যবহার করতে পারি।

পরবর্তী, আমরা সমস্ত ভিন্ন QAE অ্যালগরিদম চালাব। এটি করার জন্য, আমরা প্রথমে অনুমানের সমস্যাটি সংজ্ঞায়িত করি যার মধ্যে \(\mathcal{A}\) এবং \(\mathcal{Q}\) অপারেটর থাকবে এবং সেইসাথে কিভাবে \(|\Psi_1\rangle\) অবস্থা চিহ্নিত করতে হবে, যা এই সহজ উদাহরণে শুধু \(|1\rangle\)।

[4]:

from qiskit.algorithms import EstimationProblem

problem = EstimationProblem(
    state_preparation=A,  # A operator
    grover_operator=Q,  # Q operator
    objective_qubits=[0],  # the "good" state Psi1 is identified as measuring |1> in qubit 0
)

সার্কিট চালানোর জন্য আমরা Qiskit এর অবস্থা ভেক্টর বা দিকরাশি সিমুলেটর ব্যবহার করব।

[5]:

from qiskit import BasicAer
from qiskit.utils import QuantumInstance

backend = BasicAer.get_backend('statevector_simulator')
quantum_instance = QuantumInstance(backend)

ক্যানোনিকাল এ.ই (Canonical AE)¶

এখন আসুন ব্রাসার্ড এট আল দ্বারা মূল QAE বাস্তবায়নের সাহায্যে এটি সমাধান করি [১]।

[6]:

from qiskit.algorithms import AmplitudeEstimation

ae = AmplitudeEstimation(
    num_eval_qubits=3,  # the number of evaluation qubits specifies circuit width and accuracy
    quantum_instance=quantum_instance
)

অ্যালগরিদম সংজ্ঞায়িত করার সাথে, আমরা estimate পদ্ধতিটি কল করতে পারি এবং এই সমস্যার সমাধান করতে পারি।

[7]:

ae_result = ae.estimate(problem)

অনুমানটি estimate key তে পাওয়া যায়:

[8]:

print(ae_result.estimation)

0.1464466

আমরা দেখতে পাচ্ছি যে এটি আমাদের লক্ষ্যের জন্য খুব ভাল অনুমান নয় \(p = 0.2\)! এটি এই কারণে যে ক্যানোনিকাল AE একটি পৃথক গ্রিডের মধ্যে সীমাবদ্ধ, যা মূল্যায়ন কিউবিটের সংখ্যা দ্বারা নির্দিষ্ট করা হয়েছে:
