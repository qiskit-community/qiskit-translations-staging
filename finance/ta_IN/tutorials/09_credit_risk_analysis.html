கடன் இடர் பகுப்பாய்வு¶

முன்னுரை¶

கடன் ஆபத்து பகுப்பாய்விற்கு குவாண்டம் வழிமுறைகள் எவ்வாறு பயன்படுத்தப்படலாம் என்பதை இந்த பயிற்சிகள் காட்டுகிறது. இன்னும் துல்லியமாக, கிளாசிக்கல் மான்டே கார்லோ உருவகப்படுத்துதலின் மீது இருபடி வேகத்துடன் ஆபத்து நடவடிக்கைகளை மதிப்பிடுவதற்கு குவாண்டம் அலைவீச்சு மதிப்பீடு (QAE) எவ்வாறு பயன்படுத்தப்படலாம். பயிற்சிகள் பின்வரும் ஆவணங்களை அடிப்படையாகக் கொண்டது:

  • குவாண்டம் இடர் பகுப்பாய்வு. ஸ்டீபன் வோர்னர், டேனியல் ஜே. எகர். [Woerner2019]

  • குவாண்டம் கம்ப்யூட்டர்களைப் பயன்படுத்தி கடன் இடர் பகுப்பாய்வு. எகர் மற்றும் பலர். (2019) [Egger2019]

QAE பற்றிய பொதுவான அறிமுகத்தை பின்வரும் தாளில் காணலாம்:

  • குவாண்டம் அலைவீச்சு பெருக்கம் மற்றும் மதிப்பீடு. கில்லஸ் பிராஸார்ட் மற்றும் பலர்.

டுடோரியலின் அமைப்பு பின்வருமாறு:

  1. சிக்கல் வரையறை

  2. நிச்சயமற்ற மாதிரி

  3. எதிர்பார்க்கப்படும் இழப்பு

  4. ஒட்டுமொத்த விநியோக செயல்பாடு

  5. ஆபத்தில் மதிப்பு

  6. ஆபத்தில் நிபந்தனை மதிப்பு

சிக்கல் வரையறை¶

இந்த டுடோரியலில்: கணிதத்தின் ஒரு போர்ட்ஃபோலியோவின் கடன் அபாயத்தை பகுப்பாய்வு செய்ய விரும்புகிறோம் \(K\) சொத்துக்கள். ஒவ்வொரு சொத்தின் இயல்புநிலை நிகழ்தகவு \(k\) ஒரு காஸியன் நிபந்தனை சுதந்திரம் மாதிரியைப் பின்பற்றுகிறது, அதாவது, ஒரு மதிப்பு கொடுக்கப்பட்டுள்ளது \(z\) ஒரு மறைந்த சீரற்ற மாறியில் இருந்து மாதிரி \(Z\) ஒரு நிலையான சாதாரண விநியோகத்தைத் தொடர்ந்து, சொத்தின் இயல்புநிலை நிகழ்தகவு \(k\) வழங்கப்படுகிறது

\[p_k (z) = F\left (\frac {F ^{-1}(p_k ^ 0)-\sqrt {\rho_k} z} {\sqrt {1- \rho_k}} \right)\]

எங்கே \(F\) என்பது ஒட்டுமொத்த விநியோக செயல்பாட்டைக் குறிக்கிறது \(Z\) , \(p_k^0\) என்பது சொத்தின் இயல்புநிலை நிகழ்தகவு \(k\) க்கு \(z=0\) மற்றும் \(\rho_k\) என்பது சொத்தின் இயல்புநிலை நிகழ்தகவின் உணர்திறன் \(k\) தொடர்பாக \(Z\). ஆகவே, கணிதத்தின் ஒரு தெளிவான உணர்தல் கொடுக்கப்பட்டால் \(Z\) தனிப்பட்ட இயல்புநிலை நிகழ்வுகள் ஒருவருக்கொருவர் சுயாதீனமாக கருதப்படுகின்றன.

மொத்த இழப்பின் ஆபத்து நடவடிக்கைகளை பகுப்பாய்வு செய்வதில் நாங்கள் ஆர்வமாக உள்ளோம்

\[L = \sum_ {k=1} ^ K \lambda_k X_ k (Z)\]

எங்கே \(\lambda_k\) என்பது சொத்தின் கொடுக்கப்பட்ட இயல்புநிலையை குறிக்கிறது \(k\), மற்றும் கொடுக்கப்பட்ட \(Z\), \(X_k(Z)\) இயல்புநிலை நிகழ்வைக் குறிக்கும் பெர்ன lli லி மாறியைக் குறிக்கிறது சொத்து \(k\). இன்னும் துல்லியமாக, எதிர்பார்க்கப்படும் மதிப்பில் நாங்கள் ஆர்வமாக உள்ளோம் \(\mathbb{E}[L]\), இடர் மதிப்பு (VaR) \(L\) மற்றும் ஆபத்தில் நிபந்தனை மதிப்பு \(L\) (எதிர்பார்க்கப்படும் பற்றாக்குறை என்றும் அழைக்கப்படுகிறது). VaR மற்றும் CVaR என வரையறுக்கப்பட்டுள்ளது

\[\text{VaR}_ {\alpha} (L) = \inf\{x \mid \mathbb{P}[L < = x] \geq 1 - \alpha\}\]

நம்பிக்கை மட்டத்துடன் \(\alpha \in [0, 1]\), மற்றும்

\[\text{CVaR}_ {\alpha} (L) = \mathbb{E}[L \mid L \geq \text{VaR}_ {\alpha} (L)].\]

கருதப்பட்ட மாதிரியைப் பற்றிய கூடுதல் விவரங்களுக்கு, எ.கா. Regulatory Capital Modeling for Credit Risk. Marek Rutkowski, Silvio Tarca

சிக்கல் பின்வரும் அளவுருக்களால் வரையறுக்கப்படுகிறது: - பிரதிநிதித்துவப்படுத்த பயன்படுத்தப்படும் குவிட்களின் எண்ணிக்கை \(Z\), இதன் மூலம் குறிக்கப்படுகிறது \(n_z\) - இதற்கான துண்டிப்பு மதிப்பு \(Z\), குறிக்கப்படுகிறது \(z_{\text{max}}\), அதாவது Z எடுக்கப்படும் என்று கருதப்படுகிறது \(2^{n_z}\) இதில் சமமான மதிப்புகள் \(\{-z_{max}, ..., +z_{max}\}\)- ஒவ்வொரு சொத்துக்கும் அடிப்படை இயல்புநிலை நிகழ்தகவுகள் \(p_0^k \in (0, 1)\), \(k=1, ..., K\) - இது தொடர்பான இயல்புநிலை நிகழ்தகவுகளின் உணர்திறன் \(Z\), குறிக்கப்படுகிறது \(\rho_k \in [0, 1)\)- சொத்துக்கான இயல்புநிலை கொடுக்கப்பட்ட இழப்பு \(k\), குறிக்கப்படுகிறது \(\lambda_k\) - VaR / CVaR \(\alpha \in [0, 1]\).

நிச்சயமற்ற மாதிரி¶

நிச்சயமற்ற மாதிரியை ஏற்றும் ஒரு சுற்று ஒன்றை இப்போது உருவாக்குகிறோம். ஒரு பதிவேட்டில் ஒரு குவாண்டம் நிலையை உருவாக்குவதன் மூலம் இதை அடைய முடியும் \(n_z\) குவிட்ஸ் குறிக்கும் \(Z\) ஒரு நிலையான சாதாரண விநியோகத்தைத் தொடர்ந்து. இந்த நிலை பின்னர் இரண்டாவது குவிட் பதிவேட்டில் ஒற்றை குவிட் ஒய்-சுழற்சிகளைக் கட்டுப்படுத்தப் பயன்படுகிறது \(K\) குவிட்ஸ், அங்கு ஒரு \(|1\rangle\) குவிட் நிலை \(k\) இயல்புநிலை நிகழ்வைக் குறிக்கிறது சொத்து \(k\). இதன் விளைவாக குவாண்டம் நிலையை இவ்வாறு எழுதலாம்

\[ |\Psi\rangle = \sum_{i=0}^{2^{n_z}-1} \sqrt{p_z^i} |z_i \rangle \bigotimes_{k=1}^K \left( \sqrt{1 - p_k(z_i)}|0\rangle + \sqrt{p_k(z_i)}|1\rangle\right),\]

இங்கு நாம் குறிக்கிறோம் \(z_i\) \(i\)-th மதிப்பிடப்பட்ட மற்றும் துண்டிக்கப்பட்ட மதிப்பு \(Z\) [Egger2019].

# analyze uncertainty circuit and determine exact solutions
p_z = np.zeros(2**n_z)
p_default = np.zeros(K)
values = []
probabilities = []
num_qubits = u.num_qubits
for i, a in enumerate(job.result().get_statevector()):

    # get binary representation
    b = ('{0:0%sb}' % num_qubits).format(i)
    prob = np.abs(a)**2

    # extract value of Z and corresponding probability
    i_normal = int(b[-n_z:], 2)
    p_z[i_normal] += prob

    # determine overall default probability for k
    loss = 0
    for k in range(K):
        if b[K - k - 1] == '1':
            p_default[k] += prob
            loss += lgd[k]
    values += [loss]
    probabilities += [prob]

values = np.array(values)
probabilities = np.array(probabilities)

expected_loss = np.dot(values, probabilities)

losses = np.sort(np.unique(values))
pdf = np.zeros(len(losses))
for i, v in enumerate(losses):
    pdf[i] += sum(probabilities[values == v])
cdf = np.cumsum(pdf)

i_var = np.argmax(cdf >= 1-alpha)
exact_var = losses[i_var]
exact_cvar = np.dot(pdf[(i_var+1):], losses[(i_var+1):])/sum(pdf[(i_var+1):])

எதிர்பார்க்கப்படும் இழப்பு¶

எதிர்பார்க்கப்படும் இழப்பை மதிப்பிடுவதற்கு, தனிப்பட்ட இழப்புகளை மொத்த இழப்புக்குச் சேர்க்க முதலில் எடையுள்ள தொகை ஆபரேட்டரைப் பயன்படுத்துகிறோம்:

\[\mathcal{S}: |x_1, ..., x_K \rangle_K |0\rangle_{n_S} \mapsto |x_1, ..., x_K \rangle_K |\lambda_1x_1 + ... + \lambda_K x_K\rangle_{n_S}.\]

முடிவைக் குறிக்க தேவையான எண்ணிக்கையிலான குவிட்கள் வழங்கப்படுகின்றன

\[n_s = \lfloor \log_2( \lambda_1 + ... + \lambda_K ) \rfloor + 1.\]

ஒரு குவாண்டம் பதிவேட்டில் மொத்த இழப்பு விநியோகத்தை நாங்கள் பெற்றவுடன், மொத்த இழப்பை வரைபடமாக்க [Woerner2019] இல் விவரிக்கப்பட்டுள்ள நுட்பங்களைப் பயன்படுத்தலாம் \(L \in \{0, ..., 2^{n_s}-1\}\) ஒரு ஆபரேட்டரால் ஒரு புறநிலை குவிட்டின் வீச்சுக்கு

\[ | L \rangle_{n_s}|0\rangle \mapsto | L \rangle_{n_s} \left( \sqrt{1 - L/(2^{n_s}-1)}|0\rangle + \sqrt{L/(2^{n_s}-1)}|1\rangle \right),\]

இது எதிர்பார்த்த இழப்பை மதிப்பிடுவதற்கு வீச்சு மதிப்பீட்டை இயக்க அனுமதிக்கிறது.

ஒட்டுமொத்த விநியோக செயல்பாடு¶

எதிர்பார்த்த இழப்புக்கு பதிலாக (இது கிளாசிக்கல் நுட்பங்களைப் பயன்படுத்தி திறமையாக மதிப்பிடப்படலாம்) இழப்பின் ஒட்டுமொத்த விநியோக செயல்பாட்டை (CDF) இப்போது மதிப்பிடுகிறோம். பாரம்பரியமாக, இயல்புநிலை சொத்துக்களின் சாத்தியமான அனைத்து சேர்க்கைகளையும் மதிப்பீடு செய்வது அல்லது மான்டே கார்லோ உருவகப்படுத்துதலில் பல கிளாசிக்கல் மாதிரிகள் ஆகியவற்றை மதிப்பீடு செய்வது இதில் அடங்கும். QAE ஐ அடிப்படையாகக் கொண்ட வழிமுறைகள் எதிர்காலத்தில் இந்த பகுப்பாய்வை கணிசமாக விரைவுபடுத்தும் திறனைக் கொண்டுள்ளன.

சி.டி.எஃப் ஐ மதிப்பிடுவதற்கு, அதாவது நிகழ்தகவு \(\mathbb{P}[L \leq x]\), நாங்கள் மீண்டும் விண்ணப்பிக்கிறோம் \(\mathcal{S}\) மொத்த இழப்பைக் கணக்கிட, பின்னர் ஒரு ஒப்பீட்டாளரைப் பயன்படுத்துங்கள் கொடுக்கப்பட்ட மதிப்புக்கு \(x\) ஆக செயல்படுகிறது

\[\begin{split} \mathcal{C}: |L\rangle_n|0> \mapsto \begin{cases} |L\rangle_n|1> & \text{if}\quad L \leq x \\ |L\rangle_n|0> & \text{if}\quad L > x. \end{cases}\end{split}\]

இதன் விளைவாக குவாண்டம் நிலையை இவ்வாறு எழுதலாம்

\[ \sum_{L = 0}^{x} \sqrt{p_{L}}|L\rangle_{n_s}|1\rangle + \sum_{L = x+1}^{2^{n_s}-1} \sqrt{p_{L}}|L\rangle_{n_s}|1\rangle,\]

நிச்சயமற்ற மாதிரியின் விவரங்களை முன்வைப்பதற்கு பதிலாக சுருக்கமான இழப்பு மதிப்புகள் மற்றும் அதனுடன் தொடர்புடைய நிகழ்தகவுகளை நாங்கள் நேரடியாக கருதுகிறோம்.

சி.டி.எஃப் (\(x\)) அளவிடும் நிகழ்தகவுக்கு சமம் \(|1\rangle\) புறநிலை குவிட்டில் மற்றும் QAE ஐ நேரடியாக மதிப்பிட பயன்படுத்தலாம்.

def bisection_search(objective, target_value, low_level, high_level, low_value=None, high_value=None):
    """
    Determines the smallest level such that the objective value is still larger than the target
    :param objective: objective function
    :param target: target value
    :param low_level: lowest level to be considered
    :param high_level: highest level to be considered
    :param low_value: value of lowest level (will be evaluated if set to None)
    :param high_value: value of highest level (will be evaluated if set to None)
    :return: dictionary with level, value, num_eval
    """

    # check whether low and high values are given and evaluated them otherwise
    print('--------------------------------------------------------------------')
    print('start bisection search for target value %.3f' % target_value)
    print('--------------------------------------------------------------------')
    num_eval = 0
    if low_value is None:
        low_value = objective(low_level)
        num_eval += 1
    if high_value is None:
        high_value = objective(high_level)
        num_eval += 1

    # check if low_value already satisfies the condition
    if low_value > target_value:
        return {'level': low_level, 'value': low_value, 'num_eval': num_eval, 'comment': 'returned low value'}
    elif low_value == target_value:
        return {'level': low_level, 'value': low_value, 'num_eval': num_eval, 'comment': 'success'}

    # check if high_value is above target
    if high_value < target_value:
        return {'level': high_level, 'value': high_value, 'num_eval': num_eval, 'comment': 'returned low value'}
    elif high_value == target_value:
        return {'level': high_level, 'value': high_value, 'num_eval': num_eval, 'comment': 'success'}

    # perform bisection search until
    print('low_level    low_value    level    value    high_level    high_value')
    print('--------------------------------------------------------------------')
    while high_level - low_level > 1:

        level = int(np.round((high_level + low_level) / 2.0))
        num_eval += 1
        value = objective(level)

        print('%2d           %.3f        %2d       %.3f    %2d            %.3f' \
              % (low_level, low_value, level, value, high_level, high_value))

        if value >= target_value:
            high_level = level
            high_value = value
        else:
            low_level = level
            low_value = value

    # return high value after bisection search
    print('--------------------------------------------------------------------')
    print('finished bisection search')
    print('--------------------------------------------------------------------')
    return {'level': high_level, 'value': high_value, 'num_eval': num_eval, 'comment': 'success'}
