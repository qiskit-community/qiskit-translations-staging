Primeros pasos con la primitiva Estimator¶

Aprende a configurar y utilizar el programa primitivo Estimator.

Descripción general¶

La primitiva Estimador te permite calcular e interpretar de manera eficiente los valores esperados de los operadores cuánticos necesarios para muchos algoritmos. Puedes especificar una lista de circuitos y observables, luego evaluar los valores esperados y las variaciones para una entrada de parámetro determinada.

Preparar el entorno¶

  1. Sigue los pasos de la guía de primeros pasos para preparar tu instancia de Quantum Service.

  2. Necesitarás al menos un circuito para enviarlo al programa. Todos nuestros ejemplos tienen circuitos, pero si deseas enviar tu propio circuito, puedes usar Qiskit para crear uno. Para aprender a crear circuitos con Qiskit, consulta el Tutorial de aspectos básicos de un circuito.

  3. Crea una lista de observables. Los observables te permiten definir las propiedades del circuito que son relevantes para tu problema y te permiten medir de manera eficiente su valor esperado. Para simplificar, puedes usar la clase PauliSumOp en Qiskit para definirlos, como se ilustra en el ejemplo de abajo.

Iniciar una sesión¶

Con las primitivas de Qiskit Runtime, presentamos el concepto de una sesión o una fábrica que te permite definir un trabajo como una colección de llamadas iterativas a la computadora cuántica. Cuando inicias una sesión, almacena en caché los datos que envías para que no tengan que transmitirse al Quantum Datacenter en cada iteración.

Especificar las entradas del programa¶

El Estimator toma los siguientes argumentos:

  • circuits: una lista de circuitos (parametrizados) que deseas investigar.

  • observables: una lista de observables para medir los valores esperados.

  • parameters: una lista de parámetros para los circuitos parametrizados. Debe omitirse si los circuitos proporcionados no están parametrizados.

  • skip_transpilation: la transpilación del circuito se omite si se establece en True. El valor predeterminado es False.

  • service: la instancia QiskitRuntimeService para ejecutar el programa. Si no se especifica, se inicializa la cuenta guardada predeterminada para QiskitRuntimeService.

  • options: Diccionario de opciones de Runtime que controlan el entorno de ejecución.

    • backend: El backend sobre el que se ejecuta. Esta opción es necesaria si ejecutas sobre IBM Quantum. Sin embargo, si ejecutas sobre IBM Cloud, puedes optar por no especificar el backend, en cuyo caso se utiliza el backend menos ocupado.

Puedes encontrar más detalles en la referencia de la API de Estimator.

Ejemplo:

[1]:

from qiskit_ibm_runtime import QiskitRuntimeService, Estimator
from qiskit.circuit.library import RealAmplitudes
from qiskit.quantum_info import SparsePauliOp

service = QiskitRuntimeService()

psi1 = RealAmplitudes(num_qubits=2, reps=2)
psi2 = RealAmplitudes(num_qubits=2, reps=3)

H1 = SparsePauliOp.from_list([("II", 1), ("IZ", 2), ("XI", 3)])
H2 = SparsePauliOp.from_list([("IZ", 1)])
H3 = SparsePauliOp.from_list([("ZI", 1), ("ZZ", 1)])

Escribir y leer desde una sesión¶

La ejecución de un trabajo y la devolución de los resultados se realizan escribiendo y leyendo desde la sesión. Una vez que se devuelven los resultados, la sesión se cierra automáticamente.

Ejecutar el trabajo e imprimir los resultados¶

Ejecuta el trabajo, especificando tus entradas y opciones previamente definidas. Usa circuits, observables, y parameter_values para usar un parámetro específico y observable con el circuito especificado.

Por ejemplo, esta línea psi1_H23_result = estimator(circuits=[0, 0], observables=[1, 2], parameter_values=[theta1]*2) especifica lo siguiente:

  • Devolver el valor por usar el observable H2 y el parámetro theta1 con el circuito psi1.

  • Devolver el valor por usar el observable H3 y el parámetro theta1 con el circuito psi1.

También puedes pasar circuitos y observables como objetos. Por ejemplo, psi1_H23_result = estimator(circuits=[ps1, ps1], observables=[H2, H3], parameter_values=[theta1]*2)

[2]:

with Estimator(
    circuits=[psi1, psi2],
    observables=[H1, H2, H3],
    service=service,
    options={ "backend": "ibmq_qasm_simulator" }
) as estimator:
    theta1 = [0, 1, 1, 2, 3, 5]
    theta2 = [0, 1, 1, 2, 3, 5, 8, 13]
    theta3 = [1, 2, 3, 4, 5, 6]

    # calculate [ <psi1(theta1)|H1|psi1(theta1)> ]
    # pass indices of circuits and observables
    psi1_H1_result = estimator(circuits=[0], observables=[0], parameter_values=[theta1])
    print(psi1_H1_result)

    # calculate [ <psi1(theta1)|H2|psi1(theta1)>, <psi1(theta1)|H3|psi1(theta1)> ]
    # alternatively you can also pass circuits and observables as objects
    psi1_H23_result = estimator(circuits=[psi1, psi1], observables=[H2, H3], parameter_values=[theta1]*2)
    print(psi1_H23_result)

    # calculate [ <psi2(theta2)|H2|psi2(theta2)> ]
    # Note that you don't need to specify the labels "circuits", "observables", or "parameter_values", as long as they are specified in that order.
    psi2_H2_result = estimator([1], [1], [theta2])
    print(psi2_H2_result)

    # calculate [ <psi1(theta1)|H1|psi1(theta1)>, <psi1(theta3)|H1|psi1(theta3)> ]
    psi1_H1_result2 = estimator([0, 0], [0, 0], [theta1, theta3])
    print(psi1_H1_result2)

    # calculate [ <psi1(theta1)|H1|psi1(theta1)>,
    #             <psi2(theta2)|H2|psi2(theta2)>,
    #             <psi1(theta3)|H3|psi1(theta3)> ]
    psi12_H23_result = estimator([0, 1, 0], [0, 1, 2], [theta1, theta2, theta3])
    print(psi12_H23_result)

Los resultados se alinean con el parámetro - circuit - tuplas de observables especificadas anteriormente. Por ejemplo, el primer resultado: EstimatorResult(values=array([1.55273438]), metadata=[{'variance': 8.897655487060547, 'shots': 1024}]) es la salida del parámetro etiquetado como theta1 y el observable H1 se envía al primer circuito.
