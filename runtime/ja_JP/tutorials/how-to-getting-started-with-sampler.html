Sampler primitive 入門¶

Sampler primitive プログラムをセットアップして使用する方法について説明します。

概要¶

Sampler primitive を使用すると、カウントをより正確にコンテキスト化できます。 これは、ユーザー回路を入力として受け取り、エラーが軽減された準確率の読み出しを生成します。 これにより、破壊的な干渉の中で複数の関連データポイントの可能性をより効率的に評価できます。

環境の準備¶

  1. getting started guide の手順に従って、使用可能な量子サービス・インスタンスを取得します。

  2. プログラムに送信するには、少なくとも 1 つの回路が必要になります。 この例ではすべて回路が設定されていますが、独自の回路を提出する場合は、 Qiskit を使用して 1 つの回路を作成できます。 Qiskitを使用して回路を作成する方法を学習するには、 Circuit basics tutorial を参照してください。

セッションを開始する¶

Qiskit Runtime primitiveを使用して、量子コンピューターへの反復呼び出しのコレクションとしてジョブを定義できるようにするセッションまたはファクトリーの概念を紹介します。 セッションを開始すると、送信したデータがキャッシュされるため、反復ごとに量子データセンターに送信する必要はありません。

入力するプログラムの指定¶

Sampler は以下の引数をとります。

  • circuits: 調べたい (パラメーター化された) 回路のリストです。

  • parameters: パラメーター回路のパラメーターのリストです。与えられた回路がパラメーター化されていない場合は省略する必要があります。

  • skip_transpilation: True が設定されている場合、回路のトランスパイルはスキップされます。デフォルト値は False です。

  • service: プログラムを実行する QiskitRuntimeService インスタンスです。指定しない場合、QiskitRuntimeService のデフォルト保存アカウントが初期化されます。

  • options: 実行環境を制御するRuntime オプション辞書。

    • backend: 実行するバックエンドです。 このオプションは、 IBM Quantum の上で実行している場合に必要です。 IBM Cloud で実行している場合は、バックエンドを指定しないことを選択できます。この場合、最も空いているバックエンドが使用されます。

詳細は Sampler API リファレンス を参照してください。

例:

[1]:

from qiskit_ibm_runtime import QiskitRuntimeService, Sampler
from qiskit import QuantumCircuit

service = QiskitRuntimeService()

bell = QuantumCircuit(2)
bell.h(0)
bell.cx(0, 1)
bell.measure_all()

セッションからの書き込みと読み取り¶

ジョブの実行と結果の返送は、セッションへの書き込みとセッションからの読み取りによって行われます。 コードが with ブロックを終了すると、セッションは閉じます。

ジョブの実行および結果の印字¶

以前に定義した入力およびオプションを指定して、ジョブを実行します。 この単純な例では、回路は 1 つしかなく、パラメーターを持っていません。

各呼び出しでは、 circuits を使用して、実行する回路を指定します。また、該当する場合は、 parameter_values は、指定された回路でどのパラメーターを使用するかを指定します。

この例では、 1 つの回路 circuits=bell を指定し、最初の回路(および唯一の回路) circuits=[0] で実行した結果を求めました。

後の例では、セッションを初期化するときに circuits=[pqc, pqc2] などの複数の回路を指定した場合、それぞれ呼び出して、 circuits=[1] または circuits=[pqc2] を指定して pqc2 回路を実行させることができます。

[2]:

# executes a Bell circuit
with Sampler(circuits=bell, service=service, options={ "backend": "ibmq_qasm_simulator" }) as sampler:
    # pass indices of circuits
    result = sampler(circuits=[0], shots=1024)
    print(result)

SamplerResult(quasi_dists=[{'00': 0.5078125, '11': 0.4921875}], metadata=[{'header_metadata': None, 'shots': 1024}])

複数回路の例¶

この例では、 3 つの回路を指定しますが、パラメーターはありません。

[4]:

from qiskit_ibm_runtime import QiskitRuntimeService, Sampler
from qiskit import QuantumCircuit

service = QiskitRuntimeService()

bell = QuantumCircuit(2)
bell.h(0)
bell.cx(0, 1)
bell.measure_all()

# executes three Bell circuits
with Sampler(circuits=[bell]*3, service=service, options={ "backend": "ibmq_qasm_simulator" }) as sampler:
    # alternatively you can also pass circuits as objects
    result = sampler(circuits=[bell]*3)
    print(result)

SamplerResult(quasi_dists=[{'00': 0.478515625, '11': 0.521484375}, {'00': 0.533203125, '11': 0.466796875}, {'00': 0.4853515625, '11': 0.5146484375}], metadata=[{'header_metadata': None, 'shots': 1024}, {'header_metadata': None, 'shots': 1024}, {'header_metadata': None, 'shots': 1024}])

複数のパラメーター化された回路の例¶

この例では、複数のパラメーター化された回路を実行します。 これが実行されると、この行 result = sampler(circuits=[0, 0, 1], parameter_values=[theta1, theta2, theta3]) により各回路に送信するパラメーターが指定されます。

この例では、 theta1 とラベル付けされたパラメーターが最初の回路に送信され、 theta2 が最初の回路に送信され、 theta3 が2番目の回路に送信されます。

[5]:

from qiskit_ibm_runtime import QiskitRuntimeService, Sampler
from qiskit import QuantumCircuit
from qiskit.circuit.library import RealAmplitudes

service = QiskitRuntimeService()

# parameterized circuit
pqc = RealAmplitudes(num_qubits=2, reps=2)
pqc.measure_all()
pqc2 = RealAmplitudes(num_qubits=2, reps=3)
pqc2.measure_all()

theta1 = [0, 1, 1, 2, 3, 5]
theta2 = [1, 2, 3, 4, 5, 6]
theta3 = [0, 1, 2, 3, 4, 5, 6, 7]

with Sampler(circuits=[pqc, pqc2], service=service, options={ "backend": "ibmq_qasm_simulator" }) as sampler:
    result = sampler(circuits=[0, 0, 1], parameter_values=[theta1, theta2, theta3])
    print(result)

SamplerResult(quasi_dists=[{'00': 0.1259765625, '10': 0.1005859375, '11': 0.408203125, '01': 0.365234375}, {'01': 0.025390625, '11': 0.3056640625, '00': 0.0693359375, '10': 0.599609375}, {'11': 0.03515625, '01': 0.67578125, '10': 0.0888671875, '00': 0.2001953125}], metadata=[{'header_metadata': None, 'shots': 1024}, {'header_metadata': None, 'shots': 1024}, {'header_metadata': None, 'shots': 1024}])

結果¶

結果は、前に指定したパラメーターと回路のペアに一致します。たとえば、最初の結果（ {'11': 0.42578125, '00': 0.14453125, '10': 0.0888671875, '01': 0.3408203125} ）は、最初の回路に送信された theta1 とラベル付けされたパラメーターの出力です。
