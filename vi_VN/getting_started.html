Cài đặt¶

Qiskit hỗ trợ Python từ phiên bản 3.6 trở lên. Tuy nhiên, Python và Qiskit là những hệ thống cập nhật liên tục, và đôi khi giữa các phiên bản của chúng, sẽ có những điểm không tương thích với nhau.

Chúng tôi gợi ý bạn nên cài đặt Anaconda, một phiên bản Python đa nền tảng cho các tính toán khoa học. Jupyter Notebook được gợi ý nên dùng để tương tác với Qiskit đã tích hợp sẵn trên Anaconda.

Qiskit được thử nghiệm và hỗ trợ trên các hệ thống 64-bit:

  • Ubuntu 16.04 trở lên

  • macOS 10.12.6 trở lên

  • Windows 7 trở lên

Chúng tôi đề nghị sử dụng môi trường ảo Python để tách rời Qiskit khỏi các ứng dụng khác và cải thiện trải nghiệm sử dụng của bạn.

Cách đơn giản nhất để sử dụng các môi trường là sử dụng lệnh conda, được bao gồm trong Anaconda. Một môi trường Conda cho phép bạn sử dụng một phiên bản Python bất kỳ và các thư viện mong muốn. Mở một terminal (chương trình giao diện cửa sổ dòng lệnh) trong thư mục mà bạn muốn làm việc.

Các bạn nên sử dụng Anaconda Prompt được cài đặt chung với Anaconda. Tất cả những gì bạn phải làm là tạo một môi trường ảo bên trong Anaconda và kích hoạt nó. Các lệnh này có thể chạy trong Anaconda Prompt bất kể Windows hoặc Linux.

Tạo một môi trường tối giản với chỉ có Python được cài đặt.

conda create -n ENV_NAME python=3

Kích hoạt môi trường mới của bạn.

conda activate ENV_NAME

Tiếp theo, cài đặt Qiskit.

pip install qiskit

Nếu các package đã được cài đặt đúng, bạn có thể chạy conda list để xem các package đang được kích hoạt trong môi trường ảo của bạn.

Nếu bạn có ý định sử dụng thêm chức năng hiển thị đồ họa hoặc Jupyter Notebooks, bạn cần cần cài đặt Qiskit với các tính năng hỗ trợ visualization:

pip install qiskit[visualization]

Ngoài ra, nếu bạn là một người sử dụng zsh (shell mặc định trên những phiên bản mới hơncủa macOS), bạn cần đặt qiskit[visualization] trong dấu nháy đơn:

pip install 'qiskit[visualization]'

Những nhà cung cấp dịch vụ điện toán đám mây sau đã có Qiskit cài đặt sẵn trong môi trường của họ:

IBM Quantum Lab

Build quantum applications and experiments with Qiskit in a cloud programming environment.

Strangeworks

A platform that enables users and organizations to easily apply quantum computing to their most pressing problems and research.

Cài đặt các thành phần nguyên tố từ nguồn cho phép bạn truy cập phiên bản mới nhất của Qiskit thay vì phiên bản ở trong kho Python Package Index (PyPI). Điều này sẽ giúp bạn có thể xem và mở rộng phiên bản mới nhất của Qiskit code một cách hiệu quả hơn.

When installing the elements and components from source, by default their development version (which corresponds to the master git branch) will be used, as opposed to the stable version (which contains the same codebase as the published pip packages). Since the development versions of an element or component usually include new features and changes, they generally require using the development version of the rest of the items as well.

Gói Terra và Aer đều yêu cầu trình biên dịch để xây dựng từ nguồn trước khi bạn có thể cài đặt. Ignis, Aqua, và IBM Quantum Provider backend không yêu cầu một trình biên dịch.

Installing elements from source requires the following order of installation to prevent installing versions of elements that may be lower than those desired if the pip version is behind the source versions:

  1. qiskit-terra

  2. qiskit-aer

  3. qiskit-ignis

  4. qiskit-aqua

  5. qiskit-ibmq-provider (Nếu bạn muốn kết nối đến các thiết bị lượng tử của IBM hoặc giả lập mô phỏng online)

Để có thể làm việc với các thành phần nguyên tố và phần tử cùng lúc, hãy làm theo các bước sau cho mỗi thành phần.

Ghi chú

Due to the use of namespace packaging in Python, care must be taken in how you install packages. If you're planning to install any element from source, do not use the qiskit meta-package. Also, follow this guide and use a separate virtual environment for development. If you do choose to mix an existing installation with your development, refer to https://github.com/pypa/sample-namespace-packages/blob/master/table.md for the set of combinations of installation methods that work together.

Set up the Virtual Development Environment

conda create -y -n QiskitDevenv python=3
conda activate QiskitDevenv

Installing Terra from Source

Cài đặt từ nguồn yêu cầu một trình biên dịch C++ trong hệ thống của bạn và phải hỗ trợ C++11.

Trên hầu hết các nền tảng Linux, các trình biên dịch GCC thường đã được cài sẵn.

Nếu bạn sử dụng macOS, bạn có thể cài đặt trình biên dịch Clang bằng cách cài đặt XCode. Kiểm tra bạn đã cài đặt XCode và Clang bằng cách mở terminal và nhập.

clang --version

Cài đặt XCode và Clang bằng câu lệnh sau.

xcode-select --install

Đối với Windows, cách dễ nhất là cài đặt trình biên dịch Visual C++ từ Build Tools for Visual Studio 2019. Ngoài ra, bạn cũng có thể cài đặt Visual Studio phiên bản 2015 hoặc 2017, hãy đảm bảo rằng bạn lựa chọn các tùy chọn cho việc cài đặt trình biên dịch C++.

Khi trình biên dịch đã được cài đặt, bạn đã sẵn sàng để cài đặt Qiskit Terra.

  1. Tạo ra bản sao của Terra repository.

    git clone https://github.com/Qiskit/qiskit-terra.git

  2. Việc tạo ra bản sao của repository sẽ tạo nên một thư mục có tên qiskit-terra.

    cd qiskit-terra

  3. Cài đặt các thư viện Python cần thiết từ thư mục qiskit-terra.

    pip install cython

  4. If you want to run tests or linting checks, install the developer requirements.

    pip install -r requirements-dev.txt

  5. Cài đặt qiskit-terra.

    pip install .

Nếu bạn muốn cài đặt nó ở chế độ chỉnh sửa, nghĩa là việc thay đổi code trong project không yêu cầu cài đặt lại để áp dụng, bạn có thể làm được với:

pip install -e .

Bạn có thể chạy các ví dụ code sau khi cài đặt Terra. Bạn có thể chạy ví dụ với những lệnh sau.

python examples/python/using_qiskit_terra_level_0.py

Ghi chú

Nếu bạn không có ý định cài đặt bất kỳ thành phần nào khác, qiskit-terra sẽ hiện ra RuntimeWarning để cảnh báo rằng cả qiskit-aer lẫn qiskit-ibmq-provider không được cài đặt. Điều này được thực hiện bởi vì người dùng thường có ý định sử dụng thêm các thành phần nguyên tố khác, nhưng không nhận ra chúng không được cài đặt, hoặc việc cài đặt Aer hay IBM Quantum Provider đã thất bại vì một số lý do nào đó. Nếu bạn muốn bỏ qua những cảnh báo này, thêm:

import warnings
warnings.filterwarnings('ignore', category=RuntimeWarning,
                        module='qiskit')

before any qiskit imports in your code. This will suppress the warning about the missing qiskit-aer and qiskit-ibmq-provider, but will continue to display any other warnings from qiskit or other packages.

Installing Aer from Source

  1. Tạo bản sao Aer repository.

    git clone https://github.com/Qiskit/qiskit-aer

  2. Cài các bản dựng cần thiết.

    pip install cmake scikit-build cython

After this, the steps to install Aer depend on which operating system you are using. Since Aer is a compiled C++ program with a Python interface, there are non-Python dependencies for building the Aer binary which can't be installed universally depending on operating system.

Linux

  1. Cài đặt các yêu cầu của trình biên dịch.

    Building Aer requires a C++ compiler and development headers.

    Nếu bạn đang sử dụng Fedora hoặc một bản Linux tương đương, cài đặt bằng cách sử dụng:

    dnf install @development-tools

    Cho Ubuntu/Debian, cài đặt bằng cách sử dụng:

    apt-get install build-essential

  2. Install OpenBLAS development headers.

    Nếu bạn đang sử dụng Fedora hoặc một bản Linux tương đương, cài đặt bằng cách sử dụng:

    dnf install openblas-devel

    Cho Ubuntu/Debian, cài đặt bằng cách sử dụng:

    apt-get install libopenblas-dev
macOS

  1. Install dependencies.

    Để sử dụng Clang trình biên dịch trên macOS, bạn cần phải tải thêm thư viện hỗ trợ OpenMP. Bạn có thể sử dụng brew để tải nó và các dependencies khác.

    brew install libomp

  2. Sau đó cài đặt một ứng dụng của BLAS; OpenBLAS là lựa chọn mặc định.

    brew install openblas

    Tiếp theo, cài đặt Xcode Command Line Tools.

    xcode-select --install
Windows

Trên Windows bạn cần sử dụng Anaconda3 hoặc Miniconda3 để tải tất cả dependencies.

  1. Cài đặt các yêu cầu của trình biên dịch.

    conda install --update-deps vs2017_win-64 vs2017_win-32 msvc_runtime

Qiskit Aer is a high performance simulator framework for quantum circuits. It provides several backends to achieve different simulation goals.

conda install --update-deps -c conda-forge -y openblas cmake

  1. Trực tiếp dựng và cài đặt qiskit-aer

    Nếu bạn có phiên bản pip <19.0.0 và môi trường của bạn không yêu cầu một bản dưng tùy chỉnh, chạy:

    cd qiskit-aer
pip install .

    Điều này sẽ dựng cả mã nhị phân cũng như cài đặt Aer cùng lúc.

    Bên cạnh đó, nếu bạn có một phiên bản pip mới hơn được cài đặt, hoặc có một vài yêu cầu tùy chỉnh, bạn có thể tự dựng thủ công một bản Python wheel.

    cd qiskit-aer
python ./setup.py bdist_wheel

    Nếu bạn cần thiết lập một tùy chọn tùy chỉnh trong quá trình dựng wheel, hãy tham khảo :ref:` aer_wheel_build_options.

    After you build the Python wheel, it will be stored in the dist/ dir in the Aer repository. The exact version will depend

    cd dist
pip install qiskit_aer-*.whl

    Tên chính xác của đầu ra của tệp wheel sẽ tùy thuộc vào phiên bản đang phát triển của Aer.

Custom options

Các bản dựng Aer sử dụng scikit-build để chạy các biên dịch khi dựng nó bằng giao diện Python. Nó hoạt động như một giao diện cho setuptools để gọi CMake và biên dịch mã nhị phân cho hệ thống của bạn.

Vì sự phức tạp để biên dịch mã nhị phân, bạn có thể sẽ cần truyền một vài lựa chọn vào một vài phần trong quá trình dựng. Cách để có thể truyền tham số là:

python setup.py bdist_wheel [skbuild_opts] [-- [cmake_opts] [-- build_tool_opts]]

nơi các thành phần ở trong ngoặc [] là tùy chọn, skbuild_opts, cmake_opts, build_tool_opts có thể được thay đổi bởi lựa chọn flags của bạn. Một danh sách các tùy chọn CMake có thể tham khảo ở đây: https://cmake.org/cmake/help/v3.6/manual/cmake.1.html#options. Ví dụ, bạn có thể chạy gì đó như:

python setup.py bdist_wheel -- -- -j8

Việc truyền flag -j8 vào việc dựng hệ thống (trong trường hợp này là Automake), được dùng để nói rằng bạn muốn dựng song song sử dụng 8 quy trình (process).

Ví dụ, một trường hợp sử dụng flag thường gặp trên Linux là để xác định xem phiên bản nào của trình biên dịch C++ để dùng (thông thường trong trường hợp trình biên dịch mặc định quá cũ):

python setup.py bdist_wheel -- -DCMAKE_CXX_COMPILER=g++-7

cái mà nói với CMake dùng trình biên dịch g++-7 thay vì sử dụng trình g++ mặc định khi biên dịch Aer.

Một trường hợp sử dụng phổ biến khác cho điều này, tùy thuộc vào môi trường của bạn, là bạn có thể cần xác định tên nền tảng của bạn và tắt liên kết tĩnh.

python setup.py bdist_wheel --plat-name macosx-10.9-x86_64 \
-- -DSTATIC_LINKING=False -- -j8

Here --plat-name is a flag to setuptools, to specify the platform name to use in the package metadata, -DSTATIC_LINKING is a flag for using CMake to disable static linking, and -j8 is a flag for using Automake to use 8 processes for compilation.

Danh sách các tùy chọn phổ biến phụ thuộc vào nền tảng:

Nền tảng

Công cụ

Tùy chọn

Ứng dụng

Tất cả

Theo sau bởi một con số, thiết lập số quá trình sử dụng cho việc biên dịch.

Linux

Dùng để xác định trình biên dịch C++, điều này thường cân thiết nếu trình biên dịch g++ của bạn quá cũ.

Ghi chú

Một vài tùy chọn trên không thuộc về nên tảng nào. Các nền tảng trên được liệt kê vì chúng được sử dụng phổ biến trong môi trường. Tham khảo tài liệu về công cụ để biết thêm thông tin.

Installing Ignis from Source

  1. Nhân bản lại Ignis repository.

    git clone https://github.com/Qiskit/qiskit-ignis.git

  2. Việc tạo bản sao repository sẽ tạo nên một thư mục trên máy của bạn có tên qiskit-ignis.

    cd qiskit-ignis

  3. If you want to run tests or linting checks, install the developer requirements. This is not required to install or use the qiskit-ignis package when installing from source.

    pip install -r requirements-dev.txt

  4. Cài đặt Ignis.

    pip install .

Nếu bạn muốn cài đặt nó ở chế độ chỉnh sửa, nghĩa là việc thay đổi code trong dự án không yêu cầu việc cài đặt lại để áp dụng:

pip install -e .
