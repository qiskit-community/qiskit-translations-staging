Giới thiệu về Qiskit¶

When using Qiskit a user workflow nominally consists of following four high-level steps:

  • Build: Thiết kế một hoặc nhiều mạch lượng tử biểu diễn cho vấn đề bạn đang muốn giải quyết.

  • ** Compile**: Biên dịch các mạch cho một dịch vụ lượng tử cụ thể, ví dụ như một hệ lượng tử hoặc mô phỏng cổ điển.

  • ** Run**: Chạy các mạch đã được biên dịch trên dịch vụ lượng tử đã xác định. Các dịch vụ này có thể là dựa trên đám mây hoặc cục bộ.

  • ** Analyze**: Tính toán các chỉ số thống kê và hiển thị các kết quả thí nghiệm.

Đây là một ví dụ của toàn bộ luồng công việc, với mỗi bước được giải thích chi tiết ở các phần sau:

import numpy as np
from qiskit import QuantumCircuit, transpile
from qiskit.providers.aer import QasmSimulator
from qiskit.visualization import plot_histogram

# Use Aer's qasm_simulator
simulator = QasmSimulator()

# Create a Quantum Circuit acting on the q register
circuit = QuantumCircuit(2, 2)

# Add a H gate on qubit 0
circuit.h(0)

# Add a CX (CNOT) gate on control qubit 0 and target qubit 1
circuit.cx(0, 1)

# Map the quantum measurement to the classical bits
circuit.measure([0,1], [0,1])

# compile the circuit down to low-level QASM instructions
# supported by the backend (not needed for simple circuits)
compiled_circuit = transpile(circuit, simulator)

# Execute the circuit on the qasm simulator
job = simulator.run(compiled_circuit, shots=1000)

# Grab results from the job
result = job.result()

# Returns counts
counts = result.get_counts(compiled_circuit)
print("\nTotal count for 00 and 11 are:",counts)

# Draw the circuit
circuit.draw()

plot_histogram(counts)

Từng bước của luồng công việc¶

Chương trình trên có thể được chia thành sáu bước:

  1. Khai báo sử dụng các gói

  2. Khởi tạo các biến

  3. Thêm các cổng lượng tử

  4. Hiển thị mạch lượng tử

  5. Mô phỏng thí nghiệm

  6. Hiển thị kết quả

Bước 1: Khai báo sử dụng các gói¶

Những gói cơ bản cần thiết cho chương trình của bạn được khai báo sử dụng như sau:

import numpy as np
from qiskit import QuantumCircuit
from qiskit.providers.aer import QasmSimulator
from qiskit.visualization import plot_histogram

Chi tiết hơn, những thành phần được khai báo sử dụng gồm

  • QuantumCircuit: có thể được coi như các chỉ dẫn cho hệ thống lượng tử. Nó nắm giữ mọi hoạt động lượng tử.

  • ` ` QasmSimulator ` `: là trình mô phỏng mạch hiệu suất cao Aer.

  • plot_histogram: hiển thị biểu đồ.

Bước 2 : Khởi tạo các biến¶

Xem xét dòng code tiếp theo

circuit = QuantumCircuit(2, 2)

Ở dòng này, bạn khởi tạo 2 qubit với trạng thái 0; với 2 bit cổ điển cũng đặt giá trị 0; và mạch có nghĩa là mạch lượng tử.

Cú pháp:

  • QuantumCircuit(int, int)

Bước 3: Thêm cổng¶

Bạn có thể thêm các cổng (thao tác) để điều khiển các thanh ghi trong mạch lượng tử của bạn.

Xem xét ba dòng code sau:

circuit.h(0)
circuit.cx(0, 1)
circuit.measure([0,1], [0,1])

Các cổng lần lượt được thêm vào mạch lượng tử để tạo nên trạng thái Bell

\[|\psi\rangle = \left(|00\rangle+|11\rangle\right)/\sqrt{2}.\]

Đoạn code ở trên áp dụng các cổng sau:

  • QuantumCircuit.h(0): cổng Hadamard \(H\) tác động lên qubit số 0, làm nó trở thành trạng thái chồng chập.

  • QuantumCircuit.cx(0, 1): A controlled-Not operation (\(CNOT\)) on control qubit 0 and target qubit 1, putting the qubits in an entangled state.

  • QuantumCircuit.measure([0,1], [0,1]): nếu chúng ta tiến hành đo trên toàn bộ qubits và bit cổ điển, kết quả đo trên qubit thứ i sẽ được lưu vào bit cổ điển thứ i.

Bước 4: Hiển thị mạch lượng tử¶

Bạn có thể sử dụng qiskit.circuit. QuantuCircuit.draw() để xem mạch lượng tử bạn đã thiết kế dưới nhiều hình thức khác nhau, được sử dụng trong các tài liệu và nghiên cứu khoa học.

circuit.draw()

     ┌───┐     ┌─┐   
q_0: ┤ H ├──■──┤M├───
     └───┘┌─┴─┐└╥┘┌─┐
q_1: ─────┤ X ├─╫─┤M├
          └───┘ ║ └╥┘
c: 2/═══════════╩══╩═
                0  1

Trong mạch lượng tử này, các qubit được sắp xếp theo thứ tự với qubit số 0 ở phía trên và qubit số 1 ở phía dưới. Mạch lượng tử được đọc từ trái sang phải, có nghĩa là các cổng được tác động lên các qubits trước hay sau theo thứ tự từ trái sang phải.

Phần backend mặc định cho hàm QuantumCircuit.draw() hoặc qiskit.visualization.circuit_drawer() là text backend. Tuy nhiên, tùy vào môi trường trên máy của mình bạn có thể thay đổi cấu hình mặc định này cho phù hợp với nhu cầu của bạn hơn. Điều này thể được làm bằng cách thay đổi file config của người dùng. Theo mặc định file này được đặt ở ~/.qiskit/settings.conf và có định dạng .ini.

Ví dụ, file settings.conf để cài đặt công cụ vẽ cho thư viện Matplotlib là:

[default]
circuit_drawer = mpl

Bạn cũng có thể sử dụng bất kì backend vẽ mạch đúng chuẩn như giá trị cho config, nó bao gồm text, mpl, latex và latex_source.

Bước 5: Mô phỏng thí nghiệm¶

Qiskit Aer là một framework mô phỏng hiệu năng cao cho các mạch lượng tử. Nó cung cấp một vài backend để đạt được các mục tiêu mô phỏng khác nhau.

Nếu bạn có các vấn đề cài đặt Aer, bạn có thể sử dụng Basic Aer bằng cách thay Aer bằng BasicAer. BasicAer đươc tích hợp trong Qiskit Terra.

import numpy as np
from qiskit import QuantumCircuit, transpile
from qiskit.providers.basicaer import QasmSimulatorPy
...

Để mô phỏng mạch lượng tử này, bạn sẽ sử dụng qasm_simulator. Mỗi lần chạy mô phỏng mạch này sẽ cho ra kết quả là chuỗi 00 hoặc 11.

simulator = QasmSimulator()
compiled_circuit = transpile(circuit, simulator)
job = simulator.run(compiled_circuit, shots=1000)
result = job.result()
counts = result.get_counts(circuit)
print("\nTotal count for 00 and 11 are:",counts)


Total count for 00 and 11 are: {'00': 514, '11': 486}

Như dự kiến, kết chuỗi bit đầu ra 00 là xấp xỉ 50% thời gian. Số lần chạy mạch có thể được xác định thông qua sô ́ shots thực thi của phương thức execute. Số shots của mô phỏng được đặt là 1000 (mặc định là 1024).

Once you have a result object, you can access the counts via the method get_counts(circuit). This gives you the aggregate outcomes of the experiment you ran.

Bước 6: Hiển thị kết quả¶

Qiskit provides many visualizations,

bao gồm hàm plot_histogram, để xem kết quả của bạn.

plot_histogram(counts)

The observed probabilities \(Pr(00)\) and \(Pr(11)\) are computed by taking the respective counts and dividing by the total number of shots.

Ghi chú

Try changing the shots keyword in the run() method to see how the estimated probabilities change.
