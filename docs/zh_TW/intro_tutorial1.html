Qiskit的簡介¶

在使用 Qiskit 時，使用者工作流程名義上是由下列四個高階步驟組成：

  • 構建： 設計一個（或多於一個）量子電路即代表你正在考慮的問題。

  • 編譯： 為特定的量子服務編譯電路，例如一個量子系統或經典的模擬器。

  • **執行 **：在指定的量子服務上執行已編譯的電路。 這些服務可以是雲端型或本端服務。

  • **分析 **：計算摘要統計量，並視覺化實驗的結果。

以下是整個工作流程的範例，每一個步驟在下一節中詳細說明：

import numpy as np
from qiskit import QuantumCircuit, transpile
from qiskit.providers.aer import QasmSimulator
from qiskit.visualization import plot_histogram

# Use Aer's qasm_simulator
simulator = QasmSimulator()

# Create a Quantum Circuit acting on the q register
circuit = QuantumCircuit(2, 2)

# Add a H gate on qubit 0
circuit.h(0)

# Add a CX (CNOT) gate on control qubit 0 and target qubit 1
circuit.cx(0, 1)

# Map the quantum measurement to the classical bits
circuit.measure([0,1], [0,1])

# compile the circuit down to low-level QASM instructions
# supported by the backend (not needed for simple circuits)
compiled_circuit = transpile(circuit, simulator)

# Execute the circuit on the qasm simulator
job = simulator.run(compiled_circuit, shots=1000)

# Grab results from the job
result = job.result()

# Returns counts
counts = result.get_counts(compiled_circuit)
print("\nTotal count for 00 and 11 are:",counts)

# Draw the circuit
circuit.draw()

# Plot a histogram
plot_histogram(counts)

工作流程逐步簡介¶

上面的程式可以拆解成六個步驟：

  1. 匯入套件

  2. 起始設定變數

  3. 新增閘道

  4. 視覺化電路

  5. 模擬實驗

  6. 視覺化結果

步驟 1 ：匯入套件¶

您的程式所需要的基本元素會被匯入如下：

import numpy as np
from qiskit import QuantumCircuit
from qiskit.providers.aer import QasmSimulator
from qiskit.visualization import plot_histogram

在更詳細的情況下，匯入為

  • 」 QuantumCircuit``：可以理解為量子系統的指示。 它擁有你所有的量子操作。

  • ` QasmSimulator`：是 Aer 高效能的電路模擬器。

  • plot_histrgram：建立直方圖。

步驟 2 ：起始設定變數¶

考慮下一行的程式碼

circuit = QuantumCircuit(2, 2)

在這裡，您正在起始設定2個狀態為零的量子化元 ；其中 2 個經典位元設為零；而 circuit 則是量子電路。

語法：

  • QuantumCircuit(int, int)

步驟 3 ：新增閘道¶

您可以新增閘道（操作）來控制電路的暫存器。

請考慮下列三行程式碼：

circuit.h(0)
circuit.cx(0, 1)
circuit.measure([0,1], [0,1])

這些閘道被逐個加入到電路中以形成貝爾態

\[|\psi\rangle = \left(|00\rangle+|11\rangle\right)/\sqrt{2}.\]

上述程式碼會套用到下列閘道：

  • QuantumCircuit.h(0): Hadamard Gate \(H\) 作用於 qubit 0，令它進入 疊加態。

  • QuantumCircuit.cx(0, 1): 控制反 (controlled-Not)(\(CNOT\))運算作用於控制位元qubit 0及目標位元 qubit 1，讓它們進入 糾纏態。

  • QuantumCircuit.measure([0,1], [0,1]): if you pass the entire quantum and classical registers to measure, the ith qubit’s measurement result will be stored in the ith classical bit.

步驟 4 ：視覺化電路¶

你可以使用 :meth:`qiskit.circuit.QuantumCircuit.draw`來檢視你從教科書及研究文章中使用以不同形態所設計的電路。

circuit.draw()

     ┌───┐     ┌─┐   
q_0: ┤ H ├──■──┤M├───
     └───┘┌─┴─┐└╥┘┌─┐
q_1: ─────┤ X ├─╫─┤M├
          └───┘ ║ └╥┘
c: 2/═══════════╩══╩═
                0  1

在此電路中，位元的排列會以qubit 0在頂端 ，並qubit 1在底端。 電路是從左到右讀取的，這意味著在電路中較早應用的閘會顯示由左至右。

QuantumCircuit.draw() or qiskit.visualization.circuit_drawer()``的預設後端為文字後端。視乎您的本端環境而定， 您可能想要將這些預設值變更為更適合使用案例的預設值。這會使用user config file來完成。user config file應預設下位於``~/.qiskit/settings.conf 而且是``.ini``的檔案。

例如，設定 Matplotlib 繪圖的 settings.conf 檔是：

[default]
circuit_drawer = mpl

您可以使用任何有效的電路繪圖後端作為此配置的值，這包括文字、 mpl、latex和 latex_source。

步驟 5 ：模擬實驗¶

Qiskit Aer 是一個用於量子電路的高效能模擬器架構。 它提供 severl backends ，以達成不同的模擬目標。

如果您有安裝 Aer 的問題，您可以替換 Aer with BasicAer，替代使用 Basic Aer 提供者。Basic Aer 則包含在 Qiskit Terra中。

import numpy as np
from qiskit import QuantumCircuit, transpile
from qiskit.providers.basicaer import QasmSimulatorPy
...

如果要模擬此電路，您將使用 ``qasm_simulator ``，此電路的每一個執行都會產生位元字串 00 或 11。

simulator = QasmSimulator()
compiled_circuit = transpile(circuit, simulator)
job = simulator.run(compiled_circuit, shots=1000)
result = job.result()
counts = result.get_counts(circuit)
print("\nTotal count for 00 and 11 are:",counts)


Total count for 00 and 11 are: {'11': 538, '00': 462}

如預期一樣，輸出位元字串約為 00 時的 50 % 。 可以透過 execute 方法的 參數``shots ` 來指定電路執行的次數。 模擬的參數次數設定為 1000（預設值為 1024）。

當您有 result'` 物件之後，您可以透過方法 ``get_counts(circuit) 存取數量。 這會給您所執行實驗的聚集結果。

步驟 6 ：視覺化結果¶

Qiskit 提供 many visualizations,

including the function plot_histogram, to view your results.

plot_histogram(counts)

The observed probabilities \(Pr(00)\) and \(Pr(11)\) are computed by taking the respective counts and dividing by the total number of shots.

備註

Try changing the shots keyword in the run() method to see how the estimated probabilities change.
