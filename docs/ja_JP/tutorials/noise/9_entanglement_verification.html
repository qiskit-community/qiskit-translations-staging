エンタングルメント検証¶

GHZ 状態の概要¶

Greenberger-Horne-Zeilinger (GHZ) 状態は、次の状態ベクトルで定義された \(n\)-qubit のエンタングル状態です。

\[|{\rm GHZ} \rangle = \frac{|0 \rangle ^{\otimes n} + |1 \rangle ^{\otimes n}}{\sqrt{2}} , n > 2\]

GHZ状態の特性は、複数量子ビットの相互作用を評価するのにとても役立ちます。そして、その堅牢性は大規模な量子コンピュータの開発での鍵となります。

量子状態の特徴¶

任意の混合量子状態は、 \(\rho = \sum_{i} p_i |\psi_{i} \rangle \langle \psi_{i}|\) として定義された密度行列で識別できます。このとき \(|\psi_{i} \rangle\) は混合状態を形成する純粋状態を表し、 \(0 < p_i \le 1\), \(\sum_{i} p_i = 1\) は \(|\psi_{i} \rangle\) の状態をとる古典の確率を示しています。理想的なGHZ状態の純粋密度行列を \(\rho_{p} \equiv |{\rm GHZ} \rangle \langle {\rm GHZ}|\) で表します。この行列が実験で生成されたGHZ状態の密度行列 \(\rho_{T}\) にどれだけ近いかを見るために、この類似性を定量化する一つの方法として、状態間の忠実度 \(F(\rho_{p},\rho_{T})\) を計算します。

\[F = \Big[ Tr \sqrt{\sqrt{\rho_{p}}\rho_{T}\sqrt{\rho_{p}}} \Big] ^{2}\]

このチュートリアルの目的は2つあります。 GHZ 状態を特徴付ける方法と、特性評価方法に関係なく、 Ignis の誤り軽減ツールを使用して読み出しの忠実度を向上させる方法を探ります。

GHZ状態の準備¶

まず、GHZ状態を準備する方法について説明します。

すべて \(|0\rangle\) 状態で準備された \(n\) 量子ビットのシステムがあるとします。

\[|\psi \rangle = |00...0\rangle\]

最初の量子ビットにアダマールゲートを適用します： \(|0\rangle \longrightarrow \frac{1}{\sqrt{2}}(|0\rangle + |1\rangle)\) 。状態は次のようになります。

\[|\psi \rangle = \frac{|00...0\rangle + |10...0\rangle}{\sqrt{2}}\]

この状態に、 \(n = 0 \ldots n-1\) の \(n\) 番目と \((n+1)\) 番目の量子ビットの間に \(n-1\) CNOTゲートのシーケンスを適用すると、 \(n\) 番目が \(|0\rangle\) の場合は \(n+1\) 番目の量子ビットは \(|0\rangle\) のまま、 \(n\) 番目が \(|1\rangle\) の場合は \(|1\rangle\) になるGHZ状態が作成されます。

\[|GHZ \rangle = \frac{|00...0\rangle + |11...1\rangle}{\sqrt{2}} \equiv \frac{|0 \rangle ^{\otimes n} + |1 \rangle ^{\otimes n}}{\sqrt{2}}\]

次の関数は、GHZ状態を測定できる “線形” 回路を生成します。

特性、Part I¶

マルチ量子コヒーレンス（Multiple Quantum Coherence、MQC）¶

多重量子コヒーレンス(MQC)は、 \(n\) 量子ビットGHZ状態の事前準備を行い、各量子ビット状態をz軸周りに位相 \(\phi\) で回転させることで動作します。その後、Xゲート、すなわちX軸周りに \(\pi\) パルスを適用します。その後、最初に適用した演算の逆を適用して、GHZ状態を得ます。理想的な状態では終状態は次のようになります。 \(|\psi \rangle = \frac{|0 \rangle ^{\otimes n} + e^{i n \phi}|1 \rangle ^{\otimes n}}{\sqrt{2}}\) 理想的には \(|\psi \rangle\) を \(|0 \rangle ^{\otimes n}\) 状態に射影して、位相を観測することができます。この技術は、エコーシーケンスを連想させるもので、読み出し時の忠実度を大幅に向上させることが示されています。

以下の関数は、線形MQC回路を生成します。これ以降のすべての回路と同様に、 full_measurement 引数を変更して、すべての量子ビットの完全な測定と制御量子ビットのみの測定を切り替えることができます。完全な測定によって最も正確な結果が得られますが、7量子ビットを超える場合はfalseに設定し、 '0' 状態と '1' 状態の間の振動のみを観察することをお勧めします。

このMQC回路で実験を行った後、 \(\phi\) を \(0\) から \(2 \pi\) までスイープしたときの振動を観察するための状態を選択できます。理論的には、信号は \(S(\phi) = \frac{1}{2}(1+\cos(n \phi))\) に従う必要があります。 次に、離散フーリエ変換（DFT： \(I_{v}=(1/N)|\sum_{\phi}e^{iv\phi}S(\phi)\) ）を実行して、境界 \(2\sqrt{I_{n}} \leq F \leq \sqrt{I_{0}/2}+ \sqrt{I_{n}}\) で定義される状態の忠実度を抽出します。必要に応じて、忠実度の実際の値を取得できます： \(F = \frac{1}{2}(P_{00...0}+P_{11...1})+\sqrt{I_{n}})\) （arXiv：1905.05720）。

パリティ振動¶

GHZ状態を特徴づけるために使用する次の方法は、パリティ振動です。GHZ状態を準備した後、x軸とy軸を中心とした回転の組み合わせを適用して、 \(\phi\) ： \(U(\phi) = \otimes_{j}^{N} e^{i\frac{\pi}{4}(\cos(\phi)\sigma_{x}^{j}+\sin(\phi)\sigma_{y}^{j})}\) の関数としてさまざまな重ね合わせ状態を作成します。次に、期待値 \(\langle \otimes_{j}^{N} \sigma_{z}^{j} \rangle_{\phi}\) を \(\phi\) の関数として測定します。これにより、理論的には1と-1の間のパリティ振動が発生するはずです。

次の関数は、上記のMQC回路と同等のパリティ発振である回路を生成します。

[8]:

circ_po = get_ghz_po_para(qn)

\(F = \frac{1}{2}(P_{00...0}+P_{11...1}+C)\) からパリティ振動 \(S_{\phi}\) の忠実度を取得できます。ここでの \(C\) について、コヒーレンスは \(2\sqrt{I_{n}}\) として定義され、MQCメソッドの場合と同じDFTの規則に従います。

トモグラフィー¶

トモグラフィーは、多くの名目上同一の状態を生成し、異なる基底の状態インスタンスを測定することによって密度行列を測定します。GHZ状態の理想的な結果は、他のすべての要素が消失するテンソル積基底の4つのコーナーで4つの等しい密度行列要素である。この方法で忠実度を容易に計算することができますが、この方法は遅く（指数関数的な数の測定を必要とします）、nが7よりも大きい場合には法外に長い時間がかかります。とはいえ、この方法は少数の量子ビットに対しても有効であるため、以下にその方法を紹介します。

回路の並列化¶

上記の「線形」回路は、シミュレーションを実行するのに適していますが、実際のデバイスを使用するときはどうすれば良いでしょうか。各システムは任意のトポロジーをもち、さまざまなエラーが発生するため、どうすればよいでしょうか。 ここではAer シミュレーションだけではなく、実際のハードウェアをターゲットにしています。 実際のハードウェアに起因する影響を減らすための 1 つの方法は、 CNOT ゲートを並列化することで、より短い回路を作成することです。 これは効率の観点からも忠実度の観点からも非常に有益です。モジュール parallelize のクラス BConfig はまさにこれを行います。

まず、使用するバックエンドを最適なものに設定しなければなりません。ここではシミュレーションバックエンドである 'ibmq_qasm_simulator' を実際のデバイスである 'ibmq_16_melbourne' に見立てて、モックバックエンドである 'FakeMelbourne' を使用します。

[10]:

from qiskit.providers.aer import QasmSimulator
from qiskit.test.mock import FakeMelbourne
backend = QasmSimulator()
backend_hardware = FakeMelbourne()

noise モジュールを使用して、'ibmq_16_melbourne' から 'ibmq_qasm_simulator' への割り当てまでノイズモデルを定義することができます。

[11]:

noise_model = noise.NoiseModel.from_backend(backend_hardware)
coupling_map = backend_hardware._configuration.coupling_map
basis_gates=noise_model.basis_gates

これで完成です。今後のチュートリアルでは、シミュレーションではなく実デバイスを使う場合は、 noise_model と coupling_map の記述と割り当てをすべて削除し、実デバイスを backend に割り当ててください。これから使うシミュレータは、実デバイスを動かすことに代わるものではありません。

BConfig は、並列化された回路を作成するための設計図を示します。 先ほど定義した実際のデバイスを取り込んだオブジェクトを初期化し、protocirc という名前を付けましょう。すべての実験で使用します:

[12]:

protocirc = BConfig(backend_hardware)

エラー軽減¶

Qiskit Ignisは、生データを取得して較正済みデータを返すための非常に正確なツールを提供しています。これは，生データをベクトル \(v_{raw}\) の形式で取得し，較正行列 \(A\) を取得することで行われます．出力は，最適化問題の解となります： \(argmin_{v_{cal}}}．||Av_{cal}-v_{raw}||^{2}\)

ここで、命令用の \(\otimes _{j}^{N} \sigma_z^{j}\) 行列を作成しますが、これは次のメソッド entanglement.analysis.composite_pauli_z_expvalue() ですでに考慮されています。

[37]:

composite_sigma_z = sigma_z = np.array([[1, 0],[0, -1]])
for a in range(1,qn):
    composite_sigma_z = np.kron(composite_sigma_z,sigma_z)

ここで、カウントのリストが \(\otimes _{j}^{N} \sigma_z^{j}\) の状態と一致するように正しく順序付けられていることを確認します。これにより、 \(\langle \otimes _{j}^{N} \sigma_z^{j} \rangle\) の計算は、この順序付けられたリストの対角線が \(\otimes _{j}^{N} \sigma_z^{j}\) の内積を取るのと同じくらい簡単になります。 composite_pauli_z_expvalue 関数はまさにそれを行います。 回路を取り、状態ベクトルカウントを適切に順序付けます。 この内積を \(\phi\) の関数としてプロットして、パリティの振動を観察できます。

トモグラフィーに関する注記

5量子ビットを超える量子トモグラフィーを実行しないでください。

結論¶

結論として、エラーを軽減することなく、MQCがGHZ状態を特徴付ける優れた方法であることがわかります。 ただし、エラーを軽減することで、すべてのメソッドは、少なくとも少数の量子ビットに対して、はるかに高い忠実度を達成でき、すべてほぼ同じ値になります。 より正確な結果を得るには、実際のデバイスを使用する以外に、ショット数を4倍から8倍に増やす価値があります。 このノートブックで使用されている並列化された回路が、忠実度に関して線形化された回路とどのように機能するかを比較する価値があるかもしれません。
