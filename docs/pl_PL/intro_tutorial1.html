Wprowadzenie do Qiskit¶

Schemat pracy z Qiskit zazwyczaj składa się z następujących czterech ogólnych kroków:

  • Budowanie: Zaprojektuj obwód (lub obwody) kwantowe, który przedstawia interesujący cię problem.

  • Kompilacja: Skompiluj obwody dla konkretnego środowiska kwantowego, np. komputera kwantowego lub klasycznego symulatora.

  • Uruchamianie: Uruchom skompilowane obwody w wybranym środowisku kwantowym. Środowisko może znajdować się w chmurze lub lokalne.

  • Analizowanie: Oblicz statystyki podsumowujące i wizualizuj wyniki eksperymentów.

Poniżej przedstawiono przykład schematu działania, gdzie każdy krok objaśniono szczegółowo w kolejnych sekcjach:

import numpy as np
from qiskit import QuantumCircuit, transpile
from qiskit.providers.aer import QasmSimulator
from qiskit.visualization import plot_histogram

# Use Aer's qasm_simulator
simulator = QasmSimulator()

# Create a Quantum Circuit acting on the q register
circuit = QuantumCircuit(2, 2)

# Add a H gate on qubit 0
circuit.h(0)

# Add a CX (CNOT) gate on control qubit 0 and target qubit 1
circuit.cx(0, 1)

# Map the quantum measurement to the classical bits
circuit.measure([0,1], [0,1])

# compile the circuit down to low-level QASM instructions
# supported by the backend (not needed for simple circuits)
compiled_circuit = transpile(circuit, simulator)

# Execute the circuit on the qasm simulator
job = simulator.run(compiled_circuit, shots=1000)

# Grab results from the job
result = job.result()

# Returns counts
counts = result.get_counts(compiled_circuit)
print("\nTotal count for 00 and 11 are:",counts)

# Draw the circuit
circuit.draw()

# Plot a histogram
plot_histogram(counts)

Workflow Step–by–Step¶

Powyższy program można podzielić na sześć etapów:

  1. Importowanie pakietów

  2. Inicjalizacja zmiennych

  3. Dodawanie bramek

  4. Wizualizacja obwodu

  5. Symulowanie eksperymentu

  6. Wizualizacja wyników

Krok 1: Importowanie pakietów¶

Podstawowe elementy wymagane przez program są importowane w następujący sposób:

import numpy as np
from qiskit import QuantumCircuit
from qiskit.providers.aer import QasmSimulator
from qiskit.visualization import plot_histogram

Importujemy pakiety:

  • QuantumCircuit: można potraktować jako instrukcje do systemu kwantowego. Zawiera wszystkie Twoje operacje kwantowe.

  • QasmSimulator: jest symulatorem obwodu o wysokiej wydajności modułu Aer.

  • plot_histogram: tworzy histogramy.

Krok 2: Inicjalizacja zmiennych¶

Spójrzmy na następującą linię kodu

circuit = QuantumCircuit(2, 2)

Inicjalizujemy dwa kubity w stanie 0 oraz dwa klasyczne bity w stanie zero. circuit jest obwodem kwantowym.

Składnia:

  • QuantumCircuit(int, int)

Krok 3: Dodawanie bramek¶

Możesz dodawać bramki (operacje), aby manipulować rejestrem obwodu.

Spójrzmy na następujące trzy linie kodu:

circuit.h(0)
circuit.cx(0, 1)
circuit.measure([0,1], [0,1])

Bramki są dodawane do obwodu jedna po drugiej, tworząc stan Bell’a

\[|\psi\rangle = \left(|00\rangle+|11\rangle\right)/\sqrt{2}.\]

Powyższy kod stosuje następujące bramki:

  • QuantumCircuit.h(0): Bramka Hadamarda \(H\) dla kubitu 0, która powoduje jego przejście w stan superpozycji.

  • QuantumCircuit.cx(0, 1): bramka kontrolowanej negacji (\(CNOT\)) na kubicie kontrolnym 0 i kubicie docelowym 1, kubity zostają wprowadzone w stan splątany.

  • QuantumCircuit.measure([0,1], [0,1]): jeśli przekażesz rejestr kwantowy i klasyczny do measure, wynik pomariu i-tego kubitu zostanie zapisany w i-tym klasycznym bicie.

Krok 4: Wizualizacja obwodu¶

Możesz użyć :meth:` qiskit.circuit.QuantumCircuit.draw ` aby zobaczyć obwód, który zaprojektowałeś w różnych formach stosowanych w wielu podręcznikach i artykułach naukowych.

circuit.draw()

     ┌───┐     ┌─┐   
q_0: ┤ H ├──■──┤M├───
     └───┘┌─┴─┐└╥┘┌─┐
q_1: ─────┤ X ├─╫─┤M├
          └───┘ ║ └╥┘
c: 2/═══════════╩══╩═
                0  1

W powyższym obwodzie kubity są uporządkowane, kubit 0 na górze oraz kubit 1 na dole. Obwód należy czytać od lewej do prawej, co oznacza, że bramki zastosowane jako pierwsze znajdują się z lewej strony.

Domyślny backend dla ` `QuantumCircuit.draw () ` ` lub ` `qiskit.visualization.circuit_drawer () ` ` backend tekstowy. Jednak w zależności od środowiska lokalnego, można zmienić ustawienia domyślne na bardziej odpowiednie dla danej sytuacji. Zmianę można wprowadzić przy użyciu pliku konfiguracyjnego użytkownika. Domyślnie plik konfiguracyjny użytkownika powinien znajdować się w ` `~/.qiskit/settings.conf ` ` ` i ma format ` `.ini ` `.

Na przykład plik settings.conf do ustawień wyjścia Matplotlib to:

[default]
circuit_drawer = mpl

Możesz użyć dowolnego z dostępnych formatów wyjściowych jako wartości dla tej konfiguracji, m.in. text, mpl, latex, i latex_source.

Krok 5: Symulowanie eksperymentu¶

Qiskit Aer is a high performance simulator framework for quantum circuits. It provides several backends to achieve different simulation goals.

Jeśli masz problemy z instalacją Aer, możesz alternatywnie użyć symulatora Basic Aer, zastępując ` Aer ` ` zwrotem ` BasicAer `. Basic Aer jest zawarty w Qiskit Terra.

import numpy as np
from qiskit import QuantumCircuit, transpile
from qiskit.providers.basicaer import QasmSimulatorPy
...

Do symulacji obwodu należy użyć ` ` qasm_simulator ` `. Każde uruchomienie tego obwodu będzie skutkować uzyskaniem wyniku w formie ciągu bitów 00 lub 11.

simulator = QasmSimulator()
compiled_circuit = transpile(circuit, simulator)
job = simulator.run(compiled_circuit, shots=1000)
result = job.result()
counts = result.get_counts(circuit)
print("\nTotal count for 00 and 11 are:",counts)


Total count for 00 and 11 are: {'00': 493, '11': 507}

Zgodnie z oczekiwaniami, wartość wyjściowa dla ciągu bitów jest równa 00 w około 50% przypadków. Liczba uruchomień obwodu może być określona za pomocą argumentu shots metody execute. Liczba powtórzeń symulacji została ustawiona na 1000 (domyślnie 1024).

Mając obiekt result, możemy otrzymać wyniki poprzez funkcję get_counts(circuit). Zwraca ona zagregowane wyniki dla danego obwodu.

Krok 6: Wizualizacja wyników¶

Qiskit zapewnia wiele form wizualizacji,

including the function plot_histogram, to view your results.

plot_histogram(counts)

The observed probabilities \(Pr(00)\) and \(Pr(11)\) are computed by taking the respective counts and dividing by the total number of shots.

Informacja

Try changing the shots keyword in the run() method to see how the estimated probabilities change.
