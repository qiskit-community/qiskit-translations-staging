Qiskit 简介¶

用户在使用 Qiskit 时，操作流程通常由以下四个高阶步骤组成：

  • Build：为目标问题设计一个（或多个）量子电路。

  • Compile：为特定的量子服务编译电路。例如：量子系统或经典模拟器。

  • Run：在指定的一个（或多个）量子服务上运行编译的电路。这些服务可能是基于云端的或本地的。

  • Analyze：计算汇总统计数据，并将实验结果可视化。

以下是整个操作流程的示例，每个步骤在后续小节中详细说明：

import numpy as np
from qiskit import QuantumCircuit, transpile
from qiskit.providers.aer import QasmSimulator
from qiskit.visualization import plot_histogram

# Use Aer's qasm_simulator
simulator = QasmSimulator()

# Create a Quantum Circuit acting on the q register
circuit = QuantumCircuit(2, 2)

# Add a H gate on qubit 0
circuit.h(0)

# Add a CX (CNOT) gate on control qubit 0 and target qubit 1
circuit.cx(0, 1)

# Map the quantum measurement to the classical bits
circuit.measure([0,1], [0,1])

# compile the circuit down to low-level QASM instructions
# supported by the backend (not needed for simple circuits)
compiled_circuit = transpile(circuit, simulator)

# Execute the circuit on the qasm simulator
job = simulator.run(compiled_circuit, shots=1000)

# Grab results from the job
result = job.result()

# Returns counts
counts = result.get_counts(compiled_circuit)
print("\nTotal count for 00 and 11 are:",counts)

# Draw the circuit
circuit.draw()

# Plot a histogram
plot_histogram(counts)

操作流程分步讲解¶

上面的程序可以分为六个步骤:

  1. 导入包

  2. 初始化变量

  3. 添加门（操作）

  4. 可视化电路

  5. 模拟实验

  6. 可视化结果

步骤 1 ：导入包¶

您的程序所需的基本元素如下导入：

import numpy as np
from qiskit import QuantumCircuit
from qiskit.providers.aer import QasmSimulator
from qiskit.visualization import plot_histogram

具体来说，这些导入是

  • QuantumCircuit ：可以被视为量子系统的指令。它包您需要的所有量子操作指令。

  • QasmSimulator：是 Aer 高性能电路模拟器。

  • plot_histogram ：创建直方图。

步骤 2 ：初始化变量¶

仔细观察下面一行代码

circuit = QuantumCircuit(2, 2)

在这里，您将在零状态下初始化 2 个量子位; 将 2 个经典位设置为零；circuit 是量子电路。

语法：

  • QuantumCircuit(int, int)

步骤 3 ：添加门（操作）¶

您可以添加门（操作）来操纵您的电路寄存器。

仔细观察下面三行代码

circuit.h(0)
circuit.cx(0, 1)
circuit.measure([0,1], [0,1])

这些门被逐一添加到电路中形成贝尔状态

\[|\psi\rangle = \left(|00\rangle+|11\rangle\right)/\sqrt{2}.\]

上面的代码使用到以下门电路：

  • QuantumCircuit.h(0) ：一个 Hadamard 门 \(H\) 处于量子位 $q_{0}$ 上，它会使量子位 $q_{0}$ 变成一个**叠加状态**。

  • QuantumCircuit.cx(0, 1)：在控制量子位 0 和目标量子位 1 进行Controlled-NOT操作（\(CNOT\)），使得它们进入**纠缠态**。

  • QuantumCircuit.measure([0,1], [0,1])：如果将整个量子和经典寄存器传递到 measure ，那么第 i 个量子位的测量结果将被存储在第 i 个经典位中。

步骤 4 ：可视化电路¶

您可以使用 qiskit.circuit.QuantumCircuit.draw() 来查看电路，这些电路在许多教程和科研文章中形态各异。

circuit.draw()

     ┌───┐     ┌─┐   
q_0: ┤ H ├──■──┤M├───
     └───┘┌─┴─┐└╥┘┌─┐
q_1: ─────┤ X ├─╫─┤M├
          └───┘ ║ └╥┘
c: 2/═══════════╩══╩═
                0  1

在此电路中，量子位是自上而下按顺序排列的，量子位 $q_{0}$ 在顶端，量子位 $q_{1}$ 在底端。电路是从左至右读取，这意味着在电路中显示在左侧门会先被使用。

QuantumCircuit.draw() 或 qiskit.visualization.circuit_drawer() 的默认后端是字符画后端。 但是，根据您的本地环境，您可能希望将这些缺省值更改为更适合您用例的配置。 这时您可以通过修改用户配置文件来实现。 默认情况下，用户配置文件将位于 ~/.qiskit/settings.conf ，并且是一个 .ini 文件。

例如，用于设置 Matplotlib 绘制器的 settings.conf 文件为：

[default]
circuit_drawer = mpl

您可以使用任何有效的电路绘制器后端作为此配置的值，其中包括 text、mpl、latex 和 latex_source。

步骤 5 ：模拟实验¶

Qiskit Aer 是量子电路的高性能模拟器框架。 它提供 多种后端 ，以实现不同的模拟目标。

如果您在安装 Aer 时遇到问题，您也可以通过使用 BasicAer 来替换 Aer。Basic Aer 已经包含在 Qiskit Terra 中。

import numpy as np
from qiskit import QuantumCircuit, transpile
from qiskit.providers.basicaer import QasmSimulatorPy
...

为了模拟此电路，您将使用 qasm_simulator。每次运行此电路都会生成位字符串00或11。

simulator = QasmSimulator()
compiled_circuit = transpile(circuit, simulator)
job = simulator.run(compiled_circuit, shots=1000)
result = job.result()
counts = result.get_counts(circuit)
print("\nTotal count for 00 and 11 are:",counts)


Total count for 00 and 11 are: {'00': 467, '11': 533}

正如预期的那样，输出位字符串约占总时间的50%。 电路运行的次数可以通过 execute 方法的 shots 参数指定。 模拟测量次数设置为 1000 (默认为 1024)。

当您有 result 对象之后，您可以通过函数 get_counts(circuit) 来访问计数。 这是您所做实验的总结果。

步骤 6 ：可视化结果¶

Qiskit 提供了 许多可视化 输出，

其中包括函数 plot_histogram 可以查看您的输出结果。

plot_histogram(counts)

\(Pr(00)\) 和 \(Pr(11)\) 的观测概率是通过分别计数并除以总测量次数来计算的。

备注

尝试修改 run() 方法中的 shots 关键字，以了解估计概率的变化。
