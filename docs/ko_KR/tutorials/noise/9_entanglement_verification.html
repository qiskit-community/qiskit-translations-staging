얽힘(Entanglement) 입증¶

GHZ 상태 소개¶

The Greenberger-Horne-Zeilinger (GHZ) 상태는 다음의 상태 벡터에 의해 가장 잘 정의된 \(n\)-큐비트가 얽힌 상태이다:

\[|{\rm GHZ} \rangle = \frac{|0 \rangle ^{\otimes n} + |1 \rangle ^{\otimes n}}{\sqrt{2}} , n > 2\]

GHz 상태의 특성화는 대규모 양자 컴퓨터 개발의 핵심인 다중 양자 상호작용을 평가하는데 매우 유용하다.

양자 상태 특성화¶

모든 혼합 양자 상태는 \(\rho = \sum_{i} p_i |\psi_{i} \rangle \langle \psi_{i}|\) 로 정의되는 밀도 행렬에 의해 식별될 수 있다. 여기서 \(|\psi_{i} \rangle\) 는 혼합물 및 \(0 < p_i \le 1\) 을 형성하는 순수한 양자 상태이고 , \(\sum_{i} p_i = 1\) 은 상태 \(|\psi_{i} \rangle\) 에 있는 고전적인 확률이다. 우리는 \(\rho_{p} \equiv |{\rm GHZ} \rangle \langle {\rm GHZ}|\) 가 이상적인 GHZ 상태의 순수 밀도 행렬을 나타낸다. 우리는 이 행렬이 실험 \(\rho_{T}\) 에서 생성된 GHZ 상태의 밀도 행렬에 얼마나 가까운지 보고 싶다. 이러한 유사성을 정량화하는 한 가지 방법은 상태 간의 충실도를 계산하는 것이다 \(F(\rho_{p},\rho_{T})\)

\[F = \Big[ Tr \sqrt{\sqrt{\rho_{p}}\rho_{T}\sqrt{\rho_{p}}} \Big] ^{2}\]

본 튜토리얼의 목표는 두 가지다: 우리는 GHZ 상태를 특성화할 수 있는 방법을 탐구할 것이며, 우리가 이그니스 (Ignis) 의 에러 완화 도구를 사용하여, 특성화 방법에 상관없이 리드아웃 충실도 (readout fidelity) 를 증가시킬 수 있다.

더 자세히 보기 전에 기본 Qiskit 에서 필요한 모든 것을 가져오도록 하자.

GHZ 상태 준비¶

먼저 GHZ 상태를 준비하는 방법을 먼저 살펴보자.

두 \(|0\rangle\) 큐비트 시스템이 있다고 가정해 보자.

\[|\psi \rangle = |00...0\rangle\]

하다마르 게이트를 처음으로 적용한다: \(|0\rangle \longrightarrow \frac{1}{\sqrt{2}}(|0\rangle + |1\rangle)\). 이제 우리의 주는 다음과 같다.

\[|\psi \rangle = \frac{|00...0\rangle + |10...0\rangle}{\sqrt{2}}\]

이 상태에서 \(n = 0 \ldots n-1\) 에 대한 \(n^{th}\) 와 \((n+1)^{th}\) 큐비트 사이의 \(n-1\) CNOT 게이트 시퀀스를 적용하면 \(n^{th}\) 가 \(|0\rangle\) 에 있는 경우 \(n+1\) 큐비트를 \(|0\rangle\) 에 남두고 에서, \(n^{th}\) 이 \(|1\rangle\) 에 있는 경우 \(|1\rangle\) 에서 GHZ 상태를 생성한다.

\[|GHZ \rangle = \frac{|00...0\rangle + |11...1\rangle}{\sqrt{2}} \equiv \frac{|0 \rangle ^{\otimes n} + |1 \rangle ^{\otimes n}}{\sqrt{2}}\]

The following function creates this “linear” circuit that can measure the GHZ state:

qn = 5 #creating a 5 qubit GHZ state

특성화, Part I¶

다중 양자 결맞음 (MQC)¶

복수 양자 결맞음 (MQC)는 \(n\) 큐비트 GHZ 상태의 예비 준비를 하고, 각 큐비트 상태들을 z축을 따라 위상 \(\phi\) 만큼 회전하는 식으로 작동한다. 그리고 나서, X 게이트 즉, x축을 따라 \(\pi\) 펄스를 작용한다. 그리고, 우리가 GHZ 상태를 얻기 위해 작용한 연산들의 역을 작용한다. 이상적으로 최종 상태는 \(|\psi \rangle = \frac{|0 \rangle ^{\otimes n} + e^{i n \phi}|1 \rangle ^{\otimes n}}{\sqrt{2}}\) 이다. 우리는 이상적으로 \(|\psi \rangle\) 를 상태 \(|0 \rangle ^{\otimes n}\) 에 투영시켜서 얻은 위상을 관측할 수 있다. 이 기술은 에코 시퀀스를 연상시키며, 판독 시 fidelity 를 크게 향상시키는 것으로 나타났다.

아래 함수는 선형 MQC 회로를 만든다. 여기서부터 모든 회로와 마찬가지로 full_measurement 인수를 변경하여 모든 퀀트의 전체 측정과 제어의 측정만을 측정할 수 있다. 전체 측정은 가장 정확한 결과를 산출하지만 7개 이상의 큐비트에 대해서는 이를 거짓으로 설정하고 0 과 1 상태 사이의 진동에서만 관찰하는 것이 좋다.

이 MQC 회로에 대한 실험을 실행한 후, 우리는 \(\phi\) 를 \(0\) 에서 \(2 \pi\) 로 스위프(sweep) 할 때 진동을 관찰할 상태를 선택할 수 있다. 이론적으로 우리의 신호는 \(S(\phi) = \frac{1}{2}(1+\cos(n \phi))\) 를 따라야 한다. 그런 다음 이산 푸리에 변환(DFT: \(I_{v}=(1/N)|\sum_{\phi}e^{iv\phi}S(\phi)\))을 수행하여 경계 \(2\sqrt{I_{n}} \leq F \leq \sqrt{I_{0}/2}+ \sqrt{I_{n}}\) 에 의해 정의된 상태의 충실도(Fidelity)를 추출한다. 원하는 경우, 충실도에 대한 실제 값은 얻을 수 있다. \(F = \frac{1}{2}(P_{00...0}+P_{11...1})+\sqrt{I_{n}})\) (arXiv:1905.05720).

패리티 진동(Parity Oscillations)¶

우리가 GHZ 상태를 특성화하기 위해 사용한 다음 방법은 패리티 진동이다. GHZ 상태를 준비한 이후, 우리는 \(\phi\) 의 함수 \(U(\phi) = \otimes_{j}^{N} e^{i\frac{\pi}{4}(\cos(\phi)\sigma_{x}^{j}+\sin(\phi)\sigma_{y}^{j})}\) 처럼 다양한 중첩 상태를 생성하기 위해 x축과 y축에 대해 다양한 조합의 회전을 적용했다. 그리고 나서 기댓값 \(\langle \otimes_{j}^{N} \sigma_{z}^{j} \rangle_{\phi}\) 를 \(\phi\) 의 함수로 측정했다. 이론적으로는 1과 -1 사이의 패리티 진동을 발생시켜야 한다.

다음 함수는 위에서 주어진 MQC 회로와 같은 패리티 진동 (Parity Oscillation) 과 등가인 회로를 생성한다.

우리는 \(F = \frac{1}{2}(P_{00...0}+P_{11...1}+C)\) 에서의 패리티 진동 \(S_{\phi}\) 의 fidelity 를 얻을 수 있다. 여기서, \(C\) 는 MQC 방법과 같이 DFT에도 같은 관습을 따라 \(2\sqrt{I_{n}}\) 로 정의된 결맞음이다.

단층촬영¶

단층 촬영은 많은 명목상 동일한 상태를 생성하고 상이한 염기에서 상태 인스턴스를 측정함으로써 밀도 매트릭스를 측정한다. GHZ 상태의 이상적인 결과는 텐서-생성물 기반의 4개의 코너에 있는 4개의 동일한 밀도 매트릭스 요소이며, 다른 모든 요소들이 소멸된다. 이러한 방법에 의해 충실도가 쉽게 계산될 수 있지만, 방법은 느리고 (n 에서 기하급수적인 수의 측정을 필요로 한다), n이 7보다 큰 경우에는 엄청나게 긴 시간이 걸린다. 그럼에도 불구하고, 이 메소드가 소수의 소수에 관련되어 있으므로 이 방법을 수행하는 방법을 아래에서 볼 수 있다.

병렬 처리 회로¶

위의 《선형》 회로는 시뮬레이션을 수행하는 데 좋지만 실제 장치를 사용할 때 수행하는 작업은 시스템에 임의의 토폴로지와 다양한 오류가 있을 수 있습니다. 저희는 단순히 에이서 시뮬레이션이 아닌 실제 하드웨어를 목표로 하고 있습니다. 실제-하드웨어 효과를 감소시키는 하나의 기술은 CNOT 게이트들을 병렬화하여 더 짧은 깊이 회로를 생성하는 것이다. 이는 매우 유익한 효율적인, 그리고 충실도 (fidelity wise) 가 될 수 있다. 모듈 병렬화 parallelize 의 BConfig 가 바로 이것이다.

먼저 사용하려는 최적의 백엔드를 구성해야 한다. 가상의 백엔드 'FakeMelbourne' 를 사용하여 시뮬레이션 백엔드인 'ibmq_qasm_simulator' 를 실제 장치 'ibmq_16_melbourne' 로 위장하여 를 고려할 것이다.

[10]:

from qiskit.providers.aer import QasmSimulator
from qiskit.test.mock import FakeMelbourne
backend = QasmSimulator()
backend_hardware = FakeMelbourne()

Using the noise module, we can now define a noise model from 'ibmq_16_melbourne' to “assign” to 'ibmq_qasm_simulator'.

[11]:

noise_model = noise.NoiseModel.from_backend(backend_hardware)
coupling_map = backend_hardware._configuration.coupling_map
basis_gates=noise_model.basis_gates

그리고 우리는 그것을 가지고 있다. 이제부터는 이 튜토리얼에서 시뮬레이션이 아닌 실제 기기를 사용할 때에는 noise_model 과 coupling_map 의 모든 언급과 과제를 꺼내어 실제 기기를 backend 에 할당한다. 지금부터 사용되는 시뮬레이터는 실제 디바이스를 실행하는 데 사용할 수 없다.

BConfig 는 병렬 회로를 만들기 위한 blueprint 를 마련한다. 우리가 방금 정의한 실제 장치를 사용한 개체를 초기화하고, protocirc 라고 명명하자. 우리의 모든 실험은 다음을 사용할 것이다.

[12]:

protocirc = BConfig(backend_hardware)

에러 경감¶

Qiskit Ignis는 원시 데이터를 가져오고 교정된 데이터를 리턴하는 매우 정확한 도구를 제공한다. 이는 원시 데이터를 벡터 \(v_{raw}\) 의 형태로 가져와서 보정 매트릭스 \(A\) 를 얻음으로써 수행된다. 출력은 최적화 문제점에 대한 솔루션이다. \(argmin_{v_{cal}} ||Av_{cal}-v_{raw}||^{2}\)

[37]:

composite_sigma_z = sigma_z = np.array([[1, 0],[0, -1]])
for a in range(1,qn):
    composite_sigma_z = np.kron(composite_sigma_z,sigma_z)

이제 우리는 개수의 리스트가 \(\otimes _{j}^{N} \sigma_z^{j}\) 상태와 일치해서 \(\langle \otimes _{j}^{N} \sigma_z^{j} \rangle\) 를 계산하는 것이 \(\otimes _{j}^{N} \sigma_z^{j}\) 의 대각 성분과 이 정렬된 리스트의 점곱을 계산하는 만큼 간단해지도록 바르게 정렬되었는지 확인하고 싶다. composite_pauli_z_expvalue 함수는 그 기능을 수행한다. 이는 회로를 사용해 상태 벡터 개수들을 적절히 정렬한다. 우리는 패리티 진동을 관찰하기 위해 이 점 곱을 \(\phi\) 에 대한 함수로 그릴 수 있다.

단층촬영에 대한 참고사항

Do not perform quantum tomography with >5 qubits

결론¶

결론적으로, 에러 완화 없이, MQC는 GHZ 상태를 특성화하기 위한 탁월한 방법이라는 것을 알 수 있다. 그러나, 에러 완화로, 모든 방법들은 적어도 소수의 큐비트들 (qubits) 에 대해, 훨씬 더 큰 fidelity를 가질 수 있고, 모두 동일한 값에 근접할 수 있다. 실제 기기를 사용하지 않아도 정확한 결과를 얻기 위해 4배에서 8배의 샷으로 늘릴 가치가 있다. 이 노트북에서 사용된 병렬화된 회로가 충실도 대 선형화된 회로를 수행하는 방법을 비교할 가치가 있을 수 있습니다.
