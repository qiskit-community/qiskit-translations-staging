잡음 모델 구축¶

소개¶

이 노트북은 Qiskit Aer noise 모듈을 사용하여 노이즈 시뮬레이션을 위한 사용자 잡음 모델을 만드는 방법을 소개한다.

[1]:

import numpy as np
from qiskit import QuantumCircuit, transpile
from qiskit.quantum_info import Kraus, SuperOp
from qiskit.providers.aer import AerSimulator
from qiskit.tools.visualization import plot_histogram

# Import from Qiskit Aer noise module
from qiskit.providers.aer.noise import NoiseModel
from qiskit.providers.aer.noise import QuantumError, ReadoutError
from qiskit.providers.aer.noise import pauli_error
from qiskit.providers.aer.noise import depolarizing_error
from qiskit.providers.aer.noise import thermal_relaxation_error

Qiskit Aer Noise 모듈¶

Qiskit Aer noise 모듈에는 시뮬레이션을 위한 맞춤형 노이즈 모델을 만들기 위한 Python 클래스가 포함되어 있다. 다음과 같은 세 가지 주요 클래스가 있다.

  1. NoiseModel 클래스는 잡음이 많은 시뮬레이션에 사용되는 잡음 모델을 저장한다.

  2. QuantumError 클래스는 CPTP 게이트 오류를 나타내는 클래스다. 다음과 같은 경우에 적용될 수 있다.

    • 게이트 혹은 초기화 명령어 이후

    • 사전 측정하기 전에 지시사항.

  3. ReadoutError 클래스는 전통적인 해독 오류를 나타낸다.

양자 오류¶

QuantumError 객체를 직접 처리하는 대신, 보조 함수가 존재하여 매개 변수화된 양자 오류의 특정 유형을 자동으로 생성한다. 이것들은 noise 모듈에 포함되어 있으며 양자 컴퓨팅 연구에 사용되는 많은 일반적인 오류 유형을 위한 함수를 포함한다. 함수 이름과 반환되는 오류 유형은 다음과 같다.

표준 오차 함수

세부 사항

kraus_error

크라우스(Kraus) 행렬 \([K_0, ...]\) 목록으로 제공된 일반적인 n-큐비트 CPTP 오류 채널

mixed_unitary_error

n-큐비트 혼합하는 통합 오류 한개로 구성된 매트릭스와 확률 \([(U_0, p_0),...]\) 들의 목록으로 주어진다.

coherent_unitary_error

n-큐비트 일관성 있는 단위 오차는 단일 단위 행렬 \(U\) 로 주어진다.

pauli_error

파울리 상태와 확률의 목록 \([(P_0, p_0),...]\) 으로 주어진 n-큐비트 파울리 오류 채널 (혼합된 단위 상태).

depolarizing_error

탈분극 확률 \(p\) 로 매개변수를 가진 n-큐비트 탈분극 오류 채널.

reset_error

단일 큐비트 초기화 에러는 \(|0\rangle\), \(|1\rangle\) 상태로 초기화할 확률 \(p_0, p_1\) 로 매개화된다.

thermal_relaxation_error

완화 시간 상수 :math:》T_1》, :math:》T_2》, 게이트 시간 :math:` t `, 및 여기된 상태 열 집단 ( :math:〉 p_1) 에 의해 파라미터화된 단일의 1비트 열 완화 채널을 포함한다.

phase_amplitude_damping_error

진폭 감쇠 매개 변수 \(\lambda\), 위상 감쇠 매개 변수 \(\gamma\) 및 여기 상태 열의 집단 \(p_1\) 가 제공하는 단일 큐비트 일반화 위상 및 진폭 감쇠 오류 채널이다.

amplitude_damping_error

진폭 감쇠 매개 변수 \(\lambda\) 및 여기 상태 열의 집단 \(p_1\) 이 제공하는 단일 큐비트 일반화 진폭 감쇠 오류 채널.

phase_damping_error

위상 감쇠 매개 변수 \(\gamma\) 이 제공하는 단일 큐비트 위상 감쇠 오류 채널

양자 오류 결합하기¶

QuantumError 인스턴스는 다음과 같이 새로운 QuantumErrors 를 생성하기 위해 구성, 텐서 곱 및 텐서 확장(반전 명령 텐서 곱) 을 사용하여 결합할 수 있다.

  • 구성: \(\cal{E}(\rho)=\cal{E_2}(\cal{E_1}(\rho))\) 을 error = error1.compose(error2) 처럼

  • 텐서 곱: \(\cal{E}(\rho) =(\cal{E_1}\otimes\cal{E_2})(\rho)\) 을 error error1.tensor(error2) 처럼

  • 확장 곱: \(\cal{E}(\rho) =(\cal{E_2}\otimes\cal{E_1})(\rho)\) 을 error error1.expand(error2) 처럼

해독오류¶

기존 해독 오류는 할당 확률 벡터 \(P(A|B)\) 목록으로 지정된다.

  • \(A\) 는 기록된 고전적 비트 값이다.

  • \(B\) 는 측정으로부터 반환되는 참 비트 값이다.

예를 들어, 1 큐비트는: $ P(A|B) = [P(A|0), P(A|1)]$.

[8]:

# Measurement miss-assignement probabilities
p0given1 = 0.1
p1given0 = 0.05

ReadoutError([[1 - p1given0, p1given0], [p0given1, 1 - p0given1]])

[8]:

ReadoutError([[0.95 0.05]
 [0.1  0.9 ]])

해독 오류들은 양자 오류와 마찬가지로 compose, tensor 및 expand 을 사용하여 결합 될 수 있다.

잡음 모델에 오류 추가하기¶

노이즈 모델에 양자 오류를 추가할 때, 우리는 그것이 작용하는 명령어 의 유형과 그것을 적용할 큐비트를 지정해야 한다. 양자 오류에는 세 가지 경우가 있다.

  1. 모든 큐비트 양자 오류

  2. 특정 큐비트 양자 오류

  3. 비국소적 양자 오류

모든 큐비트 양자 오류¶

이는 어떠한 명령어의 발생에 어떤 큐비트에 동작하는것과 관계없이 동일한 오류를 적용한다.

이것은 noise_model.add_all_qubit_quantum_error(error, instructions) 로 추가된다.

[9]:

# Create an empty noise model
noise_model = NoiseModel()

# Add depolarizing error to all single qubit u1, u2, u3 gates
error = depolarizing_error(0.05, 1)
noise_model.add_all_qubit_quantum_error(error, ['u1', 'u2', 'u3'])

# Print noise model info
print(noise_model)

NoiseModel:
  Basis gates: ['cx', 'id', 'u1', 'u2', 'u3']
  Instructions with noise: ['u3', 'u2', 'u1']
  All-qubits errors: ['u1', 'u2', 'u3']

특정 큐비트 양자 오류¶

이 오류는 지정된 큐비트 목록에 작용하는 명령어의 발생에 적용된다. 큐비트 순서가 중요하다. 2-큐비트 게이트의 경우, 큐비트 [0, 1] 에 적용되는 오류는 예를 들어 큐비트 [1, 0] 에 적용되는 오류와는 다르다.

이것은 noise_model.add_quantum_error(error, instructions, qubits) 로 추가된다.

[10]:

# Create an empty noise model
noise_model = NoiseModel()

# Add depolarizing error to all single qubit u1, u2, u3 gates on qubit 0 only
error = depolarizing_error(0.05, 1)
noise_model.add_quantum_error(error, ['u1', 'u2', 'u3'], [0])

# Print noise model info
print(noise_model)

NoiseModel:
  Basis gates: ['cx', 'id', 'u1', 'u2', 'u3']
  Instructions with noise: ['u3', 'u2', 'u1']
  Qubits with noise: [0]
  Specific qubit errors: [('u1', [0]), ('u2', [0]), ('u3', [0])]

비국소적 큐비트 양자 오류¶

이는 특정 게이트 큐비트에 작동하는 모든 명령어 이후에 특정 잡음 큐비트에 오류를 적용한다.

noise_model.add_nonlocal_quantum_error(error, instructions, instr_qubits, error_qubits) 로 더해진다:

[11]:

# Create an empty noise model
noise_model = NoiseModel()

# Add depolarizing error on qubit 2 forall single qubit u1, u2, u3 gates on qubit 0
error = depolarizing_error(0.05, 1)
noise_model.add_nonlocal_quantum_error(error, ['u1', 'u2', 'u3'], [0], [2])

# Print noise model info
print(noise_model)

NoiseModel:
  Basis gates: ['cx', 'id', 'u1', 'u2', 'u3']
  Instructions with noise: ['u3', 'u2', 'u1']
  Qubits with noise: [0, 2]
  Non-local specific qubit errors: [('u1', [0], [2]), ('u2', [0], [2]), ('u3', [0], [2])]

잡음을 포함한 노이즈 모델 시뮬이션 실행하기¶

Aer Simulator(noise_model=noise_model) 명령은 지정된 노이즈 모델에 구성된 시뮬레이터를 반환한다. 잡음 모델의 게이트에 따라 시뮬레이터의 기본 게이트를 대체할 뿐만 아니라 잡음 모델의 게이트에 따라 잡음 모델의 기본 게이트도 무시된다.

노이즈 예 1: 기본 비트-플립 오류 노이즈 모델¶

양자 정보 이론 연구에서 흔히 볼 수 있는 간단한 장난감 소음 모델 예를 생각해 보자.

  • 단일 큐비트 게이트를 적용할 때 큐비트 상태를 확률 p_gate1 로 뒤집는다.

  • 2-큐비트 게이트를 적용하는 것이 단일 큐비트 오류를 각 큐비트에 적용하는 경우.

  • 큐비트 재설정을 0이 아닌 1로 확률 p_reset 로 재설정하는 경우.

  • 큐비트를 측정할때 큐비트의 상태를 확률 p_meas 로 뒤집는다.

[14]:

# Example error probabilities
p_reset = 0.03
p_meas = 0.1
p_gate1 = 0.05

# QuantumError objects
error_reset = pauli_error([('X', p_reset), ('I', 1 - p_reset)])
error_meas = pauli_error([('X',p_meas), ('I', 1 - p_meas)])
error_gate1 = pauli_error([('X',p_gate1), ('I', 1 - p_gate1)])
error_gate2 = error_gate1.tensor(error_gate1)

# Add errors to noise model
noise_bit_flip = NoiseModel()
noise_bit_flip.add_all_qubit_quantum_error(error_reset, "reset")
noise_bit_flip.add_all_qubit_quantum_error(error_meas, "measure")
noise_bit_flip.add_all_qubit_quantum_error(error_gate1, ["u1", "u2", "u3"])
noise_bit_flip.add_all_qubit_quantum_error(error_gate2, ["cx"])

print(noise_bit_flip)

NoiseModel:
  Basis gates: ['cx', 'id', 'u1', 'u2', 'u3']
  Instructions with noise: ['reset', 'u3', 'cx', 'u2', 'u1', 'measure']
  All-qubits errors: ['reset', 'measure', 'u1', 'u2', 'u3', 'cx']

sim_noise = AerSimulator(noise_model=noise_bit_flip)

# Transpile circuit for noisy basis gates

# Plot noisy output

실시예 2: T1/T2열 완화¶

이제 큐비트 환경에서 열 완화를 기반으로 한 보다 현실적인 오류 모델을 생각해 보자. * 각 큐비트는 열 완화 시간 상수 \(T_1\) 와 위상이완 시간 상수 \(T_2\) 에 의해 매개 변수화된다. * 우리는 \(T_2 \le 2 T_1\) 를 가지고 있어야만 한다. * 명령어의 착오율은 게이트 시간과 큐비트인 \(T_1\), \(T_2\) 값에 의해 결정된다.

[16]:

# T1 and T2 values for qubits 0-3
T1s = np.random.normal(50e3, 10e3, 4) # Sampled from normal distribution mean 50 microsec
T2s = np.random.normal(70e3, 10e3, 4)  # Sampled from normal distribution mean 50 microsec

# Truncate random T2s <= T1s
T2s = np.array([min(T2s[j], 2 * T1s[j]) for j in range(4)])

# Instruction times (in nanoseconds)
time_u1 = 0   # virtual gate
time_u2 = 50  # (single X90 pulse)
time_u3 = 100 # (two X90 pulses)
time_cx = 300
time_reset = 1000  # 1 microsecond
time_measure = 1000 # 1 microsecond

# QuantumError objects
errors_reset = [thermal_relaxation_error(t1, t2, time_reset)
                for t1, t2 in zip(T1s, T2s)]
errors_measure = [thermal_relaxation_error(t1, t2, time_measure)
                  for t1, t2 in zip(T1s, T2s)]
errors_u1  = [thermal_relaxation_error(t1, t2, time_u1)
              for t1, t2 in zip(T1s, T2s)]
errors_u2  = [thermal_relaxation_error(t1, t2, time_u2)
              for t1, t2 in zip(T1s, T2s)]
errors_u3  = [thermal_relaxation_error(t1, t2, time_u3)
              for t1, t2 in zip(T1s, T2s)]
errors_cx = [[thermal_relaxation_error(t1a, t2a, time_cx).expand(
             thermal_relaxation_error(t1b, t2b, time_cx))
              for t1a, t2a in zip(T1s, T2s)]
               for t1b, t2b in zip(T1s, T2s)]

# Add errors to noise model
noise_thermal = NoiseModel()
for j in range(4):
    noise_thermal.add_quantum_error(errors_reset[j], "reset", [j])
    noise_thermal.add_quantum_error(errors_measure[j], "measure", [j])
    noise_thermal.add_quantum_error(errors_u1[j], "u1", [j])
    noise_thermal.add_quantum_error(errors_u2[j], "u2", [j])
    noise_thermal.add_quantum_error(errors_u3[j], "u3", [j])
    for k in range(4):
        noise_thermal.add_quantum_error(errors_cx[j][k], "cx", [j, k])

print(noise_thermal)
