장치 백엔드 잡음 모델 시뮬레이션¶

소개¶

이 노트북은 Qiskit Aer noise 모듈을 사용하여 IBMQ 하드웨어 장치에 대한 기본 잡음 모델을 자동으로 생성하는 방법을 설명하고, 이 모델을 사용하여 실제 장치에서 발생하는 오류의 효과를 연구하기 위해 QuantumCircuits 의 소음 시뮬레이션을 수행한다.

이러한 자동 모델은 실제 디바이스에서 발생하는 실제 오류의 근사치 입니다. 이는 게이트에 평균 오류 비율 와 관련된 제한된 입력 매개변수 세트에서 빌드해야 하므로 실제 디바이스에서 발생하는 실제 오류의 근사치이다. 실제 장치에서의 양자 오류에 대한 연구는 활발한 연구 영역이며, 다른 노트북에서 보다 자세한 소음 모델을 구성하는 Qiskit Aer 도구에 대해 설명한다.

[1]:

from qiskit import IBMQ, transpile
from qiskit import QuantumCircuit
from qiskit.providers.aer import AerSimulator
from qiskit.tools.visualization import plot_histogram

디바이스 백엔드 소음 모델¶

Qiskit Aer 장치 잡음 모델은 실제 장치를 위한 단순화된 노이즈 모델을 자동으로 생성한다. 이 모델은 기기의 BackendProperties 에 보고된 교정 정보를 사용하여 생성되며 이를 고려한다.

  • 각 큐비트의 기본 게이트들의 gate_error 확률.

  • 각 큐비트에 있는 각 기본 게이트의 gate_length.

  • 각 큐비트의 \(T_1\), \(T_2\) 완화 시간 상수이다.

  • 각 비트의 읽기 오류 확률이다.

테라 Mock 백엔드¶

Qiskit Terra에 저장된 데이터를 사용하여 IBM 양자 장치의 실제 노이즈 데이터를 사용한다. 구체적으로 본 지침서에서 사용된 장치는 ibmq_vigo 이다.

[2]:

from qiskit.test.mock import FakeVigo
device_backend = FakeVigo()

디바이스 및 시뮬레이션 비교를 위한 테스트 회로¶

이제 실제 장치의 출력과 Qiskit Aer의 AerSimulator 에서 시뮬레이션한 잡음을 포함한 출력을 비교하기 위한 테스트 회로를 생성하였다. 3-큐비트 GHZ 상태 \(\frac{1}{\sqrt22}(|0,0,0\rangle + |1,1,1\rangle)\) 를 큐비트 0, 1, 2에 준비할 것이다. 비교 실험을 수행하기 전, 잡음이 포함되지 않은 이상적인 출력을 확인할 수 있다.

[3]:

# Construct quantum circuit
circ = QuantumCircuit(3, 3)
circ.h(0)
circ.cx(0, 1)
circ.cx(1, 2)
circ.measure([0, 1, 2], [0, 1, 2])

sim_ideal = AerSimulator()

# Execute and get counts
result = sim_ideal.run(transpile(circ, sim_ideal)).result()
counts = result.get_counts(0)
plot_histogram(counts, title='Ideal counts for 3-qubit GHZ state')

[3]:

장치를 모방하는 시뮬레이터 생성하기¶

우리는 ibmq_vigo 시뮬레이터를 생성하기 위하여 from_backend 를 호출하고자:

[4]:

sim_vigo = AerSimulator.from_backend(device_backend)

디바이스 속성들을 vigo_simulator 에 저장함으로써, 우리는 시뮬레이션을 위한 회로들을 컴파일링할 때 적절한 기준 게이트들 및 커플링 맵이 사용되는 것을 보장하며, 이에 의해 실제 디바이스 상에서 실행될 게이트들을 가장 가깝게 모방한다. 이밖에 vigo_simulator 에는 다음으로 구성된 대략적인 소음 모델이 포함되어 있다.

  • 단일 큐비트 게이트 오류 는 단일 큐비트 편극 소거화 오류와 단일 큐비트 열 이완 오류로 구성된다.

  • 2-큐비트 게이트 오류 는 2-큐비트 편극 소거(depolarizing) 오류로 구성되며, 그 다음 게이트의 두 큐비트 모두 단일 큐비트 열 완화 오류로 구성된다.

  • 개별 큐비트의 측정에서 얻은 클래식 비트에 대한 단일 큐비트 읽기 오류 이다.

게이트 오류의 경우 열 완화 오류의 오류 매개변수는 개별 큐비트 \(T_1\) 및 \(T_2\) 매개 변수와 함께 aer.noise.errors 모듈의 thermal_relaxation_error 함수를 사용하여 도출되고 장치 백엔드 속성에서 gate_time 매개 변수를 사용한다. 그런 다음 편극 소거 오류의 확률은 편극 소거 오류와 열 완화에 따른 결합된 평균 게이트 부정행위가 백엔드 속성의 gate_error 값가 동일하도록 설정된다.

판독 오류의 경우 큐비트 readout_errors 가 측정을 제공한 후 기록된 클래식 비트 값이 참 결과에서 플립될 확률이다.

잡음 시뮬레이션 실행하기¶

실제 장치를 기반으로 하는 노이즈를 포함한 시뮬레이터 백엔드를 생성하면 이를 사용하여 노이즈를 포함한 시뮬레이션을 실행할 수 있다.

중요 사항: 노이즈를 포함한 시뮬레이션을 실행하는 경우 백엔드에 대한 회로를 transpile 하여 백엔드에 대해 올바른 노이즈 기저 게이트 셋로 전송하는 것이 중요하다.

[5]:

# Transpile the circuit for the noisy basis gates
tcirc = transpile(circ, sim_vigo)

# Execute noisy simulation and get counts
result_noise = sim_vigo.run(tcirc).result()
counts_noise = result_noise.get_counts(0)
plot_histogram(counts_noise,
               title="Counts for 3-qubit GHZ state with device noise model")

[5]:

장치 노이즈 모델의 트랜스파일링을 건너뛰면 시뮬이터가 지원하는 회로의 게이트에는 노이즈가 적용되지 않지만 모방된 백엔드가 지원되지 않는다.
