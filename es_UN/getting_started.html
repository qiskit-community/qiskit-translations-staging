Primeros Pasos¶

Instalación¶

Qiskit es compatible con Python 3.6 o posterior. Sin embargo, tanto Python como Qiskit son ecosistemas en evolución y, a veces, cuando se producen nuevas versiones en uno u otro, puede haber problemas de compatibilidad.

Recomendamos instalar Anaconda, una distribución de Python multiplataforma para computación científica. Se recomienda Jupyter, incluido en Anaconda, para interactuar con Qiskit.

Qiskit está probado y es compatible con los siguientes sistemas de 64 bits:

  • Ubuntu 16.04 o posterior

  • macOS 10.12.6 o posterior

  • Windows 7 o posterior

Recomendamos utilizar entornos virtuales de Python para separar claramente Qiskit de otras aplicaciones y mejorar tu experiencia.

La forma más sencilla de utilizar los entornos es mediante el comando conda, incluido con Anaconda. Un entorno de Conda te permite especificar una versión específica de Python y un conjunto de bibliotecas. Abre una ventana de terminal en el directorio donde deseas trabajar.

Es preferible que utilices el prompt de Anaconda instalado con Anaconda. Todo lo que tienes que hacer es crear un entorno virtual dentro de Anaconda y activar el entorno. Estos comandos se pueden ejecutar en el prompt de Anaconda independientemente de la máquina con Windows o Linux.

Crea un entorno mínimo con solo Python instalado en él.

conda create -n ENV_NAME python=3

Activa tu nuevo entorno.

conda activate ENV_NAME

A continuación, instala el paquete Qiskit.

pip install qiskit

Si los paquetes se instalaron correctamente, puedes ejecutar conda list para ver los paquetes activos en tu entorno virtual.

Si tienes la intención de utilizar la funcionalidad de visualización o los cuadernos de Jupyter, se recomienda instalar Qiskit con el soporte adicional de visualization:

pip install qiskit[visualization]

Vale la pena señalar que si eres usuario de zsh (que es el shell predeterminado en las versiones más recientes de macOS), deberás poner qiskit[visualization] entre comillas:

pip install 'qiskit[visualization]'

Los siguientes proveedores de nube tienen Qiskit preinstalado en sus entornos:

IBM Quantum Lab

Build quantum applications and experiments with Qiskit in a cloud programming environment.

Strangeworks

A platform that enables users and organizations to easily apply quantum computing to their most pressing problems and research.

La instalación de los elementos desde el código fuente te permite acceder a la versión actualizada más recientemente de Qiskit en lugar de usar la versión en el repositorio del índice de paquetes de Python (PyPI). Esto te dará la capacidad de inspeccionar y ampliar la última versión del código Qiskit de manera más eficiente.

Al instalar los elementos y componentes desde el código fuente, por defecto se utilizará su versión de development (que corresponde a la rama de git master), a diferencia de la versión stable (que contiene el mismo codebase como los paquetes pip publicados). Dado que las versiones de development de un elemento o componente generalmente incluyen nuevas características y cambios, generalmente también requieren el uso de la versión de development del resto de los elementos.

Los paquetes Terra y Aer requieren un compilador para compilar desde el código fuente antes de que puedas instalarlos. Ignis, Aqua y el backend de IBM Quantum Provider no requieren un compilador.

La instalación de elementos desde el código fuente requiere el siguiente orden de instalación para evitar la instalación de versiones de elementos que pueden ser inferiores a las deseadas si la versión de pip es anterior a las versiones fuente:

  1. qiskit-terra

  2. qiskit-aer

  3. qiskit-ignis

  4. qiskit-aqua

  5. qiskit-ibmq-provider (si deseas conectarte a los dispositivos IBM Quantum o al simulador en línea)

Para trabajar con varios componentes y elementos simultáneamente, utiliza los siguientes pasos para cada elemento.

Nota

Debido al uso de paquetes de espacio de nombres (namespaces) en Python, se debe tener cuidado al instalar los paquetes. Si planeas instalar cualquier elemento desde el código fuente, no uses el metapaquete `qiskit. Además, sigue esta guía y utiliza un entorno virtual independiente para el desarrollo. Si eliges mezclar una instalación existente con tu desarrollo, consulta https://github.com/pypa/sample-namespace-packages/blob/master/table.md para ver el conjunto de combinaciones de métodos de instalación que funcionan juntos.

Set up the Virtual Development Environment

conda create -y -n QiskitDevenv python=3
conda activate QiskitDevenv

Installing Terra from Source

La instalación desde el código fuente requiere que tengas un compilador C ++ en tu sistema que admita C++11.

En la mayoría de las plataformas Linux, el compilador GCC necesario ya está instalado.

Si usas macOS, puede instalar el compilador Clang instalando XCode. Verifica si tienes XCode y Clang instalados abriendo una ventana de terminal e ingresando lo siguiente.

clang --version

Instala XCode y Clang usando el siguiente comando.

xcode-select --install

En Windows, es más fácil instalar el compilador de Visual C++ desde Build Tools for Visual Studio 2019. En su lugar, puedes instalar Visual Studio versión 2015 o 2017, asegurándote de seleccionar las opciones para instalar el compilador de C++.

Una vez instalados los compiladores, estarás listo para instalar Qiskit Terra.

  1. Clona el repositorio de Terra.

    git clone https://github.com/Qiskit/qiskit-terra.git

  2. La clonación del repositorio crea una carpeta local llamada qiskit-terra.

    cd qiskit-terra

  3. Instala las bibliotecas de requerimientos de Python desde tu directorio qiskit-terra.

    pip install cython

  4. Si deseas ejecutar pruebas o verificaciones linting, instala los requerimientos del desarrollador.

    pip install -r requirements-dev.txt

  5. Instala qiskit-terra.

    pip install .

Si deseas instalarlo en modo editable, lo que significa que los cambios de código en el proyecto no requieren una reinstalación para ser aplicados, puedes hacerlo con:

pip install -e .

A continuación, puedes ejecutar los ejemplos de código después de instalar Terra. Puedes ejecutar el ejemplo con el siguiente comando.

python examples/python/using_qiskit_terra_level_0.py

Nota

Si no tienes la intención de instalar ningún otro componente, qiskit-terra emitirá una advertencia RuntimeWarning debido a que tanto qiskit-aer como qiskit-ibmq-provider no están instalados. Esto se hace porque los usuarios normalmente tienen la intención de utilizar los elementos adicionales, pero no se dan cuenta de que no están instalados o de que la instalación de Aer o IBM Quantum Provider falló por algún motivo. Si deseas suprimir estas advertencias, agrega:

import warnings
warnings.filterwarnings('ignore', category=RuntimeWarning,
                        module='qiskit')

antes de cualquier importación de qiskit en su código. Esto suprimirá la advertencia sobre qiskit-aer y qiskit-ibmq-provider que faltan, pero continuará mostrando cualquier otra advertencia de qiskit u otros paquetes.

Installing Aer from Source

  1. Clona el repositorio Aer.

    git clone https://github.com/Qiskit/qiskit-aer

  2. Instala los requerimientos de construcción.

    pip install cmake scikit-build cython

Después de esto, los pasos para instalar Aer dependen del sistema operativo que estes utilizando. Dado que Aer es un programa en C++ compilado con una interfaz en Python, existen dependencias externas a Python para construir el binario de Aer que no se pueden instalar universalmente dependiendo del sistema operativo.

Linux

  1. Instala los requisitos del compilador.

    La construcción de Aer requiere un compilador C++ y cabeceras de desarrollo.

    Si estás usando Fedora o una distribución de Linux equivalente, instala usando:

    dnf install @development-tools

    Para Ubuntu/Debian instálalo usando:

    apt-get install build-essential

  2. Instala las cabeceras de desarrollo de OpenBLAS.

    Si estás usando Fedora o una distribución de Linux equivalente, instala usando:

    dnf install openblas-devel

    Para Ubuntu/Debian instálalo usando:

    apt-get install libopenblas-dev
macOS

  1. Instala dependencias.

    Para utilizar el compilador Clang en macOS, debes instalar una librería adicional para admitir OpenMP. Puedes usar brew para instalar esta y otras dependencias.

    brew install libomp

  2. Luego instala una implementación de BLAS; OpenBLAS es la opción predeterminada.

    brew install openblas

    A continuación, instala las Herramientas de línea de comandos de Xcode.

    xcode-select --install
Windows

En Windows es necesario utilizar Anaconda3 o Miniconda3 para instalar todas las dependencias.

  1. Instala los requisitos del compilador.

    conda install --update-deps vs2017_win-64 vs2017_win-32 msvc_runtime

Qiskit Aer es un framework de simulación de alto rendimiento para circuitos cuánticos. Proporciona distintos backends para lograr diferentes objetivos de simulación.

conda install --update-deps -c conda-forge -y openblas cmake

  1. Construye e instala qiskit-aer directamente

    Si tienes pip <19.0.0 instalado y tu entorno no requiere una compilación personalizada, ejecuta:

    cd qiskit-aer
pip install .

    Esto construirá los binarios e instalará Aer.

    Alternativamente, si tienes un pip más nuevo instalado o tienes algún requisito personalizado, puedes construir una Python wheel manualmente.

    cd qiskit-aer
python ./setup.py bdist_wheel

    Si necesitas configurar una opción personalizada durante la construcción de la Python wheel, consulta Opciones personalizadas durante la construcción de la wheel.

    Después de construir la Python wheel, se almacenará en el directorio dist/ en el repositorio de Aer. La versión exacta dependerá

    cd dist
pip install qiskit_aer-*.whl

    El nombre exacto del archivo de salida de la Python wheel depende de la versión actual de Aer en desarrollo.

Custom options

El sistema de construcción de Aer usa scikit-build para ejecutar la compilación cuando lo construye con la interfaz de Python. Actúa como una interfaz para setuptools para llamar a CMake y compilar los binarios para tu sistema local.

Debido a la complejidad de compilar los binarios, es posible que debas pasar opciones a una determinada parte del proceso de compilación. La forma de pasar variables es:

python setup.py bdist_wheel [skbuild_opts] [-- [cmake_opts] [-- build_tool_opts]]

donde los elementos entre corchetes [] son opcionales, y skbuild_opts, cmake_opts, build_tool_opts deben ser reemplazados por banderas de tu elección. Una lista de opciones de CMake está disponible aquí: https://cmake.org/cmake/help/v3.6/manual/cmake.1.html#options. Por ejemplo, podrías ejecutar algo como:

python setup.py bdist_wheel -- -- -j8

Esto es pasar la bandera -j8 al sistema de compilación subyacente (que en este caso es Automake), indicándole que deseas compilar en paralelo usando 8 procesos.

Por ejemplo, un caso de uso común para estas banderas en Linux es especificar una versión particular del compilador de C++ a ser usado (normalmente si el valor predeterminado es demasiado viejo):

python setup.py bdist_wheel -- -DCMAKE_CXX_COMPILER=g++-7

que le dirá a CMake que use el comando g++-7 en lugar del g++ predeterminado al compilar Aer.

Otro caso de uso común para esto, dependiendo de tu ambiente, es que es posible que debas especificar el nombre de tu plataforma y desactivar los enlaces estáticos.

python setup.py bdist_wheel --plat-name macosx-10.9-x86_64 \
-- -DSTATIC_LINKING=False -- -j8

Aquí --plat-name es una bandera para setuptools, para especificar el nombre de la plataforma a usar en los metadatos del paquete, -DSTATIC_LINKING es una bandera para CMake que deshabilita el enlace estático, y -j8 es una bandera para Automake que indica que se deben usar 8 procesos para la compilación.

Una lista de opciones comunes según la plataforma son:

Algunas de estas opciones no son específicas de la plataforma. Estas plataformas en particular se enumeran porque se usan comúnmente en un entorno local. Consulte la documentación de la herramienta para obtener más información.

Installing Ignis from Source

  1. Clonar el repositorio de Ignis.

    git clone https://github.com/Qiskit/qiskit-ignis.git

  2. Clonar el repositorio crea un directorio local llamado qiskit-ignis.

    cd qiskit-ignis

  3. Si deseas ejecutar pruebas o verificaciones lint, instala los requerimientos de desarrollador. Esto no es necesario para instalar o utilizar el paquete qiskit-ignis cuando se instala desde la fuente.

    pip install -r requirements-dev.txt

  4. Instalar Ignis.

    pip install .

Si deseas instalarlo en modo editable, es decir que los cambios en el código del proyecto no requieran una reinstalación para ser aplicados:

pip install -e .
