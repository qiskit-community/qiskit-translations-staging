Crear un histograma¶

Para visualizar los datos de un circuito cuántico ejecutado en un dispositivo real o «quasm_simulator», hemos hecho una función simple

plot_histogram(data)

Como ejemplo, generaremos un estado de Bell de 2 qubits

[9]:

# quantum circuit to make a Bell state
bell = QuantumCircuit(2, 2)
bell.h(0)
bell.cx(0, 1)

meas = QuantumCircuit(2, 2)
meas.measure([0,1], [0,1])

# execute the quantum circuit
backend = BasicAer.get_backend('qasm_simulator') # the device to run on
circ = bell.compose(meas)
result = backend.run(transpile(circ, backend), shots=1000).result()
counts  = result.get_counts(circ)
print(counts)

{'11': 480, '00': 520}

[10]:

plot_histogram(counts)

Opciones al graficar un histograma¶

La función plot_histogram() tiene algunas opciones para ajustar la gráfica de salida. La primera opción es el argumento kwarg llamado legend. Se utiliza para proporcionar una etiqueta para las ejecuciones. Se necesita una lista de cadenas que se utilizan para etiquetar los resultados de cada ejecución. Esto es sobre todo útil cuando se grafican varios resultados de ejecución en el mismo histograma. El kwarg llamado sort se usa para ajustar el orden en que se representan las barras en el histograma. Se puede configurar en orden ascendente con asc o en orden descendente con desc. El kwarg number_to_keep toma un número entero para el número de términos a mostrar, el resto se agrupa en una sola barra llamada resto. Puedes ajustar el color de las barras con el kwarg color que toma una cadena o una lista de cadenas para los colores que se utilizarán para las barras para cada ejecución. Puedes ajustar si las etiquetas se imprimen encima de las barras o no con el kwarg bar_labels. La última opción disponible es el kwarg figsize que toma una tupla con el tamaño en pulgadas para hacer la figura de salida.

[12]:

# Execute 2-qubit Bell state again
second_result = backend.run(transpile(circ, backend), shots=1000).result()
second_counts  = second_result.get_counts(circ)
# Plot results with legend
legend = ['First execution', 'Second execution']
plot_histogram([counts, second_counts], legend=legend)

Usando la salida de plot_histogram()¶

Al usar la función plot_histogram() esta devuelve un objeto de clase matplotlib.Figure para la visualización renderizada. Los cuadernos de Jupyter entienden este tipo de objeto y hacen el render dentro de este tutorial, pero al correr la función fuera de Jupyter no se tiene esta característica automáticamente. Sin embargo, la clase matplotlib.Figure tiene métodos nativos para mostrar y guardar la visualización. Puedes llamar al método .show() en el objeto devuelto por plot_histogram() para abrir la imagen en una nueva ventana (asumiendo el backend de matplotlib que tienes configurado es interactivo). O alternativamente, puedes llamar .savefig('out.png') para guardar la figura a out.png. El método savefig() tiene de entrada una ruta para que puedas ajustar la ubicación y nombre del archivo donde estés guardando la salida.

Graficar el Estado¶

En muchas ocasiones, querrás ver el estado de una computadora cuántica. Para depurar tu código por ejemplo. Aquí asumimos que tienes un estado cuántico (ya sea por una simulación o por tomografía de estado) y que el objetivo es visualizarlo. Esto requiere una cantidad de recursos exponencial, por lo que aconsejamos visualizar únicamente el estado de pequeños sistemas cuánticos. Hay varias funciones para generar distintos tipos de visualización de un estado cuántico

plot_state_city(quantum_state)
plot_state_qsphere(quantum_state)
plot_state_paulivec(quantum_state)
plot_state_hinton(quantum_state)
plot_bloch_multivector(quantum_state)

Un estado cuántico es una matriz de estado \(\rho\) (matriz hermitiana) o vector de estado \(|\psi\rangle\) (vector complejo). La matriz de estado está relacionada con el vector de estado por

\[\rho = |\psi\rangle\langle \psi|,\]

y es más general, ya que puede representar estados mixtos (suma positiva de vectores de estado)

\[\rho = \sum_k p_k |\psi_k\rangle\langle \psi_k |.\]

Las visualizaciones generadas por las funciones son:

  • 'plot_state_city': La visualización estándar para los estados cuánticos donde las partes reales e imaginarias (imag) de la matriz de estado se grafican asemejando una ciudad.

  • 'plot_state_qsphere': La visualización única de Qiskit para un estado cuántico, en ella la amplitud y la fase del vector de estado se trazan en una esfera. La amplitud es el grosor de la flecha y la fase es el color. Para los estados mixtos mostrará diferentes 'qsphere' para cada componente.

  • 'plot_state_paulivec': La representación de la matriz de estado utilizando operadores de Pauli como base \(\rho = \sum_ {q= 0} ^ {d ^ 2-1} p_jP_j/d\).

  • 'plot_state_hinton': Igual que 'city' pero aquí, el tamaño del elemento representa el valor del elemento de matriz.

  • 'pl_bloch_multivector': La proyección del estado cuántico sobre el espacio de un solo qubit graficado en una esfera de Bloch.

Opciones al usar funciones para graficar estados¶

Las diversas funciones para graficar estados cuánticos proporcionan una serie de opciones para ajustar cómo se renderizan cada una de las gráficas. Las opciones disponibles dependen de la función que se utilice.

Opciones plot_state_city()

  • ** title** (str): una cadena que representa el título de la gráfica

  • figsize (tuple): tamaño de la figura en pulgadas (ancho, alto).

  • color (lista): una lista de len = 2 que da colores para los componentes reales e imaginarios de los elementos de la matriz.

[22]:

Gráfica del Vector de Bloch¶

Una forma estándar de trazar un sistema cuántico es usar el vector de Bloch. Esto sólo funciona para un solo qubit y toma como entrada el vector de Bloch.

El vector de Bloch se define como \([x = \masthrm{Tr}[X \rho], y = \mathrm{Tr}[Y \rho], z = \mathrm{Tr}[Z \rho]] \), \(Y\), y \(Z\) son los operadores de Pauli para un único qubit y \(\rho\) es la matriz de estado.
