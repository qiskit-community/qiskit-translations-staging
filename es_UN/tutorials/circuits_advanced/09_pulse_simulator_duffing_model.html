Qiskit Aer: Simulación de pulso de dos qubits usando un modelo de oscilador Duffing¶

Este cuaderno muestra cómo utilizar el simulador de pulso Qiskit Aer, que simula experimentos especificados como objetos Schedule de pulso a nivel Hamiltoniano. El simulador resuelve la ecuación de Schrodinger para un modelo Hamiltoniano específico y un Schedule de pulso en el marco del Hamiltoniano de deriva.

En particular, en este tutorial:

  • Construir un modelo de un sistema superconductor de dos qubits.

  • Calibrar pulsos \(\pi\) en cada qubit en el sistema simulado.

  • Observar oscilaciones de resonancia cruzada al controlar el qubit 1 con el qubit objetivo 0.

La Introducción describe los conceptos y el flujo de este cuaderno.

1. Introducción¶

Las secciones principales proceden de la siguiente manera.

Sección 3: Modelo de oscilador Duffing¶

Para simular un sistema físico, es necesario especificar un modelo. En este cuaderno, modelaremos qubits superconductores como una colección de osciladores Duffing. El modelo se especifica en términos de los siguientes parámetros:

  • Cada oscilador Duffing es especificado por una frecuencia \(\nu\), una anarmonicidad \(\alpha\), y una fuerza de dirección \(r\), que resultan en los términos Hamiltonianos:

\[\begin{equation} 2\pi\nu a^\dagger a + \pi \alpha a^\dagger a(a^\dagger a - 1) + 2 \pi r (a + a^\dagger) \times D(t), \end{equation}\]

donde \(D(t)\) es la señal en el canal de dirección para el qubit, y \(a^\dagger\) y \(a\) son, respectivamente, los operadores de creación y aniquilación para el qubit. Ten en cuenta que la fuerza de dirección \(r\) establece la escala del término de control, con \(D(t)\) se asume que es un número complejo y sin unidades que satisface \(|D(t)| \leq 1\).

  • Un acoplamiento entre un par de osciladores \((l,k)\) se especifica mediante la fuerza de acoplamiento \(J\), lo que da como resultado un término de acoplamiento de intercambio:

\[\begin{equation} 2 \pi J (a_l a_k^\dagger + a_l^\dagger a_k), \end{equation}\]

donde el subíndice denota en qué qubit actúan los operadores.

  • Además, para la simulación numérica, es necesario especificar una dimensión de corte; el modelo del oscilador de Duffing es de dimensión infinita, y una simulación por computadora requiere de la restricción de los operadores a un subespacio de dimensión finita.

En el código: Definiremos un modelo de la forma anterior para dos qubits acoplados usando la función auxiliar duffing_system_model.

Sección 4: calibración de pulso \(\pi\) usando Ignis¶

Una vez definido el modelo, calibraremos pulsos \(\pi\) en cada qubit. Un pulso \(\pi\) se define como un pulso en el canal de dirección de un qubit que «voltea» el qubit; es decir, lleva el estado fundamental al primer estado excitado y el primer estado excitado al estado fundamental.

Experimentalmente encontraremos un pulso \(\pi\) para cada qubit usando el siguiente procedimiento:

  • Se establece una forma de pulso fija; en este caso, será un pulso gaussiano.

  • Se ejecuta una secuencia de experimentos, cada uno de los cuales consta de un pulso gaussiano en el qubit, seguido de una medición, y cada experimento de la secuencia tiene una amplitud posteriormente mayor para el pulso gaussiano.

  • Los datos de medición se ajustan y se encuentra la amplitud de pulso que invierte completamente el qubit (es decir, la amplitud de pulso the \(\pi\)).

En el código: Usando Ignis construiremos objetos Schedule para los experimentos anteriores, luego ajustaremos los datos para encontrar las amplitudes de pulso \(\pi\).

Sección 5: Oscilaciones de resonancia cruzada¶

Una vez calibrados los pulsos \(\pi\), simularemos los efectos de la conducción por resonancia cruzada en el qubit \(1\) con el qubit \(0\) como objetivo. Esto significa que conduciremos el qubit \(1\) a la frecuencia del qubit \(0\), con el objetivo de observar que la trayectoria y las oscilaciones del qubit \(0\) dependen del estado del qubit \(1\). Este fenómeno proporciona una base para la creación de puertas controladas de dos qubits. Nota: Esta sección requiere la calibración de pulsos \(\pi\) en la Sección 4.

Para observar la conducción por resonancia cruzada, usaremos experimentos muy similares al caso de calibración de pulso \(\pi\):

  • Inicialmente, el qubit \(1\) se deja en el estado fundamental o se conduce a su primer estado excitado usando el pulso \(\pi\) que se encuentra en la Sección 4.

  • Se ejecuta una secuencia de experimentos, cada uno de los cuales consta de un pulso gaussiano en el qubit \(1\) dirigido a la frecuencia del qubit \(0\), seguido de una medición de ambos qubits, y cada experimento de la secuencia tiene subsecuentemente una mayor amplitud para el pulso gaussiano.

En el código: Se construyen funciones para definir los experimentos y visualizar los datos, incluida una visualización de la trayectoria del qubit objetivo en la esfera de Bloch.

3. Modelo de sistema de oscilador Duffing¶

Se puede construir un objeto que representa un modelo para una colección de osciladores Duffing usando la función duffing_system_model. Aquí construimos un modelo de oscilador \(2\) Duffing con dimensión de corte \(3\).

[4]:

# cutoff dimension
dim_oscillators = 3

# frequencies for transmon drift terms, harmonic term and anharmonic term
# Number of oscillators in the model is determined from len(oscillator_freqs)
oscillator_freqs = [5.0e9, 5.2e9]
anharm_freqs = [-0.33e9, -0.33e9]

# drive strengths
drive_strengths = [0.02e9, 0.02e9]

# specify coupling as a dictionary (qubits 0 and 1 are coupled with a coefficient 0.002e9)
coupling_dict = {(0,1): 0.002e9}

# sample duration for pulse instructions
dt = 1e-9

# create the model
two_qubit_model = duffing_system_model(dim_oscillators=dim_oscillators,
                                       oscillator_freqs=oscillator_freqs,
                                       anharm_freqs=anharm_freqs,
                                       drive_strengths=drive_strengths,
                                       coupling_dict=coupling_dict,
                                       dt=dt)

4 Calibrando pulsos \(\pi\) en cada qubit usando Ignis¶

Como se describe en la introducción, ahora calibramos pulsos \(\pi\) en cada qubit en two_qubit_model. Los experimentos en este procedimiento de calibración se conocen como experimentos Rabi, y los datos que observaremos se conocen como oscilaciones Rabi.

4.1 Construir los planificadorers¶

Construimos los planificadores utilizando la función rabi_schedules en Ignis. Para ello, necesitamos proporcionar un InstructionScheduleMap que contenga un planificador de medición.

[5]:

# list of qubits to be used throughout the notebook
qubits = [0, 1]

# Construct a measurement schedule and add it to an InstructionScheduleMap
meas_amp = 0.025
meas_samples = 1200
meas_sigma = 4
meas_width = 1150
meas_pulse = GaussianSquare(duration=meas_samples, amp=meas_amp,
                            sigma=meas_sigma, width=meas_width)

acq_sched = pulse.Acquire(meas_samples, pulse.AcquireChannel(0), pulse.MemorySlot(0))
acq_sched += pulse.Acquire(meas_samples, pulse.AcquireChannel(1), pulse.MemorySlot(1))

measure_sched = pulse.Play(meas_pulse, pulse.MeasureChannel(0)) | pulse.Play(meas_pulse, pulse.MeasureChannel(1)) | acq_sched

inst_map = pulse.InstructionScheduleMap()
inst_map.add('measure', qubits, measure_sched)

A continuación, construye los planificadores Rabi.

5. Oscilaciones por impulso de resonancia cruzada¶

A continuación, simulamos los efectos de un impulso de resonancia cruzada en el qubit \(1\) con el qubit \(0\) como objetivo, observando que la trayectoria y las oscilaciones del qubit \(0\) depende del estado del qubit \(1\).

Nota: Esta sección depende de las calibraciones del pulso \(\pi\) de la Sección 2.

5.1 Índices de resonancia cruzada ControlChannel¶

Conducir el qubit \(1\) a la frecuencia del qubit \(0\) requiere el uso de un pulso ControlChannel. La función de generación de modelo duffing_system_model, configura automáticamente ControlChannels para realizar controles de resonancia cruzada entre pares de qubits acoplados. El índice del ControlChannels para realizar un control de resonancia cruzada en particular se puede recuperar utilizando el método de clase control_channel_index en el objeto PulseSystemModel regresado. Por ejemplo, para obtener el índice ControlChannel correspondiente a una unidad CR en el qubit 1 con el objetivo 0, llama a la función control_channel_index con la tupla (1,0):

[11]:

two_qubit_model.control_channel_index((1,0))

[11]:

1

Por lo tanto, para realizar un control de resonancia cruzada en qubit \(1\) con el qubit \(0\) como objetivo, usa ControlChannel(1). Esto se utilizará al construir objetos Schedule en esta sección.

5.2 Funciones para generar la lista de experimentos y analizar el resultado¶

Primero, definimos la función cr_drive_experiments, que, dados los índices de control (drive) y objetivo (target), y la opción de comenzar con el qubit de control en el estado fundamental o en el estado excitado, regresa una lista de experimentos para observar las oscilaciones.

[12]:

# store the pi amplitudes from Section 2 in a list
pi_amps = [rabifit.pi_amplitude(0), rabifit.pi_amplitude(1)]

def cr_drive_experiments(drive_idx,
                         target_idx,
                         flip_drive_qubit = False,
                         cr_drive_amps=np.linspace(0, 0.9, 16),
                         cr_drive_samples=800,
                         cr_drive_sigma=4,
                         pi_drive_samples=128,
                         pi_drive_sigma=16):
    """Generate schedules corresponding to CR drive experiments.

    Args:
        drive_idx (int): label of driven qubit
        target_idx (int): label of target qubit
        flip_drive_qubit (bool): whether or not to start the driven qubit in the ground or excited state
        cr_drive_amps (array): list of drive amplitudes to use
        cr_drive_samples (int): number samples for each CR drive signal
        cr_drive_sigma (float): standard deviation of CR Gaussian pulse
        pi_drive_samples (int): number samples for pi pulse on drive
        pi_drive_sigma (float): standard deviation of Gaussian pi pulse on drive

    Returns:
        list[Schedule]: A list of Schedule objects for each experiment
    """

    # Construct measurement commands to be used for all schedules
    meas_amp = 0.025
    meas_samples = 1200
    meas_sigma = 4
    meas_width = 1150
    meas_pulse = GaussianSquare(duration=meas_samples, amp=meas_amp,
                               sigma=meas_sigma, width=meas_width)

    acq_sched = pulse.Acquire(meas_samples, pulse.AcquireChannel(0), pulse.MemorySlot(0))
    acq_sched += pulse.Acquire(meas_samples, pulse.AcquireChannel(1), pulse.MemorySlot(1))

    # create measurement schedule
    measure_sched = (pulse.Play(meas_pulse, pulse.MeasureChannel(0)) |
                     pulse.Play(meas_pulse, pulse.MeasureChannel(1))|
                     acq_sched)

    # Create schedule
    schedules = []
    for ii, cr_drive_amp in enumerate(cr_drive_amps):

        # pulse for flipping drive qubit if desired
        pi_pulse = Gaussian(duration=pi_drive_samples, amp=pi_amps[drive_idx], sigma=pi_drive_sigma)

        # cr drive pulse
        cr_width = cr_drive_samples - 2*cr_drive_sigma*4
        cr_rabi_pulse = GaussianSquare(duration=cr_drive_samples,
                                       amp=cr_drive_amp,
                                       sigma=cr_drive_sigma,
                                       width=cr_width)

        # add commands to schedule
        schedule = pulse.Schedule(name='cr_rabi_exp_amp_%s' % cr_drive_amp)

        # flip drive qubit if desired
        if flip_drive_qubit:
            schedule += pulse.Play(pi_pulse, pulse.DriveChannel(drive_idx))

        # do cr drive
        # First, get the ControlChannel index for CR drive from drive to target
        cr_idx = two_qubit_model.control_channel_index((drive_idx, target_idx))
        schedule += pulse.Play(cr_rabi_pulse, pulse.ControlChannel(cr_idx))  << schedule.duration


        schedule += measure_sched << schedule.duration

        schedules.append(schedule)
    return schedules

def plot_cr_pop_data(drive_idx,
                     target_idx,
                     sim_result,
                     cr_drive_amps=np.linspace(0, 0.9, 16)):
    """Plot the population of each qubit.

    Args:
        drive_idx (int): label of driven qubit
        target_idx (int): label of target qubit
        sim_result (Result): results of simulation
        cr_drive_amps (array): list of drive amplitudes to use for axis labels
    """

    amp_data_Q0 = []
    amp_data_Q1 = []

    for exp_idx in range(len(cr_drive_amps)):
        exp_mem = sim_result.get_memory(exp_idx)
        amp_data_Q0.append(np.abs(exp_mem[0]))
        amp_data_Q1.append(np.abs(exp_mem[1]))

    plt.plot(cr_drive_amps, amp_data_Q0, label='Q0')
    plt.plot(cr_drive_amps, amp_data_Q1, label='Q1')
    plt.legend()
    plt.xlabel('Pulse amplitude, a.u.', fontsize=20)
    plt.ylabel('Signal, a.u.', fontsize=20)
    plt.title('CR (Target Q{0}, driving on Q{1})'.format(target_idx, drive_idx), fontsize=20)
    plt.grid(True)

def bloch_vectors(drive_idx, drive_energy_level, sim_result):
    """Plot the population of each qubit.

    Args:
        drive_idx (int): label of driven qubit
        drive_energy_level (int): energy level of drive qubit at start of CR drive
        sim_result (Result): results of simulation

    Returns:
        list: list of Bloch vectors corresponding to the final state of the target qubit
              for each experiment
    """

    # get the dimension used for simulation
    dim = int(np.sqrt(len(sim_result.get_statevector(0))))


    # get the relevant dressed state indices
    idx0 = 0
    idx1 = 0
    if drive_idx == 0:
        if drive_energy_level == 0:
            idx0, idx1 = 0, dim
        elif drive_energy_level == 1:
            idx0, idx1 = 1, dim + 1
    if drive_idx == 1:
        if drive_energy_level == 0:
            idx0, idx1 = 0, 1
        elif drive_energy_level == 1:
            idx0, idx1 = dim, dim + 1

    # construct Pauli operators for correct dressed manifold
    state0 = np.array([two_qubit_model.hamiltonian._estates[idx0]])
    state1 = np.array([two_qubit_model.hamiltonian._estates[idx1]])

    outer01 = np.transpose(state0)@state1
    outer10 = np.transpose(state1)@state0
    outer00 = np.transpose(state0)@state0
    outer11 = np.transpose(state1)@state1

    X = outer01 + outer10
    Y = -1j*outer01 + 1j*outer10
    Z = outer00 - outer11

    # function for computing a single bloch vector
    bloch_vec = lambda vec: np.real(np.array([np.conj(vec)@X@vec, np.conj(vec)@Y@vec, np.conj(vec)@Z@vec]))

    return [bloch_vec(sim_result.get_statevector(idx)) for idx in range(len(sim_result.results))]

def plot_bloch_sphere(bloch_vectors):
    """Given a list of Bloch vectors, plot them on the Bloch sphere

    Args:
        bloch_vectors (list): list of bloch vectors
    """
    sphere = Bloch()
    sphere.add_points(np.transpose(bloch_vectors))
    sphere.show()
