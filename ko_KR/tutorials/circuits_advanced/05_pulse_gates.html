펄스 게이트들¶

대부분의 양자 알고리즘은 회로 작동만으로 설명할 수 있다. 프로그램의 저수준 구현에 대한 제어가 필요할 때 펄스 게이트 를 사용할 수 있다. 펄스 게이트는 기본 게이트 만을 사용해 회로를 실행하는 제약을 뛰어넘어 어떤 기본 게이트라도 기본 구현된 것을 재정의할 수 있다

펄스 게이트를 사용하면 논리 회로 게이트(예: X )를 Schedule 이라고 하는 Qiskit 펄스 프로그램에 사상할 수 있다. 이 사상 작업을 교정 이라고 한다. 높은 정확도의 교정은 사상 된 논리 연산(예: X 게이트 교정은 \(|0\rangle\) 을 \(|1\rangle\) 로 바꾼다, 등등)을 충실히 구현하는 교정이다.

스케줄은 장치에 대한 모든 입력 채널들 에 걸쳐 입력 신호의 정확한 시간 역학을 지정한다. 일반적으로 큐비트 마다 구동 및 측정과 같은 여러 채널이 있다. 이 인터페이스는 더 강력하며 기본적인 장치의 물리학적 원리를 더 깊이 이해해야 한다.

펄스 프로그램은 물리적인 큐비트에 작용하는 것임을 주목하는 것이 중요하다. 큐비트 \(a\) 에 가해진 구동 펄스는 큐비트 \(b\) 의 상태에 같은 논리적 연산을 하지 않는다 - 바꾸어 말하면 게이트 교정은 큐비트들 사이에 상호 교환이 가능하지 않다. 이것은 X 게이트가 대상 큐비트에 독립적으로 정의되는 회로 레벨과는 대조적이다.

이 페이지는 회로에 교정을 추가하는 법을 보여준다.

참고: 펄스 게이트가 있는 프로그램을 실행하기 위해서는 OpenPulse를 지원하는 벡엔드를 사용해야 한다. backend.configuration().open_pulse 의 옵션이 True 인 것으로 OpenPulse를 사용할 수 있는지 확인할 수 있다. OpenPulse가 활성화 되어 있고 펄스 게이트 기능이 활성화되지 않은 경우, 입력 회로에 스케쥴 을 사용할 수 있다.

회로 구성하기¶

아주 간단한 예인 벨 상태(Bell state) 회로로부터 시작하자.

[1]:

from qiskit import QuantumCircuit

circ = QuantumCircuit(2, 2)
circ.h(0)
circ.cx(0, 1)
circ.measure(0, 0)
circ.measure(1, 1)

circ.draw('mpl')

[1]:

교정 구성하기¶

이제 회로가 있으므로 큐비트 0에 대한 Hadamard 게이트의 교정을 정의하자.

실제로, 펄스의 모양과 매개변수들은 Rabi 실험(Rabi experiments)의 반복을 통해 최적화된다 (Qiskit 교과서 를 따라가라). 이를 시연하고자 Hadamard는 가우시안(Gaussian)펄스가 될 것이다. 큐비트 0의 구동 채널에 펄스를 발생 시킬 것이다.

교정을 구성하는 세부사항 자체에 대해 너무 걱정하지 마라. 펄스 스케줄 작성 페이지에서 이에 대해 모두 학습할 수 있다.

[2]:

from qiskit import pulse
from qiskit.pulse.library import Gaussian
from qiskit.test.mock import FakeValencia

backend = FakeValencia()

with pulse.build(backend, name='hadamard') as h_q0:
    pulse.play(Gaussian(duration=128, amp=0.1, sigma=16), pulse.drive_channel(0))

무엇이 구성되었는지를 보고자 새로운 스케줄을 그리자.

[3]:

h_q0.draw()

[3]:

회로에 교정을 링크하기¶

등록을 마무리하는 것만이 남았다. 회로 메서드 add_calibration 은 사상을 완료하고자 게이트 정보와 스케줄에 대한 참조를 필요로 한다:

QuantumCircuit.add_calibration(gate, qubits, schedule, parameters)

gate 는 circuit.Gate 객체이거나 게이트의 이름 그대로가 될 수 있다. 보통 qubits 와 parameters 의 각각의 독특한 집합마다 다른 스케줄이 필요할 것이다. Hadamard 게이트에는 어떠한 매개변수도 없기 때문에, 어떤 것도 넘길 필요가 없다.

[4]:

circ.add_calibration('h', [0], h_q0)

마지막으로 트랜스파일러는 당신의 교정을 준수하다는 걸 주의하자. 그럴 거라고 생각하고 사용하자 (우리의 예는 트랜스파일러가 최적화하기에는 너무 간단하므로 출력이 같다).

[5]:

from qiskit import transpile
from qiskit.test.mock import FakeAlmaden

backend = FakeAlmaden()

circ = transpile(circ, backend)

print(backend.configuration().basis_gates)
circ.draw('mpl', idle_wires=False)

['id', 'u1', 'u2', 'u3', 'cx']

[5]:

h 가 모조 백엔드 FakeAlmaden 의 기저 게이트가 아닌 것에 주목하자. 보정이 추가되었기 때문에 트랜스파일러는 해당 게이트를 기저 게이트로 취급할 것이다; 하지만 오직 정의되었던 큐비트들에 대해서만 취급할 것이다. 다른 큐비트에 적용된 Hadamard는 기저 게이트들로 전개될 것이다.

다 됐다!

사용자 설정 게이트¶

비표준의 완전한 사용자 설정 게이트에 대해 같은 과정을 간단히 볼 것이다. 이번 시연에서는 매개변수가 있는 게이트가 포함되었다.

[6]:

from qiskit import QuantumCircuit
from qiskit.circuit import Gate

circ = QuantumCircuit(1, 1)
custom_gate = Gate('my_custom_gate', 1, [3.14, 1])
# 3.14 is an arbitrary parameter for demonstration
circ.append(custom_gate, [0])
circ.measure(0, 0)

circ.draw('mpl')

[6]:

[7]:

with pulse.build(backend, name='custom') as my_schedule:
    pulse.play(Gaussian(duration=64, amp=0.2, sigma=8), pulse.drive_channel(0))

circ.add_calibration('my_custom_gate', [0], my_schedule, [3.14, 1])
# Alternatively: circ.add_calibration(custom_gate, [0], my_schedule)

만약 교정을 추가하고자 Gate 인스턴스 변수 custom_gate 를 사용한다면, 그 매개변수들은 인스턴스로부터 도출된다. 매개변수 순서가 의미가 있음을 유념하라.

[8]:

circ = transpile(circ, backend)
circ.draw('mpl', idle_wires=False)

[8]:

보통 우리가 circ 를 트랜스파일링하려고 하면 에러를 보게 될 것이다. "my_custom_gate" 에 제공되는 함수적 정의가 없기 때문에, 트랜스파일러는 이를 대상 장치의 기본 게이트 집합으로 전개할 수 없다. "my_custom_gate" 를 교정되지 않은 또 다른 큐비트에 추가하려고 시도하면 이런 현상을 볼 수 있다.

[9]:

circ = QuantumCircuit(2, 2)
circ.append(custom_gate, [1])


from qiskit import QiskitError
try:
    circ = transpile(circ, backend)
except QiskitError as e:
    print(e)

"Cannot unroll the circuit to the given basis, ['id', 'u1', 'u2', 'u3', 'cx']. Instruction my_custom_gate not found in equivalence library and no rule found to expand."
