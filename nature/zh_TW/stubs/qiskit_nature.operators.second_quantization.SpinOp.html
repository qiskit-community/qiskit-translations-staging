SpinOp¶

class SpinOp(data, spin=Fraction(1, 2), register_length=None)[原始碼]¶

基礎類別：qiskit_nature.operators.second_quantization.second_quantized_op.SecondQuantizedOp

XYZ-ordered Spin operators.

Label

Allowed characters for primitives of labels are I, X, Y, Z, +, and -.

There are two types of label modes for SpinOp. The label mode is automatically detected.

  1. Dense Label (default, register_length = None)

Dense labels are strings in which each character maps to a unique spin mode. This is similar to Qiskit’s string-based representation of qubit operators. For example,

"X"
"IIYYZ-IX++"

are possible labels. Note, that dense labels are less powerful than sparse ones because they cannot represent all possible SpinOp. You will, for example, not be able to apply multiple operators on the same index within a single label.

  1. Sparse Label (register_length is passed)

A sparse label is a string consisting of a space-separated list of words. Each word must look like [XYZI+-]_<index>^<power>, where the <index> is a non-negative integer representing the index of the spin mode and the <power> is a positive integer indicating the number of times the given operator is applied to the mode at <index>. You can omit <power>, implying a single application of the operator (power = 1). For example,

"X_0"
"Y_0^2"
"Y_0^2 Z_0^3 X_1^1 Y_1^2 Z_1^2"

are possible labels. For each index the operations X, Y and Z can only be specified exclusively in this order. + and - cannot be used with X and Y because ladder operators will be parsed into X and Y. Thus, "Z_0 X_0", "Z_0 +_0", and "+_0 X_0" are invalid labels. The indices must be ascending order.

"+_i -_i" is supported. This pattern is parsed to +_i -_i = X_i^2 + Y_i^2 + Z_i.

Initialization

The SpinOp can be initialized by the list of tuples. For example,

from qiskit_nature.operators.second_quantization import SpinOp

x = SpinOp("X", spin=3/2)
y = SpinOp("Y", spin=3/2)
z = SpinOp("Z", spin=3/2)

are \(S^x, S^y, S^z\) for spin 3/2 system. Two qutrit Heisenberg model with transverse magnetic field is

SpinOp(
    [
        ("XX", -1),
        ("YY", -1),
        ("ZZ", -1),
        ("ZI", -0.3),
        ("IZ", -0.3),
    ],
    spin=1
)

SpinOp([('X_0 X_1', (-1+0j)), ('Y_0 Y_1', (-1+0j)), ('Z_0 Z_1', (-1+0j)), ('Z_0', (-0.3+0j)), ('Z_1', (-0.3+0j))], spin=1, register_length=2)

This means \(- S^x_0 S^x_1 - S^y_0 S^y_1 - S^z_0 S^z_1 - 0.3 S^z_0 - 0.3 S^z_1\).

SpinOp can be initialized with internal data structure (numpy.ndarray) directly. In this case, data is a tuple of two elements: spin_array and coeffs. spin_array is 3-dimensional ndarray. 1st axis has three elements 0, 1, and 2 corresponding to x, y, and z. 2nd axis represents the index of terms. 3rd axis represents the index of register. coeffs is one-dimensional ndarray with the length of the number of terms.

Algebra

SpinOp supports the following basic arithmetic operations: addition, subtraction, scalar multiplication, and adjoint. For example,

Raising Operator (addition and scalar multiplication)

x + 1j * y

SpinOp([('X_0', (1+0j)), ('Y_0', 1j)], spin=3/2, register_length=1)

Adjoint

~(1j * z)

SpinOp(('Z_0', -1j), spin=3/2, register_length=1)
參數

  • data (Union[str, List[Tuple[str, complex]], Tuple[ndarray, ndarray]]) – label string, list of labels and coefficients. See the label section in the documentation of SpinOp for more details.

  • spin (Union[float, Fraction]) – positive half-integer (integer or half-odd-integer) that represents spin.

  • register_length (Optional[int]) – length of the particle register.

引發

  • QiskitNatureError – invalid spin value
