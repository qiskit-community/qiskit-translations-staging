Property (বৈশিষ্ট্য) ফ্রেমওয়ার্ক¶

[1]:

import numpy as np

Qiskit Nature 0.2.0 Property ফ্রেমওয়ার্ক প্রবর্তন করে। এই ফ্রেমওয়ার্কটি আরো মডুলার এবং এক্সটেনসিবল পদ্ধতির সাথে QMolecule এবং WatsonHamiltonian এর মতো লিগ্যাসি ড্রাইভার রিটার্ন টাইপগুলি প্রতিস্থাপন করে।

এই টিউটোরিয়ালে, আমরা ফ্রেমওয়ার্কের মধ্য দিয়ে চলব, এর সবচেয়ে গুরুত্বপূর্ণ উপাদানগুলি ব্যাখ্যা করব এবং আপনাকে দেখাব, কিভাবে আপনি এটিকে একটি কাস্টম property দিয়ে নিজেই প্রসারিত করতে পারেন।

Property কি?¶

এর মূল অংশে, একটি Property এমন একটি বস্তু যা কিছু অপরিমার্জিত ডেটাকে ফাংশনগুলির সাথে পরিপূরক করে যা আপনাকে এই অপরিমার্জিত ডেটা রূপান্তর/হেরফের/ব্যাখ্যা করতে দেয়। এই "সংজ্ঞা" উদ্দেশ্যগতভাবে বরং বিমূর্ত রাখা হয়েছে, কিন্তু এর অর্থ কী তা মূলত নিম্নলিখিত। Property মানে:

  • একটি বাহ্যিক পর্যবেক্ষণযোগ্য প্রতিনিধিত্ব করে (এটি অপরিমার্জিত তথ্য)

  • অপারেটর হিসেবে প্রকাশ করা যায়

  • ওয়েভ ফাংশন দিয়ে মূল্যায়ন করা যায়

  • একটি interpret পদ্ধতি প্রদান করে যা মূল্যায়নকৃত কিউবিট অপারেটরের স্বকীয়মান (আইগেনভ্যালু) এর অর্থ প্রদান করে

[2]:

from qiskit_nature.properties import Property, GroupedProperty

Qiskit_nature.properties মডিউল দুটি ক্লাস প্রদান করে:

  1. Property: এটি মৌলিক ইন্টারফেস। এটি প্রয়োগ করার জন্য শুধুমাত্র একটি নাম এবং একটি ব্যাখ্যা পদ্ধতি প্রয়োজন।

  2. GroupedProperty: এই শ্রেণীটি Composite pattern এর একটি বাস্তবায়ন যা আপনাকে একাধিক গ্রুপের বৈশিষ্ট্যগুলিকে এক করতে দেয়।

দ্রষ্টব্য: গোষ্ঠীভুক্ত বৈশিষ্ট্যগুলির অনন্য name অ্যাট্রিবিউট থাকতে হবে!

দ্বিতীয় কোয়ান্টাইজেশন বৈশিষ্ট্য¶

লেখার সময়, Qiskit Nature বৈশিষ্ট্যগুলির একক রূপের সাথে আসে: দ্বিতীয় কোয়ান্টাইজেশন বৈশিষ্ট্য বস্তু।

এই সাব-টাইপটি একটি অতিরিক্ত প্রয়োজনীয়তা যুক্ত করে কারণ যেকোনো SecondQuantizedProperty

  • অবশ্যই একটি second_q_ops পদ্ধতি প্রয়োগ করতে হবে যা SecondQuantizedOpগুলির একটি (তালিকা) তৈরি করে।

qiskit_nature.properties.second_quantization মডিউলটি আরও ইলেকট্রনিক এবং ভাইব্রেশনাল মডিউলে বিভক্ত (Qiskit Nature এর অন্যান্য মডিউলের মতো)। আসুন প্রথমে ইলেকট্রনিক সাব-মডিউলে ডুব দেই।

ইলেকট্রনিক দ্বিতীয় কোয়ান্টাইজেশন বৈশিষ্ট্য¶

আপনি দেখতে পাচ্ছেন, প্রথম যুক্তিটি কেবল ইন্টিগ্রালের ভিত্তি নির্দিষ্ট করে। দ্বিতীয় যুক্তির আরেকটু ব্যাখ্যা প্রয়োজন:

  1. "AO" বা "MO" ভিত্তির ক্ষেত্রে, এই যুক্তিটি অবশ্যই numpy অ্যারেগুলির একটি জোড়া হতে হবে, যেখানে প্রথম এবং দ্বিতীয়টি যথাক্রমে আলফা- এবং বিটা-স্পিন অবিচ্ছেদ্য।

দ্রষ্টব্য: দ্বিতীয় যুক্তিটি None হতে পারে, সেই ক্ষেত্রে যেখানে বিটা-স্পিন ইন্টিগ্রালগুলি আলফা-স্পিন ইন্টিগ্রালগুলির সমান (তাই দুইবার একই মান দেওয়ার প্রয়োজন নেই)।

  1. "SO" ভিত্তির ক্ষেত্রে, এই যুক্তি অবশ্যই একটি একক সংখ্যাযুক্ত অ্যারে হতে হবে, যা আলফা- এবং বিটা-স্পিন ইন্টিগ্রালগুলিকে ব্লক অর্ডারে সংরক্ষণ করে, যেমন:

spin_basis = np.block([[alpha_spin, zeros], [zeros, beta_spin]])

আমাদের কয়েকটি পর্যবেক্ষণ লক্ষ্য করা উচিত:

  • numpy array কে ("alpha-alpha", "beta-alpha", "beta-beta", "alpha-beta") এভাবে সাজানো হবে

  • alpha-alpha ম্যাট্রিক্স None হতে পারে না

  • যদি alpha-beta ম্যাট্রিক্স None হয়, কিন্তু beta-alpha না হয়, তবে এর transpose ব্যবহার করা হবে (উপরের মত)

  • অন্য কোন ক্ষেত্রে, None ম্যাট্রিক্স alpha-alpha ম্যাট্রিক্স দিয়ে ভরা হবে

  • SO ভিত্তি ক্ষেত্রে, একটি একক numpy অ্যারে নির্দিষ্ট করা আবশ্যক। এর সঠিক বিন্যাসের জন্য TwoBodyElectronicIntegrals.to_spin() পড়ুন।

ফলাফল ব্যাখ্যা¶

একটি অতিরিক্ত সুবিধা যা আমরা বৈশিষ্ট্য কাঠামো থেকে লাভ করি, তা হল একটি গণিত আইগেনভ্যালুর ফলাফল ব্যাখ্যা প্রতিটি বৈশিষ্ট্য নিজেই পরিচালনা করতে পারে। এটি সুন্দর এবং যৌক্তিকভাবে সামঞ্জস্যপূর্ণ ক্লাসে ফলাফল দেয় কারণ ফলাফলটি একই প্রসঙ্গে ব্যাখ্যা করা হয় যেখানে অপরিমার্জিত তথ্য (ডেটা) পাওয়া যায়।

কণাসংখ্যা¶

ParticleNumber বৈশিষ্ট্য অন্তর্নির্মিত বৈশিষ্ট্যের মধ্যে একটি বিশেষ স্থান নেয় কারণ এটি একটি দ্বৈত উদ্দেশ্যে কাজ করে:

  • ইলেকট্রনিক সিস্টেমে কণার সংখ্যা সংরক্ষণ করা

  • অপারেটরদের সমস্যার প্রদত্ত eigensolution এর জন্য কণার সংখ্যা মূল্যায়ন করার জন্য প্রদান করা

অতএব, যদি আপনি ActiveSpaceTransformer বা ElectronicStructureProblem.default_filter_criterion() এর মতো অতিরিক্ত কার্যকারিতা ব্যবহার করতে চান তবে এই বৈশিষ্ট্য প্রয়োজন।

বিশেষায়িত (কাস্টম) প্রোপার্টি লেখা¶

আপনি আপনার নিজস্ব বাস্তবায়নের সাথে বৈশিষ্ট্য কাঠামো প্রসারিত করতে পারেন। এখানে, আমরা ইলেকট্রনিক ঘনত্ব এর জন্য একটি বৈশিষ্ট্য (Property) তৈরির সহজ উদাহরণ দিয়ে যাব। যেহেতু এই পর্যবেক্ষণযোগ্যটি মূলত ম্যাট্রিক্সের উপর ভিত্তি করে, তাই আমরা প্রকৃত ঘনত্বের ম্যাট্রিক্স সংরক্ষণের জন্য OneBodyElectronicIntegrals ধারকটি ব্যবহার করব।

class ElectronicDensity(ElectronicProperty):
    """A simple electronic density property.

    This basic example works only in the MO basis!
    """

    def __init__(self, num_molecular_orbitals: int) -> None:
        super().__init__(self.__class__.__name__)
        self._num_molecular_orbitals = num_molecular_orbitals

    def __str__(self) -> str:
        string = [super().__str__() + ":"]
        string += [f"\t{self._num_molecular_orbitals} MOs"]
        return "\n".join(string)

    @classmethod
    def from_legacy_driver_result(cls, result) -> "ElectronicDensity":
        cls._validate_input_type(result, QMolecule)

        qmol = cast(QMolecule, result)

        return cls(qmol.num_molecular_orbitals)


    def second_q_ops(self) -> List[FermionicOp]:
        ops = []

        # iterate all pairs of molecular orbitals
        for mo_i, mo_j in product(range(self._num_molecular_orbitals), repeat=2):

            # construct an auxiliary matrix where the only non-zero entry is at the current pair of MOs
            number_op_matrix = np.zeros((self._num_molecular_orbitals, self._num_molecular_orbitals))
            number_op_matrix[mo_i, mo_j] = 1

            # leverage the OneBodyElectronicIntegrals to construct the corresponding FermionicOp
            one_body_ints = OneBodyElectronicIntegrals(ElectronicBasis.MO, (number_op_matrix, number_op_matrix))
            ops.append(one_body_ints.to_second_q_op())

        return ops

    def interpret(self, result) -> None:
        # here goes the code which interprets the eigenvalues returned for the auxiliary operators
        pass

** দ্রষ্টব্য: ** কিস্কিট প্রকৃতি সংস্করণ 0.2.0 অনুযায়ী, সহায়ক অপারেটর বিশ্লেষণের সীমাবদ্ধতার কারণে স্ট্যাকের মধ্যে কাস্টম প্রপার্টি অবজেক্টের সরাসরি সংহতকরণ এখনও বাস্তবায়িত হয়নি। আরো বিস্তারিত জানার জন্য https://github.com/Qiskit/qiskit-nature/issues/312 দেখুন।

আপাতত, আপনি এখনও একটি কাস্টম সম্পত্তির মূল্যায়ন করতে পারেন, এটি aux_operators নির্মাণের মাধ্যমে সরাসরি Eigensolver.solve পদ্ধতিতে উৎপাদিত অপারেটরদের পাস করে। তবে মনে রাখবেন, উপরের সমস্যাটি সমাধান না হওয়া পর্যন্ত আপনাকে আপনার বৈশিষ্ট্যগুলিতে ম্যানুয়ালি প্রয়োগ করা রূপান্তরগুলি মোকাবেলা করতে হবে।
