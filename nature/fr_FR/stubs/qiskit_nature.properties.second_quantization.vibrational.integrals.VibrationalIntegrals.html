VibrationalIntegrals¶

class VibrationalIntegrals(num_body_terms, integrals)[source]¶

Bases : abc.ABC

A container for arbitrary n-body vibrational integrals.

When these integrals are printed the output will be truncated based on the VibrationalIntegrals._truncate value (defaults to 5). Use VibrationalIntegrals.set_truncation to change this value.

Paramètres

  • num_body_terms (int) – n, as in the n-body terms stored in these integrals.

  • integrals (List[Tuple[float, Tuple[int, …]]]) – a sparse list of integrals. The data format corresponds to a list of pairs, with its first entry being the integral coefficient and the second entry being a tuple of integers of length num_body_terms. These integers are the indices of the modes associated with the integral. If the indices are negative, the integral is treated as a kinetic term of the vibrational hamiltonian.

Lève

ValueError – if the number of body terms is less than 1.

Attributes

basis

Returns the basis.

integrals

Returns the integrals.

Methods

set_truncation(max_num_entries)

Set the maximum number of integral values to display before truncation.

to_basis()

Maps the integrals into a basis which permits mapping into second-quantization.

to_second_q_op()

Creates the operator representing the Hamiltonian defined by these vibrational integrals.
