பொருள் கட்டமைப்பு¶

[1]:

import numpy as np

Qiskit Nature 0.2.0 Property கட்டமைப்பை அறிமுகப்படுத்துகிறது. இந்த கட்டமைப்பானது QMolecule மற்றும் WatsonHamiltonian போன்ற பாரம்பரிய டிரைவர் ரிட்டர்ன் வகைகளை அதிக மட்டு மற்றும் விரிவாக்க அணுகுமுறையுடன் மாற்றுகிறது.

இந்த பயிற்சியில், நாங்கள் கட்டமைப்பின் வழியாக நடந்து, அதன் மிக முக்கியமான கூறுகளை விளக்கி, தனிப்பயன் property நீங்களே எப்படி நீட்டிக்க முடியும் என்பதை உங்களுக்குக் காண்பிப்போம்.

Property என்றால் என்ன?¶

அதன் மையத்தில், Property என்பது இந்த மூல தரவை மாற்ற/கையாள/விளக்குவதற்கு அனுமதிக்கும் செயல்பாடுகளுடன் சில மூல தரவுகளை பூர்த்தி செய்யும் ஒரு பொருள். இந்த "வரையறை" வேண்டுமென்றே சுருக்கமாக வைக்கப்பட்டுள்ளது, ஆனால் இதன் பொருள் அடிப்படையில் பின்வருமாறு. ஒரு Property:

  • ஒரு உடல் காணக்கூடியதைக் குறிக்கிறது (அது பதனிடப்படாத தரவு)

  • ஒரு ஆபரேட்டராக வெளிப்படுத்தலாம்

  • ஒரு அலை செயல்பாட்டுடன் மதிப்பீடு செய்யலாம்

  • மதிப்பீடு செய்யப்பட்ட க்யூபிட் ஆபரேட்டரின் முக்கிய மதிப்பை வழங்கும் interpret முறையை வழங்குகிறது

[2]:

from qiskit_nature.properties import Property, GroupedProperty

qiskit_nature.properties தொகுதி இரண்டு வகுப்புகளை வழங்குகிறது:

  1. Property: இது அடிப்படை இடைமுகம். அதை செயல்படுத்த name மற்றும் interpret முறை மட்டுமே தேவைப்படுகிறது.

  2. GroupedProperty: இந்த வகுப்பு Composite pattern இன் செயல்பாடாகும், இது உங்களை பல பண்புகளை ஒன்றாக இணைக்க அனுமதிக்கிறது.

குறிப்பு: தொகுக்கப்பட்ட பண்புகள் தனித்துவமான name பண்புகளைக் கொண்டிருக்க வேண்டும்!

இரண்டாவது அளவீட்டு பண்புகள்¶

எழுதும் நேரத்தில், Qiskit Nature பண்புகளின் ஒற்றை மாறுபாட்டுடன் அனுப்பப்படுகிறது: SecondQuantizedProperty பொருள்கள்.

இந்த துணை வகை ஒரு கூடுதல் தேவையை சேர்க்கிறது, ஏனெனில் எந்த SecondQuantizedProperty

  • ஒரு second_q_ops முறையை அமல்படுத்த வேண்டும், இது ஒரு (பட்டியல்) SecondQuantizedOp ஐ உருவாக்குகிறது.

qiskit_nature.properties.second_quantization தொகுதி மேலும் electronic மற்றும் vibrational தொகுதிகளாக பிரிக்கப்பட்டுள்ளது (Qiskit Nature மற்ற தொகுதிகள் போன்றது). முதலில் electronic துணை-தொகுதிக்குள் நுழைவோம்.

மின்னணு இரண்டாம் அளவு பண்புகள்¶

ElectronicEnergy என்பது மிக முக்கியமான சொத்து, ஏனெனில் இது மின்னணு கட்டமைப்பு சிக்கலைக் குறிக்கும் Hamiltonian ஐக் கொண்டுள்ளது, அதன் ElectronicStructureProblem ஐ தீர்க்கும்போது நாம் ஆர்வமாக உள்ளோம். இந்த ஹாமில்டோனியனை அது சேமிக்கும் விதம், மீதமுள்ள கட்டமைப்பைப் போலவே, மிகவும் மட்டு. ElectronicEnergy இன் துவக்கத்தில் ஒரு தேவையான வாதம் உள்ளது, electronic_integrals, இது ஒரு List[ElectronicIntegrals].

மின்னணு ஒருங்கிணைப்பு¶

[4]:

from qiskit_nature.properties.second_quantization.electronic.integrals import (

    OneBodyElectronicIntegrals,
    TwoBodyElectronicIntegrals,
    IntegralProperty,
)

ElectronicIntegrals என்பது கொடுக்கப்பட்ட அடிப்படையில் n-body தொடர்புகளுக்கான ஒரு கொள்கலன் வகுப்பாகும், இது பின்வருவனவற்றில் ஏதேனும் இருக்கலாம்:

உதாரணமாக OneBodyElectronicIntegrals ஐ எடுத்துக் கொள்வோம். பெயர் குறிப்பிடுவது போல, இந்த கொள்கலன் 1-body தொடர்பு ஒருங்கிணைப்புகளுக்கானது. இது போன்ற ஒரு உதாரணத்தை நீங்கள் உருவாக்கலாம்:

நீங்கள் பார்க்க முடியும் என, முதல் வாதம் வெறுமனே ஒருங்கிணைப்புகளின் அடிப்படையைக் குறிப்பிடுகிறது. இரண்டாவது வாதத்திற்கு இன்னும் கொஞ்சம் விளக்கம் தேவை:

  1. AO அல்லது MO அடிப்படையில், இந்த வாதம் ஒரு ஜோடி எண் வரிசைகளாக இருக்க வேண்டும், அங்கு முதல் மற்றும் இரண்டாவது முறையே ஆல்பா மற்றும் பீட்டா-ஸ்பின் ஒருங்கிணைப்புகள்.

குறிப்பு: இரண்டாவது வாதம் None என்று இருக்கலாம், பீட்டா-ஸ்பின் ஒருங்கிணைந்தவை ஆல்ஃபா-ஸ்பின் ஒருங்கிணைந்தவை (எனவே ஒரே மதிப்புகளை இரண்டு முறை வழங்க வேண்டிய அவசியம் இல்லை).

  1. SO அடிப்படையைப் பொறுத்தவரை, இந்த வாதம் ஒற்றை எண் வரிசையாக இருக்க வேண்டும், ஆல்பா மற்றும் பீட்டா-ஸ்பின் ஒருங்கிணைப்புகளை தடுக்கப்பட்ட வரிசையில் சேமிக்கிறது, அதாவது. அப்படி:

spin_basis = np.block([[alpha_spin, zeros], [zeros, beta_spin]])

TwoBodyElectronicIntegrals என்பது ஒரே மாதிரியாக வேலை செய்கிறது, தவிர அவை நம்பி வரிசைகளின் டப்பிளில் சாத்தியமான அனைத்து ஸ்பின்-இணைப்புகளையும் கொண்டிருக்கின்றன. உதாரணத்திற்கு:

ஒரு சில அவதானிப்புகளை நாம் கவனத்தில் கொள்ள வேண்டும்:

  • எண் வரிசைகள் ("alpha-alpha", "beta-alpha", "beta-beta", "alpha-beta") என வரிசைப்படுத்தப்பட வேண்டும்

  • alpha-alpha அணி not None

  • alpha-beta மேட்ரிக்ஸ் None, ஆனால் beta-alpha இல்லையென்றால், அதன் இடமாற்றம் பயன்படுத்தப்படும் (மேலே உள்ளதைப் போல)

  • வேறு எந்த விஷயத்திலும், None என்ற அணி alpha-alpha அணிகளால் நிரப்பப்படும்

  • SO அடிப்படை வழக்கில், ஒற்றை எண் வரிசை குறிப்பிடப்பட வேண்டும். அதன் சரியான வடிவமைப்பிற்காக TwoBodyElectronicIntegrals.to_spin () ஐப் பார்க்கவும்.

முடிவு விளக்கம்¶

Property கட்டமைப்பிலிருந்து நாம் பெறும் கூடுதல் நன்மை என்னவென்றால், கணக்கிடப்பட்ட ஈஜென்வெல்யூவின் முடிவு விளக்கத்தை ஒவ்வொரு சொத்தாலும் கையாள முடியும். இது நல்ல மற்றும் தர்க்கரீதியாக சீரான வகுப்புகளில் விளைகிறது, ஏனெனில் பதனிடப்படாதத் தரவு கிடைக்கும் அதே சூழலில் இதன் விளைவாக விளக்கப்படுகிறது.

[11]:

from qiskit_nature.results import ElectronicStructureResult

# some dummy result
result = ElectronicStructureResult()
result.eigenenergies = np.asarray([-1])
result.computed_energies = np.asarray([-1])


# now, let's interpret it
electronic_energy.interpret(result)
print(result)

இந்த குறிப்பிட்ட உதாரணம் இன்னும் மிகவும் ஈர்க்கக்கூடியதாக இல்லை என்றாலும், ஒரே நேரத்தில் அதிக பண்புகளைப் பயன்படுத்தும் வரை காத்திருக்கவும்.

பகுதி எண்¶

ParticleNumber சொத்து கட்டப்பட்ட பண்புகளில் ஒரு சிறப்பு இடத்தைப் பெறுகிறது, ஏனெனில் இது இரட்டை நோக்கத்திற்காக செயல்படுகிறது:

  • மின்னணு அமைப்பில் உள்ள துகள்களின் எண்ணிக்கையை சேமித்தல்

  • கொடுக்கப்பட்ட பிரச்சனைக்குத் தேவையான துகள்களின் எண்ணிக்கையை மதிப்பீடு செய்ய ஆபரேட்டர்களை வழங்குதல்

எனவே, ActiveSpaceTransformer அல்லது ElectronicStructureProblem.default_filter_criterion() போன்ற கூடுதல் செயல்பாடுகளை நீங்கள் பயன்படுத்த விரும்பினால் இந்த சொத்து தேவைப்படுகிறது.

தனிப்பயன் பண்புகளை எழுதுதல்¶

உங்கள் சொந்த செயலாக்கங்களுடன் சொத்து கட்டமைப்பை நீட்டிக்க முடியும். electronic density ஒரு சொத்தை உருவாக்குவதற்கான எளிய எடுத்துக்காட்டு வழியாக இங்கே நடப்போம். இந்த கவனிக்கத்தக்கது அடிப்படையில் மெட்ரிக்குகளை அடிப்படையாகக் கொண்டிருப்பதால், உண்மையான அடர்த்தி மேட்ரிக்ஸை சேமிக்க OneBodyElectronicIntegrals கொள்கலனை நாங்கள் மேம்படுத்துவோம்.

class ElectronicDensity(ElectronicProperty):
    """A simple electronic density property.

    This basic example works only in the MO basis!
    """

    def __init__(self, num_molecular_orbitals: int) -> None:
        super().__init__(self.__class__.__name__)
        self._num_molecular_orbitals = num_molecular_orbitals

    def __str__(self) -> str:
        string = [super().__str__() + ":"]
        string += [f"\t{self._num_molecular_orbitals} MOs"]
        return "\n".join(string)

    @classmethod
    def from_legacy_driver_result(cls, result) -> "ElectronicDensity":
        cls._validate_input_type(result, QMolecule)

        qmol = cast(QMolecule, result)

        return cls(qmol.num_molecular_orbitals)


    def second_q_ops(self) -> List[FermionicOp]:
        ops = []

        # iterate all pairs of molecular orbitals
        for mo_i, mo_j in product(range(self._num_molecular_orbitals), repeat=2):

            # construct an auxiliary matrix where the only non-zero entry is at the current pair of MOs
            number_op_matrix = np.zeros((self._num_molecular_orbitals, self._num_molecular_orbitals))
            number_op_matrix[mo_i, mo_j] = 1

            # leverage the OneBodyElectronicIntegrals to construct the corresponding FermionicOp
            one_body_ints = OneBodyElectronicIntegrals(ElectronicBasis.MO, (number_op_matrix, number_op_matrix))
            ops.append(one_body_ints.to_second_q_op())

        return ops

    def interpret(self, result) -> None:
        # here goes the code which interprets the eigenvalues returned for the auxiliary operators
        pass

நிச்சயமாக, மேலே உள்ள எடுத்துக்காட்டு மிகக் குறைவு மற்றும் விருப்பப்படி நீட்டிக்கப்படலாம்.

Note: Qiskit நேச்சர் பதிப்பு 0.2.0 படி, துணை ஆபரேட்டர் பாகுபடுத்தலின் வரம்புகள் காரணமாக, தனிப்பயன் சொத்து பொருள்களை ஸ்டாக்கில் நேரடியாக ஒருங்கிணைப்பது இன்னும் செயல்படுத்தப்படவில்லை. மேலும் விவரங்களுக்கு https://github.com/Qiskit/qiskit-nature/issue/312 ஐப் பார்க்கவும்.

இப்போதைக்கு, நீங்கள் ஒரு தனிப்பயன் சொத்தை மதிப்பீடு செய்யலாம், அது உருவாக்கப்பட்ட ஆப்பரேட்டர்களை நேரடியாக Eigensolver.solve முறைக்கு aux_operators ஐ உருவாக்குவதன் மூலம் மதிப்பீடு செய்யலாம். எவ்வாறாயினும், மேலே உள்ள சிக்கல் தீர்க்கப்படும் வரை, உங்கள் சொத்துக்களுக்குப் பயன்படுத்தப்படும் மாற்றங்களை நீங்கள் கைமுறையாக கையாள வேண்டும் என்பதை நினைவில் கொள்க.
