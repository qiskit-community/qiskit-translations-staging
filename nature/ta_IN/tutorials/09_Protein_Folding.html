புரத மடிப்பு¶

முன்னுரை¶

பல இயற்கை மற்றும் மனிதனால் உருவாக்கப்பட்ட புரதங்களின் அமைப்பு மற்றும் செயல்பாடு இன்னும் சரியாக புரிந்து கொள்ளப்படவில்லை. இதன் விளைவாக, அல்சைமர் நோய், தடுப்பூசி வளர்ச்சி மற்றும் பயிர் மேம்பாட்டு ஆராய்ச்சி போன்ற புரத மடிப்புன் தொடர்புடைய செயல்முறைகளைப் பற்றிய நமது புரிதல் குறைவாகவே உள்ளது.

விரிவடையாத பாலிபெப்டைடுகள் மிகப் பெரிய அளவிலான சுதந்திரத்தைக் கொண்டுள்ளன, இதனால் ஏராளமான சாத்தியமான இணக்கங்கள் உள்ளன. உதாரணமாக, ஒரு கணிதம் \(100\) அமினோஅசிட்களின் வரிசையில் உள்ளது \(10^{47}\) இணக்கங்கள். உண்மையில், இருப்பினும், பல புரதங்கள் அவற்றின் சொந்த அமைப்பை நொடிகளில் மடித்து விடுகின்றன. இது லெவிந்தலின் முரண்பாடு என்று அழைக்கப்படுகிறது [1].

சங்கிலி நீளத்துடன் சாத்தியமான இணக்கங்களின் அதிவேக வளர்ச்சி கிளாசிக்கல் கணினிகளுக்கு சிக்கலை தீர்க்க முடியாததாக ஆக்குகிறது. குவாண்டம் கட்டமைப்பில், அமினோஅசிட்கள் N இன் எண்ணிக்கையுடன் எங்கள் வள-திறனுள்ள வழிமுறை அளவிடப்படுகிறது.

இந்த வேலையின் குறிக்கோள் ஒரு புரதத்தின் குறைந்தபட்ச ஆற்றல் இணக்கத்தை தீர்மானிப்பதாகும். ஒரு சீரற்ற கட்டமைப்பிலிருந்து தொடங்கி, ஆற்றலைக் குறைக்க புரதத்தின் அமைப்பு உகந்ததாக உள்ளது. புரத மடிப்பு சிக்கலை ஒரு க்யூபிட் ஆபரேட்டராக குறியாக்கம் செய்வதன் மூலமும், அனைத்து உடல் தடைகளும் திருப்தி அடைவதை உறுதி செய்வதன் மூலமும் இதை அடைய முடியும்.

சிக்கல் குறியாக்கத்திற்கு நாங்கள் பயன்படுத்துகிறோம்:

  • உள்ளமைவு வினாக்கள்: உள்ளமைவுகள் மற்றும் வெவ்வேறு மணிகளின் சார்புடைய நிலையை விவரிக்கப் பயன்படும் க்யூபிட்கள்

  • இன்டராக்ஷன் கியூபிட்ஸ்: வெவ்வேறு அமினோஆசிட்களுக்கு இடையேயான தொடர்புகளை குறியாக்கம் செய்யும் கியூபிட்ஸ்

எங்கள் விஷயத்தில் நாங்கள் டெட்ராஹெட்ரல் லட்டீஸை (வைர வடிவ லேட்டிஸ்) பயன்படுத்துகிறோம், அங்கு உள்ளமைவு கியூபிட்களின் மூலம் இயக்கத்தை குறியாக்குகிறோம் (கீழே உள்ள படத்தைப் பார்க்கவும்).

கியூபிட்களின் தொகுப்பிற்கான அமைப்பின் ஹாமில்டோனியன் \(\mathbf{q}=\{\mathbf{q}_{cf}, \mathbf{q}_{in}\}\)

\[H(\mathbf{q}) = H_{gc}(\mathbf{q}_{cf}) + H_{ch}(\mathbf{q}_{cf}) + H_{in}(\mathbf{q}_{cf}, \mathbf{q}_{in})\]

எங்கே

  • \(H_{gc}\) என்பது வடிவியல் கட்டுப்பாட்டு சொல் (பைஃபர்கேஷன்ஸ் இல்லாமல் அமினோஅசிட்களின் முதன்மை வரிசையின் வளர்ச்சியை நிர்வகிக்கிறது)

  • \(H_{gc}\) என்பது சிராலிட்டி கட்டுப்பாடு (கணினிக்கான சரியான ஸ்டீரியோ கெமிஸ்ட்ரியை செயல்படுத்துகிறது)

  • \(H_{in}\) என்பது அமைப்பின் தொடர்பு ஆற்றல் விதிமுறைகள். எங்கள் விஷயத்தில் அருகிலுள்ள அண்டை தொடர்புகளை மட்டுமே நாங்கள் கருதுகிறோம்.

பயன்படுத்தப்பட்ட மாதிரி மற்றும் சிக்கலின் குறியாக்கம் பற்றிய கூடுதல் விவரங்களை [2] இல் காணலாம்.

புரத முக்கிய சங்கிலி¶

புரதம் ஒரு முக்கிய சங்கிலியைக் கொண்டுள்ளது, இது அமினோஅசிட்களின் நேரியல் சங்கிலி ஆகும். வெவ்வேறு எச்சங்களுக்கு பெயரிடுவதற்கு, குறிப்பு [3] ல் வரையறுக்கப்பட்டபடி ஒரு எழுத்து குறியீட்டைப் பயன்படுத்துகிறோம். பெயரிடுதல் மற்றும் அமினோஅசிட்களின் வகை பற்றிய கூடுதல் விவரங்களையும் [4] ல் காணலாம்.

இந்த குறிப்பிட்ட வழக்கில், நியூரோபெப்டைட்டில் உள்ள க்யூபிட் ஆபரேட்டரின் தலைமுறையை பிரதான சங்கிலியுடன் 7 அமினோஅசிட்களைக் கொண்ட APRLRFY என்ற எழுத்து குறியீடுகளுடன் நாங்கள் காண்பிக்கிறோம் (மேலும் [2] பார்க்கவும்).

[2]:

main_chain = "APRLRFY"

பக்கச் சங்கிலிகள்¶

புரதத்தின் முக்கியச் சங்கிலியைத் தாண்டி, முக்கியச் சங்கிலியின் எச்சங்களுடன் அமினோஅசிட்கள் இணைக்கப்படலாம். எங்கள் மாதிரி ஒன்றின் அதிகபட்ச நீளத்தின் பக்கச் சங்கிலிகளை அனுமதிக்கிறது. நீட்டிக்கப்பட்ட பக்கச் சங்கிலிகளுக்கு இன்னும் வளர்ச்சியில் இருக்கும் கூடுதல் அபராத விதிமுறைகளை அறிமுகப்படுத்த வேண்டும். இந்த எடுத்துக்காட்டில், நியூரோபெப்டைட்டின் உண்மையான அமைப்பை வைத்திருக்க எந்த பக்க சங்கிலிகளையும் நாங்கள் கருத்தில் கொள்ளவில்லை.

side_chains = [""]*7

அமினோஅசிட்களுக்கு இடையிலான தொடர்பு¶

புரோட்டீன்களுக்கு இடையேயான தொடர்புகளின் விளக்கத்திற்கு, அரை-வேதியியல் தோராயத்தைப் பயன்படுத்தி பெறப்பட்ட அறிவு அடிப்படையிலான (புள்ளிவிவர) ஆற்றலைப் பயன்படுத்துகிறோம். இங்கு பயன்படுத்தப்படும் ஆற்றல்கள் மியாசாவா, எஸ். மற்றும் ஜெர்னிகன், ஆர். எல் [5] இல் அறிமுகப்படுத்தப்பட்டது.

இந்த மாதிரிக்கு அப்பால், சீரற்ற தொடர்பு வரைபடத்தை வழங்கும் சீரற்ற தொடர்பு வரைபடங்களையும் (இடைவினைகள்) அனுமதிக்கிறோம். புரதத்தின் சில உள்ளமைவுகளை மேம்படுத்தும் தனிப்பயன் தொடர்பு வரைபடத்தையும் ஒருவர் அறிமுகப்படுத்தலாம் (எ.கா. ஆல்பா ஹெலிக்ஸ், பீட்டா தாள் போன்றவை).

[4]:

random_interaction = RandomInteraction()
mj_interaction = MiyazawaJerniganInteraction()

இயற்பியல் கட்டுப்பாடு¶

அனைத்து இயற்பிய தடைகளும் மதிக்கப்படுவதை உறுதி செய்வதற்காக நாங்கள் அபராதம் செயல்பாடுகளை அறிமுகப்படுத்துகிறோம். பயன்படுத்தப்படும் வெவ்வேறு தண்டனை விதிமுறைகள்:

  • பெனால்டி_சிரல்: சரியான சிராலிட்டியைத் திணிக்கப் பயன்படுத்தப்படும் பெனால்டி அளவுரு.

  • பெனால்டி_பேக்: பெனால்டி விதிக்கப் பயன்படுத்தப்படும் பெனால்டி அளவுரு அதே அச்சில் சுழல்கிறது. ஒரே அச்சு தொடர்ச்சியாக இரண்டு முறை தேர்ந்தெடுக்கப்படும் காட்சிகளை அகற்ற இந்த சொல் பயன்படுத்தப்படுகிறது. இந்த வழியில் ஒரு சங்கிலி மீண்டும் மடிவதை நாம் அனுமதிக்க மாட்டோம்.

  • பெனால்டி_1: அருகிலுள்ள அண்டை தொடர்புகளுக்குள் உள்ள மணிகளுக்கு இடையில் உள்ளூர் ஒன்றுடன் ஒன்று தண்டிக்கப் பயன்படும் பெனால்டி அளவுரு.

[5]:

penalty_back = 10
penalty_chiral = 10
penalty_1 = 10

penalty_terms = PenaltyParameters(penalty_chiral, penalty_back, penalty_1)

பெப்டைட் வரையறை¶

முக்கிய வழி மற்றும் சாத்தியமான பக்க வழிகளின் அடிப்படையில், மாதிரி அமைப்பின் அனைத்து கட்டமைப்பு தகவல்களையும் உள்ளடக்கிய பெப்டைட் பொருளை வரையறுக்கிறோம்.

[6]:

peptide = Peptide(main_chain, side_chains)

Protein Folding பிரச்சினை¶

வரையறுக்கப்பட்ட பெப்டைடு, தொடர்பு (தொடர்பு வரைபடம்) மற்றும் எங்கள் மாதிரிக்கு நாங்கள் வரையறுக்கப்பட்ட அபராத விதிமுறைகளின் அடிப்படையில், க்யூபிட் ஆபரேட்டர்களை வழங்கும் புரத மடிப்பு பிரச்சினை நாங்கள் வரையறுக்கிறோம்.

சிக்கலின் தீர்வுக்கு CVaR எதிர்பார்ப்பு மதிப்புடன் VQE ஐப் பயன்படுத்துதல்¶

நாங்கள் சமாளிக்கும் சிக்கல் இப்போது அனைத்து உடல் தடைகளையும் செயல்படுத்தியுள்ளது மற்றும் ஒரு மூலைவிட்ட ஹாமில்டோனியனைக் கொண்டுள்ளது. குறிப்பிட்ட விஷயத்தில், எங்களுக்கு குறைந்தபட்ச ஆற்றலை (புரதத்தின் மடிந்த கட்டமைப்பிற்கு ஒத்ததாக) வழங்கும் ஒற்றை பிட்ஸ்ட்ரிங்கை நாங்கள் குறிவைக்கிறோம். எனவே, சிக்கலைத் தீர்ப்பதற்கும் குறைந்தபட்ச உள்ளமைவு ஆற்றலைக் கண்டுபிடிப்பதற்கும் நிபந்தனை மதிப்பில் (CVaR) எதிர்பார்ப்பு மதிப்புகளுடன் மாறுபாடு குவாண்டம் ஐஜென்சோல்வரைப் பயன்படுத்தலாம் [6] . Ref இல் உள்ள அதே அணுகுமுறையை நாங்கள் பின்பற்றுகிறோம். [2] ஆனால் இங்கே கிளாசிக்கல் தேர்வுமுறை பகுதிக்கு கோபிலாவைப் பயன்படுத்துகிறோம். Ref இல் விவாதிக்கப்பட்டாலும், சிக்கலைத் தீர்க்க நிலையான VQE அல்லது QAOA வழிமுறையையும் ஒருவர் பயன்படுத்தலாம். [2] CVaR மிகவும் பொருத்தமானது.

from qiskit.circuit.library import RealAmplitudes
from qiskit.algorithms.optimizers import COBYLA
from qiskit.algorithms import NumPyMinimumEigensolver, VQE
from qiskit.opflow import PauliExpectation, CVaRExpectation
from qiskit import execute, Aer

# set classical optimizer
optimizer = COBYLA(maxiter=50)

# set variational ansatz
ansatz = RealAmplitudes(reps=1)

#set the backend
backend_name = 'aer_simulator'
backend = QuantumInstance(
    Aer.get_backend(backend_name),
    shots=8192,
    seed_transpiler=algorithm_globals.random_seed,
    seed_simulator=algorithm_globals.random_seed)

counts = []
values = []

def store_intermediate_result(eval_count, parameters, mean, std):
    counts.append(eval_count)
    values.append(mean)

# initialize CVaR_alpha objective with alpha = 0.1
cvar_exp = CVaRExpectation(0.1, PauliExpectation())

# initialize VQE using CVaR
vqe = VQE(expectation=cvar_exp, optimizer=optimizer, ansatz=ansatz, quantum_instance=backend,
          callback=store_intermediate_result)

result = vqe.compute_minimum_eigenvalue(qubit_op)
print(result)
