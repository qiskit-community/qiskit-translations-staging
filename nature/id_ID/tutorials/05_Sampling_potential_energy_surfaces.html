Penyampelan energi potensial permukaan¶

Introduction¶

This interactive notebook demonstrates how to utilize the Potential Energy Surface (PES) samplers algorithm of Qiskit Nature to generate the dissociation profile of a molecule. We use the Born-Oppenheimer Potential Energy Surface (BOPES) and demonstrate how to exploit bootstrapping and extrapolation to reduce the total number of function evaluations in computing the PES using the Variational Quantum Eigensolver (VQE).

Kalkulasi disosiasi ikatan menggunakan Penyampel BOPES¶

Di sini, kita meneruskan informasi molekuler dan VQE ke tipe bawaan yang disebut Penyampel BOPES. Penyampel BOPES memungkinkan komputasi energi potensial permukaan untuk set tertentu dari derajat kebebasan/titik yang diinginkan.

Pertama-tama, kita bandingkan antara tanpa bootstrapping vs bootstrapping¶

Bootstrapping penyampel BOPES ini melibatkan pemanfaatan parameter variasional optimal untuk derajat kebebasan tertentu, katakanlah r (misal jarak antaratom) sebagai titik awal untuk VQE pada derajat kebebasan setelahnya, katakanlah r + :math:` epsilon `. Secara default, jika boostrapping diatur ke True, semua parameter optimal sebelumnya digunakan sebagai titik awal untuk langkah berikutnya.

distance1 = partial(Molecule.absolute_distance, atom_pair=(1, 0))
mol = Molecule(geometry=[('H', [0., 0., 0.]),
                        ('H', [0., 0., 0.3])],
                       degrees_of_freedom=[distance1],
                       )

# pass molecule to PSYCF driver
driver = ElectronicStructureMoleculeDriver(mol, driver_type=ElectronicStructureDriverType.PYSCF)

es_problem = ElectronicStructureProblem(driver)


# Specify degree of freedom (points of interest)
points = np.linspace(0.25, 2, 30)
results_full = {} # full dictionary of results for each condition
results = {} # dictionary of (point,energy) results for each condition
conditions = {False: 'no bootstrapping', True: 'bootstrapping'}


for value, bootstrap in conditions.items():
    # define instance to sampler
    bs = BOPESSampler(
        gss=me_gsc
        ,bootstrap=value
        ,num_bootstrap=None
        ,extrapolator=None)
    # execute
    res = bs.sample(es_problem,points)
    results_full[f'{bootstrap}'] =  res.raw_results
    results[f'points_{bootstrap}'] = res.points
    results[f'energies_{bootstrap}'] = res.energies

Ekstrapolasi¶

Di sini, sebuah ekstrapolator ditambahkan yang akan mencoba menyesuaikan setiap (param, titik) yang disetel ke beberapa fungsi tertentu dan menyarankan set parameter awal untuk titik (derajat kebebasan) berikutnya.

  • Ekstrapolator didasarkan pada ekstrapolator eksternal yang menetapkan 'jendela', yaitu jumlah titik sebelumnya yang digunakan untuk ekstrapolasi, sedangkan ekstrapolator internal melanjutkan dengan ekstrapolasi sesungguhnya.

  • Dalam praktiknya, pengguna menetapkan jendela dengan menentukan nilai integer ke num_bootstrap - yang juga merupakan jumlah poin sebelumnya untuk memakai bootstrapping. Selain itu, ekstrapolator eksternal mendefinisikan ruang di dalam bagaimana untuk mengekstrapolasi - di sini PCA, pengelompokan dan pendekatan jendela standar.

Dalam praktiknya, jika tidak ada ekstrapolator yang ditentukan dan bootstrapping adalah True, maka semua titik sebelumnya akan di-bootstrap. Jika daftar ekstrapolator ditentukan dan tidak ada titik yang ditentukan untuk bootstrapping, maka ekstrapolasi akan dilakukan berdasarkan semua titik sebelumnya.

  1. Ekstrapolator Jendela: Ekstrapolator yang membungkus ekstrapolator lain, membatasi parameter kebenaran dasar ekstrapolator internal yang disetel ke ukuran jendela tetap

  2. Ekstrapolator PCA: Ekstrapolator pembungkus yang mengurangi dimensionalitas titik dengan PCA, melakukan ekstrapolasi dalam ruang pca tertransformasi, dan membatalkan transformasi hasilnya sebelum mengembalikannya.

  3. Ekstrapolator Sieve: Ekstrapolator pembungkus yang melakukan ekstrapolasi, kemudian mengelompokkan nilai parameter yang diekstrapolasi menjadi dua kluster besar dan kecil, dan menetapkan parameter kluster kecil ke nol.

  4. Ekstrapolator Polinomial: Ekstrapolator yang berdasarkan pada penyesuaian setiap parameter ke fungsi polinomial pada derajat yang ditentukan pengguna.

  5. Ekstrapolator Diferensial: Ekstrapolator yang didasarkan pada perlakuan setiap himpunan parameter sebagai titik dalam ruang, dan melakukan regresi untuk memprediksi himpunan parameter untuk titik berikutnya. Derajat yang ditentukan pengguna juga menambahkan turunan ke nilai di vektor titik yang berfungsi sebagai fitur dalam data pelatihan untuk regresi linier.

# define different extrapolators
degree = 1
extrap_poly = Extrapolator.factory("poly", degree = degree)
extrap_diff = Extrapolator.factory("diff_model", degree = degree)
extrapolators = {'poly': extrap_poly, 'diff_model': extrap_diff}

for key in extrapolators:
    extrap_internal = extrapolators[key]
    extrap = Extrapolator.factory("window", extrapolator = extrap_internal)
    # define extrapolator
    # BOPES sampler
    bs_extr = BOPESSampler(
        gss=me_gsc
        ,bootstrap=True
        ,num_bootstrap=None
        ,extrapolator=extrap)
    # execute
    res = bs_extr.sample(es_problem, points)

    results_full[f'{key}']= res.raw_results
    results[f'points_{key}'] = res.points
    results[f'energies_{key}'] = res.energies
