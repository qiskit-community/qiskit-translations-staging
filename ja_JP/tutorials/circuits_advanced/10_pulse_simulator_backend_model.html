Qiskit Aer: バックエンド・モデルの Pulse シミュレーション¶

このnotebookでは、バックエンドから生成されたモデルを使用して Aer パルス・シミュレーターを使用する方法を紹介します。具体的には、FakeArmonk モック・バックエンドから生成された、Armonk の1量子ビット・バックエンド・モデル上で \(\pi\)-パルスの振幅を見つけるためのラビ実験を行います。

1. インポート¶

一般的なライブラリをインポートします:

[1]:

import numpy as np

Ignisからラビ実験生成器とフィッターをインポートし、ジョブ送信のためのその他の機能をインポートします:

[2]:

from qiskit.ignis.characterization.calibrations import rabi_schedules, RabiFitter

from qiskit.pulse import DriveChannel
from qiskit.compiler import assemble
from qiskit.qobj.utils import MeasLevel, MeasReturnType

パルス・シミュレーション用の PulseSimulator と PulseSystemModel とモックのArmonk バックエンドをインポートします。

[3]:

# The pulse simulator
from qiskit.providers.aer import PulseSimulator

# Object for representing physical models
from qiskit.providers.aer.pulse import PulseSystemModel

# Mock Armonk backend
from qiskit.test.mock.backends.armonk.fake_armonk import FakeArmonk

2. バックエンドからのモデルの構築¶

このセクションでは、バックエンドから PulseSystemModel オブジェクトを生成するための PulseSystemModel.from_backend 関数の使い方について説明します。

注記 ：バックエンドで報告されるハミルトニアン・パラメーターは、時間の経過とともに変化します。 これらの変更に対してこのチュートリアルを安定させるために、 FakeArmonk をインスタンス化した後、バックエンド構成で報告されたハミルトニアンを静的な値に手動でオーバーライドします。

モックバックエンドをインスタンス化します:

[4]:

armonk_backend = FakeArmonk()

ハミルトニアン・パラメーターを手動でオーバーライドします。

[5]:

freq_est = 4.97e9
drive_est = 6.35e7
armonk_backend.defaults().qubit_freq_est = [freq_est]
armonk_backend.configuration().hamiltonian['h_str']= ['wq0*0.5*(I0-Z0)', 'omegad0*X0||D0']
armonk_backend.configuration().hamiltonian['vars'] = {'wq0': 2 * np.pi * freq_est, 'omegad0': drive_est}
armonk_backend.configuration().hamiltonian['qub'] = {'0': 2}
armonk_backend.configuration().dt = 2.2222222222222221e-10

バックエンドからモデルを生成します:

[6]:

armonk_model = PulseSystemModel.from_backend(armonk_backend)

3. ラビ実験の実行と :math:`pi`パルス振幅のフィッティング¶

次に、Armonkバックエンドから生成されたシステムモデルを使用して、シミュレータ上のIgnisを使用して生成されたラビ実験を実行します。

まず、ラビ実験のスケジュールを作成します。

[7]:

# qubit list
qubits = [0]

# drive amplitudes to use
num_exps = 64
drive_amps = np.linspace(0, 1.0, num_exps)

# drive shape parameters
drive_duration = 2048
drive_sigma = 256

# list of drive channels
drive_channels = [DriveChannel(0)]

# construct the schedules
rabi_schedules, xdata = rabi_schedules(amp_list=drive_amps,
                                       qubits=qubits,
                                       pulse_width=drive_duration,
                                       pulse_sigma=drive_sigma,
                                       drives=drive_channels,
                                       inst_map=armonk_backend.defaults().instruction_schedule_map,
                                       meas_map=armonk_backend.configuration().meas_map)

ジョブ送信のために``qobj`` を組み立てます。パルス・シミュレーターと一緒に使用するパルス・スケジュールを組み立てる場合は、バックエンドとして PulseSimulator を渡します。

[8]:

backend_sim = PulseSimulator(system_model=armonk_model)

rabi_qobj = assemble(rabi_schedules,
                     backend=backend_sim,
                     meas_level=1,
                     meas_return='avg',
                     shots=512)

シミュレーションを実行します:

[9]:

sim_result = backend_sim.run(rabi_qobj).result()

[10]:

rabi_fit = RabiFitter(sim_result, xdata, qubits, fit_p0 = [1.5, 2, 0, 0])

# get the pi amplitude
pi_amp = rabi_fit.pi_amplitude(0)

# plot
rabi_fit.plot(0)
print('Pi Amp: %f'%pi_amp)
