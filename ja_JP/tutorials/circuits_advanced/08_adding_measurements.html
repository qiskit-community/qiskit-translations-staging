測定を Schedule に追加する¶

明らかに測定は、パルススケジュールを作成する上で非常に重要な部分です。これは、プログラム実行の結果を取得するために必要です。パルスによって付与される強力な低レベルの制御により、 QuantumCircuit よりも自由に測定を指定できるため、読み出しエラーの軽減を検討できます。もちろん、この制御には責任が伴います。測定がどのように機能するかを理解し、特定のハードウェアの制約に対応する必要があります。

このページでは、複雑さの増大についていくつかの異なるアプローチを使用して、測定を作成する方法を詳細に検討します。

注：パルスを使用すると、生の読み出しデータ、カーネル化された読み出しデータおよび識別された読み出しデータを受信することができます (一方、回路は識別されたデータのみを返します) 。

measure を使用したバックエンドデフォルト測定を追加する¶

QuantumCircuit と同じくらい簡単に Schedule に測定値を追加するには、測定する量子ビット（以下、量子ビット0および1）を知って、オープンパルス対応の backend を用意する必要があります。

# Appending a measurement schedule to a Schedule, sched
from qiskit.scheduler import measure
sched += measure([0, 1], backend) << sched.duration

backend には測定のデフォルト定義が含まれており、測定する量子ビットに合わせて調整されています。

基本的な測定パターンと measure_all¶

デフォルトの測定機能を使用して測定調査を行い、各パルスが何をするのかを学びましょう。以下では、バックエンド上のすべての量子ビットを測定する measure_all を使用します。

[1]:

from qiskit import IBMQ
from qiskit.pulse import Schedule, DriveChannel, MeasureChannel, AcquireChannel
from qiskit.pulse.macros import measure_all
from qiskit.test.mock import FakeAlmaden

backend = FakeAlmaden()

[2]:

sched = Schedule(name="Measurement scheduling example")
sched += measure_all(backend)

# For clarity, plot a subset of the channels
sched.draw(plot_range=[0, 1000],
           channels=[chan(i) for chan in [DriveChannel, MeasureChannel, AcquireChannel]
                             for i in range(3)])


[2]:

Each qubit has two channels related to readout, as we see above. These are the readout transmit MeasureChannels, and the readout receive AcquireChannels. In superconducting qubit architectures, qubits are coupled to readout resonators. The MeasureChannel and AcquireChannels label signal lines which connect to the readout resonator. The coupling between the qubit and the readout resonator hybridizes their state, so when a stimulus pulse is sent to the readout resonator, the reflected pulse is dependent on the state of the qubit. The acquisition “pulse” is truly a trigger specifying to the analog-to-digital converter (ADC) to begin collecting data, and for how long. That data is used to classify the qubit state.

古典的なメモリースロットの指定¶

測定結果の行き先を指定したい場合は、 measure に qubit_mem_slot というオプションがあります。量子ビットのインデックスを古典的なメモリスロットにマッピングする辞書が必要です。たとえば、量子ビット0をメモリスロット1で測定したい場合は、次のようにします。:

[3]:

from qiskit.pulse.macros import measure

sched = measure(qubits=[0], backend=backend, qubit_mem_slots={0: 1})

これは回路測定の circuit.measure(qubit_reg[0], classical_reg[1]) と同じです。

パルスから測定シーケンスを作成する¶

バックエンドから提供されるデフォルトの測定値を使用するのではなく、基本的なPulseスケジュールとして測定シーケンスを作成することもできます。 以下の例は、IBMシステムの一般的な測定に似ています。

まず、測定したい各量子ビットの測定刺激パルスを構築します。以下では、ガウシアン矩形波のパラメトリックパルスを使用します。

[4]:

from qiskit.pulse import MeasureChannel, AcquireChannel, MemorySlot, Play, GaussianSquare, Acquire

# Duration (in number of cycles) for readout
duration = 1600

# Stimulus pulses for qubits 0 and 1
measure_tx = Play(GaussianSquare(duration=duration, amp=0.2, sigma=10, width=duration - 50),
                  MeasureChannel(0))
measure_tx += Play(GaussianSquare(duration=duration, amp=0.2, sigma=10, width=duration - 50),
                   MeasureChannel(1))

取得パルスを作る前に、測定マップを理解する必要があります。

量子ビットの取得: 測定マップ measure_map¶

制御ラックハードウェアの制約により、一部の量子ビットを一緒に取得する必要がある場合があります。 これは、読み出しチャネルが多重化されている量子ビットの場合に使用できます。OpenPulse対応のバックエンドは、これをユーザーに通知するために measure_map を提供します。

例えば、これが 5量子ビット の backend で表示された場合

backend.configuration().meas_map

Out: [[0, 1, 2, 3, 4]]

この装置のすべての量子ビットが一緒に取得されなければならないことがわかります。一方、この出力は、

Out: [[0], [1], [2], [3, 4]]

量子ビット0、1、2は個別に取得できるが、量子ビット3と4は一緒に取得する必要がある、ということです。

パルス・スケジュールを作成する場合は、必ず、実行する予定のバックエンドが必要とするすべての取得用パルスを追加してください。 これはアセンブル時に検証されます。

例に戻りますが、この測定マップ [[0, 1, 2]] を使用して、バックエンドで実行する計画を考えてみましょう。まず取得パルスを作ることができます。 これは期間 (duration) だけをとる Acquire コマンドで実行できます。acquire では、チャネルとメモリスロットを指定します。

[5]:

# Acquisition instructions
measure_rx = Acquire(duration, AcquireChannel(0), MemorySlot(0))
measure_rx += Acquire(duration, AcquireChannel(1), MemorySlot(1))

最後に、2つのパーツを結合します。すべての命令は別のチャネルにありため、時間0で命令をスケジュールします。次に、 measure_schedule を任意のパルススケジュールの最後に追加して、量子ビット0および1を測定し、古典的なメモリスロット0および1に入れます。

[6]:

measure_sched = measure_tx + measure_rx

measure_sched.draw()
