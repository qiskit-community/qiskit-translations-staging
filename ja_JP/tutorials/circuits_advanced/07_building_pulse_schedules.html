Pulseスケジュールの構築¶

Schedule を構築するのはとても簡単ですが、それらを構成する方法はいくつかあります。ここでは、これらの様々な方法を見ていきます。

In this tutorial we use the Play instruction, with a ConstantPulse pulse argument. To learn about these and other Instructions, first check out this page.

ヒント: Instruction は Schedule と同じように扱うことができます。つまり、ここで使用するすべてのメソッドは Schedule と Schedule 、 Instruction と Instruction 、あるいはその２つの間で構成する場合にも同様に適切に処理されます。

構成メソッドを試すための基礎として、ふたつのダミー Schedule を初期化しておきましょう。

[1]:

from qiskit.pulse import Schedule, Play, Constant, Delay, DriveChannel

sched_a = Schedule(name="A")
sched_b = Schedule(name="B")

insert または |¶

insert メソッドは特定の時間に Instruction または Schedule をスケジュールします。それぞれのスケジュールに異なる命令を与えるために使いましょう。

insertスケジュールメソッドは、 | (or) 演算子を Schedule にオーバーロードするため、スケジュール B でこの特別な構文を使用します。この糖衣構文を使用する場合、 time 引数は暗黙的にゼロになります。

[2]:

sched_a = sched_a.insert(0, Play(Constant(duration=5, amp=1), DriveChannel(0)))
sched_b |= Play(Constant(duration=5, amp=-1), DriveChannel(0))

sched_a.draw()

[2]:

shift または <<¶

時々、あるスケジュールを time=0 からオフセットする必要があります。 例えば、 | を使用する場合は、ゼロ以外の時刻に命令を追加するために shift を使用します。

[3]:

sched_a |= Play(Constant(duration=5, amp=0.5), DriveChannel(0)).shift(10)
sched_b |= Play(Constant(duration=5, amp=-0.5), DriveChannel(0)) << 10

sched_b.draw()

[3]:

insert を使って A と B のスケジュールを一緒に構成する方法を見てみましょう。

[4]:

sched_a_and_b = sched_a.insert(20, sched_b)  # A followed by B at time 20
sched_b_and_a = sched_a | sched_b << 20      # B followed by A at time 20

sched_a_and_b.draw()

[4]:

append または +¶

append メソッドは insert` に似ていますが、挿入時間は私たちのために決定されます。追加される ``Instruction または Schedule は、2つに共通のすべてのチャネルが解放されたときに開始されます。共通のチャネルが含まれていない場合、 Schedule は time=0 で追加されます。 擬似コードの場合：

time = 0
for channel in intersection(original_sched_channels, appended_sched_channels):
    time = max(time, original_sched.duration(channel))

appendスケジュールメソッドは、Scheduleの + (add) 演算子をオーバーロードします。

スケジュール A とスケジュール B を続けて行きましょう。どちらも同じチャネル、 DriveChannel(0) でスケジュールされた命令があることを覚えておいてください。スケジュール A は正の振幅パルスを含み、スケジュール B は負の振幅パルスを含みます。

[5]:

sched_a_plus_b = sched_a.append(sched_b)

sched_a_plus_b.draw()

[5]:

共通のチャネルがない場合に何が起こるか見てみましょう。

警告: 共通チャネルは共通量子ビットと同じではありません。ひとつの量子ビットだったとしても、測定にはゲート操作とは異なるチャネルが必要です。単純に測定をスケジュールに追加すると、 time=0 で測定をスケジュールすることになり、期待していたものではないではなくなってしまいます。代わりに、時刻の引数として schedule.duration とあわせて shift 操作が使えます。

[6]:

sched_a_plus_c = sched_a + Play(Constant(duration=20, amp=0.2), DriveChannel(1))

sched_a_plus_c.draw()

[6]:

今回、新たな命令は time=0 に追加されました。それは、 DriveChannel(0) に関する命令がなかったためです。

注意してください: append は、追加されたスケジュールを、それらの間で共通ではない元のチャネルに 「スライド」 させるだけです。 次の例は、この点を示しています。

[7]:

sched_d = Play(Constant(duration=20, amp=-0.2), DriveChannel(0)) \
          + (Play(Constant(duration=10, amp=-0.2), DriveChannel(1)) << 10)

(sched_a_plus_c + sched_d).draw()

[7]:

おめでとうございます。スケジュール構成をマスターしました。
