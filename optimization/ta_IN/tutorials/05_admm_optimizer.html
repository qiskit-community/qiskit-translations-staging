ADMM ஆப்டிமைசர்¶

முன்னுரை¶

ADMM ஆப்டிமைசர் கலப்பு-பைனரி கட்டுப்படுத்தப்பட்ட தேர்வுமுறை சிக்கல்களின் வகுப்புகளை தீர்க்க முடியும், இனி (MBCO), இது பெரும்பாலும் தளவாட, நிதி மற்றும் செயல்பாட்டு ஆராய்ச்சிகளில் தோன்றும். குறிப்பாக, இங்கு வடிவமைக்கப்பட்ட ADMM ஆப்டிமைசர் பின்வரும் தேர்வுமுறை சிக்கலைச் சமாளிக்கும்:math: (P):

\[\min_{x \in \mathcal{X},u\in\mathcal{U} \subseteq \mathbb{R}^l } \quad q(x) + \varphi(u),\]

தடைகளுக்கு உட்பட்டு:

\[\mathrm{s.t.:~} \quad G x = b, \quad g(x) \leq 0, \quad \ell(x, u) \leq 0,\]

தொடர்புடைய செயல்பாட்டு அனுமானங்களுடன்.

  1. செயல்பாடு \(q: \mathbb{R}^n\to \mathbb{R}\) என்பது இருபடி, அதாவது, \(q(x) = x^{\ intercal} Q x + a^{\ intercal } x\) கொடுக்கப்பட்ட சமச்சீர் ஸ்கொயர் மேட்ரிக்ஸுக்கு:math:Q inmathbb {R} ^ n times mathbb {R} ^ n, Q = Q ^ {intercal} `, மற்றும் திசையன்:math: a inmathbb{R} ^ n`;

  2. தொகுப்பு \(\mathcal{X} = \{0,1\}^n = \{x_{(i)} (1-x_{(i)}) = 0, \forall i\}\) பைனரி தடைகளை செயல்படுத்துகிறது;

  3. மேட்ரிக்ஸ்:math:Ginmathbb{R}^n times mathbb{R}^{n'}, திசையன் \(b \in \mathbb{R}^{n'}\),மற்றும் செயல்பாடு:math:`g: mathbb{R}^n to mathbb{R} குவிந்ததாகும்;

  4. செயல்பாடு \(\varphi: \mathbb{R}^l \to \mathbb{R}\) குவிந்த மற்றும் :math:`mathcal{U}`ஒரு குவிந்த தொகுப்பு;

  5. செயல்பாடு \(\ell: \mathbb{R}^n\times \mathbb{R}^l \to \mathbb{R}\) * கூட்டாக * குவிந்த நிலையில் உள்ளது \(x, u\).

MBO சிக்கல்களைத் தீர்க்க, [1] முன்மொழியப்பட்ட ஹூரிஸ்டிக்ஸ்:math: (P) பெருக்கிகளின் மாற்று திசை முறையின் அடிப்படையில் (ADMM) [2]. ADMM என்பது குவிந்த தேர்வுமுறையில் நீண்ட வரலாற்றைக் கொண்ட ஒரு ஆபரேட்டர் பிரிக்கும் வழிமுறையாகும், மேலும் இது மீதமுள்ள, புறநிலை மற்றும் இரட்டை மாறி குவிப்பு பண்புகளைக் கொண்டிருப்பதாக அறியப்படுகிறது, இது குவிவு அனுமானங்களைக் கொண்டுள்ளது.

[1] (3-ADMM-H) என்ற முறையீடு ADMM ஆபரேட்டர்-பிளக்கும் நடைமுறையை சில வகை MBO களுக்கு சிதைவை உருவாக்க உதவுகிறது: - குவாண்டம் சாதனத்தில் மாறுபட்ட வழிமுறைகள் மூலம் தீர்க்கப்பட வேண்டிய QUBO துணைப்பிரிவு, VQE அல்லது QAOA போன்றவை; - தொடர்ச்சியான குவிந்த கட்டுப்படுத்தப்பட்ட துணைப் பிரச்சினை, இது கிளாசிக்கல் தேர்வுமுறை தீர்வுகளுடன் திறமையாக தீர்க்கப்படலாம்.

3-ADMM-H வழிமுறை பின்வருமாறு செயல்படுகிறது:

  1. துவக்க கட்டம் (அளவுருக்கள் மற்றும் QUBO மற்றும் குவிந்த தீர்வுகளை அமைக்கும்);

  2. ஒவ்வொரு ADMM மறு செய்கைகளுக்கும் ($k = 1, 2, \ldots, $) நிறுத்தப்படும் வரை:

    • சரியாக வரையறுக்கப்பட்ட QUBO துணைப் பிரச்சினையைத் தீர்க்கவும் (கிளாசிக்கல் அல்லது குவாண்டம் தீர்வி கொண்டு);

    • சரியாக வரையறுக்கப்பட்ட குவிந்த சிக்கல்களைத் தீர்க்கவும் (கிளாசிக்கல் தீர்வி கொண்டு);

    • இரட்டை மாறிகள் புதுப்பிக்கவும்.

  3. ஆப்டிமைசர்கள் மற்றும் செலவை வழங்குகிறது.

வழிமுறையின் ஒருங்கிணைப்பு, சாத்தியக்கூறு மற்றும் உகந்த தன்மைக்கான நிபந்தனைகள் பற்றிய விரிவான கலந்துரையாடலைக் காணலாம் [1]. 2 ADMM தொகுதிகள் கொண்ட ஒரு மாறுபாடு, அதாவது QUBO subproblem, மற்றும் தொடர்ச்சியான குவிந்த கட்டுப்படுத்தப்பட்ட துணைப்பொருள் ஆகியவை [1] இல் அறிமுகப்படுத்தப்பட்டுள்ளன.

குறிப்புகள்¶

[1] C. Gambella and A. Simonetto, Multi-block ADMM heuristics for mixed-binary optimization, on classical and quantum computers, arXiv preprint arXiv:2001.02069 (2020).

[2] S. Boyd, N. Parikh, E. Chu, B. Peleato, and J. Eckstein, Distributed optimization and statistical learning via the alternating direction method of multipliers, Foundations and Trends in Machine learning, 3, 1–122 (2011).

இந்த டுடோரியலில் பின்னர் பயன்படுத்த திட்டமிட்ட அனைத்து வழிமுறைகளையும் முதலில் துவக்குகிறோம்.

QUBO சிக்கல்களைத் தீர்க்க நாம் இடையில் தேர்வு செய்யலாம் - MinimumEigenOptimizer வெவ்வேறு பயன்படுத்தி MinimumEigensolver, போன்றவை VQE, QAOA அல்லது NumpyMinimumEigensolver (கிளாசிக்கல்) - GroverOptimizer - CplexOptimizer (கிளாசிக்கல், CPLEX நிறுவப்பட்டிருந்தால்)

மேலும் குவிந்த தொடர்ச்சியான சிக்கல்களைத் தீர்க்க பின்வரும் கிளாசிக்கல் தீர்வுகளுக்கு இடையில் நாம் தேர்வு செய்யலாம்: - `` CplexOptimizer`` (CPLEX நிறுவப்பட்டிருந்தால்) - `` CobylaOptimizer``

CPLEX கிடைக்கவில்லை எனில், CobylaOptimizer (குவிந்த தொடர்ச்சியான சிக்கல்களுக்கு) மற்றும் NumpyMinimumEigensolver (QUBO க்காக) ஐப் பயன்படுத்தி MinimumEigenOptimizer ஆகியவை சோதனை, சரிபார்ப்பு, மற்றும் இலக்குக் குறி.

[2]:

# define COBYLA optimizer to handle convex continuous problems.
cobyla = CobylaOptimizer()

# define QAOA via the minimum eigen optimizer
qaoa = MinimumEigenOptimizer(QAOA(quantum_instance=BasicAer.get_backend('statevector_simulator')))

# exact QUBO solver as classical benchmark
exact = MinimumEigenOptimizer(NumPyMinimumEigensolver()) # to solve QUBOs

# in case CPLEX is installed it can also be used for the convex problems, the QUBO,
# or as a benchmark for the full problem.
#
# cplex = CplexOptimizer()

எடுத்துக்காட்டு¶

சமத்துவம் மற்றும் சமத்துவமின்மை கட்டுப்பாடுகள் கொண்ட எளிய கலப்பு-பைனரி இருபடி சிக்கலில் 3-ADMM-H வழிமுறையை சோதிக்கிறோம் (எடுத்துக்காட்டு 6 [1] இல் தெரிவிக்கப்பட்டுள்ளது). நாங்கள் முதலில் டாக்ளெக்ஸ் சிக்கலை உருவாக்கி, அதை `` குவாட்ராடிக் புரோகிராம்`` இல் ஏற்றுவோம்.

[3]:

# construct model using docplex
mdl = Model('ex6')

v = mdl.binary_var(name='v')
w = mdl.binary_var(name='w')
t = mdl.binary_var(name='t')
u = mdl.continuous_var(name='u')

mdl.minimize(v + w + t + 5 * (u-2)**2)
mdl.add_constraint(v + 2 * w + t + u <= 3, "cons1")
mdl.add_constraint(v + w + t >= 1, "cons2")
mdl.add_constraint(v + w == 1, "cons3")

# load quadratic program from docplex model
qp = from_docplex_mp(mdl)
print(qp.export_as_lp_string())

பாரம்பரிய தீர்வு¶

3-ADMM-H க்கு QUBO துணைப்பிரச்சினைத் தீர்க்க ஒரு QUBO உகப்பாக்கி தேவை, மற்றும் தொடர்ச்சியான குவிந்த கட்டுப்படுத்தப்பட்ட துணைப் பிரச்சினையைத் தீர்க்க தொடர்ச்சியான தேர்வுமுறை தேவை. நாங்கள் முதலில் சிக்கலை கிளாசிக்கலாக தீர்க்கிறோம்: கிளாசிக்கல் மற்றும் துல்லியமான QUBO தீர்வாக NumPyMinimumEigenSolver உடன் MinimumEigenOptimizer ஐப் பயன்படுத்துகிறோம், மேலும் CobylaOptimizer ஐ தொடர்ச்சியான குவிந்த தீர்வாகப் பயன்படுத்துகிறோம். 3-ADMM-H கிஸ்கிட்டில் கிடைக்கக்கூடிய வேறு பொருத்தமான தீர்வை ஆதரிக்கிறது. உதாரணமாக, VQE, QAOA மற்றும் க்ரோவர் ஆப்டிமைசரை குவாண்டம் தீர்வுகளாகப் பயன்படுத்தலாம், பின்னர் நிரூபிக்கப்பட்டுள்ளது. CPLEX நிறுவப்பட்டிருந்தால், CplexOptimizer இரண்டையும் பயன்படுத்தலாம், QUBO மற்றும் குவிந்த தீர்வி.

அளவுருக்கள்¶

3-ADMM-H ADMMParameters வகுப்பில் மூடப்பட்டிருக்கும். தனிப்பயனாக்கப்பட்ட அளவுரு மதிப்புகள் வகுப்பின் வாதங்களாக அமைக்கப்படலாம். இந்த எடுத்துக்காட்டில், அளவுருக்கள் \(\rho, \beta\) முறையே \(1001\) மற்றும் \(1000\). சமத்துவக் கட்டுப்பாடுகளின் factor_c அபராதம் \(Gx = b\) என அமைக்கப்பட்டுள்ளது \(900\). முதன்மை மீதமுள்ள ஒருங்கிணைப்புக்கான tol சகிப்புத்தன்மை 1.e-6 என அமைக்கப்பட்டுள்ளது. இந்த வழக்கில், 3-தொகுதி செயல்படுத்தல் [1] இன் தேற்றம் 4 க்கு ஒன்றிணைவதற்கு உத்தரவாதம் அளிக்கப்படுகிறது, ஏனெனில் தொடர்ச்சியான மாறியுடன் சமத்துவமின்மை கட்டுப்பாடு எப்போதும் செயலில் இருக்கும். 2-தொகுதி செயலாக்கத்தை three_block=False அமைப்பதன் மூலம் இயக்க முடியும், மேலும் நடைமுறையில் இது ஒரு உகந்த தீர்வாக மாறாது.

[4]:

admm_params = ADMMParameters(
                            rho_initial=1001,
                            beta=1000,
                            factor_c=900,
                            maxiter=100,
                            three_block=True, tol=1.e-6
                        )

3-ADMM-H வழிமுறையை அழைக்கிறது¶

3-ADMM-H வழிமுறையைப் பயன்படுத்த, ADMMOptimizer வகுப்பின் ஒரு உதாரணத்தை உருவாக்க வேண்டும். இது ADMM- குறிப்பிட்ட அளவுருக்கள் மற்றும் சப்ரோப்ளம் ஆப்டிமைசர்களை தனித்தனியாக கட்டமைப்பாளருக்குள் எடுக்கும். வழங்கப்பட்ட தீர்வு OptimizationResult வகுப்பின் ஒரு எடுத்துக்காட்டு.

[5]:

# define QUBO optimizer
qubo_optimizer = exact
# qubo_optimizer = cplex  # uncomment to use CPLEX instead

# define classical optimizer
convex_optimizer = cobyla
# convex_optimizer = cplex  # uncomment to use CPLEX instead

# initialize ADMM with classical QUBO and convex optimizer
admm = ADMMOptimizer(params=admm_params,
                     qubo_optimizer=qubo_optimizer,
                     continuous_optimizer=convex_optimizer)

[6]:

# run ADMM to solve problem
result = admm.solve(qp)

கிளாசிக்கல் சால்வர் முடிவு¶

3-ADMM-H தீர்வு பின்னர் அச்சிடப்பட்டு காட்சிப்படுத்தப்படலாம். தீர்வின் x பண்பு முறையே, பைனரி முடிவு மாறிகள் மற்றும் தொடர்ச்சியான முடிவு மாறிகளின் மதிப்புகள் ஆகியவற்றைக் கொண்டுள்ளது. fval என்பது தீர்வின் புறநிலை மதிப்பு.

[7]:

print("x={}".format(result.x))
print("fval={:.2f}".format(result.fval))

x=[1. 0. 0. 2.]
fval=1.00

தீர்வு புள்ளிவிவரங்களை state புலத்தில் அணுகலாம் மற்றும் காட்சிப்படுத்தலாம். முதன்மை எச்சங்களின் அடிப்படையில், 3-ADMM-H இன் ஒருங்கிணைப்பை இங்கு காண்பிக்கிறோம்.

[8]:

plt.plot(result.state.residuals)

plt.ylabel("Residuals")
plt.show()

குவாண்டம் தீர்வு¶

QUBO ஆப்டிமைசரின் QAOA உடன் அதே தேர்வுமுறை சிக்கலை நாங்கள் இப்போது தீர்க்கிறோம், உருவகப்படுத்தப்பட்ட குவாண்டம் சாதனத்தில் இயங்குகிறது. முதலில், ஈஜென்சால்வர் QAOA இன் கிளாசிக்கல் ஆப்டிமைசரைத் தேர்ந்தெடுக்க வேண்டும். பின்னர், உருவகப்படுத்துதல் பின்தளமானது அமைக்கப்பட்டது. இறுதியாக, ஈஜென்சால்வர் "மினிமம் ஐஜென் ஆப்டிமைசர்" வகுப்பில் மூடப்பட்டுள்ளது. "ADMMஆப்டிமைசர்" இன் ஒரு புதிய உதாரணம் QAOA உடன் QUBO ஆப்டிமைசராக நிரம்பியுள்ளது.

[9]:

# define QUBO optimizer
qubo_optimizer = qaoa

# define classical optimizer
convex_optimizer = cobyla
# convex_optimizer = cplex  # uncomment to use CPLEX instead

# initialize ADMM with quantum QUBO optimizer and classical convex optimizer
admm_q = ADMMOptimizer(params=admm_params,
                       qubo_optimizer=qubo_optimizer,
                       continuous_optimizer=convex_optimizer)

[10]:

# run ADMM to solve problem
result_q = admm_q.solve(qp)

குவாண்டம் சால்வர் முடிவுகள்¶

குவாண்டம் தீர்விலிருந்து பெறப்பட்ட முடிவுகளை இங்கே முன்வைக்கிறோம். மேலே உள்ள எடுத்துக்காட்டில் `` x`` என்பது தீர்வைக் குறிக்கிறது, `` fval`` என்பது புறநிலை மதிப்புக்கு.
