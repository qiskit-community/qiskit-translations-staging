வாகன வழித்தடம்¶

முன்னுரை¶

தளவாடங்கள் ஒரு முக்கிய தொழிலாகும், சில மதிப்பீடுகள் 2015 ஆம் ஆண்டில் உலகளவில் 8183 பில்லியன் அமெரிக்க டாலராக மதிப்பிடப்பட்டுள்ளன. பெரும்பாலான சேவை வழங்குநர்கள் பல வாகனங்களை (எ.கா., லாரிகள் மற்றும் கொள்கலன் கப்பல்கள்), பல டிப்போக்களை இயக்குகின்றனர், அங்கு வாகனங்கள் ஒரே இரவில் அமைந்திருக்கின்றன, மேலும் சேவை செய்கின்றன ஒவ்வொரு நாளும் ஒவ்வொரு வாகனத்துடனும் பல வாடிக்கையாளர் இருப்பிடங்கள். இந்த அளவுருக்களைக் கருத்தில் கொள்ளும் பல தேர்வுமுறை மற்றும் கட்டுப்பாட்டு சிக்கல்கள் உள்ளன. கணக்கீட்டு ரீதியாக, முக்கிய சவால் என்னவென்றால், டிப்போக்களிலிருந்து பல கிளையன்ட் இருப்பிடங்களுக்கான பாதைகளை எவ்வாறு வடிவமைப்பது மற்றும் மீண்டும் டிப்போவுக்குச் செல்வது, இதனால் வாகன மைல்கள் பயணிப்பது, செலவழித்த நேரம் அல்லது இதே போன்ற புறநிலை செயல்பாடுகளை குறைத்தல். இந்த நோட்புக்கில், சிக்கலின் இலட்சியப்படுத்தப்பட்ட பதிப்பை நாங்கள் முறைப்படுத்துகிறோம் மற்றும் ஃபர்ஹி, கோல்ட்ஸ்டோன் மற்றும் குட்மேன் (2014) ஆகியவற்றின் குவாண்டம் தோராயமான தேர்வுமுறை அணுகுமுறையைப் பயன்படுத்தி அதன் தீர்வை வெளிப்படுத்துகிறோம்.

நாங்கள் நிரூபிக்கும் ஒட்டுமொத்த பணிப்பாய்வு பின்வருமாறு:

  1. வாடிக்கையாளர் இடங்களை நிறுவவும். பொதுவாக, இவை தரவுத்தளத்தில் இருந்து டெலிவரி செய்யப்படும் நாளுக்கு முன்னதாகவே கிடைக்கும். எங்கள் பயன்பாட்டு வழக்கில், நாங்கள் இதை தோராயமாக உருவாக்குகிறோம்.

  2. ஜோடி வாரியாக தூரங்கள், பயண நேரங்கள் அல்லது ஒத்த கணக்கிட. எங்கள் விஷயத்தில், நாம் யூக்ளிடியன் தூரத்தை, "காகம் பறப்பது போல்" கருதுகிறோம், இது ஒருவேளை மிகவும் எளிதானது.

  3. உண்மையான வழிகளைக் கணக்கிடுங்கள். இந்த படி உண்மையில் இரண்டு முறை இயக்கப்படுகிறது. முதலில், கிளாசிக்கல் கணினியில் கிளாசிக்கல் சொல்வர் (IBM CPLEX) மூலம் ஒரு குறிப்பு மதிப்பைப் பெறுகிறோம். இரண்டாவதாக, குவாண்டம் கணினியில் ஒரு மாற்று, கலப்பின வழிமுறையை இயக்குகிறோம்.

  4. முடிவுகளின் காட்சிப்படுத்தல். எங்கள் விஷயத்தில், இது மீண்டும் ஒரு எளிய வரைதல்.

பின்வருவனவற்றில், முன்-தேவைகளை நிறுவுதல் மற்றும் தரவை ஏற்றுவதற்கு முன் மாதிரியை முதலில் விளக்குகிறோம்.

மாதிரி¶

கணித ரீதியாகப் பார்த்தால், வாகன ரூட்டிங் சிக்கல் (VRP) ஒரு கூட்டுப் பிரச்சினையாகும், இதில் ஒரு டெப்போவிலிருந்து பல வாடிக்கையாளர்களுக்கும், மீண்டும் டிப்போவிற்கும் சிறந்த வழிகள் தேடப்படுகின்றன, கிடைக்கக்கூடிய பல வாகனங்கள் கொடுக்கப்பட்டுள்ளன. பயண விற்பனையாளர் பிரச்சினையின் பல சூத்திரங்களை விரிவுபடுத்தும் பல சூத்திரங்கள் உள்ளன [ஆப்பிள்கேட் மற்றும் பலர், 2006]. இங்கே, MTZ [மில்லர், டக்கர், ஜெம்லின், 1960]. எனப்படும் ஒரு சூத்திரத்தை நாங்கள் முன்வைக்கிறோம்.

\(n\) வாடிக்கையாளர்களின் எண்ணிக்கையாக இருக்க வேண்டும் (indexed as \(1,\dots,n\)), மற்றும் \(K\) கிடைக்கக்கூடிய வாகனங்களின் எண்ணிக்கை. \(x_{ij} = \{0,1\}\) பைனரி முடிவு மாறியாக இருக்க வேண்டும், அது இருந்தால்: \(1\) , பிரிவை முனை: கணிதத்திலிருந்து செயல்படுத்துகிறது \(i\). முனை குறியீடு \(0\) முதல் \(n\), எங்கே \(0\) என்பது (மாநாட்டின் படி) டிப்போ ஆகும். விளிம்புகளை விட இரண்டு மடங்கு தனித்துவமான முடிவு மாறிகள் உள்ளன. எடுத்துக்காட்டாக, முழுமையாக இணைக்கப்பட்ட வரைபடத்தில், \(n(n+1)\) பைனரி முடிவு மாறிகள்.

இரண்டு முனைகள் இருந்தால் \(i\) மற்றும் \(j\) முதல் \(j\), நாங்கள் எழுதுகிறோம் \(i \sim j\). \(\delta(i)^+\) எந்த முனைகளின் தொகுப்பு \(i\) ஒரு இணைப்பு உள்ளது, அதாவது, \(j \in \delta(i)^+\) என்றால் மற்றும் இருந்தால் மட்டுமே \(i \sim j\). இதேபோல், \(\delta(i)^-\) உடன் இணைக்கப்பட்டுள்ள முனைகளின் தொகுப்பு \(i\), அதாவது \(j \in \delta(i)^-\) என்றால் மற்றும் மட்டும் என்றால் \(j \sim i\).

கூடுதலாக, எல்லா முனைகளுக்கும் தொடர்ச்சியான மாறிகளைக் கருதுகிறோம் \(i = 1,\dots, n\), குறிக்கப்படுகிறது \(u_i\) வாடிக்கையாளர்களிடையே துணை சுற்றுப்பயணங்களை அகற்ற சிக்கலின் MTZ வடிவமைப்பில் இந்த மாறிகள் தேவைப்படுகின்றன.

VRP இவ்வாறு வடிவமைக்கப்படலாம்:

\[(VRP) \quad f = \min_{\{x_{ij}\}_{i\sim j}\in \{0,1\}, \{u_i\}_{i=1,\dots,n}\in \mathbb{R}} \quad \sum_{i \sim j} w_{ij} x_{ij}\]

முனை-வருகை தடைக்கு உட்பட்டது:

\[\sum_{j \in \delta(i)^+} x_{ij} = 1, \,\sum_{j \in \delta(i)^-} x_{ji} = 1,\, \forall i \in \{1,\dots,n\},\]

டிப்போ-வருகை கட்டுப்பாடுகள்:

\[\sum_{i \in \delta(0)^+} x_{0i} = K, \, \sum_{j \in \delta(0)^+} x_{j0} = K,\]

மற்றும் சுற்றுப்பயணம் நீக்குவதற்கான தடைகள்:

\[u_i - u_j + Q x_{ij} \leq Q-q_j, \, \forall i \sim j, \,i ,j \neq 0, \quad q_i \leq u_i \leq Q,\, \forall i, i \neq 0.\]

குறிப்பாக, - செலவுச் செயல்பாடு செலவுச் செயல்பாடுகளில் நேர்கோட்டு மற்றும் நேர்மறையான எடையின் அடிப்படையில் வெவ்வேறு வளைவுகளை எடைபோடுகிறது \(w_{ij}>0\) (பொதுவாக முனை \(i\) மற்றும் முனை \(j\) இடையிலான தூரம்); - முதல் கிளையண்டுகள் ஒவ்வொரு வாடிக்கையாளரிடமிருந்தும், ஒரு இணைப்பு மட்டுமே அனுமதிக்கப்படுவதை அமல்படுத்துகின்றன; - இரண்டாவது கட்டுப்பாடுகள் டிப்போவிலிருந்து மற்றும் அதைச் சரியாகச் செயல்படுத்துகின்றன \(K\) இணைப்புகள் அனுமதிக்கப்படுகின்றன; - மூன்றாவது தொகுப்பு கட்டுப்பாடுகள் துணை-சுற்றுப்பயண நீக்குதல் தடைகளைச் செயல்படுத்துகின்றன, மேலும் அவை \(u_i\), உடன் \(Q>q_j>0\), மற்றும் \(Q,q_i \in \mathbb{R}\).

கிளாசிக்கல் தீர்வு¶

CPLEX பயன்படுத்துவதன் மூலம் VRP யை கிளாசிக்கலாக தீர்க்க முடியும், எ.கா. CPLEX VRP யின் தோராயமான தீர்வைக் கண்டுபிடிக்க ஒரு கிளை மற்றும் பிணைப்பு மற்றும் வெட்டு முறையைப் பயன்படுத்துகிறது, இது இந்த சூத்திரத்தில், ஒரு கலப்பு-முழு எண் நேரியல் நிரல் (MILP) ஆகும். குறியீட்டின் பொருட்டு, ஒரு திசையனில் முடிவு மாறிகள் பேக் செய்கிறோம்

\[{\bf z} = [x_{01},x_{02},\ldots,x_{10}, x_{12},\ldots,x_{n(n-1)}]^T,\]

இதில் \({\bf z} \in \{0,1\}^N\) , உடன் \(N = n (n+1)\). எனவே சிக்கலின் பரிமாணம் முனைகளின் எண்ணிக்கையுடன் இருபடி அளவிடுகிறது. இதன் மூலம் உகந்த தீர்வைக் குறிப்பிடுவோம் \({\bf z}^*\), மற்றும் அதனுடன் தொடர்புடைய உகந்த செலவு: \(f^*\).

குவாண்டம் தீர்வு¶

இங்கே, ஃபார்ஹி, கோல்ட்ஸ்டோன் மற்றும் குட்மேன் (2014) ஆகியவற்றின் குவாண்டம் தோராயமான தேர்வுமுறை அணுகுமுறையைப் பின்பற்றி, கிளாசிக்கல் மற்றும் குவாண்டம் கம்ப்யூட்டிங் படிகளை ஒருங்கிணைக்கும் ஒரு அணுகுமுறையை நாங்கள் நிரூபிக்கிறோம். குறிப்பாக, நாம் மாறுபாடு குவாண்டம் ஐஜென்சால்வர் (VQE). பயன்படுத்தப்படும் குவாண்டம் சுற்றுகளின் வரையறுக்கப்பட்ட ஆழத்தைப் பயன்படுத்துவதால் (மாறுபாடு வடிவங்கள்), அல்காரிதத்தின் வேகத்தைப் பற்றி விவாதிப்பது கடினம், ஏனெனில் பெறப்பட்ட தீர்வு இயற்கையில் ஹியூரிஸ்டிக் ஆகும். அதே நேரத்தில், இலக்கு பிரச்சினைகளின் தன்மை மற்றும் முக்கியத்துவம் காரணமாக, சில சிக்கல் வகுப்புகளுக்கு பயனுள்ள துரபூர்வ அணுகுமுறைகளை ஆராய்வது மதிப்புக்குரியது.

[5] ஐத் தொடர்ந்து, வழிமுறையை பின்வருமாறு சுருக்கலாம்:

  • தயாரிப்பு படிகள்:

    • ஒருங்கிணைந்த பிரச்சனையை சமநிலை கட்டுப்பாடுகளுடன் மட்டுமே பைனரி பல்லுறுப்பு தேர்வுமுறை பிரச்சனையாக மாற்றவும்;

    • இதன் விளைவாக வரும் சிக்கலை ஐசிங் ஹாமில்டோனியனாக வரைபடமாக்குங்கள் (\(H\)) மாறிகளுக்கு \({\bf z}\) மற்றும் அடிப்படை \(Z\), தேவைப்பட்டால் அபராதம் முறைகள் மூலம்;

    • குவாண்டம் சுற்றுகளின் ஆழத்தைத் தேர்வுசெய்க \(m\). ஆழத்தை தகவமைப்புக்கு மாற்றியமைக்க முடியும் என்பதை நினைவில் கொள்க.

    • கட்டுப்பாடுகளின் தொகுப்பைத் தேர்ந்தெடுக்கவும் \(\theta\) மற்றும் ஒரு சோதனை செயல்பாடு \(\big|\psi(\boldsymbol\theta)\rangle\),சி-கட்ட கேட்ஸ் மற்றும் ஒற்றை-க்யூபிட் Y சுழற்சிகளால் செய்யப்பட்ட குவாண்டம் சர்க்யூட்டைப் பயன்படுத்தி கட்டப்பட்டது \(\boldsymbol\theta\).

  • வழிமுறை படிகள்:

    • மதிப்பீடு \(C(\boldsymbol\theta) = \langle\psi(\boldsymbol\theta)\big|H\big|\psi(\boldsymbol\theta)\rangle\) சுற்றுவட்டத்தின் மாதிரியை மாதிரியாகக் கொண்டு Z- அடிப்படை மற்றும் தனிப்பட்ட ஐசிங் சொற்களின் எதிர்பார்ப்பு மதிப்புகளை ஒன்றாகச் சேர்ப்பது. பொதுவாக, வெவ்வேறு கட்டுப்பாட்டு புள்ளிகள் \(\boldsymbol\theta\) மதிப்பிடப்பட வேண்டும், இது தேர்ந்தெடுக்கப்பட்ட கிளாசிக்கல் ஆப்டிமைசரைப் பொறுத்து.

    • புதிய கட்டுப்பாடுகளின் தொகுப்பைத் தேர்வுசெய்ய கிளாசிக்கல் ஆப்டிமைசரைப் பயன்படுத்தவும்.

    • \(C(\boldsymbol\theta)\) குறைந்தபட்சத்தை அடையும் வரை தொடரவும், தீர்வுக்கு போதுமானதாக இருக்கிறது \(\boldsymbol\theta^*\).

    • விநியோகத்திலிருந்து இறுதி மாதிரிகளை உருவாக்க கடைசி \(\boldsymbol \theta\) ஐப் பயன்படுத்தவும் \(\Big|\langle z_i\big|\psi(\boldsymbol\theta)\rangle\Big|^2\;\forall i\) பதிலைப் பெற.

முழுவதும் பல அளவுருக்கள் உள்ளன, குறிப்பாக சோதனை அலை செயல்பாட்டின் தேர்வு. கீழே, நாங்கள் கருதுகிறோம்:

\[\big|\psi(\theta)\rangle = [U_\mathrm{single}(\boldsymbol\theta) U_\mathrm{entangler}]^m \big|+\rangle\]

எங்கே \(U_\mathrm{entangler}\) என்பது C-கட்ட கேட்ஸ்களின் தொகுப்பு (முழுமையாக சிக்க வைக்கும் வாயில்கள்), மற்றும் \(U_\mathrm{single}(\theta) = \prod_{i=1}^N Y(\theta_{i})\), எங்கே \(N\) என்பது க்யூபிட்களின் எண்ணிக்கை மற்றும் \(m\) என்பது குவாண்டம் சுற்றுகளின் ஆழம்.

ஐசிங் ஹாமில்டோனியன் உருவாக்கு¶

இருந்து \(VRP\) ஒருவர் பைனரி பல்லுறுப்புறுப்பு தேர்வுமுறையை சமத்துவக் கட்டுப்பாடுகளுடன் உருவாக்க முடியும்: இதில் கணிதத்தை கருத்தில் கொண்டு மட்டுமே \(K=n-1\). இந்த சந்தர்ப்பங்களில் துணை-சுற்று நீக்குதல் கட்டுப்பாடுகள் தேவையில்லை மற்றும் சிக்கல் மாறியில் மட்டுமே உள்ளது \({\bf z}\). குறிப்பாக, நாம் ஒரு வளர்ந்த லக்ராங்கியன் என எழுதலாம்

\[(IH) \quad H = \sum_{i \sim j} w_{ij} x_{ij} + A \sum_{i \in \{1,\dots,n\}} \Big(\sum_{j \in \delta(i)^+} x_{ij} - 1\Big)^2 + A \sum_{i \in \{1,\dots,n\}}\Big(\sum_{j \in \delta(i)^-} x_{ji} - 1\Big)^2 +A \Big(\sum_{i \in \delta(0)^+} x_{0i} - K\Big)^2 + A\Big(\sum_{j \in \delta(0)^+} x_{j0} - K\Big)^2\]

எங்கே \(A\) என்பது ஒரு பெரிய அளவுரு.

ஹாமில்டோனியன் இருந்து QP வடிவமைப்பில் இருந்து¶

திசையனில் \({\bf z}\), மற்றும் ஒரு முழுமையான வரைபடத்திற்காக (\(\delta(i)^+ = \delta(i)^- = \{0,1,\dots,i-1,i+1,\dots,n\}\)) , \(H\) ஐ பின்வருமாறு எழுதலாம்.

\[\min_{{\bf z}\in \{0,1\}^{n(n+1)}} {\bf w}^T {\bf z} + A \sum_{i \in \{1,\dots,n\}} \Big({\bf e}_i \otimes {\bf 1}_n^T {\bf z} - 1\Big)^2 + A \sum_{i \in \{1,\dots,n\}}\Big({\bf v}_i^T {\bf z} - 1\Big)^2 + A \Big(({\bf e}_0 \otimes {\bf 1}_n)^T{\bf z} - K\Big)^2 + A\Big({\bf v}_0^T{\bf z} - K\Big)^2.\]

அதாவது:

\[\min_{\bf z\in \{0,1\}^{n(n+1)}} \bf z^T {\bf Q} \bf z + {\bf g}^T \bf z + c,\]

எங்கே: முதல் கால எல்லை:

\[{\bf Q} = A \sum_{i \in \{0,1,\dots,n\}} \Big[({\bf e}_i \otimes {\bf 1}_n)({\bf e}_i \otimes {\bf 1}_n)^T + {\bf v}_i{\bf v}_i^T \Big]\]

இரண்டாவது கால எல்லை:

\[{\bf g} = {\bf w} -2 A \sum_{i \in \{1,\dots,n\}} \Big[({\bf e}_i \otimes {\bf 1}_n) + {\bf v}_i \Big] -2 A K \Big[({\bf e}_0 \otimes {\bf 1}_n) + {\bf v}_0 \Big]\]

மூன்றாம் கால எல்லை:

\[c = 2An +2AK^2.\]

ஐஸிங் ஹாமில்டோனியனின் QP உருவாக்கம் VQE ஐப் பயன்படுத்த தயாராக உள்ளது. Qiskit-டில் கிடைக்கும் தேர்வுமுறை அடுக்கைப் பயன்படுத்தி QP ஐ தீர்ப்போம்.

        # np.random.seed(33)

                instance[jj, ii] = instance[ii, jj]

class ClassicalOptimizer:

    def __init__(self, instance,n,K):

        self.instance = instance
        self.n = n  # number of nodes
        self.K = K  # number of vehicles


    def compute_allowed_combinations(self):
        f = math.factorial
        return f(self.n) / f(self.K) / f(self.n-self.K)


    def cplex_solution(self):

        # refactoring
        instance = self.instance
        n = self.n
        K = self.K

        my_obj = list(instance.reshape(1, n**2)[0])+[0. for x in range(0,n-1)]
        my_ub = [1 for x in range(0,n**2+n-1)]
        my_lb = [0 for x in range(0,n**2)] + [0.1 for x in range(0,n-1)]
        my_ctype = "".join(['I' for x in range(0,n**2)]) + "".join(['C' for x in range(0,n-1)])

        my_rhs = 2*([K] + [1 for x in range(0,n-1)]) + [1-0.1 for x in range(0,(n-1)**2-(n-1))] + [0 for x in range(0,n)]
        my_sense = "".join(['E' for x in range(0,2*n)]) + "".join(['L' for x in range(0,(n-1)**2-(n-1))])+"".join(['E' for x in range(0,n)])

        try:
            my_prob = cplex.Cplex()
            self.populatebyrow(my_prob,my_obj,my_ub,my_lb,my_ctype,my_sense,my_rhs)

            my_prob.solve()

        except CplexError as exc:
            print(exc)
            return

        x = my_prob.solution.get_values()
        x = np.array(x)
        cost = my_prob.solution.get_objective_value()

        return x,cost


    def populatebyrow(self,prob,my_obj,my_ub,my_lb,my_ctype,my_sense,my_rhs):

        n = self.n

        prob.objective.set_sense(prob.objective.sense.minimize)
        prob.variables.add(obj = my_obj, lb = my_lb, ub = my_ub, types = my_ctype)

        prob.set_log_stream(None)
        prob.set_error_stream(None)
        prob.set_warning_stream(None)
        prob.set_results_stream(None)

        rows = []
        for ii in range(0,n):
            col = [x for x in range(0+n*ii,n+n*ii)]
            coef = [1 for x in range(0,n)]
            rows.append([col, coef])

        for ii in range(0,n):
            col = [x for x in range(0+ii,n**2,n)]
            coef = [1 for x in range(0,n)]

            rows.append([col, coef])

        # Sub-tour elimination constraints:
        for ii in range(0, n):
            for jj in range(0,n):
                if (ii != jj)and(ii*jj>0):

                    col = [ii+(jj*n), n**2+ii-1, n**2+jj-1]
                    coef = [1, 1, -1]

                    rows.append([col, coef])

        for ii in range(0,n):
            col = [(ii)*(n+1)]
            coef = [1]
            rows.append([col, coef])

        prob.linear_constraints.add(lin_expr=rows, senses=my_sense, rhs=my_rhs)

குவாண்டம் தீர்வுக்கு, நாங்கள் Qiskit பயன்படுத்துகிறோம்.

முதலில், குவாண்டம் அணுகுமுறையை குறியீடாக்கும் ஒரு வகுப்பு குவாண்டம் ஆப்டிமைசரைப் பயன்படுத்தி சிக்கலைத் தீர்க்க தரையில் இருந்து தீர்வைப் பெறுகிறோம், பின்னர் அதை உடனடியாக நிறுவி தீர்க்கிறோம். வகுப்பினுள் பின்வரும் முறைகளை நாங்கள் வரையறுக்கிறோம்: - binary_representation: சிக்கலைக் குறிக்கிறது \((M)\) ஒரு QP சொற்களாக (அது அடிப்படையில் நேரியல் இயற்கணிதம்); - construct_problem: QuadraticProgram இன் உதாரணமாக QUBO தேர்வுமுறை பிரச்சனை உருவாக்குகிறது; - solve_problem: சிக்கலைத் தீர்க்கிறது \((M)\) முந்தைய கட்டத்தில் MinimunEigenOptimizer வழியாக இயல்புநிலை அளவுருக்களுடன் VQE ஐப் பயன்படுத்துவதன் மூலம் கட்டப்பட்டது;

class QuantumOptimizer:

    def __init__(self, instance, n, K):

        self.instance = instance

        self.K = K

    def binary_representation(self,x_sol=0):

        instance = self.instance
        n = self.n
        K = self.K

        A = np.max(instance) * 100  # A parameter of cost function

        # Determine the weights w
        instance_vec = instance.reshape(n ** 2)
        w_list = [instance_vec[x] for x in range(n ** 2) if instance_vec[x] > 0]
        w = np.zeros(n * (n - 1))
        for ii in range(len(w_list)):
            w[ii] = w_list[ii]

        # Some variables I will use
        Id_n = np.eye(n)
        Im_n_1 = np.ones([n - 1, n - 1])
        Iv_n_1 = np.ones(n)
        Iv_n_1[0] = 0
        Iv_n = np.ones(n-1)
        neg_Iv_n_1 = np.ones(n) - Iv_n_1

        v = np.zeros([n, n*(n-1)])
        for ii in range(n):
            count = ii-1
            for jj in range(n*(n-1)):

                if jj//(n-1) == ii:
                    count = ii

                if jj//(n-1) != ii and jj%(n-1) == count:
                    v[ii][jj] = 1.

        vn = np.sum(v[1:], axis=0)

        # Q defines the interactions between variables
        Q = A*(np.kron(Id_n, Im_n_1) + np.dot(v.T, v))

        # g defines the contribution from the individual variables
        g = w - 2 * A * (np.kron(Iv_n_1,Iv_n) + vn.T) - \
                2 * A * K * (np.kron(neg_Iv_n_1, Iv_n) + v[0].T)

        # c is the constant offset
        c = 2 * A * (n-1) + 2 * A * (K ** 2)

        try:
            max(x_sol)
            # Evaluates the cost distance from a binary representation of a path
            fun = lambda x: np.dot(np.around(x), np.dot(Q, np.around(x))) + np.dot(g, np.around(x)) + c
            cost = fun(x_sol)
        except:
            cost = 0

        return Q, g, c, cost

    def construct_problem(self, Q, g, c) -> QuadraticProgram:
        qp = QuadraticProgram()
        for i in range(n * (n - 1)):
            qp.binary_var(str(i))
        qp.objective.quadratic = Q
        qp.objective.linear = g
        qp.objective.constant = c
        return qp

    def solve_problem(self, qp):
        algorithm_globals.random_seed = 10598
        quantum_instance = QuantumInstance(BasicAer.get_backend('qasm_simulator'),
                                           seed_simulator=algorithm_globals.random_seed,
                                           seed_transpiler=algorithm_globals.random_seed)

        vqe = VQE(quantum_instance=quantum_instance)
        optimizer = MinimumEigenOptimizer(min_eigen_solver=vqe)
        result = optimizer.solve(qp)
        # compute cost of the obtained result
        _,_,_,level = self.binary_representation(x_sol=result.x)
        return result.x, level

அம்புகள் கொண்ட வாகனங்களுக்கான depot ஒரு நட்சத்திரத்தையும், தேர்ந்தெடுக்கப்பட்ட பாதைகளையும் இந்த இடங்கள் வழங்குகின்றன. இந்த குறிப்பிட்ட விஷயத்தில், QP உருவாக்கத்தின் உகந்த தீர்வை நாம் காணலாம், இது ILP இன் உகந்த தீர்வோடு ஒத்துப்போகிறது.

VQE என்பது Ising Hamiltonianனின் QP உருவாக்கத்தில் ஒரு heuristic வேலை என்பதை நினைவில் கொள்ளுங்கள். A இன் பொருத்தமான தேர்வுகளுக்கு, QP உருவாக்கத்தின் உள்ளூர் உகந்த தன்மை ILP க்கு சாத்தியமான தீர்வுகளாக இருக்கும். சில சிறிய நிகழ்வுகளுக்கு, மேலே குறிப்பிட்டபடி, QP சூத்திரத்தின் உகந்த தீர்வுகளை நாம் காணலாம், இது ILP யின் உகந்த தன்மையுடன் ஒத்துப்போகிறது, ஐஎல்பியின் உகந்த தீர்வுகளைக் கண்டறிவது QP சூத்திரத்தின் உள்ளூர் உகந்ததைக் கண்டுபிடிப்பதை விட கடினமானது, பொதுவாக, இது ILP இன் சாத்தியமான தீர்வுகளைக் கண்டுபிடிப்பதை விட கடினம். VQE க்குள் கூட, குறிப்பிட்ட மாறுபட்ட வடிவங்களுக்கு (சோதனை அலை செயல்பாடுகள்) ஒருவர் வலுவான உத்தரவாதங்களை வழங்கலாம்.
