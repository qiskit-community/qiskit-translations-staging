குவாண்டம் மேம்படுத்தலுக்கான இதமான துவக்கம்¶

முன்னுரை¶

முழு மாறிகள் அல்லது கட்டுப்பாடுகளுடன் கூடிய தேர்வுமுறை சிக்கல்களைத் தீர்ப்பது பெரும்பாலும் கடினம். உதாரணமாக, குவாட்ராடிக் கட்டுப்பாடற்ற பைனரி ஆப்டிமைசேஷன் (QUBO) பிரச்சனை, அதாவது.

\begin{align} \min_{x\in\{0,1\}^n}x^T\Sigma x + \mu^Tx, \end{align}

nP- கடினமானது.இங்கே, \(\Sigma\) என்பது ஒரு :\(n\times n\) மேட்ரிக்ஸ் மற்றும் \(x\) என்பது \(n\) பைனரி மாறிகளின் ஒரு திசையன்.நேரியல் சொல்லை \(\mu\) என மூலைவிட்டத்தில் சேர்த்திருக்கலாம் என்பதை நினைவில் கொள்க \(x_i^2=x_i\) என்பதற்கு \(x_i\in\{0, 1\}\).தீர்க்க கடினமாக இருக்கும் போது தீர்க்க எளிதான துகள்களுக்கு அவற்றை தளர்த்த பல வழிகள் உள்ளன.உதாரணமாக, என்றால் \(\Sigma\) அரை-உறுதியான நேர்மறையானது QUBO தளர்த்தப்பட்டு ஒரு குவிந்த இருபடி திட்டத்தில் விளைகிறது

\begin{align} \min_{x\in[0,1]^n}x^T\Sigma x, \end{align}

இது தீர்க்க எளிதானது:math: x இப்போது குறிக்கிறது:math:` n` வரம்பிற்கு கட்டுப்பட்ட தொடர்ச்சியான மாறிகள்:math: [0, 1]. இத்தகைய தளர்வுகள் [1] இல் காட்டப்பட்டுள்ளபடி இதமான-துவக்க குவாண்டம் மேம்படுத்தல் வழிமுறைகளுக்கு உதவலாம்.

முதல் கட்டம்: QUBOகளின் ஆசுவாசப்படுத்தல்¶

முதலில், ஒரு சுலபமாக தீர்க்கக்கூடிய QP ஐப் பெற அரை-வரையறுக்கப்பட்ட நேர்மறை மேட்ரிக்ஸுடன் கட்டப்பட்ட ஒரு QUBO ஐ எவ்வாறு தளர்த்துவது என்பதை நாங்கள் காண்பிக்கிறோம்.

[2]:

def create_problem(mu: np.array, sigma: np.array, total: int = 3) -> QuadraticProgram:
    """ Solve the quadratic program using docplex. """

    mdl = Model()
    x = [mdl.binary_var('x%s' % i) for i in range(len(sigma))]

    objective = mdl.sum([mu[i]*x[i] for i in range(len(mu))])
    objective -= 2*mdl.sum([sigma[i, j]*x[i]*x[j] for i in range(len(mu)) for j in range(len(mu))])
    mdl.maximize(objective)
    cost = mdl.sum(x)
    mdl.add_constraint(cost == total)

    qp = from_docplex_mp(mdl)
    return qp

def relax_problem(problem) -> QuadraticProgram:
    """Change all variables to continuous."""
    relaxed_problem = copy.deepcopy(problem)
    for variable in relaxed_problem.variables:
        variable.vartype = VarType.CONTINUOUS

    return relaxed_problem

QAOA¶

மேலே காட்டப்பட்டுள்ள தளர்வான சிக்கலை மேம்படுத்துவதன் மூலம் குவாண்டம் தோராயமான ஆப்டிமைசேஷன் வழிமுறை (QAOA) எப்படி சூடாகத் தொடங்குவது என்பதை இங்கு விளக்குகிறோம்.

நிலையான QAOA¶

முதலில், QUBO ஐத் தீர்க்க நாங்கள் நிலையான QAOA ஐப் பயன்படுத்துகிறோம். இதைச் செய்ய, நாங்கள் QUBO ஐ Qiskit டின் இருபடி நிரல் வகுப்பிற்கு மாற்றுகிறோம் (இதன் விளைவாக வரும் பிரச்சினை இன்னும் பைனரி பிரச்சனை என்பதை நினைவில் கொள்க).

QAOA உடன் இதமான தொடக்கம்¶

அடுத்து, இந்த முடிவை ஒரு சூடான தொடக்க QAOA உடன் ஒப்பிடுகிறோம், அதில் சிக்கலின் தொடர்ச்சியான தளர்வுக்கான தீர்வைப் பயன்படுத்துகிறோம். முதலில், நாம் ஆரம்ப நிலையை உருவாக்குகிறோம்

\begin{align} |\phi^*\rangle=\bigotimes_{i=0}^{n-1}R_y(\theta_i)|0\rangle_n . \end{align}

இது விண்ணப்பிப்பதன் மூலம் வழங்கப்படுகிறது \(R_y\) ஒரு கோணத்துடன் சுழற்சி:math:theta=2arcsin(sqrt{c^*_i}) அது தளர்வான பிரச்சனைக்கான தீர்வைப் பொறுத்தது. இங்கே, \(c^*_i\) the value of variable \(i\) தளர்வான பிரச்சனை.

அடுத்து, நாங்கள் QAOA க்கான கலவை ஆபரேட்டரை உருவாக்குகிறோம். QAOA- வை சூடு-தொடங்கும் போது, ​​மிக்சர் ஆபரேட்டருக்கு ஆரம்ப நிலை நில நிலமாக இருப்பதை உறுதி செய்ய வேண்டும். எனவே நாங்கள் ஹாமில்டோனியனைத் தேர்ந்தெடுத்தோம்

\begin{align} H_{M,i}^{(ws)}= \begin{pmatrix} 2c_i^*-1 & -2\sqrt{c_i^*(1-c_i^*)} \\ -2\sqrt{c_i^*(1-c_i^*)} & 1-2c_i^* \end{pmatrix} \end{align}

க்யூபிட்டுக்கான மிக்சர் ஆபரேட்டராக \(i\). \(-i\beta\) ஆல் ஒருமுறை பெருக்கப்படுகிறது மேலும் இந்த மிக்சர் பின்வரும் மிக்சர் சர்க்யூட்டை உருவாக்குகிறது.

பகுப்பாய்வு¶

def format_qaoa_samples(samples, max_len: int = 10):
    qaoa_res = []
    for s in samples:
        if sum(s.x) == 3:
            qaoa_res.append((''.join([str(int(_)) for _ in s.x]), s.fval, s.probability))

    res = sorted(qaoa_res, key=lambda x: -x[1])[0:max_len]

    return [(_[0] + f': value: {_[1]:.3f}, probability: {1e2*_[2]:.1f}%') for _ in res]

QAOA உடன் இதமான தொடக்கம்¶

மேலே உள்ள இதமான-தொடக்க அம்சங்கள் க்விஸ்கிட் ஆப்டிமைசேஷன் தொகுதியில் WarmStartQAOAOptimizer என பெயரிடப்பட்ட ஒற்றை ஆப்டிமைசராகக் கிடைக்கின்றன இது கீழே விளக்கப்பட்டுள்ளது. இந்த தீர்வு ஒரு QUBO ஐ ஒரு இதமான-தொடக்க QAOA உடன் தீர்க்கும். இது \(c^*\) ஐ பிரச்சனையை தளர்த்துவதன் மூலம் கணக்கிடுகிறது. இந்த நடத்தை relax_for_pre_solver க்கு ``True``அமைப்பால் கட்டுப்படுத்தப்படுகிறது.

[19]:

from qiskit_optimization.algorithms import WarmStartQAOAOptimizer

[20]:

qaoa_mes = QAOA(quantum_instance=quantum_instance, initial_point=[0., 1.])
ws_qaoa = WarmStartQAOAOptimizer(pre_solver=CplexOptimizer(), relax_for_pre_solver=True,
                                 qaoa=qaoa_mes, epsilon=0.0)
