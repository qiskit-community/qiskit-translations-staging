மேக்ஸ்-கட் மற்றும் பயண விற்பனையாளர் பிரச்சனை¶

முன்னுரை¶

பொருளாதார மற்றும் பொறியியல் போன்றவற்றில் பல பிரச்சினைகள் பெரும்பாலான பிரச்சினைகளாகும். ஒப்பீட்டளவு சிக்கல்கள் சிக்கலான முடிவெடுப்பது மற்றும் உத்திகளின் வரையறையின் மையத்தில் உள்ளன.

மிகையாக்கல் (அல்லது ஒருங்கிணைந்த தேர்வுமுறை) என்பது ஒரு வரையறுக்கப்பட்ட அல்லது எண்ணற்ற அளவிலான சாத்தியமான தீர்வுகளில் உகந்த தீர்வைத் தேடுவது. உகந்த தன்மை சில அளவுகோல் செயல்பாட்டைப் பொறுத்து வரையறுக்கப்படுகிறது, இது குறைக்கப்பட வேண்டும் அல்லது அதிகரிக்கப்பட வேண்டும். இது பொதுவாக செலவு செயல்பாடு அல்லது புறநிலை செயல்பாடு என்று அழைக்கப்படுகிறது.

**Typical optimization problems * *

குறைத்தல்: செலவு, தூரம், ஒரு பயணத்தின் நீளம், எடை, செயலாக்க நேரம், பொருள், ஆற்றல் நுகர்வு, பொருட்களின் எண்ணிக்கை

அதிகரிப்பு: லாபம், மதிப்பு, வெளியீடு, வருமானம், மகசூல், பயன்பாடு, செயல்திறன், திறன், பொருட்களின் எண்ணிக்கை

பல துறைகளில் நடைமுறை ஆர்வத்தின் அதிகபட்ச சிக்கல்களை நாங்கள் இங்கு கருதுகிறோம், மேலும் அவை குவாண்டம் கணினிகளில் கைமுறையாக எவ்வாறு வரைபடமாக்கப்படலாம் என்பதையும், Qiskit’டின் தேர்வுமுறை தொகுதி இதை எவ்வாறு ஆதரிக்கிறது என்பதையும் காண்பிப்போம்.

வெயிட்டட் மேக்ஸ்-கட்¶

மேக்ஸ்-கட் என்பது NP- முழுமையான சிக்கலாகும், இதில் க்ளஸ்டரிங், நெட்வொர்க் சயின்ஸ் மற்றும் புள்ளிவிவர இயற்பியல் ஆகியவற்றில் பயன்பாடுகள் உள்ளன. கொடுக்கப்பட்ட மேக்ஸ்-கட் நிகழ்வுகளில் நடைமுறை பயன்பாடுகள் எவ்வாறு மாற்றப்படுகின்றன என்பதைப் புரிந்துகொள்ள, ஒருவருக்கொருவர் தொடர்புகொண்டு செல்வாக்கு செலுத்தக்கூடிய பல நபர்களின் அமைப்பைக் கவனியுங்கள். தனிநபர்களை ஒரு வரைபடத்தின் செங்குத்துகளால் குறிக்க முடியும், மேலும் அவற்றின் தொடர்புகள் வரைபடத்தின் செங்குத்துகள் அல்லது விளிம்புகளுக்கு இடையில் ஜோடிவரிசை இணைப்புகளாகக் காணப்படுகின்றன. இந்த பிரதிநிதித்துவத்தை மனதில் கொண்டு, வழக்கமான சந்தைப்படுத்தல் சிக்கல்களை மாதிரியாக்குவது எளிது. எடுத்துக்காட்டாக, தனிநபர்கள் ஒருவருக்கொருவர் வாங்கும் முடிவுகளை பாதிக்கும் என்று கருதப்படுகிறது என்று வைத்துக்கொள்வோம், மேலும் அவர்கள் ஒருவருக்கொருவர் எவ்வளவு வலுவாக செல்வாக்கு செலுத்துவார்கள் என்பது பற்றிய அறிவு வழங்கப்படுகிறது. வரைபடத்தின் ஒவ்வொரு விளிம்பிலும் ஒதுக்கப்பட்டுள்ள எடைகளால் செல்வாக்கை வடிவமைக்க முடியும். சில தனிநபர்களுக்கு இலவசமாக தயாரிப்புகள் வழங்கப்படும் சந்தைப்படுத்தல் மூலோபாயத்தின் விளைவுகளை கணிக்க முடியும், பின்னர் வருவாயை அதிகரிக்க, இலவச தயாரிப்புகளைப் பெற வேண்டிய தனிநபர்களின் உகந்த துணைக்குழு எது என்று கேளுங்கள்.

இந்த சிக்கலின் முறையான வரையறை பின்வருமாறு:

இதைக் கவனியுங்கள் \(n\)- முனை திருப்பிவிடப்படாத வரைபடம் G = (V, E) எங்கே |V| = n விளிம்பு எடையுடன் \(w_{ij}>0\), \(w_{ij}=w_{ji}\), இதற்கு \((i, j)\in E\) இல். ஒரு வெட்டு அசல் தொகுப்பு V இன் பகிர்வாக இரண்டு துணைக்குழுக்களாக வரையறுக்கப்படுகிறது.உகந்ததாக்கப்பட வேண்டிய செலவுச் செயல்பாடு, இந்த வழக்கில் இரண்டு வெவ்வேறு துணைக்குழுக்களில் உள்ள புள்ளிகளை இணைக்கும் விளிம்புகளின் எடைகளின் கூட்டுத்தொகையாகும், குறுக்கு வெட்டு. ஒதுக்குவதன் மூலம் \(x_i=0\) அல்லது \(x_i=1\) ஒவ்வொரு முனைக்கும் \(i\), ஒருவர் உலகளாவிய இலாப செயல்பாட்டை அதிகரிக்க முயற்சிக்கிறார் (இங்கே மற்றும் பின்வரும் சுருக்கங்களில் குறியீடுகள் 0,1 ,… N-1)

\[\tilde{C}(\textbf{x}) = \sum_{i,j} w_{ij} x_i (1-x_j).\]

எங்கள் எளிய சந்தைப்படுத்தல் மாதிரியில்,:math: w_ {ij} நபர் :math: j இதற்குப் பிறகு ஒரு பொருளை வாங்குவதற்கான நிகழ்தகவைக் குறிக்கிறது \(i\) இலவசம் கிடைக்கும். எடைகள்:math: w_ {ij} கொள்கையளவில் \(1\) (அல்லது எதிர்மறை கூட) இதைவிட அதிகமாக இருக்கலாம், இது தனிநபர்:math:` j` ஒன்றுக்கு மேற்பட்ட தயாரிப்புகளை வாங்கும் வழக்கோடு தொடர்புடையது. மொத்த கொள்முதல் நிகழ்தகவை அதிகரிப்பது மொத்த எதிர்கால வருவாயை அதிகரிப்பதை ஒத்துள்ளது. ஆரம்ப இலவச மாதிரிகளின் விலையை விட இலாப நிகழ்தகவு அதிகமாக இருக்கும் விஷயத்தில், மூலோபாயம் ஒரு வசதியானது. இந்த மாதிரியின் நீட்டிப்பானது, எங்களது மார்க்கெட்டிங் மாதிரியில், எடைகளை சுமந்து செல்லும் முனைகளைக் கொண்டுள்ளது, இது ஒரு இலவச மாதிரியுடன் வழங்கப்பட்ட ஒரு நபர் எதிர்காலத்தில் அதை மீண்டும் வாங்குவதற்கான வாய்ப்பாகும். எங்கள் மாதிரியில் இந்த கூடுதல் தகவலுடன், அதிகரிப்பதற்கான புறநிலை செயல்பாடு ஆகிறது

\[C(\textbf{x}) = \sum_{i,j} w_{ij} x_i (1-x_j)+\sum_i w_i x_i.\]

ஒரு குவாண்டம் கணினியில் இந்த சிக்கலுக்கு தீர்வு காண, முதலில் அதை ஒரு ஐசிங் ஹாமில்டோனியனுடன் வரைபடமாக்க வேண்டும்.:math: x_i rightarrow (1-Z_i)/2, இது:math:` Z_i` என்பது பவுலி இசட் ஆபரேட்டராகும், இது சமமான மதிப்புகளைக் கொண்டுள்ளது \(\pm 1\). இதைச் செய்வதன் மூலம் அதைக் காணலாம்

\[C(\textbf{Z}) = \sum_{i,j} \frac{w_{ij}}{4} (1-Z_i)(1+Z_j) + \sum_i \frac{w_i}{2} (1-Z_i) = -\frac{1}{2}\left( \sum_{i<j} w_{ij} Z_i Z_j +\sum_i w_i Z_i\right)+\mathrm{const},\]

எங்கே:math:mathrm{const} = sum_{i <j}w_ {ij}/2+sum_i w_i/2. வேறு வார்த்தைகளில் கூறுவதானால், எடையுள்ள மேக்ஸ்-கட் சிக்கல் ஐசிங் ஹாமில்டோனியனைக் குறைப்பதற்கு சமம்

\[H = \sum_i w_i Z_i + \sum_{i<j} w_{ij} Z_iZ_j.\]

Qiskit டின் தேர்வுமுறை தொகுதி முதல் இலாபச் செயல்பாட்டிற்காக ஐசிங் ஹாமில்டோனியனை உருவாக்க முடியும்:math:tilde{C}. இந்த அளவிற்கு, செயல்பாடு:math:tilde{C} ஐ QuadraticProgram என வடிவமைக்க முடியும், இது ``to_ising ()``முறையை வழங்குகிறது.

மிகையாக்கல் சிக்கல்களுக்கான தோராயமான யுனிவர்சல் குவாண்டம் கம்ப்யூட்டிங்¶

ஒருங்கிணைந்த தேர்வுமுறை சிக்கல்களுக்கு ஒரு தீர்வைக் காண குவாண்டம் கணினிகளைப் பயன்படுத்துவது குறித்து சமீபத்திய காலங்களில் கணிசமான அளவு ஆர்வம் உள்ளது. ஒருங்கிணைந்த சிக்கல்களின் கிளாசிக்கல் தன்மையைக் கருத்தில் கொண்டு, சிறந்த கிளாசிக்கல் வழிமுறைகளுடன் ஒப்பிடும்போது குவாண்டம் கணினிகளைப் பயன்படுத்துவதில் அதிவேக வேகம் உறுதி செய்யப்படவில்லை என்று சொல்வது முக்கியம். இருப்பினும், இலக்கு சிக்கல்களின் தன்மை மற்றும் முக்கியத்துவம் காரணமாக, ஒரு குவாண்டம் கணினியில் ஹூரிஸ்டிக் அணுகுமுறைகளை ஆராய்வது மதிப்புக்குரியது, இது உண்மையில் சில சிக்கல் நிகழ்வுகளை துரிதப்படுத்தக்கூடும். ஃபர்ஹி, கோல்ட்ஸ்டோன் மற்றும் குட்மேன் (2014) எழுதிய Quantum Approximate Optimization Algorithm (QAOA) அடிப்படையிலான ஒரு அணுகுமுறையை இங்கே நாங்கள் நிரூபிக்கிறோம். அல்காரிதத்தை அதன் தோராயமான தன்மையைக் கருத்தில் கொண்டு,*approximate quantum computing* பின்னணியில் வடிவமைக்கிறோம்.

வழிமுறை பின்வருமாறு செயல்படுகிறது:

  1. இலக்கு ஐசிங் சிக்கலில்:math: w_i மற்றும்:math:` w_ {ij} `ஐத் தேர்வுசெய்க. கொள்கையளவில், Z இன் உயர் சக்திகள் கூட அனுமதிக்கப்படுகின்றன.

  2. குவாண்டம் சுற்றுகளின் ஆழத்தைத் தேர்வுசெய்க:math: m. ஆழத்தை தகவமைப்புக்கு மாற்றியமைக்க முடியும் என்பதை நினைவில் கொள்க.

  3. கட்டுப்பாடுகளின் தொகுப்பைத் தேர்வுசெய்க:math:theta மற்றும் ஒரு சோதனைச் செயல்பாட்டைச் செய்யுங்கள்:math:|psi(boldsymboltheta)rangle, இது சி-கட்ட கேட்கள் மற்றும் ஒற்றை-க்யூபிட் Y ஆகியவற்றால் செய்யப்பட்டகுவாண்டம் சுற்று பயன்படுத்தி கட்டப்பட்டது சுழற்சிகள், இதன் \(\boldsymbol\theta\) கூறுகளால் அளவுருவாக்கப்படுகின்றன.

  4. மதிப்பீடு செய்யுங்கள்

    \[C(\boldsymbol\theta) = \langle\psi(\boldsymbol\theta)~|H|~\psi(\boldsymbol\theta)\rangle = \sum_i w_i \langle\psi(\boldsymbol\theta)~|Z_i|~\psi(\boldsymbol\theta)\rangle+ \sum_{i<j} w_{ij} \langle\psi(\boldsymbol\theta)~|Z_iZ_j|~\psi(\boldsymbol\theta)\rangle\]

    z- அடிப்படையில் சுற்றுகளின் முடிவை மாதிரிப்படுத்துவதன் மூலமும், தனிப்பட்ட ஐசிங் சொற்களின் எதிர்பார்ப்பு மதிப்புகளை ஒன்றாகச் சேர்ப்பதன் மூலமும். பொதுவாக, வெவ்வேறு கட்டுப்பாட்டு புள்ளிகள்:math: boldsymboltheta மதிப்பிடப்பட வேண்டும், இது தேர்ந்தெடுக்கப்பட்ட கிளாசிக்கல் ஆப்டிமைசரைப் பொறுத்து.

  5. புதிய கட்டுப்பாடுகளின் தொகுப்பைத் தேர்வுசெய்ய கிளாசிக்கல் ஆப்டிமைசரைப் பயன்படுத்தவும்.

  6. Continue until \(C(\boldsymbol\theta)\) reaches a minimum, close enough to the solution \(\boldsymbol\theta^*\).

  7. \(| \langle z_i ~ | \ psi (\boldsymbol\theta)\rangle|^2 \;\ forall i\) ஐப் பயன்படுத்தவும்.

நல்ல ஹியூரிஸ்டிக் வழிமுறைகளைக் கண்டுபிடிப்பதில் உள்ள சிரமம் பொருத்தமான சோதனை அலைவடிவத்தைத் தேர்ந்தெடுப்பதற்கு வரும் என்பது எங்கள் நம்பிக்கை. எடுத்துக்காட்டாக, ஒரு சோதனைச் செயல்பாட்டை ஒருவர் கருத்தில் கொள்ளலாம், அதன் சிக்கலானது இலக்கு சிக்கலுடன் சிறப்பாக ஒத்துப்போகிறது, அல்லது சிக்கலின் அளவை ஒரு மாறியாக மாற்றலாம். இந்த டுடோரியலில், படிவத்தின் எளிய சோதனை செயல்பாட்டை நாங்கள் கருத்தில் கொள்வோம்

\[|\psi(\theta)\rangle = [U_\mathrm{single}(\boldsymbol\theta) U_\mathrm{entangler}]^m |+\rangle\]

எங்கே \(U_\ mathrm {entangler}\) என்பது சி-கட்ட கேட்களின் தொகுப்பாகும் (முழுமையாக எண்டங்கள்டு கேட்கள்), மற்றும் \(U_\mathrm {single} (\theta) = \ prod_ {i = 1} ^ n Y (\theta_ {i})\), எங்கே \(n\) என்பது க்யூபிட்களின் எண்ணிக்கை மற்றும்:math:m என்பது குவாண்டம் சுற்றுகளின் ஆழம். இந்த தேர்விற்கான உந்துதல் என்னவென்றால், இந்த கிளாசிக்கல் சிக்கல்களுக்கு இந்த தேர்வு உண்மையான குணகங்களை மட்டுமே கொண்ட குவாண்டம் நிலைகளின் இடத்தைத் தேட அனுமதிக்கிறது, மேலும் தீர்வுக்கு விரைவாக ஒன்றிணைவதற்கான சிக்கலை இன்னும் பயன்படுத்துகிறது.

அடிபயாடிக் அணுகுமுறைகளுடன் ஒப்பிடும்போது இந்த மாதிரி முறையைப் பயன்படுத்துவதன் ஒரு நன்மை என்னவென்றால், இலக்கு ஐசிங் ஹாமில்டோனியன் நேரடியாக வன்பொருளில் செயல்படுத்தப்பட வேண்டியதில்லை, இது இந்த வழிமுறையை சாதனத்தின் இணைப்பிற்கு மட்டுப்படுத்தக்கூடாது. மேலும், செலவுச் செயல்பாட்டில் உயர்-வரிசை சொற்கள்,:math:Z_iZ_jZ_k, திறம்பட மாதிரியாகவும் எடுக்கப்படலாம், அதேசமயம் அடிபயாடிக் அல்லது வருடாந்திர அணுகுமுறைகளில் அவை சமாளிக்க நடைமுறைக்கு சாத்தியமற்றவை.

மேற்கோள்கள்: - ஏ. லூகாஸ், இயற்பியலில் எல்லைகள் 2, 5 (2014) - E. ஃபர்ஹி, ஜே. கோல்ட்ஸ்டோன், எஸ். குட்மேன் மின் அச்சு arXiv 1411.4028 (2014) - D. வெக்கர், எம். பி. ஹேஸ்டிங்ஸ், எம். ரெவ். A 94, 022309 (2016) - ஈ. பார்ஹி, ஜே. கோல்ட்ஸ்டோன், எஸ். குட்மேன், எச். நெவன் மின்-அச்சு arXiv 1703.06199 (2017)

பயன்பாட்டு வகுப்புகள்¶

இந்த பக்கத்தில் அதிகபட்ச வெட்டு பிரச்சனை மற்றும் பயண விற்பனையாளர் பிரச்சனைக்கான விண்ணப்ப வகுப்புகளை நாங்கள் பயன்படுத்துகிறோம். பிற தேர்வுமுறை சிக்கல்களுக்கு விண்ணப்ப வகுப்புகள் உள்ளன. தேர்வுமுறை சிக்கல்களுக்கான பயன்பாடு வகுப்புகள் விவரங்களுக்கு பார்க்கவும்.

ட்ராவலிங் சாலேஸ்மேன் ப்ரோப்ளேம்¶

இரண்டு நூற்றாண்டுகளுக்கும் மேலாக கணினி விஞ்ஞானிகள் மற்றும் கணிதவியலாளர்களின் கவனத்தை ஈர்த்த ஒரு மோசமான NP-முழுமையான பிரச்சனையாக இருப்பதைத் தவிர, டிராவலிங் சேல்ஸ்மேன் பிரச்சனை (TSP) நிதி மற்றும் சந்தைப்படுத்துதலில், முக்கிய தாங்குதல்களைக் கொண்டுள்ளது. பேச்சு வார்த்தையில், பயண விற்பனையாளர் நகரத்திலிருந்து நகரத்திற்கு பொருட்களை விற்கச் செல்லும் ஒரு நபர். இந்த வழக்கில் குறிக்கோள் என்னவென்றால், விற்பனையாளர் அனைத்து நகரங்களுக்கும் சென்று அதன் சொந்த ஊருக்கு, அதாவது அவர் பயணிக்கத் தொடங்கிய நகரத்திற்குத் திரும்புவதற்கு உதவும் குறுகிய பாதையைக் கண்டுபிடிப்பதாகும். இதைச் செய்வதன் மூலம், விற்பனையாளர் குறைந்த நேரத்தில் சாத்தியமான விற்பனையை அதிகரிக்க முடியும்.

சிக்கல் அதன் "hardness" மற்றும் நடைமுறையில் எழும் பிற தொடர்புடைய ஒருங்கிணைப்பு தேர்வுமுறை சிக்கல்களுக்கு எங்கும் நிறைந்த சமநிலையிலிருந்து அதன் முக்கியத்துவத்தைப் பெறுகிறது.

சில ஆரம்ப பகுப்பாய்வுகளுடன் கணித உருவாக்கம் 19 ஆம் நூற்றாண்டின் ஆரம்பத்தில் டபிள்யூ.ஆர். ஹாமில்டனால் முன்மொழியப்பட்டது. கணித ரீதியாக சிக்கல் என்னவென்றால், மேக்ஸ்-கட் விஷயத்தைப் போலவே, வரைபடங்களின் அடிப்படையில் மிகச் சுருக்கமாக உள்ளது. ஒரு வரைபடத்தின் முனைகளில் உள்ள TSP ஒவ்வொரு முனைகளிலும் எடுக்கக்கூடிய மிகக் குறுகிய ஹாமில்டோனிய சுழற்சி ஐ கேட்கிறது. ஒரு ஹாமில்டன் சுழற்சி என்பது ஒரு மூடிய பாதையாகும், இது ஒரு வரைபடத்தின் ஒவ்வொரு உச்சியையும் ஒரு முறை பயன்படுத்துகிறது. பொதுவான தீர்வு தெரியவில்லை மற்றும் அதை திறமையாகக் கண்டுபிடிக்கும் ஒரு வழிமுறை (எ.கா., பல்லுறுப்புறுப்பு நேரத்தில்) இருப்பதாக எதிர்பார்க்கப்படவில்லை.

ஒரு வரைபடத்தில் குறுகிய ஹாமில்டோனிய சுழற்சியைக் கண்டறியவும்:math:G=(V,E) உடன் \(n = |V|\) முனைகள் மற்றும் தூரங்கள், \(w_{ij}\) (தூரம்:math:i வெர்டெக்ஸிலிருந்து vertex \(j\) வெர்டெக்ஸிக்கு). ஒரு ஹாமில்டோனிய சுழற்சி விவரிக்கிறது:math:N^2 மாறிகள்:math:x_{i,p} `, எங்கே:math:`i கணுவைக் குறிக்கிறது மற்றும்:math:p அதன் வரிசையை வருங்கால சுழற்சி குறிக்கிறது. தீர்வு முனை:math:i நேர வரிசையில்:math:p இல் தீர்வு ஏற்பட்டால் முடிவு மாறி மதிப்பு 1 ஐ எடுக்கும். ஒவ்வொரு முனையும் சுழற்சியில் ஒரு முறை மட்டுமே தோன்ற வேண்டும், ஒவ்வொரு முறையும் ஒரு முனை ஏற்பட வேண்டும். இது இரண்டு தடைகளுக்கு சமம் (இங்கே மற்றும் பின்வருவனவற்றில், குறிப்பிடப்படாத போதெல்லாம், சம்மன்கள் 0,1,…N-1 க்கு மேல் இயங்கும்)

\[\sum_{i} x_{i,p} = 1 ~~\forall p\]
\[\sum_{p} x_{i,p} = 1 ~~\forall i.\]

எங்கள் வருங்கால வரிசையில் உள்ள முனைகளுக்கு :math: x_ {i,p} மற்றும்:math: x_{j, p+1} இரண்டும் 1 என்றால், ஆற்றல் அபராதம் இருக்க வேண்டும்:math: (i,j) notin E (வரைபடத்தில் இணைக்கப்படவில்லை). இந்த தண்டனையின் வடிவம்

\[\sum_{i,j\notin E}\sum_{p} x_{i,p}x_{j,p+1}>0,\]

இது ஹாமில்டோனிய சுழற்சிகளின் எல்லை நிலை என்று கருதப்படுகிறது:math: (p=N) equiv (p=0). இருப்பினும், இங்கே இது முழுமையாக இணைக்கப்பட்ட வரைபடமாக கருதப்படும் மற்றும் இந்த வார்த்தையை சேர்க்காது. குறைக்க வேண்டிய தூரம்

\[C(\textbf{x})=\sum_{i,j}w_{ij}\sum_{p} x_{i,p}x_{j,p+1}.\]

குறைக்கப்பட வேண்டிய ஒற்றை புறநிலை செயல்பாட்டில் இவை அனைத்தையும் ஒன்றாக இணைத்து, பின்வருவனவற்றைப் பெறுகிறோம்:

\[C(\textbf{x})=\sum_{i,j}w_{ij}\sum_{p} x_{i,p}x_{j,p+1}+ A\sum_p\left(1- \sum_i x_{i,p}\right)^2+A\sum_i\left(1- \sum_p x_{i,p}\right)^2,\]

எங்கே:math: A ஒரு இலவச அளவுரு. ஒருவர் இதை உறுதிப்படுத்த வேண்டும்:math: A போதுமானதாக இருப்பதால் இந்த தடைகள் மதிக்கப்படுகின்றன. இதைச் செய்வதற்கான ஒரு வழி:math: A அதாவது:math:` A> mathrm {max} (w_ {ij}) `.

மீண்டும், இந்த வடிவத்தில் சிக்கலை ஒரு குவாண்டம் கணினியில் வரைபடமாக்குவது எளிதானது, மேலும் ஒரு ஐசிங் ஹாமில்டோனியனைக் குறைப்பதன் மூலம் தீர்வு காணப்படும்.

from itertools import permutations

def brute_force_tsp(w, N):
    a = list(permutations(range(1,N)))
    last_best_distance = 1e10
    for i in a:
        distance = 0
        pre_j = 0
        for j in i:
            distance = distance + w[j,pre_j]
            pre_j = j
        distance = distance + w[pre_j,0]
        order = (0,) + i
        if distance < last_best_distance:
            best_order = order
            last_best_distance = distance
            print('order = ' + str(order) + ' Distance = ' + str(distance))
    return last_best_distance, best_order

best_distance, best_order = brute_force_tsp(adj_matrix , n)
print('Best order from brute force = ' + str(best_order) + ' with total distance = ' + str(best_distance))

def draw_tsp_solution(G, order, colors, pos):
    G2 = nx.DiGraph()
    G2.add_nodes_from(G)
    n = len(order)
    for i in range(n):
        j = (i + 1) % n
        G2.add_edge(order[i], order[j], weight=G[order[i]][order[j]]['weight'])
    default_axes = plt.axes(frameon=True)
    nx.draw_networkx(G2, node_color=colors, edge_color='b', node_size=600, alpha=.8, ax=default_axes, pos=pos)
    edge_labels = nx.get_edge_attributes(G2, 'weight')
    nx.draw_networkx_edge_labels(G2, pos, font_color='b', edge_labels=edge_labels)
