이차계획법 문제를 위한 변환기¶

Qiskit의 최적화 모듈에서 최적화 문제는 QuadraticProgram 클래스로 표현되는데, 이는 최적화 문제에 대한 일반적이고 강력한 표현이다. 일반적으로 최적화 알고리즘은 이차계획법 문제라는 특수한 형태의 수식에 의해 정의되므로 해결하고자 하는 문제를 올바른 유형으로 변환할 필요가 있다.

예를 들어, Qiskit은 이차 비한정 이진 최적화 (Quadratic Unconstrained Binary Optimization; QUBO) 문제들을 처리할 수 있는 몇 가지 최적화 알고리즘들을 제공한다. 이 문제들은 키스킷의 qiskit.aqua.operators 모듈을 통해 이징 해밀토니언 (Ising Hamiltonians)에 매핑되며, 이후 이들의 바닥상태 (ground sate) 가 근사화된다. VQE 또는 QAOA와 같이 일반적으로 잘 알려진 알고리즘들에 대하여 위와 같은 방식으로 최적화가 수행된다. 자세한 내용은 다음의 사용 지침서 Minimum Eigen Optimizer 를 참조하도록 한다. 물론 GroverOptimizer 과 같이 다른 방식으로 작동하는 알고리즘들도 있음에 유의한다.

문제를 올바른 입력 형식으로 보이기 위하여, Qiskit의 최적화 모듈은 다양한 변환기들을 제공한다. 본 사용 지침서에서는 최적화 모듈이 제공하는 각 변환기의 기능들을 개괄적으로 정리하여 설명할 것이다. 현재 Qiskit에 포함되어 있는 변환기들은 다음과 같다 - InequalityToEquality: 부등식 제약조건을 등식 제약조건 및 변환 과정에서 추가된 완화 제약조건들로 변환한다. - IntegerToBinary: 정수형 변수를 바이너리형 변수 및 각 변수에 대응되는 계수로 변환한다. - LinearEqualityToPenalty: 등식 제약조건을 목적함수의 추가 항으로 변환한다. 사용자의 편의를 위화여 QuadraticProgramToQubo 는 InequalityToEquality, IntegerToBinary, LinearEqualityToPenalty 의 wrapper이다.

부등식의 등식화¶

QuadraticProgram 에서 부등식의 제약조건을 제거하기 위하여 InequalityToEqualityConverter 는 여유변수(slack variable)를 추가함으로써 부등식을 등식으로 변환시킨다. 여유변수의 상한과 하한은 식의 좌변과 우변의 차이로 계산된다. 여유변수의 부호는 \(\leq\) 와 \(\geq\) 같이 부등호에 의하여 결정된다.

다음은 부등식 제약조건이 2 개인 최대화 문제의 예이다. 변수 \(x\) 와 \(y\) 는 이진 변수이고 변수 \(z\) 는 정수 변수이다.

``QuadraticProgram``을 이용하여 작성한 최적화 문제의 모델은 다음과 같다.

[1]:

from qiskit_optimization import QuadraticProgram

[2]:

qp = QuadraticProgram()
qp.binary_var('x')
qp.binary_var('y')
qp.integer_var(lowerbound=0, upperbound=7, name='z')

qp.maximize(linear={'x': 2, 'y': 1, 'z': 1})
qp.linear_constraint(linear={'x': 1, 'y': 1, 'z': 1}, sense='LE', rhs=5.5,name='xyz_leq')
qp.linear_constraint(linear={'x': 1, 'y': 1, 'z': 1}, sense='GE', rhs=2.5,name='xyz_geq')
print(qp.export_as_lp_string())

모델을 변환하기 위하여 InequalityToEquality``의 ``convert 메서드를 아래와 같이 호출한다.

[3]:

from qiskit_optimization.converters import InequalityToEquality

[4]:

ineq2eq = InequalityToEquality()
qp_eq = ineq2eq.convert(qp)
print(qp_eq.export_as_lp_string())

\ ENCODING=ISO-8859-1
\Problem name: CPLEX

Maximize
 obj: 2 x + y + z
Subject To
 xyz_leq: x + y + z + xyz_leq@int_slack = 5
 xyz_geq: x + y + z - xyz_geq@int_slack = 3

 0 <= x <= 1
 0 <= y <= 1
       z <= 7
       xyz_leq@int_slack <= 5
       xyz_geq@int_slack <= 6

Binaries
 x y

Generals
 z xyz_leq@int_slack xyz_geq@int_slack
End

변환 후, 문제의 수식은 위 코드 실행결과와 같이 바뀌었다. 위에서 살펴본 것처럼, 부등식의 부호들이 \(xyz\_leg\text{@}int\_slack\) and \(xyz\_geq\text{@}int\_slack\) 의 정수 여유변수를 추가함으로써 부등식이 등식으로 변환되었다.

변환이 어떻게 작동하는지 살펴보자. 예를 들어, 좌변 첫 번째 제약 조건의 하한(lower bound)은 \(0\) 이며, 이는 \(x=0\), \(y=0\) 및 \(z=0\) 의 경우이다. 따라서 추가된 정수형 변수의 상한(upperbound)은 \(5\) 가 되어야 \(x=0\), \(y=0\), \(z=0\) 의 경우를 만족할 수 있다. 원래 공식에서 좌변 첫 번째 제약 조건은 정수 값만 가질 수 있기 때문에, 변환된 공식에서 소수점 뒤의 값들이 버려진 것을 확인할 수 있다. 두 번째 제약의 경우 같은 방식을 적용한다. 그러나 두 번째 제약 조건의 부호는 \(\geq\) 이므로 \(xyz\_geq\text{@}int\_slack\) 앞에 빼기 부호를 추가하여야 :math: x=1, y=1 및 \(z=7\) 의 경우도 만족시킬 수 있다.

정수변수의 이진변수화¶

QuadraticProgram 에서 정수형 변수를 제거하기 위하여 IntegerToBinary 는 정수형 변수를 이진 변수와 계수로 변환한다. 이러한 변환은, 논문 arxiv:1706.01945 (Eq. (5)) 에서 제안한 구속-계수 인코딩 (bounded-coefficient encoding) 기법을 이용하여 변환되었다. 인코딩 기법과 관련한 자세한 내용은 논문을 통해 확인할 수 있다.

InequalityToEquality 의 출력을 사용하여 다음 예제를 수행한다. 변수 \(x\) 와 \(y\) 는 이진 변수이고, 변수 \(z\) 와 여유변수인 \(xyz\_leq\text{@}int\_slack\) 및 \(xyz\_geq\text{@}int\_slack\) 는 정수형 변수이다. 값을 확인하기 위하여 문제를 다시 출력하면 아래와 같다.

변환 후, 위와 같이 변수 \(z\) 는 이진 변수 \(z\text{@}0\), \(z\text{@}1\) 와 \(z\text{@}2\) 와 계수 1, 2, 4로 각각 대체된다. InequalityToEquality 에서 소개되었던 여유변수인 \(xyz\_leq\text{@}int\_slack\) 와 \(xyz\_geq\text{@}int\_slack\) 역시 세 개의 이진 변수와 계수 1, 2, 2 및 계수 1, 2, 3으로 각각 대체된다.

참고 : 기본적으로 계수가 의미하는 바는, 이러한 계수를 갖는 이진 변수들의 합이 원래의 정수형 변수의 하한과 상한인 \(\{0, \ldots, 7\}\), \(\{0, \ldots, 5\}\), \(\{0, \ldots, 6\}\) 를 나타내기 위하여 \(\{1, 2, 4\}\), \(\{1, 2, 2\}\), \(\{1, 2, 3\}\) 의 하위 집합의 합이 될 수 있음을 의미한다.

IntegerToBinary 는 이진화된 결과를 원래의 정수 표현으로 다시 변환하는 기능인 interpret 메서드도 제공한다.

LinearEqualityToPenalty¶

LinearEqualityToPenalty 는 QuadraticProgram 을 제약 조건이 없는 형태로 매핑하기 위하여, 선형 등식 제약 조건들을 (목적 함수에 추가되는) 이차 패널티 항(term)으로 변환한다. 이러한 등식 제약 조건( \(\sum_i a_i x_i = b\) 여기서 \(a_i\) 와 \(b\) 는 숫자이며 \(x_i\) 는 변수)들은 목적 함수에 \(M(b - \sum_i a_i x_i)^2\) 와 같은 형태로 추가될 것이다. 이때 \(M\) 은 아주 큰 숫자로 패널티 요소(factor)이며 기본적으로 \(M= 1e5\) 이다. 이차 패널티 항의 부호는 문제가 최대화 문제인지 최소화 문제인지의 여부에 따라 결정된다.

IntegerToBinary 의 결과를 시작 지점으로 사용한다. 여기서 모든 변수는 바이너리 변수이고 모든 부등식 제약조건은 등식 제약조건에 매핑되었다. 참조를 위하여 문제를 다시 출력하겠다.
