দ্বিঘাত (কোয়াড্র্যাটিক) প্রোগ্রামস¶

ভূমিকা¶

এই পাঠসমূহে (টিউটোরিয়াল), আমরা Qiskit অপ্টিমাইজেশন মডিউলটি ব্যবহার করে কীভাবে অনুকূলায়ন (অপ্টিমাইজেশন) সমস্যাসমূহ তৈরি করতে পারি তার জন্য সংক্ষেপে পরিচয় করিয়ে দেব। কিসকিট একটি অপ্টিমাইজেশান সমস্যার মডেল তৈরি করতে QuadraticProgram শ্রেণীর সাথে পরিচয় করিয়ে দিয়েছে। আরও স্পষ্টভাবে, এটি দ্বিঘাত ভাবে সীমাবদ্ধ দ্বিঘাত প্রোগ্রামগুলির (Quadratic Program) সাথে সম্পর্কিত যা নীচে দেওয়া হয়েছে:

\[\begin{split}\begin{align} \text{minimize}\quad& x^\top Q_0 x + c^\top x\\ \text{subject to}\quad& A x \leq b\\ & x^\top Q_i x + a_i^\top x \leq r_i, \quad 1,\dots,i,\dots,q\\ & l_i \leq x_i \leq u_i, \quad 1,\dots,i,\dots,n, \end{align}\end{split}\]

যেখানে \(Q_i\) হ’ল: \(n \times n\) ম্যাট্রিক্স, \(A\) একটি \(m \times n\) ম্যাট্রিক্স,: \(x\), , এবং: \(c\) হ’ল: \(n\)-মাত্রিক ভেক্টর,: \(b\) একটি \(m\)-ডাইমেনশনাল ভেক্টর এবং যেখানে \(x\) বাইনারি, পূর্ণসংখ্যা বা অবিচ্ছিন্ন ভেরিয়েবল হিসাবে সংজ্ঞায়িত করতে পারে । “\(\leq\)” ছাড়াও ‘QuadraticProgram’ “\(\geq\)” এবং “\(=\)” সমর্থন করে।

একটি এল.পি (LP) ফাইল থেকে একটি Quadratic Program লোড করা¶

সেটআপ হিসাবে, আপনাকে নিম্নলিখিত মডিউলটি আমদানি (ইম্পোর্ট) করতে হবে।

[1]:

from qiskit_optimization import QuadraticProgram
from qiskit_optimization.translators import from_docplex_mp

আপনি একটি খালি নকশা ( মডেল ) দিয়ে শুরু করুন। কোনও নকশায় চলরাশি (ভেরিয়েবল) এবং সীমাবদ্ধতা (কন্সট্রেইন্টস ) কীভাবে যুক্ত করবেন সে সম্পর্কে "সরাসরি একটি ``QuadraticProgram` তৈরি " বিভাগে ব্যাখ্যা করা হয়েছে।

Qiskit এর অনুকূলায়ক (অপ্টিমাইজেশন) মডিউল ডকপ্লেক্স নকশা ( মডেল) থেকে রূপান্তরকে সমর্থন করে। ডকপ্লেক্সের সাহায্যে আপনি খুব সহজেই একটি অনুকূলায়ক (অপ্টিমাইজেশান ) সমস্যার নকশা (মডেল) তৈরি করতে পারেন। আপনি ডকপ্লেক্সের ডকুমেন্টেশন https://ibmdecisionoptimization.github.io/docplex-doc/mp/index.html এ খুঁজে পেতে পারেন

from_docplex ফাংশন ব্যবহার করে QuadraticProgram -এ একটা ডকপ্লেক্স মডেল লোড করা হয়।

একটি ডকপ্লেক্স নকশা (মডেল) থেকে একটি Quadratic Program লোড করা¶

# Make a Docplex model
from docplex.mp.model import Model

mdl = Model('docplex model')
x = mdl.binary_var('x')
y = mdl.integer_var(lb=-1, ub=5, name='y')
mdl.minimize(x + 2 * y)
mdl.add_constraint(x - y == 3)
mdl.add_constraint((x + y) * (x - y) <= 1)
print(mdl.export_as_lp_string())

সরাসরি একটি QuadraticProgram তৈরি করা¶

তারপরে আমরা কীভাবে সরাসরি QuadraticProgram ব্যবহার করে একটি অনুকূলায়ন (অপ্টিমাইজেশন) সমস্যাসমূহর নকশা (মডেল) তৈরি করব তা ব্যাখ্যা করবো। একটি খালি মডেল থেকে শুরু করা যাক।

QuadraticProgram তিন ধরণের চল রাশি ( ভেরিয়েবল) সমর্থন করে: - বাইনারি ভেরিয়েবল - পূর্ণসংখ্যার ভেরিয়েবল - অবিচ্ছিন্ন ভেরিয়েবল

আপনি যখন চল রাশি (ভেরিয়েবল) যুক্ত করবেন, আপনি নাম, প্রকার, নিম্ন সীমা(আবদ্ধ ) এবং উপরের সীমা ((আবদ্ধ ) নির্দিষ্ট করতে পারেন।

আপনি যখন এল.পি ফর্ম্যাট হিসাবে আপনার সমস্যাটি প্রদর্শন করেন, তখন Binaries বাইনারি চল রাশি (ভেরিয়েবল) গুলি এবং` Generals পূর্ণ সংখ্যা (ইন্টিজার) চল রাশি(ভেরিয়েবল) কে চিহ্নিত করে। যদি চল রাশিগুলি Binaries বা Generals - এর মধ্যে অন্তর্ভুক্ত না করা হয়, তবে এই ধরণের পরিবর্তনগুলি অবিচ্ছিন্নভাবে ডিফল্ট লোয়ার সীমানা (বাউন্ড) = 0 এবং উপরের সীমানা (বাউন্ড)= অসীম থাকে। মনে রাখবেন যে এল.পি ফর্ম্যাটের নির্দিষ্টকরণের কারণে আপনি নামের প্রথম অক্ষর হিসাবে ‘e’ বা ‘E’ ব্যবহার করতে পারবেন না।

আপনি QuadraticProgram.minimize বা QuadraticProgram.maximize- দ্বারা নৈর্ব্যক্তিক অন্বয় (অব্জেক্টিভ ফাংশন) টি সেট করতে পারেন। আপনি তালিকা, ম্যাট্রিক্স বা অভিধানের সাথে লিনিয়ার এবং দ্বিঘাত (কোয়াড্রাটিক) পদগুলি নির্দিষ্ট করে একটি ধ্রুবক পদ পাশাপাশি লিনিয়ার এবং দ্বিঘাত (কোয়াড্রাটিক) নৈর্ব্যক্তিক অন্বয় (অব্জেক্টিভ ফাংশন) যুক্ত করতে পারেন।

টীকা: এল.পি বিন্যাসে দ্বিঘাত (কোয়াড্রাটিক) অংশটি একটি গুণক \(1/2\) দ্বারা মাপতে হবে সুতরাং, এল.পি ফর্ম্যাট হিসাবে মুদ্রণের সময়, দ্বিঘাত (কোয়াড্রাটিক) অংশটি প্রথমে ২ দ্বারা গুণিত হয় এবং তারপরে আবার ২ দিয়ে বিভক্ত হয়।

দ্বিঘাত (কোয়াড্রাটিক) প্রোগ্রামগুলির জন্য, এখানে 3 টি অংশ নির্দিষ্ট করতে হবে: একটি ধ্রুবক (অফসেট), একটি রৈখিক পদ (\(c^{T}x\)), এবং একটি দ্বিঘাত (কোয়াড্রাটিক) পদ (\(x^{T}Qx\))।

নীচের সেলটি দেখায় যে কীভাবে একটি অভিধান ব্যবহার করে কোনও নৈর্ব্যক্তিক অন্বয় (অব্জেক্টিভ ফাংশন) ঘোষণা করতে হয়। রৈখিক পদটির জন্য, অভিধানের কী-গুলি ভেরিয়েবল নামের সাথে মিলে যায় এবং সংশ্লিষ্ট মানগুলি সহগ হয়। দ্বিঘাত (কোয়াড্রাটিক) পদটির জন্য অভিধানের কী-গুলি দুটি চল রাশির (ভেরিয়েবল) সাথে গুণিত হওয়ার সাথে মিলে যায় এবং মানগুলি আবার সহগ হয়।

দ্বিঘাত (কোয়াড্রাটিক) প্রোগ্রাম নির্দিষ্ট করার আরেকটি উপায় হ’ল অ্যারে (array) ব্যবহার করা। রৈখিক (লিনিয়ার) পদের জন্য অ্যারে - টি ` \(c\) এর সাথে গাণিতিক গঠনের সাথে মিলে যায়। দ্বিঘাত (কোয়াড্রাটিক) পদটির জন্য অ্যারে-টি ম্যাট্রিক্স \(Q\) এর সাথে মিলে যায়। টীকা: যে চল রাশি (ভেরিয়েবল)-র ক্রম (\(x\) গাণিতিক গঠনের ক্ষেত্রে) হ’ল ক্রমটি যেখানে চল রাশি (ভেরিয়েবল) গুলি মূলত QuadraticProgram অবজেক্টে ঘোষিত হয়েছিল।

আপনি যথাক্রমে``Quadratic.objective.{constant, linear, quadratic}``, দেখে ধ্রুবক, রৈখিক পদ এবং দ্বিঘাত (কোয়াড্রাটিক) পদ ব্যবহার করতে পারেন। রৈখিক এবং দ্বিঘাত শর্তাবলী হিসাবে, আপনি একটি ডেন্স ম্যাট্রিক্স (to_array), একটি স্পার্স ম্যাট্রিক্স (coefficients`), এবং একটি অভিধান (``to_dict) পেতে পারেন। অভিধানগুলির জন্য, আপনি "কী" হিসাবে পরিবর্তনশীল সূচক বা নাম ব্যবহার করতে হবে তা নির্দিষ্ট করতে পারেন। টীকা: দ্বিঘাত (কোয়াড্রাটিক) পদগুলি সংকুচিতভাবে সংরক্ষণ করা হয়, উদাহরণস্বরূপ, {('x', 'y'): 1, ('y', 'x'): 2} , {('x', 'y'): 3}. হিসাবে সঞ্চিত হয়। আপনি to_array(symmetric=True) বা to_dict(symmetric=True) কল করে দ্বিঘাত (কোয়াড্রাটিক) পদকে প্রতিসম ম্যাট্রিক্স হিসাবে পেতে পারেন। আপনি যদি to_dict(name=True) কল করেন তবে আপনি একটি অভিধান পেতে পারেন যার কী-গুলি চলক নামের জোড়া।

রৈখিক এবং দ্বিঘাত (কোয়াড্রাটিক) সীমাবদ্ধতা যুক্ত করা / সরানো¶

নাম, রৈখিক (লিনিয়ার) এক্সপ্রেশন, সেন্স এবং ডান দিকের মান (আরএইচএস) সেট করে আপনি রৈখিক সীমাবদ্ধতা যুক্ত করতে পারেন। ডকপ্লেক্স সমর্থন হিসাবে আপনি সেন্স-গুলিতে ‘ইসিউ’, ‘এলই’ এবং ‘জিই’ ব্যবহার করতে পারেন।.

ধ্রুবক বা অন্যান্য চল রাশিগুলির সাথে আপনি কয়েকটি চল রাশির বিকল্প করতে পারেন। আরও স্পষ্টভাবে, নিম্নোক্ত দুটি ক্ষেত্রে মোকাবেলা করার জন্য QuadraticProgram এর substitute_variables(constants=..., variables=...) নামে একটি পদ্ধতি রয়েছে : \(x \leftarrow c\): যখন constants এর একটি {x: c} ডিকশনারি থাকে এবং - \(x \leftarrow c y\): যখন variables এর একটি {x: (y, c)} ডিকশনারি থাকে ডিকশনারি থাকে।
