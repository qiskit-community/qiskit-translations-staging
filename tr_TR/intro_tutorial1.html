Qiskit’e Giriş¶

Qiskit ‘i kullanırken kullanıcı iş akışı, aşağıdaki dört yüksek seviyeli adımdan oluşur:

  • Oluştur: Üzerine düşündüğün problemi temsil eden bir kuantum devre(ler) tasarla.

  • Derle: Devreleri belirli bir kuantum hizmeti için derle, örneğin bir kuantum sistem ya da klasik simülatör için.

  • Çalıştır: Belirtilen kuantum hizmet(ler)inde derlenen devreleri çalıştırın. Bu hizmetler bulut tabanlı ya da yerel olabilir.

  • Analiz Et: Özet istatistikleri hesapla ve denemelerin sonuçlarını görselleştir.

Burası sonraki bölümlerde her adımının detaylı şekilde anlatılacağı iş akışının örneğidir:

import numpy as np
from qiskit import QuantumCircuit, transpile
from qiskit.providers.aer import QasmSimulator
from qiskit.visualization import plot_histogram

# Use Aer's qasm_simulator
simulator = QasmSimulator()

# Create a Quantum Circuit acting on the q register
circuit = QuantumCircuit(2, 2)

# Add a H gate on qubit 0
circuit.h(0)

# Add a CX (CNOT) gate on control qubit 0 and target qubit 1
circuit.cx(0, 1)

# Map the quantum measurement to the classical bits
circuit.measure([0,1], [0,1])

# compile the circuit down to low-level QASM instructions
# supported by the backend (not needed for simple circuits)
compiled_circuit = transpile(circuit, simulator)

# Execute the circuit on the qasm simulator
job = simulator.run(compiled_circuit, shots=1000)

# Grab results from the job
result = job.result()

# Returns counts
counts = result.get_counts(compiled_circuit)
print("\nTotal count for 00 and 11 are:",counts)

# Draw the circuit
circuit.draw()


Total count for 00 and 11 are: {'11': 505, '00': 495}

     └───┘┌─┴─┐└╥┘┌─┐

c: 2/═══════════╩══╩═
                0  1 
# Plot a histogram
plot_histogram(counts)

Adım Adım iş akışı¶

Yukarıdaki program altı adıma ayrılabilir:

  1. Paketleri içe aktar

  2. Değişkenlere İlk Değerlerini Ata

  3. Kapılar Ekle

  4. Devreyi Görselleştir

  5. Deneyi Simüle Et

  6. Sonuçları görselleştir

Adım 1: Paketleri Yükle¶

Programınız için gerekli olan temel öğeler takip eden şekilde içe aktarılır:

import numpy as np
from qiskit import QuantumCircuit
from qiskit.providers.aer import QasmSimulator
from qiskit.visualization import plot_histogram

Daha ayrıntılı bir şekilde, içe aktarımlar

  • QuantumCircuit: Kuantum sisteminin talimatları olarak düşünülebilir. Sizin bütün kuantum operatörlerinizi taşır.

  • QasmSimulator: Aer yüksek performanslı devre simülatörüdür.

  • plot_histogram: histogramlar oluşturur.

Adım 2: Değişkenlere İlk Değerlerini Atama¶

Aşağıdaki kod satırına bakalım

circuit = QuantumCircuit(2, 2)

Burada sıfır durumunda 2 kübit ve sıfır atanmış iki klasik bitle başlıyorsunuz; circuit ise kuantum devreniz.

Sözdizimi:

  • QuantumCircuit(int, int)

Adım 3: Kapılar Ekle¶

Devrenizin yazmaçlarını manipüle etmek için kapılar (işlemler) ekleyebilirsiniz.

Aşağıdaki üç kod satırına bakalım:

circuit.h(0)
circuit.cx(0, 1)
circuit.measure([0,1], [0,1])

Kapılar Bell durumunu oluşturmak için devreye teker teker eklenir

\[|\psi\rangle = \left(|00\rangle+|11\rangle\right)/\sqrt{2}.\]

Yukarıdaki kod aşağıdaki kapıları uygular:

  • QuantumCircuit.h(0): Kübit 0’a uygulanıp onu süperpozisyon durumuna sokan bir Hadamard kapısı \(H\).

  • QuantumCircuit.cx(0, 1): Kübit 0 kontrol ve kübit 1 hedef olacak şekilde uygulanan Kontrollü Not işlemi (\(CNOT\)) kübitleri **dolanıklık durumu**na getirir.

  • QuantumCircuit.measure([0,1], [0,1]): bütün kuantum ve klasik yazmaçları measure’a geçirirseniz, i’nci kübitin ölçüm sonucu i’nci klasik bite kaydedilir.

Adım 4: Devreyi Görselleştir¶

Birçok ders kitabı ve araştırma makalesinde kullanılan çeşitli şekillerde tasarladığınız devreyi görmek için qiskit.circuit.QuantumCircuit.draw()’u kullanabilirsiniz.

circuit.draw()

     ┌───┐     ┌─┐   
q_0: ┤ H ├──■──┤M├───
     └───┘┌─┴─┐└╥┘┌─┐
q_1: ─────┤ X ├─╫─┤M├
          └───┘ ║ └╥┘
c: 2/═══════════╩══╩═
                0  1

Bu devrede kubitler, üstte kubit sıfır ve altta kubit bir olacak şekilde sıralanmıştır. Devre soldan sağa doğru okunur, yani daha önce uygulanacak kapılar devrenin daha solunda olanlardır.

QuantumCircuit.draw() veya qiskit.visualization.circuit_drawer() için varsayılan backend metin backendidir. Ancak yerel çevrenize bağlı olarak bu varsayılanları sizin kullanımınıza uygun olacak şekilde değiştirmek isteyebilirsiniz. Bu, kullanıcı yapılandırma dosyası (user config file) ile yapılır. Kullanıcı yapılandırma dosyası varsayılan olarak ~/.qiskit/settings.conf içerisinde konumlandırılmıştır ve bir .ini dosyasıdır.

Örneğin, bir Matplotlib çizimcisi ayarlamaya yarayan settings.conf dosyası:

[default]
circuit_drawer = mpl

Bu yapılandırmanın değeri olarak, text, mpl, latex, ve latex_source dahil tüm geçerli devre çizimci backendlerini kullanabilirsiniz.

Adım 5: Deneyi Simüle Etmek¶

Qiskit Aer, kuantum devreleri için yüksek performanslı bir simülatör çerçevesidir. Farklı simülasyon hedeflerine ulaşmak için several backends sağlar.

Aer’i yüklemekte sorun yaşıyorsanız, Aer yerine BasicAer koyarak Basic Aer sağlayıcısını kullanabilirsiniz. Basic Aer, Qiskit Terra içerisinde bulunmaktadır.

import numpy as np
from qiskit import QuantumCircuit, transpile
from qiskit.providers.basicaer import QasmSimulatorPy
...

Bu devreyi simüle etmek için qasm_simulator kullanacaksınız. Devrenin her çalışması 00 veya 11 bit dizilerinden biriyle sonuçlanacaktır.

simulator = QasmSimulator()
compiled_circuit = transpile(circuit, simulator)
job = simulator.run(compiled_circuit, shots=1000)
result = job.result()
counts = result.get_counts(circuit)
print("\nTotal count for 00 and 11 are:",counts)


Total count for 00 and 11 are: {'11': 507, '00': 493}

Beklendiği gibi çıktılarının yaklaşık yarısı 00 bit dizisinden oluşuyor. Devrenin kaç kere çalıştırılacağı execute metodunun shots argümanı aracılığıyla belirlenebilir. Simülasyonun deneme sayısı 1000 olarak belirlenmişti (varsayılan 1024).

Bir defa result objesine sahip olmanız durumunda, get_counts(circuit) metodu aracılığıyla sayılara erişebilirsiniz. Bu size çalıştırdığınız deneyin çıktılarını toplu halde verir.

Adım 6: Sonuçları Görselleştir¶

Qiskit many visualizations sağlar,

sonuçları görmek için plot_histogram fonksiyonu da dahil olmak üzere.

plot_histogram(counts)

Gözlemlenen \(Pr(00)\) ve \(Pr(11)\) olasılıkları, durumların gözlemlenme miktarlarının toplam deneme miktarına bölünmesiyle hesaplanır.

Not

Tahmini olasılıkların nasıl değişeceğini görmek için ` ` run () ` ` yöntemindeki ` ` shots ` ` anahtar sözcüğünü değiştirmeyi deneyin.
