Gerbang Pulsa¶

Most quantum algorithms can be described with circuit operations alone. When we need more control over the low-level implementation of our program, we can use pulse gates. Pulse gates remove the constraint of executing circuits with basis gates only, and also allow you to override the default implementation of any basis gate.

Gerbang pulsa memungkinkan Anda untuk memetakan gerbang sirkuit logis (misalnya, ` ` X ` ) ke sebuah program Qiskit Pulse, yang disebut ` ` Jadwal ` `. Pemetaan ini disebut sebagai *kalibrasi *. Sebuah kalibrasi yang tinggi adalah salah satu yang dengan setia mengimplementasikan operasi logis itu dipetakan dari (misalnya, apakah kalibrasi ` ` X ` ` ` gerbang drive :math: | 0rsudut ` ke :math:` | 1rsudut `, dll).

Sebuah jadwal menjelaskan dinamika waktu yang tepat dari sinyal masukan di semua saluran masukan * ke perangkat. Biasanya ada beberapa saluran per qubit, seperti drive dan measure. Antarmuka ini lebih kuat, dan membutuhkan pemahaman yang lebih dalam tentang perangkat fisika yang mendasarinya.

Penting untuk dicatat bahwa program Pulse beroperasi pada qubit fisik. Sebuah pulsa drive pada qubit :math:` a ` tidak akan memberlakukan operasi logis yang sama pada keadaan qubit :math:` b `-dengan kata lain, kalibrasi gerbang tidak dapat dipertukarkan di seluruh qubit. Hal ini berbeda dengan tingkat sirkuit, di mana sebuah gerbang ` ` X ` ` didefinisikan independen dari operan qubit nya.

Halaman ini menunjukkan Anda bagaimana untuk menambahkan kalibrasi di sirkuit Anda.

** Catatan:** Untuk melaksanakan sebuah program dengan gerbang pulsa, backend harus diaktifkan dengan OpenPulse. Anda dapat memeriksa melalui ` `backend.konfigurasi ().open_pulse ` ` `, akan ` ` True ` ` ketika OpenPulse diaktifkan. Jika diaktifkan dan fitur gerbang pulsa tidak diaktifkan, anda dapat ` schedule <07_pulse_scheduler.ipynb> ` __ rangkaian masukan anda.

Bangun sirkuit Anda¶

Mari kita mulai dengan contoh yang sangat sederhana, sebuah sirkuit Bell state.

[1]:

from qiskit import QuantumCircuit

circ = QuantumCircuit(2, 2)
circ.h(0)
circ.cx(0, 1)
circ.measure(0, 0)
circ.measure(1, 1)

circ.draw('mpl')

[1]:

Bangun kalibrasi Anda¶

Sekarang kita memiliki sirkuit kita, mari kita mendefinisikan kalibrasi untuk gerbang Hadamard pada qubit 0.

Dalam praktiknya, bentuk pulsa dan parameter-parameternya akan dioptimalkan melalui serangkaian eksperimen Rabi (lihat ` Qiskit Textbook <https://qiskit.org/textbook/ch-quantum-hardware/calibrating-qubits-openpulse.html>` __ untuk mempelajari lebih jauh). Untuk demonstrasi ini, Hadamard kita akan menjadi pulsa Gaussian. Kita akan * play * pulsa kita pada * drive * saluran qubit 0.

Jangan terlalu khawatir dengan detail dari membangun kalibrasinya; Anda dapat mempelajari semua hal ini pada halaman berikut: ` membangun jadwal pulsa <06_building_pulse_schedules.ipynb> ` __.

[2]:

from qiskit import pulse
from qiskit.pulse.library import Gaussian
from qiskit.test.mock import FakeValencia

backend = FakeValencia()

with pulse.build(backend, name='hadamard') as h_q0:
    pulse.play(Gaussian(duration=128, amp=0.1, sigma=16), pulse.drive_channel(0))

Mari kita gambar jadwal baru untuk melihat apa yang telah kita bangun.

[3]:

h_q0.draw()

[3]:

Hubungkan kalibrasi Anda ke sirkuit Anda¶

Semua yang tersisa adalah untuk menyelesaikan registrasi. Metode sirkuit ` ` add_calibration ` ` ` membutuhkan informasi tentang gerbang dan referensi untuk jadwal untuk menyelesaikan pemetaan:

QuantumCircuit.add_calibration(gate, qubits, schedule, parameters)

` ` gate ` ` dapat berupa sebuah ` `circuit.Gate ` ` objek atau nama dari gerbang. Biasanya, Anda akan membutuhkan jadwal yang berbeda untuk setiap set unik dari ` ` qubits ` ` dan ` ` ` parameter ` `. Karena gerbang Hadamard tidak memiliki parameter apapun, kita tidak harus memasok apapun.

[4]:

circ.add_calibration('h', [0], h_q0)

Terakhir, catatan bahwa transpiler akan menghormati kalibrasi Anda. Gunakan itu sebagaimana biasanya. (contoh kami terlalu sederhana untuk transpiler untuk dioptimasikan, sehingga keluaran adalah sama).

[5]:

from qiskit import transpile
from qiskit.test.mock import FakeAlmaden

backend = FakeAlmaden()

circ = transpile(circ, backend)

print(backend.configuration().basis_gates)
circ.draw('mpl', idle_wires=False)

['id', 'u1', 'u2', 'u3', 'cx']

[5]:

Perhatikan bahwa ` ` h ` ` bukan merupakan sebuah gerbang dasar untuk backend tiruan ` ` FakeAlmaden ` `. Karena kami telah menambahkan kalibrasi untuk itu, transpiler akan memperlakukan gerbang kami sebagai gerbang basis; * tapi hanya pada kubit yang telah didefinisikan *. Sebuah Hadamard diterapkan pada qubit yang berbeda akan dibongkar untuk gerbang basis.

Itu saja!

Gerbang khusus¶

Kami akan menunjukkan secara singkat proses yang sama untuk nonstandar, sepenuhnya gerbang khusus. Demonstrasi ini mencakup sebuah gerbang dengan parameter.

[6]:

from qiskit import QuantumCircuit
from qiskit.circuit import Gate

circ = QuantumCircuit(1, 1)
custom_gate = Gate('my_custom_gate', 1, [3.14, 1])
# 3.14 is an arbitrary parameter for demonstration
circ.append(custom_gate, [0])
circ.measure(0, 0)

circ.draw('mpl')

[6]:

[7]:

with pulse.build(backend, name='custom') as my_schedule:
    pulse.play(Gaussian(duration=64, amp=0.2, sigma=8), pulse.drive_channel(0))

circ.add_calibration('my_custom_gate', [0], my_schedule, [3.14, 1])
# Alternatively: circ.add_calibration(custom_gate, [0], my_schedule)

Jika kita menggunakan ` ` Gate ` `variable instansi ` ` custom_gate ` ` untuk menambahkan kalibrasi, parameter berasal dari instansi tersebut. Ingat bahwa urutan parameter bermakna.

[8]:

circ = transpile(circ, backend)
circ.draw('mpl', idle_wires=False)

[8]:

Biasanya, jika kita mencoba untuk transpile ` ` circ ` `, kita akan mendapatkan kesalahan. Tidak ada definisi fungsional yang disediakan untuk ` ` "my_custom_gate" ` `, sehingga transpiler tidak dapat membukanya ke gerbang basis set dari perangkat target. Kita dapat menunjukkan ini dengan mencoba menambahkan ` ` "my_custom_gate" ` ` ` ke qubit lain yang belum dikalibrasi.

[9]:

circ = QuantumCircuit(2, 2)
circ.append(custom_gate, [1])


from qiskit import QiskitError
try:
    circ = transpile(circ, backend)
except QiskitError as e:
    print(e)

"Cannot unroll the circuit to the given basis, ['id', 'u1', 'u2', 'u3', 'cx']. Instruction my_custom_gate not found in equivalence library and no rule found to expand."
