QAOA Runtime¶

Qiskit runtime என்பது ஒரு எக்ஸிகியூஷன் மாடலாகும், இது ஒரு முழு நிரலையும் பின்தளத்தில் இயக்க அனுமதிக்கிறது. QAOA அல்காரிதத்தை Qiskit runtime எவ்வாறு இயக்குவது என்பதை இங்கே விவாதிக்கிறோம். QAOA runtime -ன் இந்த முதல் பதிப்பில் கிடைக்கும் பல அம்சங்களைப் பற்றி நாம் விவாதிப்போம்.

[1]:

from qiskit import IBMQ

IBMQ.load_account()

provider = IBMQ.get_provider(hub="ibm-q", group="open", project="main")

பூர்வாங்கங்கள்¶

முதலில், நிரலை ஏற்றி, அது என்ன உள்ளீடுகளை எதிர்பார்க்கிறது என்பதை அறிய அதன் ஆர்கியுமென்ட்களை ஆய்வு செய்கிறோம். நாம் என்ன திரும்பப் பெறப்போகிறோம் என்பதைப் புரிந்துகொள்வதற்காக ரிட்டர்ன் வகைகளையும் நாம் ஆராய்வோம். QAOA திட்டத்தை நாம் ப்ரொவைடரிடமிருந்து பின்வருமாறு பெறுகிறோம்.

[2]:

program_id = "qaoa"
qaoa_program = provider.runtime.program(program_id)

இங்கே, நாம் இப்போது விரிவுபடுத்தும் வெவ்வேறு உள்ளீட்டு வாதங்களைக் காண்கிறோம்.

  • operator என்பது மிக முக்கியமான ஆர்கியுமென்ட் ஆகும், ஏனெனில் இது நாம் குறைக்க விரும்பும் காஸ்ட் ஆபரேட்டரைக் குறிக்கிறது. QAOA க்கு இது PauliSumOp -க்கு ஒத்திருக்கிறது. ஒவ்வொரு முறையும் நாம் குறைக்க விரும்பும் சிக்கலில் இரண்டு முடிவு மாறிகளுக்கு இடையே இருபடிச் சொற்கூறு இருக்கும் போது, ​​``PauliSumOp`` இல் தொடர்புடைய ZZ` சொற்கூறு இருக்கும். எடுத்துக்காட்டாக, ஆபரேட்டர் ``PauliSumOp.from_list([("ZIZ", 1), ("IZZ", -1), ("ZZI", 1)]) ஒரு முக்கோணத்தில் ஒரு Maxcut சிக்கலுடன் ஒத்திருக்கலாம், அங்கு ஒரு விளிம்பில் எதிர்மறை எடை இருக்கும்.

  • optimizer என்பது க்ளோஸ்ட்-லூப் கிளாசிக்கல் ஆப்டிமைசேஷனில் பயன்படுத்தப்படும் ஆப்டிமைசரைக் குறிக்கிறது. வன்பொருளின் இரைச்சலைச் சமாளிக்க இது SPSAக்கு இயல்புநிலையாக இருக்கும்.

  • reps இந்த முழு எண் QAOA அடுக்குகளின் எண்ணிக்கை.

  • initial_point என்பது \(\gamma\) மற்றும் \(\beta\) க்கான ஆரம்ப மதிப்புகள். இது பின்வரும் வரிசையில் கொடுக்கப்பட்ட பட்டியல் \([\gamma_1, \beta_1, \gamma_2, \beta_2, ...]\). எனவே இந்தப் பட்டியலின் நீளம் QAOA ஆழத்தை விட இரு மடங்கு இருக்க வேண்டும் (அதாவது reps அளவுரு).

  • shots என்பது ஒவ்வொரு சர்க்யூட்டுக்கும் சேகரிக்க வேண்டிய ஷாட்களின் எண்ணிக்கை.

  • alpha என்பது CVaR optimization [1] இன் \(\alpha\) ஆகும். \(\alpha\) என்பது ஒன்றை விடச் சிறியதாக இருந்தால், சிறந்த \(\alpha\) ஷாட்களின் பின்னம் மட்டுமே தேர்வுமுறையில் தக்கவைக்கப்படும், அதாவது இது CVaRExpectation(alpha, PauliExpectation()) ஐப் பயன்படுத்துவதற்கு ஒத்திருக்கிறது, qiskit.opflow இல் காணலாம்.

  • measurement_error_mitigation வாசிப்புப் பிழைத் தணிப்பைப் பயன்படுத்தலாமா வேண்டாமா என்பதைத் தீர்மானிக்கிறது. True எனில், இதைப் பயன்படுத்திய அல்காரிதம் TensoredMeasFitter ஐப் பயன்படுத்தும்.

  • use_swap_strategies என்பது True எனில், QAOA நிரல், QAOA சர்கியூட்டை அர்ப்பணிப்பு ஸ்வாப் உத்திகளுடன்மாற்றி அமைக்க QAOA இல் உள்ள ZZ ஆபரேட்டர்களின் பரிமாற்றத் தன்மையையும், நிரல் இயங்கும் பின்தளத்தின் இணைப்பு வரைபடத்தையும் கணக்கில் எடுத்துக் கொள்ளும். இந்த விருப்பம் தவறு எனில், QAOA நிரல் மேம்படுத்தல் நிலை 1 உடன் நிலையான டிரான்ஸ்பைலருக்கு இயல்புநிலையாக இருக்கும். இந்த தேர்வுமுறை நிலையை optimization_level உள்ளீட்டு விருப்பத்தைப் பயன்படுத்தி தேர்வு செய்யலாம்.

  • use_pulse_efficiency என்பது True என அமைக்கப்பட்டால், ஒவ்வொரு QAOA மறு செய்கையிலும் ஒரு பல்ஸ் எஃபிஷியன்ட் டிரான்ஸ்பிலேஷன் இயக்கப்படும் [2]. இது இரண்டு-கியூபிட் தொகுதிகளைச் சேகரித்து, சர்கியூட்டின் RZXGate பிரதிநிதித்துவத்தைப் பெற Cartan’s KAK சிதைவைப் பயன்படுத்துகிறது. அளவிடப்பட்ட குறுக்கு-அதிர்வு பல்ஸ்கள் பின்னர் பயன்படுத்தப்படுகின்றன. மேலும் விவரங்கள் பின்னிணைப்பு A இல் கொடுக்கப்பட்டுள்ளன.

  • use_initial_mapping என்பது True எனில், சாதனத்தின் இணைப்பு வரைபடத்தைச் சிறப்பாகப் பொருத்துவதற்கு, காஸ்ட் ஆபரேட்டரில் உள்ள Pauli ஆபரேட்டர்களை வரிசைப்படுத்த ஒரு ஹூரிஸ்டிக் அல்காரிதம் இயக்கப்படும். use_swap_strategies விருப்பம் True என அமைக்கப்பட்டால் மட்டுமே இந்த வரிசைமாற்றம் பயன்படுத்தப்படும், மேலும் முழுமையாக இணைக்கப்படாத சிக்கல்களுக்கு மட்டுமே இது பொருந்தும்.

  • optimization_level இது ஸ்வாப் உத்திகள் பயன்படுத்தப்படாவிட்டால் வரும் Qiskit டிரான்ஸ்பைலரின் மேம்படுத்தல் நிலை. இதன் இயல்புநிலை மதிப்பு 1.

QAOA இயக்க நேர நிரலை இயக்குகையில்¶

QAOA இயக்க நேர நிரலை எவ்வாறு பயன்படுத்துவது என்பதை இப்போது ஐந்து கியூபிட் உதாரணத்துடன் காட்டுகிறோம்.

கிளவுடில் நிரலை எவ்வாறு நேரடியாக அழைப்பது என்பதை இங்கே காண்பிக்கிறோம். இருப்பினும், Qiskit ஆப்டிமைசேஷன் QAOAClient ஐ வழங்குகிறது, இது Qiskit ஆப்டிமைசேஷன் பணிப்பாய்வுக்கு தடையின்றி பொருந்துகிறது மற்றும் உங்கள் லோக்கல் QAOA அல்காரிதத்தின் நேரடி மாற்றாகப் பயன்படுத்தப்படலாம். இந்த QAOAClient இந்த டுடோரியலில் கீழே விவாதிக்கப்பட்டுள்ளது.

[4]:

import numpy as np

from qiskit.tools import job_monitor
from qiskit.opflow import PauliSumOp, Z, I
from qiskit.algorithms.optimizers import SPSA

# Define the cost operator to run.
op = (
    (Z ^ Z ^ I ^ I ^ I)
    - (I ^ I ^ Z ^ Z ^ I)
    + (I ^ I ^ Z ^ I ^ Z)
    - (Z ^ I ^ Z ^ I ^ I)
    - (I ^ Z ^ Z ^ I ^ I)
    + (I ^ Z ^ I ^ Z ^ I)
    + (I ^ I ^ I ^ Z ^ Z)
)

# SPSA helps deal with noisy environments.
optimizer = SPSA(maxiter=100)

# We will run a depth two QAOA.
reps = 2

# The initial point for the optimization, chosen at random.
initial_point = np.random.random(2 * reps)

# The backend that will run the programm.
options = {"backend_name": "ibmq_qasm_simulator"}

# The inputs of the program as described above.
runtime_inputs = {
    "operator": op,
    "reps": reps,
    "optimizer": optimizer,
    "initial_point": initial_point,
    "shots": 2**13,
    # Set to True when running on real backends to reduce circuit
    # depth by leveraging swap strategies. If False the
    # given optimization_level (default is 1) will be used.
    "use_swap_strategies": False,
    # Set to True when optimizing sparse problems.
    "use_initial_mapping": False,
    # Set to true when using echoed-cross-resonance hardware.
    "use_pulse_efficient": False,
}

ஒவ்வொரு மறு செய்கையிலும் வன்பொருளில் அளவிடப்பட்ட ஆற்றல் மற்றும் திரும்பிய ஐகென்ஸ்டேட் ஆகியவை குறிப்பான ஆர்வமாக உள்ளன. இந்த எடுத்துக்காட்டில், நாங்கள் குறைத்த காஸ்ட் ஆபரேட்டரின் ஆற்றல், MaxCut சிக்கலின் நிகழ்வுக்கு ஒத்திருக்கிறது என்று கருதுகிறோம்.

from collections import defaultdict


def op_adj_mat(op: PauliSumOp) -> np.array:
    """Extract the adjacency matrix from the op."""
    adj_mat = np.zeros((op.num_qubits, op.num_qubits))
    for pauli, coeff in op.primitive.to_list():
        idx = tuple([i for i, c in enumerate(pauli[::-1]) if c == "Z"])  # index of Z
        adj_mat[idx[0], idx[1]], adj_mat[idx[1], idx[0]] = np.real(coeff), np.real(coeff)

    return adj_mat


def get_cost(bit_str: str, adj_mat: np.array) -> float:
    """Return the cut value of the bit string."""
    n, x = len(bit_str), [int(bit) for bit in bit_str[::-1]]
    cost = 0
    for i in range(n):
        for j in range(n):
            cost += adj_mat[i, j] * x[i] * (1 - x[j])

    return cost


def get_cut_distribution(result) -> dict:
    """Extract the cut distribution from the result.

    Returns:
        A dict of cut value: probability.
    """

    adj_mat = op_adj_mat(PauliSumOp.from_list(result["inputs"]["operator"]))

    state_results = []
    for bit_str, amp in result["eigenstate"].items():
        state_results.append((bit_str, get_cost(bit_str, adj_mat), amp**2 * 100))

    vals = defaultdict(int)

    for res in state_results:
        vals[res[1]] += res[2]

    return dict(vals)

பின்னிணைப்பு A: பல்ஸ்-எஃபிஷியன்ட் இடமாற்றம்¶

QAOA இயக்க நேர நிரல் ஒரு பல்ஸ்-எஃபிஷியன்ட் டிரான்ஸ்பிலேஷனைப் பயன்படுத்துகிறது, இது ஒற்றை-கியூபிட் பல்ஸ்களின் எண்ணிக்கையைக் குறைக்கிறது மற்றும் IBM Quantum Systems -ன் எதிரொலி-குறுக்கு-அதிர்வு கேட்களுக்குப் பொருந்தக்கூடிய குறுக்கு-அதிர்வு அளவிடுதல் முறையைப் பயன்படுத்துகிறது. முழுமைக்காக, செயல்படுத்தப்பட்ட பாஸ் மேனேஜரைக் காட்டுகிறோம்.

KAK சிதைவிலிருந்து பெறப்பட்ட RZXGates -க்கு இணைக்கப்பட்ட அளவிடப்பட்ட குறுக்கு-அதிர்வு கேட்களுக்கு பல்ஸ்-செயல்படுத்தப்பட்ட பின்தளத்தின் புரிதல் தேவை. இந்த அளவிடப்பட்ட கேட்கள் அளவீடு செய்யப்பட்ட CNOT கேட்களின் அட்டவணையை அடிப்படையாகக் கொண்டவை.

inst_map = backend.defaults().instruction_schedule_map
channel_map = backend.configuration().qubit_channel_mapping
rzx_basis = ["rzx", "rz", "x", "sx"]

pulse_efficient = PassManager(
    [
        # Consolidate consecutive two-qubit operations.
        Collect2qBlocks(),
        ConsolidateBlocks(basis_gates=["rz", "sx", "x", "rxx"]),
        # Rewrite circuit in terms of Weyl-decomposed echoed RZX gates.
        EchoRZXWeylDecomposition(backend.defaults().instruction_schedule_map),
        # Attach scaled CR pulse schedules to the RZX gates.
        RZXCalibrationBuilderNoEcho(
            instruction_schedule_map=inst_map, qubit_channel_mapping=channel_map
        ),
        # Simplify single-qubit gates.
        UnrollCustomDefinitions(std_eqlib, rzx_basis),
        BasisTranslator(std_eqlib, rzx_basis),
        Optimize1qGatesDecomposition(rzx_basis),
    ]
)
