Runtime QAOA¶

O Qiskit runtime é um modelo de execução que nos permite executar um programa inteiro no backend. Aqui, discutimos como executar o algoritmo QAOA do Qiskit runtime. Discutiremos vários dos recursos que esta primeira versão do QAOA Runtime disponibiliza.

[1]:

from qiskit import IBMQ

IBMQ.load_account()

provider = IBMQ.get_provider(hub="ibm-q", group="open", project="main")

Preliminares¶

Primeiro, carregamos o programa e inspecionamos seus argumentos para saber quais entradas ele está esperando. Também investigamos os tipos de devolução para entender o que nos será devolvido. Obtemos o programa QAOA do fornecedor da seguinte forma.

[2]:

program_id = "qaoa"
qaoa_program = provider.runtime.program(program_id)

Os argumentos para o runtime são obtidos da seguinte maneira.

[3]:

print(f"Program name: {qaoa_program.name}, Program id: {qaoa_program.program_id}")
print(qaoa_program.parameters())

Aqui, vemos os diferentes argumentos de entrada que acabamos de elaborar.

  • operador é de longe o argumento mais importante, pois representa o operador custo que queremos minimizar. Para QAOA isso corresponde a um PauliSumOp. Cada vez que houver um termo quadrático entre duas variáveis de decisão no problema que desejamos minimizar, haverá um termo ZZ correspondente no PauliSumOp. Por exemplo, o operador PauliSumOp.from_list([("ZIZ", 1), ("IZZ", -1), ("ZZI", 1)]) pode corresponder a um problema de Maxcut em um triângulo onde uma aresta tem um peso negativo.

  • optimizer representa o otimizador usado na otimização clássica de malha fechada. Este padrão será o SPSA para lidar com o ruído do hardware.

  • reps esse número inteiro é o número de camadas QAOA.

  • initial_point são os valores iniciais para \(\gamma\) e \(\beta\). Esta é uma lista dada na seguinte ordem \([\gamma_1, \beta_1, \gamma_2, \beta_2, ...]\). O comprimento desta lista deve, portanto, ser duas vezes a profundidade QAOA (i.e. o parâmetro reps).

  • tiros é o número de shots a serem coletados para cada circuito.

  • alpha é o \(\alpha\) da otimização CVaR [1]. Se \(\alpha\) for menor que um, então apenas a melhor fração \(\alpha\) de shots será retida na otimização, i.e. isso corresponde ao uso de CVaRExpectation(alpha, PauliExpectation()) que pode ser encontrado em qiskit.opflow.

  • measurement_error_mitigation determina ou não o uso da mitigação de erros de leitura. Se for True, o algoritmo usado empregará o TensoredMeasFitter.

  • Se use_swap_strategies for True o programa QAOA irá transpilar o circuito QAOA com estratégias de swap dedicadas que levam em conta a natureza comutativa dos operadores ZZ em QAOA e o mapa de acoplamento do backend no qual o programa será executado. Se esta opção for False, o programa QAOA assumirá como padrão o transpilador padrão com nível de otimização 1. Esse nível de otimização pode ser escolhido usando a opção de entrada optimization_level.

  • Se use_pulse_efficient for definido como True, uma transpilação com eficiência de pulso será executada em cada iteração QAOA [2]. Isso coleta blocos de dois qubits e aplica a decomposição KAK de Cartan para obter uma representação RZXGate do circuito. Pulsos de ressonância cruzada escalonados são então usados. Mais detalhes são fornecidos no Apêndice A abaixo.

  • Se use_initial_mapping for True então um algoritmo heurístico será executado para permutar os operadores Pauli no operador de custo para melhor ajustar o mapa de acoplamento do dispositivo. Esta permutação só é aplicada se a opção use_swap_strategies estiver configurada para True e só faz sentido para problemas não totalmente conectados.

  • optimization_level Este é o nível de otimização do transpilador Qiskit se as estratégias de troca não forem usadas. O valor padrão é 1.

Referências¶

[1] P. Kl. Barkoutsos, G. Nannicini, A. Robert, I. Tavernelli, and S. Woerner, Improving Variational Quantum Optimization using CVaR, Quantum ** 4 * *, 256 (2020).

[2] N. Earnest, C. Tornow, and D. J. Egger, Pulse-efficient circuit transpilation for quantum applications on cross-resonance-based hardware, Phys. Rev. Research 3, 043088 (2021).

Executando o programa QAOA runtime¶

Agora mostramos como usar o programa QAOA runtime com um exemplo de cinco qubits.

Aqui, mostramos como chamar diretamente o programa na nuvem. No entanto, o Qiskit Optimization fornece o QAOAClient que se encaixa perfeitamente no fluxo de trabalho do Qiskit Optimization e pode ser usado como um substituto direto do seu algoritmo local QAOA. Este QAOAClient é discutido abaixo neste tutorial.

[4]:

import numpy as np

from qiskit.tools import job_monitor
from qiskit.opflow import PauliSumOp, Z, I
from qiskit.algorithms.optimizers import SPSA

# Define the cost operator to run.
op = (
    (Z ^ Z ^ I ^ I ^ I)
    - (I ^ I ^ Z ^ Z ^ I)
    + (I ^ I ^ Z ^ I ^ Z)
    - (Z ^ I ^ Z ^ I ^ I)
    - (I ^ Z ^ Z ^ I ^ I)
    + (I ^ Z ^ I ^ Z ^ I)
    + (I ^ I ^ I ^ Z ^ Z)
)

# SPSA helps deal with noisy environments.
optimizer = SPSA(maxiter=100)

# We will run a depth two QAOA.
reps = 2

# The initial point for the optimization, chosen at random.
initial_point = np.random.random(2 * reps)

# The backend that will run the programm.
options = {"backend_name": "ibmq_qasm_simulator"}

# The inputs of the program as described above.
runtime_inputs = {
    "operator": op,
    "reps": reps,
    "optimizer": optimizer,
    "initial_point": initial_point,
    "shots": 2**13,
    # Set to True when running on real backends to reduce circuit
    # depth by leveraging swap strategies. If False the
    # given optimization_level (default is 1) will be used.
    "use_swap_strategies": False,
    # Set to True when optimizing sparse problems.
    "use_initial_mapping": False,
    # Set to true when using echoed-cross-resonance hardware.
    "use_pulse_efficient": False,
}

Agora, executamos o programa usando o provedor.

[5]:

job = provider.runtime.run(
    program_id=program_id,
    options=options,
    inputs=runtime_inputs,
)

Uma vez que o trabalho tenha sido executado com sucesso, podemos recuperar o resultado do trabalho. O objeto de resultado contém informações sobre a otimização. Para auto consistência, ele também contém as entradas para o programa QAOA que são acessíveis através de result["inputs"]. O valor em "inputs" é um dicionário com as chaves descritas acima.

[8]:

result = job.result()

De particular interesse é a energia que foi medida no hardware em cada iteração, bem como o autoestado retornado. Para este exemplo, assumimos que a energia do operador de custo que minimizamos corresponde a uma instância de um problema MaxCut.

[9]:

from collections import defaultdict


def op_adj_mat(op: PauliSumOp) -> np.array:
    """Extract the adjacency matrix from the op."""
    adj_mat = np.zeros((op.num_qubits, op.num_qubits))
    for pauli, coeff in op.primitive.to_list():
        idx = tuple([i for i, c in enumerate(pauli[::-1]) if c == "Z"])  # index of Z
        adj_mat[idx[0], idx[1]], adj_mat[idx[1], idx[0]] = np.real(coeff), np.real(coeff)

    return adj_mat


def get_cost(bit_str: str, adj_mat: np.array) -> float:
    """Return the cut value of the bit string."""
    n, x = len(bit_str), [int(bit) for bit in bit_str[::-1]]
    cost = 0
    for i in range(n):
        for j in range(n):
            cost += adj_mat[i, j] * x[i] * (1 - x[j])

    return cost


def get_cut_distribution(result) -> dict:
    """Extract the cut distribution from the result.

    Returns:
        A dict of cut value: probability.
    """

    adj_mat = op_adj_mat(PauliSumOp.from_list(result["inputs"]["operator"]))

    state_results = []
    for bit_str, amp in result["eigenstate"].items():
        state_results.append((bit_str, get_cost(bit_str, adj_mat), amp**2 * 100))

    vals = defaultdict(int)

    for res in state_results:
        vals[res[1]] += res[2]

    return dict(vals)

Otimização Qiskit¶

A funcionalidade discutida acima é implementada na otimização Qiskit onde o QAOA-runtime é acessível através do módulo runtime. Agora mostramos como usar este módulo com o QAOAClient.

[11]:

from qiskit_optimization.runtime import QAOAClient
from qiskit_optimization.algorithms import MinimumEigenOptimizer
from qiskit_optimization import QuadraticProgram

qubo.binary_var("x")
qubo.binary_var("y")
qubo.binary_var("z")

Apêndice A: Transpilação com eficiência de pulso¶

O programa QAOA runtime pode alavancar uma transpilação com eficiência de pulso que minimiza o número de pulsos de qubit único e faz uso da metodologia de escala de ressonância cruzada aplicável às portas de ressonância cruzada ecoada do IBM Quantum Systems. Para completar, mostramos aqui o gerenciador de passes implementado.

[15]:

from qiskit.transpiler import PassManager
from qiskit.circuit.library.standard_gates.equivalence_library import (
    StandardEquivalenceLibrary as std_eqlib,
)
from qiskit.transpiler.passes import (
    Collect2qBlocks,
    ConsolidateBlocks,
    UnrollCustomDefinitions,
    BasisTranslator,
    Optimize1qGatesDecomposition,
)
from qiskit.transpiler.passes.calibration.builders import RZXCalibrationBuilderNoEcho
from qiskit.transpiler.passes.optimization.echo_rzx_weyl_decomposition import (
    EchoRZXWeylDecomposition,
)

from qiskit.test.mock import FakeBelem

[16]:

backend = FakeBelem()

O passe eficiente de pulso é implementado usando passos transpiladores do qiskit. Requer conhecimento do backend ativado por pulso para portas de ressonância cruzada escaladas anexadas aos RZXGates obtidos da decomposição KAK. Essas portas escalonadas são baseadas nos cronogramas das portas CNOT calibradas.

[17]:

inst_map = backend.defaults().instruction_schedule_map
channel_map = backend.configuration().qubit_channel_mapping
rzx_basis = ["rzx", "rz", "x", "sx"]

pulse_efficient = PassManager(
    [
        # Consolidate consecutive two-qubit operations.
        Collect2qBlocks(),
        ConsolidateBlocks(basis_gates=["rz", "sx", "x", "rxx"]),
        # Rewrite circuit in terms of Weyl-decomposed echoed RZX gates.
        EchoRZXWeylDecomposition(backend.defaults().instruction_schedule_map),
        # Attach scaled CR pulse schedules to the RZX gates.
        RZXCalibrationBuilderNoEcho(
            instruction_schedule_map=inst_map, qubit_channel_mapping=channel_map
        ),
        # Simplify single-qubit gates.
        UnrollCustomDefinitions(std_eqlib, rzx_basis),
        BasisTranslator(std_eqlib, rzx_basis),
        Optimize1qGatesDecomposition(rzx_basis),
    ]
)
