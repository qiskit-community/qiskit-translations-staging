மறுபடியும் குறியீடுகள்¶

முன்னுரை¶

குவாண்டம் பிழை திருத்தத்தில் நாம் பல சத்தமில்லாத குவிட்களை எடுத்துக்கொள்கிறோம் (அவற்றை * physical qubits* என்று அழைக்கிறோம்) மற்றும் குறைந்த எண்ணிக்கையிலான logical qubits சேமிக்க அவற்றைப் பயன்படுத்துகிறோம். பிழை திருத்தும் செயல்முறை சத்தத்தின் விளைவுகளை கண்டறிந்து சரிசெய்ய வடிவமைக்கப்பட்டுள்ளது. இது தர்க்கரீதியான வினாடிகளை அவை கட்டியெழுப்பிய தீகங்களைக் காட்டிலும் மிகக் குறைவான சத்தமாகவும் நம்பகத்தன்மையுடனும் ஆக்குகிறது.

மறுபடியும் குறியீடு குவாண்டம் பிழை திருத்தம் செய்வதற்கான எளிய எடுத்துக்காட்டு, இதில் ஒரு தருக்க bit ஒரு தருக்க quibit ஐ விட சேமிக்கப்படுகிறது. மறுபடியும் குறியீட்டின் கொடுக்கப்பட்ட நிகழ்வு இரண்டு எண்களால் வரையறுக்கப்படுகிறது, இதை நாம் அழைக்கிறோம் \(d\) மற்றும் \(T\).

\(d\) அளவுரு எத்தனை உடல் வினாக்கள் பயன்படுத்தப்படுகின்றன என்பதை தீர்மானிக்கிறது. தருக்க பிட் பற்றிய உண்மையான தகவல்கள் இதில் சேமிக்கப்பட்டுள்ளன: \(d\) குவிட்ஸ் (இதை நாங்கள் code qubits என்று அழைக்கிறோம்). குறியாக்கம் மிகவும் எளிமையான முறையில் செய்யப்படுகிறது: 0 ஐ குறியாக்க, இந்த வினாடிகள் அனைத்தும் மாநிலத்தில் அமைக்கப்பட்டுள்ளன \(| 0 \rangle\), அவை அமைக்கப்பட்ட 1 ஐ குறியாக்க \(| 1 \rangle\). மதிப்பைப் படிக்க, நீங்கள் எந்த குவிட்டையும் பார்க்கலாம். ஒற்றை குவிட் பிழைகள் இருந்து பாதுகாக்கும் வகையில் இதைப் படிக்க, நீங்கள் அனைத்து வினாடிகளையும் படித்து பெரும்பான்மை வாக்குகளைப் பெறலாம்.

பிழைகளை சரிசெய்ய உதவும் பயனுள்ள தகவல்களைப் பிரித்தெடுக்க முடியும் என்பது வாசிப்பில் மட்டுமல்ல. தருக்க பிட் செயலற்ற நிலையில் இருக்கும்போது அல்லது கணக்கீட்டில் ஈடுபடும்போது கூட தகவல்களைப் பிரித்தெடுக்க முடியும்.

இது குவாண்டம் பிழை திருத்தம் செய்வதற்கான ஒரு பயிற்சி என்பதால், இதை நாங்கள் தர்க்கரீதியான * qubits * வேலை செய்யும் வகையில் செய்வோம். குறிப்பாக, பிழைகள் பற்றிய தகவல்களைப் பிரித்தெடுக்கும் எங்கள் முறை சேமிக்கப்பட்ட தர்க்கரீதியான தகவல்களைப் பற்றிய எந்த தகவலையும் பிரித்தெடுக்கக்கூடாது. குவாண்டம் வழக்கில், இது தேவைப்படுகிறது, இதனால் சேமிக்கப்பட்ட குவிட்களின் சூப்பர் பொசிஷன் நிலைகளை நாம் தொந்தரவு செய்யக்கூடாது.

\(T\) syndrome measurements. மறுபடியும் குறியீட்டைப் பொறுத்தவரை, இவை எல்லா குறியீடு வினாக்களும் ஒரே நிலையில் இருக்க வேண்டும் என்ற உண்மையை அடிப்படையாகக் கொண்டவை (அனைத்தும் \(| 0 \rangle\) அல்லது அனைத்தும் \(| 1 \rangle\)). இதிலிருந்து எந்தவொரு புறப்பாடும் பிழையின் கையொப்பமாகும். குறிப்பாக, எங்கள் \(d\) குறியீடு ஒரு வரியுடன் அமர்ந்திருப்பதை நாங்கள் கற்பனை செய்கிறோம். ஒவ்வொரு ஜோடி அண்டை குறியீடு வினாடிகளிலும் ஒரு அளவீடு செய்வோம். அவற்றின் மதிப்புகள் என்ன என்பது பற்றிய எந்த தகவலையும் பிரித்தெடுக்காமல், அவை ஒரே மாதிரியானவை அல்லது வேறுபட்டவை என்பதை இது நமக்குத் தெரிவிக்கும்.

இந்த அளவீடுகளை செயல்படுத்துவதற்கு இது தேவைப்படுகிறது \(d-1\) கூடுதல் குவிட்கள், அவை மீண்டும் குறியீட்டு குறியீட்டிற்கு link qubits என்று அழைக்கிறோம். ஜோடி குறியீட்டு குவிட்டுகளுக்கும் அதனுடன் தொடர்புடைய இணைப்பு குவிட்டிற்கும் இடையில் இரண்டு CNOT களைச் செய்வதன் மூலம், தேவையான தகவல்கள் (மற்றும் தேவையான தகவல்கள் மட்டுமே) இணைப்பு குவிட்டில் வைக்கப்பட்டு பின்னர் அளவிடப்படலாம்.

[1]:

from qiskit import *
cq = QuantumRegister(2,'code_qubit')
lq = QuantumRegister(1,'link_qubit')
qc = QuantumCircuit(cq,lq)
qc.cx(cq[0],lq[0])
qc.cx(cq[1],lq[0])
print(qc)

மீண்டும் ஒரு குறியீட்டை உருவாக்குதல்¶

repetition_code வகுப்பு கொடுக்கப்பட்ட மதிப்புகளுக்கு ஒரு குறியீட்டை உருவாக்குகிறது \(d\) மற்றும் \(T\).

[3]:

d = 3
T = 2
code = RepetitionCode(d,T)

இதன் மூலம் குறியீட்டின் பல்வேறு பண்புகளை நாம் ஆராயலாம், அதாவது குறியீட்டிற்கு பயன்படுத்தப்படும் குவிட் பதிவேடுகளின் பெயர்கள் மற்றும் துணை குவிட்கள்.

code.qubit_registers

இந்த பதிவேடுகள்``repetition_code``` பொருளின் பண்புகளாகும்.

அளவீட்டு சுற்றுகள் மற்றும் தர்க்கரீதியான X வாயில்களையும் நீங்களே செயல்படுத்தலாம். எடுத்துக்காட்டாக, நோய்க்குறி அளவீட்டு சுற்றுகள் இல்லாத குறியீட்டை அமைப்போம்.

மறுபடியும் குறியீட்டை இயக்குகிறது¶

குறியீடு பொருள் குறியீட்டிற்கு தேவையான சுற்றுகளை உருவாக்குகிறது. பயனர் பின்னர் அவர்கள் விரும்பும் எந்தவொரு முறையையும் பயன்படுத்தி அவற்றை இயக்க முடியும், இது தொகுப்பு, பின்தளத்தில், சத்தம் மாதிரிகள் மற்றும் பலவற்றில் முழு கட்டுப்பாட்டை அனுமதிக்கிறது.

எடுத்துக்காட்டாக, இங்கே qasm_simulator இல் சத்தமின்றி அவற்றை இயக்குகிறோம்.

[12]:

circuits = code.get_circuit_list()
job = execute( circuits, Aer.get_backend('qasm_simulator') )
raw_results = {}
for log in ['0','1']:
    raw_results[log] = job.result().get_counts(log)
    print('\n========= logical',log,'=========\n')
    print(raw_results[log])


========= logical 0 =========

{'000 00 00': 1024}

========= logical 1 =========

{'111 00 00': 1024}

இங்கே வலமிருந்து இடமாக உள்ள சரங்கள் நோய்க்குறி அளவீட்டு சுற்றுகளின் வெளியீடுகளைக் குறிக்கின்றன, அதைத் தொடர்ந்து குறியீட்டு வினாடிகளின் இறுதி அளவீடு.

டிகோட் செய்ய இந்த முடிவுகளை வேறு வடிவத்தில் மீண்டும் எழுத வேண்டும். இதைச் செய்ய, குறியீடு பொருளின் process_results முறையைப் பயன்படுத்தலாம்.

[13]:

code.process_results( raw_results )

[13]:

{'0': {'0 0  00 00 00': 1024}, '1': {'1 1  00 00 00': 1024}}

இதன் விளைவாக ஒரு அகராதி அதன் விசைகள் பிட் சரங்களாக இருக்கின்றன, அவை சுற்றுகளின் வெளியீடுகளைக் குறிக்கின்றன, மேலும் மதிப்புகள் இந்த வெளியீடு நிகழ்ந்த காட்சிகளின் எண்ணிக்கையைக் குறிக்கும்.

சரங்கள் சுற்றுகளிலிருந்து நேரடி வெளியீடு அல்ல. பிழைகளை சரிசெய்ய உதவும் படிவத்தை எடுக்க அவை செயலாக்கப்பட்டுள்ளன. ஒரு குறுகிய வழிகாட்டுதல் சுற்றுப்பயணம் இங்கே.

  • தர்க்கரீதியான 0 முடிவுக்கு இடதுபுறத்தில் உள்ள 0 0, மற்றும் தர்க்கரீதியான 1 இன் இடதுபுறத்தில் உள்ள 1 1 ஆகியவை தர்க்கரீதியான வாசிப்பு ஆகும். எந்தவொரு குறியீடு குவிட்டையும் இந்த வாசிப்புக்கு பயன்படுத்தலாம், ஏனெனில் அவை (பிழைகள் இல்லாமல்) அனைத்தும் சமமாக இருக்க வேண்டும். எனவே ஒரு தன்னிச்சையாக தேர்ந்தெடுக்கப்பட்ட குறியீடு குவிட்டிற்கு இங்கே ஒரு முடிவை மட்டுமே நாம் பெற முடியும். அல்லது நம்மிடம் இருக்க முடியும் \(d\), ஒவ்வொரு குவிட்டிற்கும் ஒன்று. அதற்கு பதிலாக, வரியின் இரு முனைகளிலும் உள்ள இரண்டு குவிட்களிலிருந்து இரண்டு உள்ளன. இது டிகோடருடன் சிறப்பாக செயல்படுவதால் (இது பின்னர் பயன்படுத்துவோம்). பிழைகள் இல்லாத நிலையில், இந்த இரண்டு மதிப்புகள் எப்போதும் சமமாக இருக்கும்.

  • பின்வரும் 0000 \(d-1\) முதல் சுற்றுக்கான நோய்க்குறி அளவீடுகளின் முடிவுகள். ஒரு 0 என்பது தொடர்புடைய ஜோடி குவிட்கள் ஒரே இடத்தில் இருப்பதையும், 1 என்பதையும் குறிக்கிறது. வேறுபட்டதைக் குறிக்கிறது. \(d-1\) முடிவுகள் \(d\) குறியீடு வினாக்கள் \(d-1\) சாத்தியமான அண்டை ஜோடிகள் உள்ளன. பிழைகள் இல்லாத நிலையில், அவை அனைத்தும் 0 ஆக இருக்கும்.

  • அதைத் தொடர்ந்து வரும் `` 0000`` என்பது முதல் மற்றும் இரண்டாவது சுற்றுகளுக்கு இடையிலான நோய்க்குறி மாற்றமாகும். எனவே இது நோய்க்குறி அளவீட்டின் பிட்வைஸ் `` OR`` என்பது இரண்டாவது சுற்றில் இருந்து முதல்வர்களுடன் முடிவடைகிறது. பிழைகள் இல்லாத நிலையில், அவை அனைத்தும் `` 0`` ஆக இருக்கும்.

  • கடைசியாக சில கருத்துக்கள் தேவைப்பட்டாலும், அடுத்தடுத்த தொகுதிகள் அதே சூத்திரத்தைப் பின்பற்றுகின்றன. இது நிலையான முறையைப் பயன்படுத்தி அளவிடப்படவில்லை (இணைப்பு குவிட்டுடன்). அதற்கு பதிலாக இது அனைத்து குறியீடு வினாடிகளின் இறுதி வாசிப்பு அளவிலிருந்து கணக்கிடப்படுகிறது. மீண்டும் இது ஒரு நோய்க்குறி மாற்றமாக வழங்கப்படுகிறது, மேலும் பிழைகள் இல்லாத நிலையில் அனைத்தும் 0 ஆக இருக்கும். இது \(T+1\)-வது நோய்க்குறி அளவீடுகள் என்பதால், இது மற்றவர்களைப் போலவே செய்யப்படாததால், இது கணக்கிடப்படவில்லை \(T\) நோய்க்குறி அளவீட்டு சுற்றுகள்.

Example 1: 0 0 0110 0000 0000 என்பது ஒரு \(d=5\), \(T=2\) குறியாக்கப்பட்ட 0 உடன் மீண்டும் குறியீடு. முதல் அளவீட்டு சுற்றுக்கு முன் நடுத்தர குறியீடு குவிட் ஒரு பிழையால் புரட்டப்பட்டதாக (பெரும்பாலும்)நோய்க்குறி காட்டுகிறது. இது மீதமுள்ள சுற்றுக்கான அண்டை குறியீடு க்விட்களுடன் உடன்படவில்லை. இது முதல் சுற்றில் நோய்க்குறியால் காட்டப்படுகிறது, ஆனால் அடுத்தடுத்த சுற்றுகளுக்கான தொகுதிகள் அதைப் புகாரளிக்காது, ஏனெனில் இது இனி மாற்றத்தைக் குறிக்காது. பிற பிழைகள் இந்த நோய்க்குறியை ஏற்படுத்தியிருக்கலாம், ஆனால் அவை மிகவும் சிக்கலானதாக இருக்க வேண்டும், எனவே அவை குறைவாகவே இருக்கும்.

Example 2: 0 0 0010 0010 0000 என்பது ஒரு \(d=5\), \(T=2\) உடன் மீண்டும் குறியீடு. இங்கே நோய்க்குறி அளவீடுகளில் ஒன்று முதல் சுற்றில் இரண்டு குறியீடு வினாடிகளுக்கு இடையிலான வேறுபாட்டைப் புகாரளித்தது, இது 1 க்கு வழிவகுத்தது. அடுத்த சுற்று அதே விளைவைக் காணவில்லை, இதனால் 0 கிடைத்தது. இருப்பினும், இதே நோய்க்குறி அளவீட்டுக்கான முந்தைய முடிவுடன் இது உடன்படவில்லை என்பதால், நோய்க்குறி மாற்றங்களை நாங்கள் கண்காணிப்பதால், இந்த மாற்றம் மற்றொரு 1 இல் விளைகிறது. அடுத்தடுத்த சுற்றுகளும் எதையும் கண்டறியவில்லை, ஆனால் இது இனி ஒரு மாற்றத்தைக் குறிக்காது, எனவே அதே நிலையில் 0 விளைகிறது. முதல் 1 க்கு வழிவகுக்கும் அளவீட்டு முடிவு பிழையாக இருக்கலாம்.

Example 3: 0 1 0000 0001 0000 என்பது ஒரு \(d=5\), \(T=2\) குறியாக்கப்பட்ட 1 உடன் மீண்டும் குறியீடு. நோய்க்குறி அளவீடுகளின் இரண்டாவது சுற்றுக்கு முன் கோட்டின் முடிவில் ஒரு குறியீடு குவிட் புரட்டப்படுகிறது. இது ஒரு நோய்க்குறி அளவீடு மூலம் மட்டுமே கண்டறியப்படுகிறது, ஏனெனில் இது கோட்டின் முடிவில் உள்ளது. அதே காரணத்திற்காக, இது தர்க்கரீதியான வாசிப்புகளில் ஒன்றையும் தொந்தரவு செய்கிறது.

இந்த எல்லா எடுத்துக்காட்டுகளிலும், ஒரு பிழையானது சரம் சரியாக இரண்டு எழுத்துக்களை பிழைகள் இல்லாமல் இருக்கும் மதிப்பிலிருந்து மாற்றுவதற்கு காரணமாகிறது என்பதை நினைவில் கொள்க. உண்மையில், இது தருக்க வெளியீடு இரு முனைப்புள்ளிகளையும் கொண்டிருப்பதற்கான காரணம். இது டிகோடரால் பயன்படுத்தப்படும் ஒரு சொத்து.

சத்தத்தின் விளைவுகளைக் காண, நாங்கள் ஒரு சத்தம் மாதிரியைக் குறிப்பிட வேண்டும். எடுத்துக்காட்டாக, கேட் மற்றும் அளவீட்டு பிழைகள் கொண்ட எளிய இரைச்சல் மாதிரியை அமைப்போம்.

from qiskit.providers.aer.noise import NoiseModel
from qiskit.providers.aer.noise.errors import pauli_error, depolarizing_error

def get_noise(p_meas,p_gate):

    error_meas = pauli_error([('X',p_meas), ('I', 1 - p_meas)])
    error_gate1 = depolarizing_error(p_gate, 1)
    error_gate2 = error_gate1.tensor(error_gate1)

    noise_model = NoiseModel()
    noise_model.add_all_qubit_quantum_error(error_meas, "measure")
    noise_model.add_all_qubit_quantum_error(error_gate1, ["u1", "u2", "u3"])
    noise_model.add_all_qubit_quantum_error(error_gate2, ["cx"])

    return noise_model

மறுபடியும் குறியீட்டை டிகோடிங் செய்கிறது¶

சத்தமில்லாத முடிவுகள் வாசிப்பில் தர்க்கரீதியான மதிப்பை மாற்றக்கூடும், எனவே தருக்க குவிட்டைப் படிக்கும் திறனை பாதிக்கும். நோய்க்குறி மதிப்புகளைப் பார்த்து இதைத் தணிக்க முடியும். தர்க்கரீதியான மதிப்புகள் பெரும்பாலும் மாறியிருக்கிறதா இல்லையா என்பதை இவை நமக்குச் சொல்லக்கூடும், எனவே பிழைகளை சரிசெய்ய அனுமதிக்கும். பிழைகளை சரிசெய்ய நோய்க்குறியை பகுப்பாய்வு செய்யும் செயல்முறையை ‘டிகோடிங்’ என்று அழைக்கப்படுகிறது. எங்கள் குறியீட்டிற்கான டிகோடிங் பொருளை உருவாக்குவதன் மூலம் இதைச் செய்கிறோம்.
