உங்கள் backend பற்றிய தகவல்களைப் பெறுதல்¶

குறிப்பு: பின்னணி அனைத்து குணங்கள்Qiskit Backend Specifications. இந்த பக்கம் வெப்பமடைதலின் ஒரு துணை விவரம்.¶

மைக்ரோவேவ் பல்ஸிலி௫நஂது ஒரு குவாண்டம் கணினியை நிரலாக்க, சர்கூடிலி௫நஂது தேவைப்படுவதை விட சாதனத்தைப் பற்றிய கூடுதல் தகவல்கள் தேவை. ஒரு குவாண்டம் சர்கூட் ஒரு சுருக்கமான குவாண்டம் கணினிக்காக கட்டப்பட்டுள்ளது - இது எந்த குவாண்டம் கணினியிலும் அதே குவாண்டம் நிலையை வழங்கும் (மாறுபட்ட செயல்திறன் நிலைகளைத் தவிர). ஒரு பல்ஸ் அட்டவணை, மறுபுறம், சாதனத்திற்கு மிகவும் குறிப்பிட்டது, இரண்டு வெவ்வேறு பின்தளத்தில் ஒரு நிரலை இயக்குவது ஒரே மாதிரியான பலனைத் தரும் என்று எதிர்பார்க்கப்படுவதில்லை, துளஂளியமான நாயஂஸஂெலஸஂ கணினிகளில் கூட.

ஒரு அடிப்படை எடுத்துக்காட்டு, ஒரு டிரைவ் பல்ஸ் q0_X180 க்யூபிட் 0 இல் அளவீடு செய்யப்பட்டதை கற்பனை செய்து பாருங்கள் \(X180\) பல்ஸ், இது க்யூபிட் 0 நிலையை புரட்டுகிறது. அந்த பல்ஸ்லிருந்து மாதிரிகளை அதே சாதனத்தில் க்யூபிட் 1 இல் அல்லது மற்றொரு சாதனத்தில் க்யூபிட் 0 ஐப் பயன்படுத்தினால், இதன் விளைவாக இறுதி நிலை என்ன இருக்கும் என தெரியாது. ஆனாலஂ உறுதியாக \(X180\) செயல்பாடாக இருகஂகாது.

அதனுடன், பின்புலத்திலிருந்து தகவல்கள் பல்ஸ் திட்டப்பணி கட்டுவதற்கு நேரத்தில் மிகவும் பயனுள்ளதாகத் தெரிவித்துள்ளோம். ஒரு backend மூன்று முக்கிய பகுதிகளாக உடைந்தது:

  • Configuration: நிலையான பின்னணி அம்சங்கள்

  • Properties: அளவு மற்றும் அறிக்கப்பட்ட பின்னணி பண்புகள்

  • ` Defaults <#Defaults>` _: OpenPulse-செயல்படுத்தப்பட்ட பின்னணிக்கு முன்னிருப்பு அமைப்புகள்

பின்வரும் பகுதிகளில் ஒவ்வொன்றும் மூடப்பட்டுள்ளன. இவற்றில் மூன்று முழுவதும் பல்ஸ் பயனர்களுக்கு ஆசிரியமான தரவு கொண்டிருக்கும் போது, முன்னிருப்புகள் only ஓபன்ஸ் (OpenPulse) செயற்படுத்தப்பட்ட பின்னணிகளுக்கு மட்டும் கொடுக்கப்பட்டுள்ளது.

நாம் செய்ய வேண்டிய முதல் விஷயம், ஆய்வு செய்ய ஒரு பின்நிலையை பெருவது.

[1]:

from qiskit.test.mock import FakeAlmaden

backend = FakeAlmaden()

உள்ளமைவு¶

சாதனத்தின் நிலையான அமைப்பைப் பற்றிய தரவை நீங்கள் காணும் இடம்தான் உள்ளமைவு, அதன் பெயர், பதிப்பு, க்யூபிட்களின் எண்ணிக்கை மற்றும் அது ஆதரிக்கும் அம்சங்கள் போன்றவை.

backend யினஂ உள்ளமைவின் தகவலைப் பயன்படுத்தி பின்தளத்தில் விளக்கத்தை உருவாக்க கூடுமஂ.

[2]:

config = backend.configuration()

# Basic Features
print("This backend is called {0}, and is on version {1}. It has {2} qubit{3}. It "
      "{4} OpenPulse programs. The basis gates supported on this device are {5}."
      "".format(config.backend_name,
                config.backend_version,
                config.n_qubits,
                '' if config.n_qubits == 1 else 's',
                'supports' if config.open_pulse else 'does not support',
                config.basis_gates))

This backend is called fake_almaden, and is on version 1.4.6. It has 20 qubits. It supports OpenPulse programs. The basis gates supported on this device are ['id', 'u1', 'u2', 'u3', 'cx'].

மேலேயுள்ள உள்ளமைவுகள் எந்தவொரு பின்தளத்திலுமஂ கிடைக்கின்றன, இது ஓப்பன் பல்ஸுடன் இயக்கப்பட்டிருந்தாலும் இல்லாவிட்டாலும், அது ஒரு முழுமையான பட்டியல் அல்ல. பல்ஸ் பின்தளத்தில் கூடுதல் பண்புக்கூறுகள் உள்ளன. அவற்றை மேலுமஂ விரிவாகப் பார்ப்போம்.

timescale, dt, பின்தளத்தில் சார்ந்துள்ளது. கட்டுப்பாட்டு ரேக்கின் தன்னிச்சையான அலைவடிவ ஜெனரேட்டர்களின் தலைகீழ் மாதிரி வீதமாக இதை நினைத்துப் பாருங்கள். ஒரு துடிப்பு Schedule ஒவ்வொரு மாதிரி புள்ளி மற்றும் கால அளவு இந்த கால அளவின் அலகுகளில் கொடுக்கப்பட்டுள்ளது.

[3]:

config.dt  # units of seconds

[3]:

2.2222222222222221e-10

கட்டமைப்பு அளவீடுகளுக்கு பயனுள்ள தகவல்களையும் உள்ளமைவு வழங்குகிறது. பல்ஸ் மூன்று அளவீட்டு நிலைகளை ஆதரிக்கிறது: `` 0: RAW``, `` 1: KERNELED``, மற்றும் `` 2: DISCRIMINATED``. இநஂத பின்தளத்தில் எவையெலஂலாமஂ ஆதரிக்கப்படுகின்றன என்பதை meas_levels பண்புக்கூறு நமக்குக் கூறுகிறது. இந்த வெவ்வேறு நிலைகளைக் கொண்ட நிரல்களை எவ்வாறு செயல்படுத்துவது என்பதை அறிய, இந்த பக்கத்தைப் பார்க்கவும் - விரைவில்.

பண்புகள்¶

backend பண்புகள் அளவிடப்பட்ட மற்றும் வழங்குநரால் விருப்பமாக அறிவிக்கப்பட்ட தரவைக் கொண்டுள்ளன. க்யூபிட் 0 க்கு என்ன வகையான தகவல் தெரிவிக்கப்படுகிறது என்பதைப் பார்ப்போம்.

[10]:

props = backend.properties()

[11]:

def describe_qubit(qubit, properties):
    """Print a string describing some of reported properties of the given qubit."""

    # Conversion factors from standard SI units
    us = 1e6
    ns = 1e9
    GHz = 1e-9

    print("Qubit {0} has a \n"
          "  - T1 time of {1} microseconds\n"
          "  - T2 time of {2} microseconds\n"
          "  - U2 gate error of {3}\n"
          "  - U2 gate duration of {4} nanoseconds\n"
          "  - resonant frequency of {5} GHz".format(
              qubit,
              properties.t1(qubit) * us,
              properties.t2(qubit) * us,
              properties.gate_error('u2', qubit),
              properties.gate_length('u2', qubit) * ns,
              properties.frequency(qubit) * GHz))

describe_qubit(0, props)

Qubit 0 has a
  - T1 time of 96.36208105210916 microseconds
  - T2 time of 43.4363963452638 microseconds
  - U2 gate error of 0.0011847011560486597
  - U2 gate duration of 35.55555555555556 nanoseconds
  - resonant frequency of 4.85648672471907 GHz

பண்புகள் புகாரளிக்க உத்தரவாதம் இல்லை, ஆனால் பல்ஸ் அணுகல் இல்லாத பின்பின்னிலையில் பொதுவாக இந்த தரவை வழங்குகிறது.

முன்னிருப்பு¶

மற்ற இரண்டு பிரிவுகளைப் போலன்றி, PulseDefaults பல்ஸ்-இயக்கப்பட்ட பின்நிலையில் மட்டுமே கிடைக்கும். இது சாதனத்தில் இயங்கும் முன்னிருப்பு நிரல் அமைப்புகளைக் கொண்டுள்ளது.

[12]:

defaults = backend.defaults()

டிரைவஂ அதிர்வெண்கள்¶

முன்னிருப்பு இயக்கி மற்றும் அளவீட்டு சமிக்ஞை சேனல்களுக்கான முன்னிருப்பு அதிர்வெண் அமைப்புகளைக் கொண்டுள்ளது:

[13]:

q0_freq = defaults.qubit_freq_est[0]  # Hz
q0_meas_freq = defaults.meas_freq_est[0]  # Hz

GHz = 1e-9
print("DriveChannel(0) defaults to a modulation frequency of {} GHz.".format(q0_freq * GHz))
print("MeasureChannel(0) defaults to a modulation frequency of {} GHz.".format(q0_meas_freq * GHz))

DriveChannel(0) defaults to a modulation frequency of 4.85648672471907 GHz.
MeasureChannel(0) defaults to a modulation frequency of 7.264856891000002 GHz.

குவாண்டம் சுற்று வழிமுறைகளுக்கான பல்ஸ் அட்டவணை வரையறைகள்¶

இறுதியாக, Schedule கட்டிடத்திற்கான backend மிக முக்கியமான அம்சங்களில் ஒன்று, InstructionScheduleMap. இது ஒரு சுற்று செயல்பாட்டின் பெயர் மற்றும் க்யூபிட்டிலிருந்து அந்த அறிவுறுத்தலின் முன்னிருப்பு துடிப்பு-நிலை செயல்படுத்தலுக்கான அடிப்படை விவரணையாக்கம் ஆகும்.

calibrations = defaults.instruction_schedule_map
print(calibrations)
