க்ரோவரின் அல்காரிதம் மற்றும் அலைவீச்சு பெருக்கம்¶

க்ரோவரின் வழிமுறை 1996 இல் லோவ் க்ரோவர் அறிமுகப்படுத்திய மிகவும் பிரபலமான குவாண்டம் வழிமுறைகளில் ஒன்றாகும் [1]. இது ஆரம்பத்தில் கட்டமைக்கப்படாத தேடல் சிக்கல்களுக்கு முன்மொழியப்பட்டது, அதாவது கட்டமைக்கப்படாத தரவுத்தளத்தில் குறிக்கப்பட்ட உறுப்பைக் கண்டுபிடிப்பதற்காக. இருப்பினும், க்ரோவரின் வழிமுறை இப்போது க்ரோவர் தகவமைப்பு தேடல் [2] போன்ற பல வழிமுறைகளுக்கு ஒரு சப்ரூட்டீன் ஆகும். க்ரோவரின் வழிமுறையின் விவரங்களுக்கு, Qiskit பாடப்புத்தகத்தில் க்ரோவரின் வழிமுறை <https://qiskit.org/textbook/ch-algorithms/grover.html> __ ஐப் பார்க்கவும்.

Qiskit க்ரோவரின் வழிமுறையை `` க்ரோவர்`` வகுப்பில் செயல்படுத்துகிறது. இந்த வகுப்பில் பொதுவான பதிப்பு, அலைவீச்சு பெருக்கம் [3] ஆகியவை அடங்கும், மேலும் க்ரோவரின் வழிமுறைக்கு தனிப்பட்ட மறு செய்கைகள் மற்றும் பிற மெட்டா அமைப்புகளை அமைக்க அனுமதிக்கிறது.

க்ரோவரின் வழிமுறை¶

குரோவரின் வழிமுறை குரோவர் ஆபரேட்டரைப் பயன்படுத்துகிறது:math:mathcal{Q} நல்ல மாநிலங்களின் பெருக்கங்களைப் பெருக்க:

\[\mathcal{Q} = \mathcal{A}\mathcal{S_0}\mathcal{A}^\dagger \mathcal{S_f}\]

இங்கே, * \(\mathcal{A}\) என்பது வழிமுறையின் ஆரம்ப தேடல் நிலை, இது வெறும் ஹடமார்ட்ஸ் , :math: க்ரோவர் தேடலுக்கான பாடநூலுக்கான H^{otimes n}, ஆனால் இன்னும் அதிகமாக இருக்கலாம் அலைவீச்சு பெருக்கத்திற்கான விரிவானது * \(\mathcal{S_0}\) என்பது அனைத்து 0 நிலைகளையும் பற்றிய பிரதிபலிப்பாகும்

\[\begin{split}|x\rangle \mapsto \begin{cases} -|x\rangle, &x \neq 0 \\ |x\rangle, &x = 0\end{cases}\end{split}\]

* \(\mathcal{S_f}\) என்பது பொருந்தும் ஆரக்கிள்

\[|x\rangle \mapsto (-1)^{f(x)}|x\rangle\]

\(x\) ஒரு நல்ல நிலை என்றால், \(f(x)\) என்பது 1, இல்லையெனில் 0.

சுருக்கமாக, க்ரோவரின் வழிமுறை :math வெவ்வேறு சக்திகளைப் பயன்படுத்துகிறது :mathcal{Q} மற்றும் ஒவ்வொரு மரணதண்டனை முடிந்தபின்னும் ஒரு நல்ல தீர்வு காணப்பட்டதா என்பதை சரிபார்க்கிறது.

க்ரோவரின் வழிமுறை இயங்குகிறது¶

க்ரோவரின் வழிமுறையை Grover வகுப்போடு இயக்க, முதலில், க்ரோவரின் வழிமுறையின் சுற்றுக்கு ஒரு ஆரக்கிளைக் குறிப்பிட வேண்டும். பின்வரும் எடுத்துக்காட்டில், க்ரோவரின் வழிமுறையின் ஆரக்கிளாக QuantumCircuit ஐப் பயன்படுத்துகிறோம். இருப்பினும், க்ரோவரின் வழிமுறையின் ஆரக்கிளாக நாம் பயன்படுத்தக்கூடிய பல வகுப்புகள் உள்ளன. இந்த டுடோரியலில் அவற்றைப் பற்றி பின்னர் பேசுகிறோம்.

Grover க்கான ஆரக்கிள் ஒரு * கட்ட-திருப்பு * ஆரக்கிளாக இருக்க வேண்டும் என்பதை நினைவில் கொள்க. அதாவது, இது “good states” பெருக்கங்களை ஒரு காரணியால் பெருக்குகிறது \(-1\). * பிட்-ஃபிளிப் * ஆரக்கிளை ஒரு கட்ட-திருப்பு ஆரக்கிளாக மாற்றுவது எப்படி என்பதை பின்னர் விளக்குகிறோம்.

[1]:

from qiskit import QuantumCircuit
from qiskit.aqua.algorithms import Grover

# the state we desire to find is '11'
good_state = ['11']

# specify the oracle that marks the state '11' as a good solution
oracle = QuantumCircuit(2)
oracle.cz(0, 1)

# define Grover's algorithm
grover = Grover(oracle=oracle, good_state=good_state)

# now we can have a look at the Grover operator that is used in running the algorithm
grover.grover_operator.draw(output='mpl')

[1]:

<matplotlib.figure.Figure at 0x7f359572eba8>

பின்னர், நாங்கள் ஒரு பின்தளத்தில் குறிப்பிடுகிறோம், Grover இன் run முறையை ஒரு பின்தளத்தில் சுற்றுகளை இயக்க அழைக்கிறோம். திரும்பிய முடிவு வகை GroverResult.

தேடல் வெற்றிகரமாக இருந்தால், முடிவின் oracle_evaluation பண்பு True. இந்த வழக்கில், மிகவும் மாதிரியான அளவீட்டு, top_measurement,“good states” ஒன்றாகும். இல்லையெனில், oracle_evaluation தவறானது.

[2]:

from qiskit import Aer
from qiskit.aqua import QuantumInstance

qasm_simulator = Aer.get_backend('qasm_simulator')
result = grover.run(quantum_instance=qasm_simulator)
print('Result type:', type(result))
print()
print('Success!' if result.oracle_evaluation else 'Failure!')
print('Top measurement:', result.top_measurement)

Result type: <class 'qiskit.aqua.algorithms.amplitude_amplifiers.grover.GroverResult'>

Success!
Top measurement: 11

எடுத்துக்காட்டில், top_measurement இன் விளைவாக `` 11`` என்பது “good state” ஒன்றாகும். இதனால், Grover ஐப் பயன்படுத்தி பதிலைக் கண்டுபிடிக்க நாங்கள் வெற்றி பெற்றோம்.

Grover இன் ஆரக்கிள் என பல்வேறு வகையான வகுப்புகளைப் பயன்படுத்துதல்¶

மேலேயுள்ள எடுத்துக்காட்டில், Grover இன் ஆரக்கிளாக QuantumCircuit ஐப் பயன்படுத்தினோம். இருப்பினும், `` qiskit.aqua.components.oracles.Oracle``, மற்றும் `` qiskit.quantum_info.Statevector`` ஆகியவற்றை ஆரக்கிள்களாகப் பயன்படுத்தலாம். பின்வரும் எல்லா எடுத்துக்காட்டுகளும் பின்வருமாறு: \(|11\rangle\) என்பது “good state”

[3]:

from qiskit.quantum_info import Statevector
oracle = Statevector.from_label('11')
grover = Grover(oracle=oracle, good_state=['11'])

result = grover.run(quantum_instance=qasm_simulator)
print('Result type:', type(result))
print()
print('Success!' if result.oracle_evaluation else 'Failure!')
print('Top measurement:', result.top_measurement)

Result type: <class 'qiskit.aqua.algorithms.amplitude_amplifiers.grover.GroverResult'>

Success!
Top measurement: 11

உட்புறமாக, ஸ்டேட்வெக்டர் ஒரு குவாண்டம் சுற்றுக்கு மாற்றப்படுகிறது:

[4]:

grover.grover_operator.oracle.draw(output='mpl')

[4]:

Qiskit அக்வாவில் உள்ள Oracle கூறுகள் மிகவும் சிக்கலான ஆரக்கிள்களை எளிதாக உருவாக்க அனுமதிக்கின்றன. Oracle வகை சுவாரஸ்யமான துணைப்பிரிவுகளைக் கொண்டுள்ளது: * LogicalExpressionOracle: `` '~ a | போன்ற தர்க்கரீதியான வெளிப்பாடுகளை பாகுபடுத்துவதற்காக. b'. 3-SAT சிக்கல்களைத் தீர்க்க இது மிகவும் பயனுள்ளதாக இருக்கும், மேலும் அதனுடன் `Grover Examples <08_grover_examples.ipynb>` __ டுடோரியலில் காட்டப்பட்டுள்ளது. \ * `` TrutheTableOracle: பைனரி உண்மை அட்டவணைகளை சுற்றுகளாக மாற்றுவதற்காக

நிலையைக் கண்டுபிடிப்பதற்கான எளிய எடுத்துக்காட்டுக்கு இங்கே `` லாஜிக்கல் எக்ஸ்பிரஷன்ஆரக்கிள்`` ஐப் பயன்படுத்துவோம் \(|11\rangle\), இது` 'a & b' உடன் ஒத்திருக்கிறது.

[5]:

from qiskit.aqua.components.oracles import LogicalExpressionOracle

# `Oracle` (`LogicalExpressionOracle`) as the `oracle` argument
expression = '(a & b)'
oracle = LogicalExpressionOracle(expression)
grover = Grover(oracle=oracle)
grover.grover_operator.oracle.draw(output='mpl')

[5]:

இந்த ஆரக்கிள் உண்மையில் இரண்டுக்கு பதிலாக மூன்று குவிட்களுடன் செயல்படுத்தப்படுகிறது என்பதை நீங்கள் அவதானிக்கலாம்!

ஏனென்றால், LogicalExpressionOracle என்பது ஒரு கட்ட-புரட்டு ஆரக்கிள் அல்ல (இது நல்ல மாநிலத்தின் கட்டத்தை புரட்டுகிறது) ஆனால் ஒரு பிட்-ஃபிளிப் ஆரக்கிள். இதன் பொருள் மற்ற குவிட்கள் நிபந்தனையை பூர்த்தி செய்தால் அது ஒரு துணை குவிட்டின் நிலையை புரட்டுகிறது. க்ரோவரின் வழிமுறைக்கு, எங்களுக்கு ஒரு கட்ட-புரட்டு ஆரக்கிள் தேவைப்படுகிறது. பிட்-ஃபிளிப் ஆரக்கிளை ஒரு கட்ட-ஃபிளிப் ஆரக்கிளாக மாற்ற, கட்டுப்படுத்தப்பட்ட- NOT: சாண்ட்விச் \(X\) மற்றும் \(H\) வாயில்கள், மேலே உள்ள சுற்றுகளில் நீங்கள் காணக்கூடியது.

** குறிப்பு: ** ஒரு பிட்-ஃபிளிப்பிலிருந்து ஒரு கட்ட-திருப்பு ஆரக்கிள் வரை இந்த மாற்றம் பொதுவாக உள்ளது, மேலும் உங்கள் ஆரக்கிளை சரியான பிரதிநிதித்துவத்திற்கு மாற்ற இதைப் பயன்படுத்தலாம்.

அலைவீச்சு பெருக்கம்¶

க்ரோவர் ஆபரேட்டரின் தொடக்கத்தில் அனைத்து மாநிலங்களின் சீரான சூப்பர் போசிஷனை உருவாக்க க்ரோவரின் வழிமுறை ஹடமார்ட் வாயில்களைப் பயன்படுத்துகிறது :math: mathcal {Q}. நல்ல மாநிலங்கள் குறித்த சில தகவல்கள் கிடைத்தால், ஒரே மாதிரியான சூப்பர் போசிஷனில் தொடங்காமல் இருப்பது பயனுள்ளதாக இருக்கும், ஆனால் குறிப்பிட்ட மாநிலங்களை மட்டுமே துவக்கலாம். இது, குரோவரின் வழிமுறையின் பொதுமைப்படுத்தப்பட்ட பதிப்பு Amplitude Amplification என குறிப்பிடப்படுகிறது.

Qiskit, state_preparation வாதத்தை அமைப்பதன் மூலம் ஆரம்ப சூப்பர் நிலையை எளிதாக சரிசெய்ய முடியும்.

நிலை தயாரிப்பு¶

வீச்சு பெருக்கத்தின் தொடக்க புள்ளிக்கு ஒரு குவாண்டம் நிலையைத் தயாரிக்கும் குவாண்டம் சுற்றுவட்டத்தைக் குறிப்பிட state_preparation வாதம் பயன்படுத்தப்படுகிறது. முன்னிருப்பாக, ஒரு சுற்று:math:H ^ {otimes n} சீரான சூப்பர் போசிஷனைத் தயாரிக்கப் பயன்படுகிறது (எனவே இது க்ரோவரின் தேடலாக இருக்கும்). வீச்சு பெருக்கத்தின் பரவல் சுற்று தானாகவே state_preparation பிரதிபலிக்கிறது.

import numpy as np

# Specifying `state_preparation`
# to prepare a superposition of |01>, |10>, and |11>

oracle.h(2)

state_preparation = QuantumCircuit(3)
state_preparation.ry(theta, 0)
state_preparation.ch(0,1)
state_preparation.x(1)
state_preparation.h(2)

grover = Grover(oracle=oracle, state_preparation=state_preparation, good_state=['110', '111'])

முழு நெகிழ்வுத்தன்மை¶

மேலும் மேம்பட்ட பயன்பாட்டிற்கு, grover_operator வாதத்தை அமைப்பதன் மூலம் முழு க்ரோவர் ஆபரேட்டரையும் குறிப்பிடலாம். பூஜ்ஜிய பிரதிபலிப்பு, ஆரக்கிள் மற்றும் மாநில தயாரிப்பு வழியாக இயல்புநிலை கட்டுமானத்தை விட \(\mathcal{Q}\) இதற்கான திறமையான செயல்படுத்தல் உங்களுக்குத் தெரிந்தால் இது பயனுள்ளதாக இருக்கும்.

இந்த ` `qiskit.circuit.library.GroverOperator ` ` ஒரு நல்ல ஆரம்பிக்கும் புள்ளி இருக்க முடியும் மற்றும் குரோவர் இயக்குநர் ஒரு தானியங்கி கட்டமைப்புக்கு அதிக விருப்பங்கள் கொடுக்கலாம். நீங்கள் உதாரணமாக முடியும்

  • set the mcx_mode

  • ` ` reflection_qubits ` ` அமைப்பால் பூஜ்ஜியத்தில் கியூபிட்டை புறக்கணிக்கும்

  • வெளிப்படையாக பரிமாறவும்:math: mathcal {S_f}, mathcal {S_0} மற்றும்:math: th mathcal {A} செயல்பாடுகள் oracle, zero_reflection மற்றும் state_ preparation ஆகிய வாதங்கள் பயன்படுத்துகின்றன

good_state குறிப்பிடுகிறது¶

அளவீட்டு முடிவு சரியானதா அல்லது உள்நாட்டில் இல்லையா என்பதைச் சரிபார்க்க good_state பயன்படுத்தப்படுகிறது. இது பைனரி சரங்களின் பட்டியல், முழு எண், Statevector, மற்றும் அழைக்கக்கூடியது. உள்ளீடு பிட்ஸ்ட்ரிங்ஸின் பட்டியல் என்றால், பட்டியலில் உள்ள ஒவ்வொரு பிட்ஸ்ட்ரிங்ஸும் ஒரு நல்ல நிலையைக் குறிக்கும். உள்ளீடு முழு எண்ணின் பட்டியலாக இருந்தால், ஒவ்வொரு முழு எண்ணும் நல்ல நிலையின் குறியீட்டைக் குறிக்கும் \(|1\rangle\). இது ஒரு Statevector என்றால், இது அனைத்து நல்ல மாநிலங்களின் சூப்பர் போசிஷனைக் குறிக்கிறது.

iterations எண்ணிக்கை¶

க்ரோவரின் வழிமுறையுடன் சரியான முடிவைப் பெற க்ரோவர் ஆபரேட்டரைப் பயன்படுத்துவதற்கான எண்ணிக்கை மீண்டும் முக்கியமானது. மறு செய்கையின் எண்ணிக்கையை Grover இன் iteration வாதத்தால் அமைக்கலாம். பின்வரும் உள்ளீடுகள் ஆதரிக்கப்படுகின்றன: * பயன்படுத்தப்படும் க்ரோவர் ஆபரேட்டரின் ஒற்றை சக்தியைக் குறிப்பிடுவதற்கான ஒரு முழு எண் * அல்லது முழு எண்களின் பட்டியல், இதில் க்ரோவர் ஆபரேட்டரின் இந்த வெவ்வேறு சக்திகள் அனைத்தும் தொடர்ச்சியாக இயங்குகின்றன, ஒவ்வொரு முறையும் நாம் சரிபார்க்கிறோம் சரியான தீர்வு கண்டறியப்பட்டுள்ளது

கூடுதலாக, sample_from_iterations வாதம் உள்ளது. இது True ஆக இருக்கும்போது, iterations இல் உள்ள குறிப்பிட்ட சக்திக்கு பதிலாக, 0 மற்றும் iterations இன் மதிப்பு ஆகியவற்றுக்கு இடையேயான ஒரு சீரற்ற முழு எண் பவர் க்ரோவரின் ஆபரேட்டராகப் பயன்படுத்தப்படுகிறது. தீர்வின் எண்ணிக்கை கூட நமக்குத் தெரியாதபோது இந்த அணுகுமுறை பயனுள்ளதாக இருக்கும்.

`` Sample_from_iterations`` ஐப் பயன்படுத்தி வழிமுறையின் கூடுதல் விவரங்களுக்கு, [4] ஐப் பார்க்கவும்.

` ` post_processing ` ` பயன்பாடு¶

வாசிப்பு எளிமைக்கு மேல் அளவீட்டுக்கு விருப்பமான இடுகை செயலாக்கத்தை நாங்கள் பயன்படுத்தலாம். இதைப் பயன்படுத்தலாம் எ.கா. அளவீட்டின் பிட்-பிரதிநிதித்துவத்திலிருந்து `` [1, 0, 1] `` DIMACS CNF வடிவத்திற்கு மாற்ற `` [1, -2, 3] ``.

def to_DIAMACS_CNF_format(bit_rep):
    return [index+1 if val==1 else -1 * (index + 1) for index, val in enumerate(bit_rep)]

oracle = QuantumCircuit(2)
oracle.cz(0, 1)
grover = Grover(oracle=oracle, good_state=['11'],post_processing=to_DIAMACS_CNF_format)
