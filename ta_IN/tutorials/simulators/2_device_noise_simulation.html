சாதன பின்தளத்தில் இரைச்சல் மாதிரி உருவகப்படுத்துதல்கள்¶

முன்னுரை¶

IBMQ வன்பொருள் சாதனத்திற்கான அடிப்படை இரைச்சல் மாதிரியை தானாக உருவாக்க Qiskit ஏர் noise தொகுதியை எவ்வாறு பயன்படுத்துவது என்பதை இந்த நோட்புக் காட்டுகிறது, மேலும் ஏற்படும் பிழைகளின் விளைவுகளை ஆய்வு செய்ய QuantumCircuits இன் சத்தமான உருவகப்படுத்துதல்களைச் செய்ய இந்த மாதிரியை உண்மையான சாதனங்களில் பயன்படுத்தவும்.

குறிப்பு, இந்த தானியங்கி மாதிரிகள் உண்மையான சாதனங்களில் நிகழும் உண்மையான பிழைகளின் * approximation *, அவை வாயில்களில் *average error rates * தொடர்பான வரையறுக்கப்பட்ட உள்ளீட்டு அளவுருக்களிலிருந்து உருவாக்கப்பட வேண்டும் என்பதன் காரணமாக. உண்மையான சாதனங்களில் குவாண்டம் பிழைகள் பற்றிய ஆய்வு ஆராய்ச்சியின் செயலில் உள்ள பகுதியாகும், மேலும் விரிவான இரைச்சல் மாதிரிகளை மற்றொரு நோட்புக்கில் உள்ளமைப்பதற்கான Qiskit ஏர் கருவிகளைப் பற்றி விவாதிக்கிறோம்.

[1]:

from qiskit import IBMQ, transpile
from qiskit import QuantumCircuit
from qiskit.providers.aer import AerSimulator
from qiskit.tools.visualization import plot_histogram

சாதன பின்தளத்தில் சத்தம் மாதிரி¶

Qiskit Aer சாதன சத்தம் மாதிரி தானாகவே ஒரு உண்மையான சாதனத்திற்கான எளிமைப்படுத்தப்பட்ட சத்தம் மாதிரியை உருவாக்குகிறது. இந்த மாதிரி ஒரு சாதனத்தின் BackendProperties இல் புகாரளிக்கப்பட்ட அளவுத்திருத்த தகவல்களைப் பயன்படுத்தி உருவாக்கப்படுகிறது மற்றும் கணக்கில் எடுத்துக்கொள்ளப்படுகிறது

  • ஒவ்வொரு குவிட்டிலும் ஒவ்வொரு அடிப்படை கேட்டின் *gate-error * நிகழ்தகவு.

  • ஒவ்வொரு குவிட்டிலும் ஒவ்வொரு அடிப்படை கேட்டின் gate_length.

  • தி \(T_1\), \(T_2\) ஒவ்வொரு குவிட்டின் தளர்வு நேர மாறிலிகள்.

  • ஒவ்வொரு குவிட்டின் வாசிப்பு பிழை நிகழ்தகவுகள்.

டெர்ரா மோக் பின்தளத்தில்¶

Qiskit Terra வில் சேமிக்கப்பட்ட தரவைப் பயன்படுத்தி ஒரு IBM குவாண்டம் சாதனத்திற்கு உண்மையான இரைச்சல் தரவைப் பயன்படுத்துவோம். குறிப்பாக, இந்த பயிற்சிகளில், சாதனம் ibmq_vigo.

[2]:

from qiskit.test.mock import FakeVigo
device_backend = FakeVigo()

சாதனம் மற்றும் உருவகப்படுத்துதல் ஒப்பீட்டுக்கான சோதனை சுற்று¶

இப்போது நாம் ஒரு சோதனை சுற்று உருவாக்க உண்மையான சாதனத்தின் வெளியீட்டை Qiskit Aer உருவகப்படுத்தப்பட்ட இரைச்சல் வெளியீடுடன் ஒப்பிடுகிறோம் AerSimulator .நாம் ஒரு 3-க்யூபிட் GHZ நிலை \(\frac{1}{\sqrt2}(|0,0,0\rangle + |1,1,1\rangle)\) க்யூபிட்கள் 0, 1 மற்றும் 2.இரைச்சலுடன் இயங்கும் முன் அல்லது சாதனத்தில் எந்த சத்தமும் இல்லாமல் சிறந்த எதிர்பார்க்கப்படும் வெளியீட்டை நாங்கள் காண்பிக்கிறோம்.

[3]:

# Construct quantum circuit
circ = QuantumCircuit(3, 3)
circ.h(0)
circ.cx(0, 1)
circ.cx(1, 2)
circ.measure([0, 1, 2], [0, 1, 2])

sim_ideal = AerSimulator()

# Execute and get counts
result = sim_ideal.run(transpile(circ, sim_ideal)).result()
counts = result.get_counts(0)
plot_histogram(counts, title='Ideal counts for 3-qubit GHZ state')

[3]:

சாதனத்தைப் பிரதிபலிக்கும் சிமுலேட்டரை உருவாக்குகிறது¶

Ibmq_vigo க்கு ஒரு சிமுலேட்டரை உருவாக்க from_backend ஐ அழைக்கிறோம்:

[4]:

sim_vigo = AerSimulator.from_backend(device_backend)

சாதன பண்புகளை vigo_simulator சேமிப்பதன் மூலம், உருவகப்படுத்துதலுக்கான சுற்றுகளை தொகுக்கும்போது பொருத்தமான அடிப்படை கேட்கள் மற்றும் இணைப்பு வரைபடம் பயன்படுத்தப்படுவதை நாங்கள் உறுதிசெய்கிறோம், இதன் மூலம் உண்மையான சாதனத்தில் செயல்படுத்தப்படும் கேட்களை மிக நெருக்கமாகப் பிரதிபலிக்கிறோம். கூடுதலாக vigo_simulator இதில் தோராயமான இரைச்சல் மாதிரியைக் கொண்டுள்ளது:

  • Single-qubit gate errors ஒற்றை குவிட் டிப்போலரைசிங் பிழையைக் கொண்டுள்ளது, அதைத் தொடர்ந்து ஒற்றை குவிட் வெப்ப தளர்வு பிழை உள்ளது.

  • Two-qubit gate errors இரண்டு-குவிட் டிப்போலரைசிங் பிழையைக் கொண்டிருக்கிறது, அதைத் தொடர்ந்து வாயிலில் உள்ள இரண்டு குவிட்களிலும் ஒற்றை-குவிட் வெப்ப தளர்வு பிழைகள் உள்ளன.

  • Single-qubit readout errors தனிப்பட்ட குவிட்களின் அளவீடுகளிலிருந்து பெறப்பட்ட கிளாசிக்கல் பிட் மதிப்பில்.

கேட் பிழைகளுக்கு, வெப்ப தளர்வு பிழைகளின் பிழை அளவுரு aer.noise.errors தொகுதியிலிருந்து thermal_relaxation_error செயல்பாட்டைப் பயன்படுத்தி பெறப்படுகிறது, தனித்தனி குவிட் \(T_1\) மற்றும் \(T_2\) அளவுருக்கள் மற்றும் சாதன பின்தளத்தில் பண்புகளிலிருந்து gate_time அளவுரு. டிப்போலரைசிங் பிழையின் நிகழ்தகவு பின்னர் அமைக்கப்படுகிறது, இதனால் வெப்ப தளர்த்தலைத் தொடர்ந்து டிப்போலரைசிங் பிழையிலிருந்து ஒருங்கிணைந்த சராசரி கேட் துரோகம் பின்தளத்தில் பண்புகளிலிருந்து gate_error மதிப்புக்கு சமமாக இருக்கும்.

ரீட்அவுட் பிழைகளுக்கு, `` readout_errors`` என்ற குவிட் மூலம் ஒரு அளவீட்டு வழங்கப்பட்ட பின்னர் பதிவுசெய்யப்பட்ட கிளாசிக்கல் பிட் மதிப்பு உண்மையான முடிவிலிருந்து புரட்டப்படும்.

இரைச்சல் உருவகப்படுத்துதலை இயக்குகிறது¶

ஒரு உண்மையான சாதனத்தின் அடிப்படையில் சத்தமில்லாத சிமுலேட்டர் பின்தளத்தில் உருவாக்கியவுடன், சத்தமான உருவகப்படுத்துதல்களை இயக்க அதைப் பயன்படுத்தலாம்.

Important: சத்தமில்லாத உருவகப்படுத்துதல்களை இயக்கும் போது, ​​பின்தளத்தில் சுற்று transpile செய்ய critical, இதன் மூலம் சுற்று பின்தளத்தில் அமைக்கப்பட்ட சரியான சத்தமான அடிப்படை வாயிலுக்கு அனுப்பப்படுகிறது.

[5]:

# Transpile the circuit for the noisy basis gates
tcirc = transpile(circ, sim_vigo)

# Execute noisy simulation and get counts
result_noise = sim_vigo.run(tcirc).result()
counts_noise = result_noise.get_counts(0)
plot_histogram(counts_noise,
               title="Counts for 3-qubit GHZ state with device noise model")

[5]:

டிரான்ஸ்பைலேஷன் தவிர்க்கப்பட்டால், சாதன இரைச்சல் மாதிரியிலிருந்து வரும் சத்தம் சிமுலேட்டரால் ஆதரிக்கப்படும் சுற்றுகளில் உள்ள கேட்களுக்குப் பயன்படுத்தப்படாது, ஆனால் பிரதிபலித்த பின்தளத்தில் ஆதரிக்கப்படாது.
