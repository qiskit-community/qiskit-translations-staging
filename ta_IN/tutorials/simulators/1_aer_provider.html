சிமுலேட்டர்கள்¶

முன்னுரை¶

இந்த குறிப்பேடு Qiskit Aer simulator பின்புலத்தை எப்படி இறக்குமதி செய்கிறது என்பதை காட்டுகிறது மற்றும் அதை இயக்க (சத்தம் இல்லாத) Qiskit Terra சுற்றுகளைப் பயன்படுத்துவது எப்படி என்பதைக் காட்டுகிறது.

[1]:

import numpy as np

# Import Qiskit
from qiskit import QuantumCircuit
from qiskit import Aer, transpile
from qiskit.tools.visualization import plot_histogram, plot_state_city
import qiskit.quantum_info as qi

ஏர் வழங்குநர்¶

Aer வழங்குநர் பலவிதமான உருவகப்படுத்துதல் முறைகளுக்கான உயர் செயல்திறன் சிமுலேட்டர் பின்தளத்தில் உள்ளது. தற்போதைய கணினியில் கிடைக்கக்கூடிய பின்தளத்தில் Aer.backends ஐப் பயன்படுத்தி பார்க்க முடியும்

Aer.backends()

ஏர் சிமுலேட்டர்¶

ஏர் வழங்குநரின் முக்கிய சிமுலேட்டர் AerSimulator பின்தளத்தில் உள்ளது. Aer.get_backend ('aer_simulator') ஐப் பயன்படுத்தி புதிய சிமுலேட்டர் பின்தளத்தில் உருவாக்க முடியும்.

[3]:

simulator = Aer.get_backend('aer_simulator')

உண்மையான சாதனத்தின் செயல்பாட்டைப் பிரதிபலிப்பதே AerSimulator பின்தளத்தில் இயல்புநிலை நடத்தை. அளவீடுகளைக் கொண்ட QuantumCircuit இயங்கினால், அது சுற்றுவட்டத்தில் உள்ள எந்தவொரு கிளாசிக்கல் பதிவுகளின் இறுதி மதிப்புகளைக் கொண்ட ஒரு எண்ணிக்கை அகராதியைத் தரும். சுற்றுக்கு வாயில்கள், அளவீடுகள், மீட்டமைப்புகள், நிபந்தனைகள் மற்றும் பிற விருப்ப சிமுலேட்டர் வழிமுறைகள் இருக்கலாம், அவை மற்றொரு நோட்புக்கில் விவாதிக்கப்படும்.

ஒரு குவாண்டம் சுற்று உருவகப்படுத்துதல்¶

அடிப்படை செயல்பாடு ஒரு குவாண்டம் சுற்று இயங்குகிறது மற்றும் அளவீட்டு விளைவுகளின் எண்ணிக்கையிலான அகராதியை வழங்குகிறது. 2-குவிட் பெல்-நிலையைத் தயாரிக்கும் எளிய சுற்று ஒன்றை இங்கே இயக்குகிறோம்:math:left|psirightrangle = frac{1}{2}left(left|0,0rightrangle + left|1,1 rightrangleright) மற்றும் இரண்டு குவிட்களையும் அளவிடும்.

[4]:

# Create circuit
circ = QuantumCircuit(2)
circ.h(0)
circ.cx(0, 1)
circ.measure_all()

# Transpile for simulator
simulator = Aer.get_backend('aer_simulator')
circ = transpile(circ, simulator)

# Run and get counts
result = simulator.run(circ).result()
counts = result.get_counts(circ)
plot_histogram(counts, title='Bell-State counts')

[4]:

ஒவ்வொரு ஷாட்டுக்கும் அளவீட்டு விளைவுகளைத் தருகிறது¶

ஒவ்வொரு தனிப்பட்ட ஷாட்டுக்கும் அளவீட்டு விளைவுகளின் பட்டியலைத் திருப்பித் தர QasmSimulator ஆதரிக்கிறது. run இல் memory=True என்ற முக்கிய வாதத்தை அமைப்பதன் மூலம் இது இயக்கப்படுகிறது.

[5]:

# Run and get memory
result = simulator.run(circ, shots=10, memory=True).result()
memory = result.get_memory(circ)
print(memory)

['00', '00', '11', '11', '00', '00', '11', '11', '00', '00']

ஏர் சிமுலேட்டர் விருப்பங்கள்¶

AerSimulator பின்தளத்தில் பல்வேறு கட்டமைக்கக்கூடிய விருப்பங்களை ஆதரிக்கிறது, அவை set_options முறையைப் பயன்படுத்தி புதுப்பிக்கப்படலாம். கூடுதல் விவரங்களுக்கு AerSimulator API ஆவணங்களைக் காண்க.

உருவகப்படுத்துதல் முறை¶

AerSimulator பலவிதமான உருவகப்படுத்துதல் முறைகளை ஆதரிக்கிறது, அவை ஒவ்வொன்றும் வெவ்வேறு வழிமுறைகளை ஆதரிக்கின்றன. simulator.set_option(method=value) விருப்பத்தைப் பயன்படுத்தி இந்த முறையை கைமுறையாக அமைக்கலாம், அல்லது முன்பே கட்டமைக்கப்பட்ட முறையுடன் கூடிய சிமுலேட்டர் பின்தளத்தில் Aer வழங்குநரிடமிருந்து Aer.get_backend ஐப் பயன்படுத்தி நேரடியாகப் பெறலாம்.

இலட்சிய சுற்றுகளை உருவகப்படுத்தும்போது, ​​சரியான உருவகப்படுத்துதல் முறைகளுக்கு இடையில் முறையை மாற்றுவது stabilizer, statevector, density_matrix மற்றும் matrix_product_state ஆகியவை உருவகப்படுத்துதல் முடிவை மாற்றக்கூடாது (மாதிரி நிகழ்தகவுகளிலிருந்து வழக்கமான வேறுபாடுகள் தவிர அளவீட்டு விளைவுகளுக்கு)

[6]:

# Increase shots to reduce sampling variance
shots = 10000

# Stabilizer simulation method
sim_stabilizer = Aer.get_backend('aer_simulator_stabilizer')
job_stabilizer = sim_stabilizer.run(circ, shots=shots)
counts_stabilizer = job_stabilizer.result().get_counts(0)

# Statevector simulation method
sim_statevector = Aer.get_backend('aer_simulator_statevector')
job_statevector = sim_statevector.run(circ, shots=shots)
counts_statevector = job_statevector.result().get_counts(0)

# Density Matrix simulation method
sim_density = Aer.get_backend('aer_simulator_density_matrix')
job_density = sim_density.run(circ, shots=shots)
counts_density = job_density.result().get_counts(0)

# Matrix Product State simulation method
sim_mps = Aer.get_backend('aer_simulator_matrix_product_state')
job_mps = sim_mps.run(circ, shots=shots)
counts_mps = job_mps.result().get_counts(0)

plot_histogram([counts_stabilizer, counts_statevector, counts_density, counts_mps],
               title='Counts for different simulation methods',
               legend=['stabilizer', 'statevector',
                       'density_matrix', 'matrix_product_state'])

[6]:

தானியங்கி உருவகப்படுத்துதல் முறை¶

இயல்புநிலை உருவகப்படுத்துதல் முறை automatic என்பது அந்த சுற்றுகளில் உள்ள வழிமுறைகளின் அடிப்படையில் ஒவ்வொரு சுற்றுக்கும் பிற உருவகப்படுத்துதல் முறைகளில் ஒன்றை தானாகத் தேர்ந்தெடுக்கும். பின்தளத்தில் பெறும்போது முறையின் பெயரைச் சேர்ப்பதன் மூலமாகவோ அல்லது பின்தளத்தில் method விருப்பத்தை அமைப்பதன் மூலமாகவோ ஒரு நிலையான சிமுலேஷன் முறையைக் குறிப்பிடலாம்.

GPU உருவகப்படுத்துதல்¶

என்விடியா ஜி.பீ.யுகளில் இயங்குவதற்கு statevector, density_matrix மற்றும் unitary சிமுலேட்டர்கள் துணைபுரிகின்றன. இந்த முறைகளுக்கு, simulator.set_options (device = 'GPU') பின்தளத்தில் விருப்பத்தைப் பயன்படுத்தி உருவகப்படுத்துதல் சாதனத்தை கைமுறையாக CPU அல்லது GPU க்கு அமைக்கலாம். ஒரு GPU சாதனம் கிடைக்கவில்லை என்றால் இந்த விருப்பம் விதிவிலக்கை எழுப்புகிறது.

[7]:

from qiskit.providers.aer import AerError

# Initialize a GPU backend
# Note that the cloud instance for tutorials does not have a GPU
# so this will raise an exception.
try:
    simulator_gpu = Aer.get_backend('aer_simulator')
    simulator_gpu.set_options(device='GPU')
except AerError as e:
    print(e)

"Invalid simulation device GPU. Available devices are: ['CPU']"

இணக்கமான கணினியில் GPU ஆதரவுடன் Qiskit ஏர் நிறுவப்பட்டிருந்தால், Aer வழங்குநர் முன்பே கட்டமைக்கப்பட்ட GPU சிமுலேட்டர் பின்தளத்தில் இருக்கும்:

  • aer_simulator_statevector_gpu

  • aer_simulator_density_matrix_gpu

  • aer_simulator_unitary_gpu

குறிப்பு: ஏரின் GPU பதிப்பை ``pip install qiskit-aer-gpu`` ஐப் பயன்படுத்தி நிறுவ முடியும்.

உருவகப்படுத்துதல் துல்லியம்¶

கிடைக்கக்கூடிய சிமுலேட்டர் விருப்பங்களில் ஒன்று, statevector, density_matrix unitary மற்றும் superop முறைகளுக்கான மிதவை துல்லியத்தை அமைக்க அனுமதிக்கிறது. இது set_precision="single" அல்லது precision="double" (இயல்புநிலை) விருப்பத்தைப் பயன்படுத்தி செய்யப்படுகிறது:

[8]:

# Configure a single-precision statevector simulator backend
simulator = Aer.get_backend('aer_simulator_statevector')
simulator.set_options(precision='single')

# Run and get counts
result = simulator.run(circ).result()
counts = result.get_counts(circ)
print(counts)

{'11': 509, '00': 515}

உருவகப்படுத்துதல் முன்னுரிமையை அமைப்பது CPU மற்றும் GPU உருவகப்படுத்துதல் சாதனங்களுக்கு பொருந்தும். ஒற்றை துல்லியம் தேவைப்படும் நினைவகத்தை பாதியாகக் குறைக்கும் மற்றும் சில கணினிகளில் செயல்திறன் மேம்பாடுகளை வழங்கக்கூடும்.

இந்த வழிமுறைகள் ஏர் சிமுலேட்டரால் மட்டுமே ஆதரிக்கப்படுகின்றன என்பதையும், அவற்றைக் கொண்ட ஒரு சுற்று IBM குவாண்டம் சாதனம் போன்ற சிமுலேட்டர் அல்லாத பின்தளத்தில் இயங்கினால் பிழை ஏற்படும் என்பதையும் நினைவில் கொள்க.

இறுதி நிலை திசைனயச் சேமிக்கிறது¶

உருவகப்படுத்துதலின் இறுதி நிலைமாற்றியைச் சேமிக்க, save_statevector அறிவுறுத்தலுடன் சுற்று சேர்க்கலாம். சரிந்த பிந்தைய அளவீட்டு நிலையை நாம் சேமிக்க விரும்பவில்லை என்றால், இந்த வழிமுறைகள் எந்த அளவீடுகளுக்கும் *before * பயன்படுத்தப்பட வேண்டும் என்பதை நினைவில் கொள்க

[9]:

# Construct quantum circuit without measure
circ = QuantumCircuit(2)
circ.h(0)
circ.cx(0, 1)
circ.save_statevector()

# Transpile for simulator
simulator = Aer.get_backend('aer_simulator')
circ = transpile(circ, simulator)

# Run and get statevector
result = simulator.run(circ).result()
statevector = result.get_statevector(circ)
plot_state_city(statevector, title='Bell state')

சுற்று ஒற்றுமையாக சேமிக்கப்படுகிறது¶

QuantumCircuit க்கான ஒற்றையாட்சி மேட்ரிக்ஸைச் சேமிக்க, `` save_unitary`` அறிவுறுத்தலுடன் சுற்று சேர்க்கலாம். `` "unitary" `` உருவகப்படுத்துதல் முறைக்கு இந்த வழிமுறைகள் வழங்கப்படாததால், இந்த சுற்று எந்த அளவீடுகளையும் மீட்டமைப்புகளையும் கொண்டிருக்க முடியாது என்பதை நினைவில் கொள்க

[10]:

# Construct quantum circuit without measure
circ = QuantumCircuit(2)
circ.h(0)
circ.cx(0, 1)
circ.save_unitary()

# Transpile for simulator
simulator = Aer.get_backend('aer_simulator')
circ = transpile(circ, simulator)

# Run and get unitary
result = simulator.run(circ).result()
unitary = result.get_unitary(circ)
print("Circuit unitary:\n", unitary.round(5))

தனிப்பயன் நிலைக்கு சிமுலேட்டரை அமைத்தல்¶

தனிப்பயன் சிமுலேட்டர் வழிமுறைகளைப் பயன்படுத்தி அதன் பல உருவகப்படுத்துதல் முறைகளுக்கு தனிப்பயன் சிமுலேட்டர் நிலையை அமைக்க AerSimulator அனுமதிக்கிறது

மின்சுற்று முறை

விளக்கம்

ஆதரிக்கப்படும் முறைகள்

set_statevector

சிமுலேட்டர் நிலையை குறிப்பிட்ட ஸ்டேட்வெக்டருக்கு அமைக்கவும்

"automatic", "statevector", "density_matrix"

set_stabilizer

சிமுலேட்டர் நிலையை குறிப்பிட்ட கிளிஃபோர்ட் நிலைப்படுத்திக்கு அமைக்கவும்

"automatic", "stabilizer"

set_density_matrix

சிமுலேட்டர் நிலையை குறிப்பிட்ட அடர்த்தி அணிக்கு அமைக்கவும்

"automatic", "density_matrix"

set_unitary

சிமுலேட்டர் நிலையை குறிப்பிட்ட யூனிட்டரி மேட்ரிக்ஸுக்கு அமைக்கவும்

"automatic", "unitary", "superop"

set_superop

சிமுலேட்டர் நிலையை குறிப்பிட்ட சூப்பர் ஆபரேட்டர் மேட்ரிக்ஸுக்கு அமைக்கவும்

"automatic", "superop"

** குறிப்புகள்: ** * இந்த வழிமுறைகள் ஒரு சுற்றுவட்டாரத்தில் உள்ள அனைத்து குவிட்டுகளுக்கும் பயன்படுத்தப்பட வேண்டும், இல்லையெனில் விதிவிலக்கு எழுப்பப்படும். * உள்ளீட்டு நிலை சரியான மாநிலமாக இருக்க வேண்டும் (ஸ்டேட்வெக்டர், டெனிஸ்டி மேட்ரிக்ஸ், யூனிட்டரி போன்றவை) இல்லையெனில் விதிவிலக்கு எழுப்பப்படும். * இந்த வழிமுறைகள் ஒரு சுற்றுவட்டத்தின் எந்த இடத்திலும் பயன்படுத்தப்படலாம், மேலும் தற்போதைய நிலையை குறிப்பிட்டவற்றுடன் மேலெழுதும். எந்தவொரு கிளாசிக்கல் பதிவு மதிப்புகளும் (எ. கா. முந்தைய அளவீடுகளிலிருந்து) பாதிக்கப்படாது * செட் நிலை அறிவுறுத்தல்கள் ஏர் சிமுலேட்டரால் மட்டுமே ஆதரிக்கப்படுகின்றன, மேலும் அவை கொண்ட ஒரு சுற்று IBM குவாண்டம் சாதனம் போன்ற சிமுலேட்டர் அல்லாத பின்தளத்தில் இயங்கினால் பிழை ஏற்படும்.

தனிப்பயன் நிலைதிசையினை அமைத்தல்¶

தனிப்பயன் Statevector நிலையை அமைக்க `` set_statevector`` அறிவுறுத்தலைப் பயன்படுத்தலாம். உள்ளீட்டு நிலைவெக்டர் செல்லுபடியாகும் (:math: |langlepsi|psirangle|=1)

[12]:

# Generate a random statevector
num_qubits = 2
psi = qi.random_statevector(2 ** num_qubits, seed=100)

# Set initial state to generated statevector
circ = QuantumCircuit(num_qubits)
circ.set_statevector(psi)
circ.save_state()

# Transpile for simulator
simulator = Aer.get_backend('aer_simulator')
circ = transpile(circ, simulator)

# Run and get saved data
result = simulator.run(circ).result()
result.data(0)
